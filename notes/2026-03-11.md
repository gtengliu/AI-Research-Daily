## 主清单（单一合并版，按技术重点分类）

### VLA 架构与可组合规划

#### 1) TiPToP: A Modular Open-Vocabulary Planning System for Robotic Manipulation
- **标题**：TiPToP: A Modular Open-Vocabulary Planning System for Robotic Manipulation
- **简称**：TiPToP
- **领域**：具身规划 / VLA / Manipulation
- **论文**：https://arxiv.org/abs/2603.09971
- **项目**：https://tiptop-robot.github.io
- **代码**：https://tiptop-robot.github.io（论文正文标注“Project website and code”）
- **评测**：正文在 IsaacSim、标准 DROID（Franka FR3+ZED Mini）以及外部 DROID 场景上对比 π0.5-DROID；并给出 UR5e、WidowX AI 的跨本体部署，实机单任务 5 次/仿真 10 次试验，报告成功率与 task progress。
- **算力**：正文未给统一训练算力预算（方法主打无需机器人训练数据）；实现层面强调单 GPU 的 cuTAMP/cuRobo 规划与执行，附录记录评测使用 RTX 4090。
- **解读**：贡献是把 VLM 感知、抓取候选与 GPU-TAMP 组合成可解释、可迁移的开词汇流水线，减少端到端策略对大规模机器人数据的依赖。局限是对感知分割与网格近似质量敏感，复杂抓取失败时恢复能力仍弱于反应式策略。

#### 2) NS-VLA: Towards Neuro-Symbolic Vision-Language-Action Models
- **标题**：NS-VLA: Towards Neuro-Symbolic Vision-Language-Action Models
- **简称**：NS-VLA
- **领域**：神经符号 VLA / 机器人策略学习
- **论文**：https://arxiv.org/abs/2603.09542
- **项目**：https://zuzuzzy.github.io/NS-VLA/
- **代码**：https://github.com/Zuzuzzy/NS-VLA
- **评测**：正文在 LIBERO、LIBERO-Plus、CALVIN 做 1-shot 与 full-demo 设置；报告 SR、Steps、CALVIN 5-task success，并在扰动场景下比较泛化鲁棒性（含背景/光照等分布偏移）。
- **算力**：正文与附录未提供统一 GPU 型号/卡数总表；给出了训练步数曲线与 chunk-level 在线 RL 配置（决策步与环境步分离），但硬件预算披露不完整。
- **解读**：贡献在于将“符号原语计划 + 单调约束切换 + 在线 RL”整合进 VLA，显著改善低数据与 OOD 设定下稳定性。局限是工程复杂度高、对原语标注与计划生成质量依赖强，真实场景闭环验证仍不足。

### 长时任务与鲁棒执行

#### 1) Beyond Short-Horizon: VQ-Memory for Robust Long-Horizon Manipulation in Non-Markovian Simulation Benchmarks
- **标题**：Beyond Short-Horizon: VQ-Memory for Robust Long-Horizon Manipulation in Non-Markovian Simulation Benchmarks
- **简称**：VQ-Memory
- **领域**：长时操作 / 记忆建模 / 仿真基准
- **论文**：https://arxiv.org/abs/2603.09513
- **项目**：https://vqmemory.github.io
- **代码**：未披露
- **评测**：正文提出 RuleSafe（20 条规则、10 类保险柜）并在单任务与多任务设置评估 RDT/CogACT/π0；以 Success Rate 与 Process Score 报告，验证非马尔可夫长程依赖下 VQ 记忆对不同 backbone 的增益。
- **算力**：正文给出明确训练配置：DP3 在 4×A100 上 batch=256、200k steps；RDT/CogACT/π0 从预训练权重继续 30k steps（batch 分别为 8/32/32）；多任务 π0 为 batch=64、60k steps。
- **解读**：贡献是将连续历史压缩为离散可聚类记忆 token，并与新基准 RuleSafe 共同刻画“长时+规则依赖”难点。局限是目前验证集中在仿真与特定锁类任务，跨场景迁移与真机收益仍待证实。

#### 2) See, Plan, Rewind: Progress-Aware Vision-Language-Action Models for Robust Robotic Manipulation
- **标题**：See, Plan, Rewind: Progress-Aware Vision-Language-Action Models for Robust Robotic Manipulation
- **简称**：SPR-VLA
- **领域**：鲁棒 VLA / 失败恢复 / 长时执行
- **论文**：https://arxiv.org/abs/2603.09292
- **项目**：https://tingjundai.github.io/SPRVLA/
- **代码**：未披露
- **评测**：正文在 LIBERO 与 LIBERO-Plus（背景/机器人初始位姿/语言/布局/光照等扰动）评估，宣称 LIBERO 平均较 MolmoAct +5%；LIBERO-Plus 平均 SR 71.8%，并在真机 Pick-up/Tidy-up/Push-T 上每配置 10 次试验。
- **算力**：附录披露仿真训练用 32×A100-80G；真机任务训练用 4×A100-80G、batch=64、约 5k steps（冻结视觉编码器、微调语言模型）。
- **解读**：贡献是把“进度识别—异常检测—rewind 回退”做成无需额外模型的闭环机制，显著改善分布外与长时失败累积问题。局限是 rewind 步长与阈值需要任务调参，且依赖高质量子任务语义分解。

### 人在环后训练与灵巧操作

#### 1) DexHiL: A Human-in-the-Loop Framework for Vision-Language-Action Model Post-Training in Dexterous Manipulation
- **标题**：DexHiL: A Human-in-the-Loop Framework for Vision-Language-Action Model Post-Training in Dexterous Manipulation
- **简称**：DexHiL
- **领域**：灵巧操作 / HiL 后训练 / VLA
- **论文**：https://arxiv.org/abs/2603.09121
- **项目**：https://chenzhongxi-sjtu.github.io/dexhil/
- **代码**：未披露
- **评测**：正文在 Franka Research 3 + DexHand021 实机平台评估 Tissue Extraction 与 Plush Toy Grasping，采用 20 次独立试验/任务；报告三轮在线迭代后成功率提升，并与离线基线和 DAgger* 对比。
- **算力**：正文明确：Being-H0.5 全参训练使用 8×NVIDIA H100、60k iterations；随后在人类干预数据上用单张 H100 微调。
- **解读**：贡献是把“人类实时纠偏采样 + 干预感知重加权训练”系统化，提升高自由度接触任务的数据效率。局限是目前任务规模较小、依赖专用遥操作与手部重定向链路，迁移到更复杂场景仍需更多证据。
