### A. VLA 鲁棒性与上下文适配

#### 1) SOMA: Strategic Orchestration and Memory-Augmented System for Vision-Language-Action Model Robustness via In-Context Adaptation
- **标题**：SOMA: Strategic Orchestration and Memory-Augmented System for Vision-Language-Action Model Robustness via In-Context Adaptation
- **简称**：SOMA
- **领域**：VLA / In-Context Adaptation / 鲁棒性增强
- **论文**：https://arxiv.org/abs/2603.24060
- **项目**：https://github.com/LZY-1021/SOMA
- **代码**：https://github.com/LZY-1021/SOMA
- **评测**：仿真 benchmark：LIBERO-PRO 与作者构建的 LIBERO-SOMA。LIBERO-SOMA 将 OOD 挑战细分为 Visual Focus、Clutter Removal、Noisy Prompt、Execution Fragility、Task Chaining 五类，并在每个任务运行 199 次 randomized rollouts。真机：正文未报告独立真机实验；论文只说明这些挑战对齐 simulated and real-world robotics 的常见失效模式。设备配置：不适用。
- **算力**：训练/后训练：方法主打冻结 VLA 主干、无需参数微调，冷启动阶段仅以标准 LIBERO 任务执行轨迹填充记忆库；正文未披露 GPU 型号、卡数与总时长。推理/增强：采用 Qwen3-VL-32B 作为部分 RAG 逻辑评分器，但未给统一 latency/FPS/显存账单。总体披露状态：正文未披露完整算力账单。
- **解读**：SOMA把“记忆检索+归因式编排+MCP干预”叠在冻结 VLA 外层，核心价值是提升 OOD 失败后的就地纠偏，而不是再训一版更大的策略。结果很亮眼，但目前仍停留在仿真基准；复现时要先把记忆库质量、失败轨迹归因和工具调用边界做稳，否则收益会明显打折。

### B. 人形遥操作与偏好控制

#### 1) MIRROR: Visual Motion Imitation via Real-time Retargeting and Teleoperation with Parallel Differential Inverse Kinematics
- **标题**：MIRROR: Visual Motion Imitation via Real-time Retargeting and Teleoperation with Parallel Differential Inverse Kinematics
- **简称**：MIRROR
- **领域**：Humanoid Teleoperation / Motion Retargeting / IK
- **论文**：https://arxiv.org/abs/2603.23995
- **项目**：https://caltech-amber.github.io/mirror/
- **代码**：项目页已公开实验视频与项目说明；正文未给独立代码仓库链接
- **评测**：仿真 benchmark：MuJoCo 中比较并行 continuation-based differential IK 与多个消融版本，采用双臂移动目标 50-step closed-loop tracking、250 randomized trials。真机：Westwood Robotics THEMIS V2 Pro humanoid，40 actuated DOF，完成 object transfer、partial occlusion、dual-arm manipulation、hand-to-hand transfer、tennis 等遥操作任务。设备配置：机器人本体为 THEMIS V2 Pro；末端为双手/双臂人形执行器；感知端使用 1 套 ZED 2i stereo camera 做人体/手部姿态估计；wrist 相机未使用；head 相机位姿数量正文未披露。
- **算力**：实验平台：Ubuntu 22 台式机，Intel Ultra 9 285K + NVIDIA RTX 5090；GPU 并行 IK 通过自定义 ADMM solver 在 PyTorch 中实现。推理延迟：从 perception front-end 到机器人接收关节命令平均 54.9 ms（tethered desktop）；真实 human-to-robot end-to-end latency 约 170 ms（PD）/ 250 ms（WBC）。训练资源：正文未披露，因为核心贡献主要是在线求解与系统集成。
- **解读**：MIRROR的卖点不是单纯“看人动机器人”，而是把并行 IK、避碰 CBF 和低延迟视觉链路合成到可实时跑的人形遥操作系统里。它对受约束姿态与局部极小值处理更稳，但仍依赖视觉前端质量；复现时应优先验证相机标定、滤波和低层控制器匹配，再谈复杂双臂任务。

#### 2) PCHC: Enabling Preference Conditioned Humanoid Control via Multi-Objective Reinforcement Learning
- **标题**：PCHC: Enabling Preference Conditioned Humanoid Control via Multi-Objective Reinforcement Learning
- **简称**：PCHC
- **领域**：Humanoid RL / 多目标控制 / Preference Conditioning
- **论文**：https://arxiv.org/abs/2603.24047
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：Isaac Gym 中两类任务——fall recovery 与平地 walking。训练使用 4096 parallel environments，每个 episode 10s；评测时 fall recovery 与 walking 均在 1000 parallel rollouts 下统计能耗、鲁棒性、成功率、stride length、trajectory deviation 等指标。真机：Unitree G1 humanoid 上验证 fall recovery 与 walking 两类偏好切换行为。设备配置：机器人本体为 Unitree G1；末端为人形双臂/双腿关节系统，非抓取型末端任务；正文未使用外部相机作为主闭环输入，wrist/head 相机位姿数量不适用。
- **算力**：训练资源：单张 NVIDIA GeForce RTX 4090，在 Isaac Gym 上训练；walking 与 fall recovery 各训练 12,000 iterations。部署：策略 50 Hz 输出目标关节位置，500 Hz PD controller 跟踪扭矩；模型转为 ONNX 后运行在机器人 onboard computer。推理显存/延迟正文未给统一账单。
- **解读**：PCHC把“单策略单折中点”改成“单策略覆盖一条可调 Pareto 曲线”，对人形系统很实用：同一套策略能随偏好在稳、快、省电之间切换。当前局限是主要面向双目标场景；复现时建议先在奖励标定和偏好采样分布上做消融，否则偏好条件很容易学成表面标签。

### C. 四足基础运动数据与语言驱动控制

#### 1) QuadFM: Foundational Text-Driven Quadruped Motion Dataset for Generation and Control
- **标题**：QuadFM: Foundational Text-Driven Quadruped Motion Dataset for Generation and Control
- **简称**：QuadFM
- **领域**：Quadruped / Motion-Language Dataset / Language-driven Control
- **论文**：https://arxiv.org/abs/2603.24021
- **项目**：https://github.com/GaoLii/QuadFM
- **代码**：https://github.com/GaoLii/QuadFM
- **评测**：仿真 benchmark：主要以作者自建数据集评测与对 MotionGPT3、DogML 训练/验证对照为主，围绕 motion generation quality、tracking、stability、robustness 做比较；正文未给统一公开仿真 benchmark 名称。真机：Unitree Go2 X 平台部署 Gen2Control。设备配置：机器人本体为 Unitree Go2 X；末端为四足本体足端执行；控制与生成均部署在 NVIDIA Jetson Orin；正文未将相机作为主评测输入，camera 位姿数量不适用。语音交互时 ASR 在云端完成后下发指令。
- **算力**：Stage I 模仿学习：单张 AMD MI308X，batch size 24，约 48 小时；VAE 300 epochs，LM 300 epochs。Stage II Gen2Control joint training：2× NVIDIA RTX 3090，20,000 iterations，4,096 parallel environments，rollout length 24 steps。部署侧：Jetson Orin 上运行，motion generator 2 Hz、低层控制 50 Hz，speech-to-motion 总延迟约 0.5 s。
- **解读**：QuadFM的贡献是把“可表达、可执行、可语言对齐”的四足运动资源做成统一底座，再用 Gen2Control 把生成器和低层控制闭环联训。优势在于 expressive motion 与部署效率兼顾；局限是评测还偏作者自建协议。复现时最好先单独验证数据清洗与 joint training 稳定性，否则生成好看但落地不稳。
