### 一、机载具身导航与安全效率

#### 1) OnFly: Onboard Zero-Shot Aerial Vision-Language Navigation toward Safety and Efficiency
- **标题**：OnFly: Onboard Zero-Shot Aerial Vision-Language Navigation toward Safety and Efficiency
- **简称**：OnFly
- **领域**：Aerial VLA / Zero-shot VLN / 安全导航
- **论文**：https://arxiv.org/abs/2603.10682
- **项目**：https://robotics-star.com/OnFly/
- **代码**：https://github.com/Robotics-STAR-Lab/OnFly
- **评测**：正文 IV-B（仿真基准）对比 TypeFly/PIVOT/SPF：OnFly 的 SR/OSR/CR/FT 为 **67.8% / 78.1% / 2.7% / 27.1s**；SPF 为 26.4% / 61.5% / 42.7% / 39.2s。正文 IV-D 给出 4 个真机机载任务验证（行人跟随、长走廊导航、室外精确停靠、二层平台避障到人）。
- **算力**：正文 IV-A（仿真）披露统一实验机：**i9-13900K + RTX 4090 + 64GB RAM**；正文 IV-D/IV-E 披露机载平台：**Jetson Orin NX 16GB**（并给出决策/监控延迟分解）。未给出端到端训练 GPU 小时（该方法主打零样本推理）。
- **解读**：通过“双智能体决策-监控解耦+语义几何校验+局部规划”把零样本空中 VLN 推向可机载落地，显著提升成功率并压低碰撞率。局限在复杂开放环境和长程指令鲁棒性仍待验证；复现应严格对齐速度约束、停止判定与机载延迟统计口径。

### 二、RL+MPC 混合控制与 Sim2Real

#### 1) RL-Augmented MPC for Non-Gaited Legged and Hybrid Locomotion
- **标题**：RL-Augmented MPC for Non-Gaited Legged and Hybrid Locomotion
- **简称**：AugMPC
- **领域**：Legged/Hybrid Locomotion / RL+MPC / Sim2Real
- **论文**：https://arxiv.org/abs/2603.10878
- **项目**：https://github.com/AndrePatri/IBRIDO
- **代码**：https://github.com/AndrePatri/AugMPC
- **评测**：正文 VII（多平台实验）报告在平地与非平地任务中出现非周期接触策略，并完成 **Centauro 零样本 sim-to-real**（无 domain randomization）；正文 VII-C 给出训练收敛区间 **4–10×10^6 steps（约 9–29 simulated days）**，并强调 hybrid 任务较 legged 任务收敛更快、方差更低。
- **算力**：正文 VII-B/VII-C 披露训练配置：**800 并行环境**、batch **16384**、SAC（actor/critic 三层网络），工作站 **Threadripper 7970 + 128GiB RAM + RTX 4090**。墙钟训练时长在正文提及不完整，故按正文可见信息记录。
- **解读**：核心价值是让高层 RL 学接触时序与目标扭转，低层 MPC 保证可行性与稳定性，在样本效率和可部署性间取得较优折中。风险在于层间耦合与注入时机敏感；复现建议先固定 MPC 超参，再扫描策略频率 nrep 与注入节点 i*。

### 三、扩散策略推理期稳健化

#### 1) PPGuide: Steering Diffusion Policies with Performance Predictive Guidance
- **标题**：PPGuide: Steering Diffusion Policies with Performance Predictive Guidance
- **简称**：PPGuide
- **领域**：Diffusion Policy / Inference-time Guidance / Manipulation
- **论文**：https://arxiv.org/abs/2603.10980
- **项目**：未披露
- **代码**：未披露
- **评测**：正文 IV-A/IV-B 在 Robomimic + MimicGen 8 个任务评测。以 Table II 可读数值示例：Stack D1(500epoch) **DP 92% → PPGuide 94%**；Coffee D2(500epoch) **28% → 34%**；Kitchen D1(500epoch) **52% → 56%**；Transport(550epoch) **68% → 72%**。正文 IV-C 还给出异构策略 checkpoint 上的泛化增益。
- **算力**：正文给出训练流程与 checkpoint（250/300/350/400/450 训练引导器，500/550 做最终评测），但**GPU 型号、训练时长、显存占用等正文未披露**。
- **解读**：方法把“成功/失败相关性”转成扩散反推阶段的梯度引导，几乎不改基座策略即可提升长程任务稳健性。局限是引导器误判会放大偏航，且超参依赖任务；复现应做阈值与引导步频消融，并单独报告失败类型迁移。

### 四、VLA 推理加速与部署

#### 1) DepthCache: Depth-Guided Training-Free Visual Token Merging for Vision-Language-Action Model Inference
- **标题**：DepthCache: Depth-Guided Training-Free Visual Token Merging for Vision-Language-Action Model Inference
- **简称**：DepthCache
- **领域**：VLA Inference Acceleration / Token Merging / Real-world Manipulation
- **论文**：https://arxiv.org/abs/2603.10469
- **项目**：未披露
- **代码**：未披露
- **评测**：正文 IV-B 在 LIBERO（40 tasks）跨 π0.5/OpenVLA/GR00T 评测，给出 **1.07–1.28×** 加速且平均 SR 损失 <1%。正文 Table II 真机核心任务（π0.5）：总成功 **55/60 → 52/60**，平均延迟 **191ms → 143ms（1.33×）**。正文 Table III 扩展场景：多物体分拣时间 **28.6s→22.1s**，扰动恢复 **17.4s→13.7s**。
- **算力**：正文 IV-A Implementation Details 披露仿真与真机实验均在 **单张 RTX 4090 24GB**；方法为 training-free 推理侧改造，未给训练 GPU 小时预算。
- **解读**：以深度先验驱动 token 合并，避免纯语义裁剪破坏几何信息，在速度和成功率间取得实用平衡。风险是深度估计误差与多视角同步误差会误导合并；复现应加深度噪声扰动实验并分场景统计延迟-成功率曲线。
