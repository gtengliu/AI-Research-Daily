### A. 抓取执行闭环与功能性抓取

#### 1) A Physical Agentic Loop for Language-Guided Grasping with Execution-State Monitoring
- **标题**：A Physical Agentic Loop for Language-Guided Grasping with Execution-State Monitoring
- **简称**：Agentic Loop Grasping
- **领域**：具身抓取 / 语言引导抓取 / 执行监测闭环
- **论文**：https://arxiv.org/abs/2604.07395
- **项目**：https://wenzewwz123.github.io/Agentic-Loop/
- **代码**：https://github.com/WenzeWWZ123/Agentic-Loop
- **评测**：真机为 **Hello Robot Stretch 3** 移动操作平台，末端为 **Stretch arm + gripper**，视觉为 **1 路 eye-in-hand Intel RealSense D405 RGB-D**。论文围绕 4 组真实抓取场景做定量：**Single Target、Ambiguity(Color/Spatial)、Distractor Robustness、Multiple Identical Targets**，每配置 **10 次试验**。基线成功率分别为 **80% / 40% / 0% / 10%**，加入执行监测与有界重试后提升到 **100% / 80% / 100% / 100%**。仿真 benchmark：正文未设置独立仿真 benchmark。
- **算力**：正文重点是系统封装与运行时闭环，给出了“额外开销很小、端到端时延与基线接近”的结论，但 **GPU 型号、训练卡数、统一训练时长未系统披露**；底层 grasp primitive 直接复用 ForceSight，不做重训。
- **解读**：亮点不在于换新抓取模型，而是把 **抓取结果监测、语义化失败状态、有限重试/升级** 做成物理执行闭环，能显著补足 open-loop grasping 的脆弱点。局限是任务仍聚焦短程抓取，视觉与场景复杂度有限。复现建议优先按论文配置复刻 **Stretch + D405 + gripper telemetry Watchdog**，先验证失败分类稳定性，再替换底层 grasp policy。

#### 2) Grasp as You Dream: Imitating Functional Grasping from Generated Human Demonstrations
- **标题**：Grasp as You Dream: Imitating Functional Grasping from Generated Human Demonstrations
- **简称**：GraspDreamer
- **领域**：功能性抓取 / 生成式示范 / 跨手型抓取迁移
- **论文**：https://arxiv.org/abs/2604.07517
- **项目**：正文未披露独立项目页
- **代码**：正文未披露代码仓库
- **评测**：仿真/离线 benchmark 主要是 **TaskGrasp** 与 **DexGraspNet 子集**。TaskGrasp 上，Shadow hand 设定中 GraspDreamer 的 success 达 **80.2%/83.1%**，高于表中 DexGYS；Allegro 设定 success 达 **78.5%/82.6%**，也明显优于 DexDiffuser。真机使用 **Franka Panda**，末端覆盖 **16-DoF Allegro Hand** 与 **Robotiq 2F-85 平行夹爪**，视觉为 **1 路 eye-to-hand 标定 RGB-D 相机**。每个真实物体 **10 次重复试验**；总计 **平行夹爪 76/100、Allegro 73/100**。并进一步测了 3 个短时操控任务：**Take Flower from Vase / Open Pot / Pull Tissue**，整体成功率约 **70% / 50% / 80%**。
- **算力**：正文阐述了 video generative model 驱动的人类示范生成、手部轨迹优化与人手到机器人手的功能重定向流程，但 **未系统披露 GPU 型号、训练卡数、统一 batch/step/时长**。
- **解读**：这篇的价值是把“功能性抓取”从静态抓取姿态预测推进到 **由生成式人类示范提供操作意图先验**，再做跨手型重定向，因而更适合开放世界物体。局限是代码未放、算力账单缺失，且真实视觉仍是单视角。复现建议先从 **TaskGrasp/Robotiq 夹爪** 版本入手，先检验 intent 对齐收益，再扩到 Allegro 等灵巧手。

### B. 人类第一视角数据与跨 embodiment 学习

#### 3) EgoVerse: An Egocentric Human Dataset for Robot Learning from Around the World
- **标题**：EgoVerse: An Egocentric Human Dataset for Robot Learning from Around the World
- **简称**：EgoVerse
- **领域**：Human-to-Robot Transfer / Egocentric Data / Cross-Embodiment Co-training
- **论文**：https://arxiv.org/abs/2604.07607
- **项目**：https://egoverse.ai/
- **代码**：https://github.com/GaTech-RL2/EgoVerse
- **评测**：论文核心是数据集 + 联盟式多实验室复现实验。人类侧采集主设备为 **Project Aria Gen 1**（前向 RGB + 两路侧向单色 SLAM/手跟踪等），并提供手机版采集子集。机器人评测跨 **3 个不同平台**，共享协议下覆盖 **4 个代表性 Flagship tasks**，正文明确点名至少 **object-in-container（单臂）、cup-on-saucer（精细双臂）、bag-grocery（长时程双臂）**。评测协议为每方法 **每任务 20 次 ID + 20 次 OOD rollout**。硬件拓扑上，论文明确至少一套平台使用 **1 路主相机 + 2 路 wrist camera + 1 路 proprioception stem**；正文图文还能核到不同实验室的组合，包括 **Aria 主视角 + 双腕 D405**、**Aria + 双腕 Logitech webcams**、以及 **ZED 2 主视角**。结论是加入 EgoVerse-A 的人类数据后，跨机器人共训在 ID 与 OOD 上 **最高带来约 30% 提升**。
- **算力**：正文/附录披露训练超参为 **150,000 optimization steps**、**global batch size 32–64**、学习率 **1e-4**、推理采用 **Euler 10 步积分**；但因多实验室并行开展，**GPU 型号、卡数与总训练时长未统一披露**。
- **解读**：EgoVerse的重要性不只是“又一个第一视角数据集”，而是它把 **跨实验室、跨本体、共享协议的 human-to-robot transfer** 做成可复验研究基座。局限是不同机器人感知拓扑并不完全一致，工程复现仍需额外对齐标定和动作空间。复现建议优先照着其 **主视角+双腕视角 + 共训 recipe** 跑小规模 co-training，再做数据量扩展。

### C. 具身任务规划与多阶段训练

#### 4) RoboAgent: Chaining Basic Capabilities for Embodied Task Planning
- **标题**：RoboAgent: Chaining Basic Capabilities for Embodied Task Planning
- **简称**：RoboAgent
- **领域**：Embodied Task Planning / VLM Agent / 多阶段训练
- **论文**：https://arxiv.org/abs/2604.07774
- **项目**：正文未披露独立项目页
- **代码**：https://github.com/woyut/RoboAgent_CVPR26
- **评测**：主评测为 **ALFWorld** 与 **EB-ALFRED**，均基于 **AI2-THOR / ALFRED** 体系；OOD 额外测 **EB-Habitat** 与 **LoTa-WAH（VirtualHome）**。指标上，ALFWorld 与 EB-ALFRED 用 **Success Rate**，LoTa-WAH 用 **Subgoal Success Rate**。论文还附了一个 **real-world demonstration**，但该部分是人类操作员在真实场景下协助执行高层动作，不构成标准化真机 benchmark，因此 **真机机器人本体、末端与相机位姿数量正文未形成统一配置表**。
- **算力**：训练采用 **Qwen2.5-VL-3B** 作为底座，三阶段分别为：**Expert-SFT 2 epochs，lr 1e-5，batch size 32**；**DAgger-SFT 同 batch/lr，1 epoch**；**RFT batch size 512，lr 5e-6，120 iterations policy updates**。全部实验运行在 **4× NVIDIA H800 80GB**。
- **解读**：RoboAgent 的强点是把 embodied planning 拆成若干基础能力链，再用 **专家轨迹 + DAgger + 专家引导 RL** 渐进优化，而不是只靠一句 prompt 让大模型硬扛长链规划。局限是主成绩仍集中在模拟器，真实部署只做展示。复现建议先完整复跑 **ALFRED-only 训练数据 + 三阶段优化**，再单独消融 capability supervision 和 EIPO 目标的贡献。

### D. 真机强化学习与现实世界落差

#### 5) Reset-Free Reinforcement Learning for Real-World Agile Driving: An Empirical Study
- **标题**：Reset-Free Reinforcement Learning for Real-World Agile Driving: An Empirical Study
- **简称**：Reset-Free Agile Driving
- **领域**：Real-World RL / Agile Driving / Reset-Free Learning
- **论文**：https://arxiv.org/abs/2604.07672
- **项目**：正文未披露独立项目页
- **代码**：正文未披露代码仓库
- **评测**：真机平台为 **RoboRacer 1/10-scale autonomous vehicle**，无机械臂末端；场地为 **slippery indoor circular track**，车辆带 **aluminum bumpers**，最高实验速度约 **3.0 m/s**。仿真 benchmark 为 **F1TENTH-Gym**，设置地面摩擦系数 **μ = 0.25** 复现湿滑条件。方法比较 **PPO / SAC / TD-MPC2**，均测试是否叠加 residual learning，并以 **MPPI** 作为基准与 reset policy。真实训练中，只有 **TD-MPC2** 能稳定超过 MPPI；论文还给出 **200 episodes** 后 TD-MPC2 对比 MPPI 的轨迹分析。相机/感知配置方面，正文在主文未给出统一传感器与相机位姿表，重点放在闭环控制与 reset-free 训练协议。
- **算力**：正文给出了控制与训练协议关键超参，如 **MPPI horizon T=10**、每 episode **500 steps** 等，但 **GPU/训练卡数/总训练耗时正文未披露**。
- **解读**：这篇最有价值的结论不是“谁赢了”，而是明确展示了 **simulation best ≠ real-world best**：SAC+residual 在仿真强，在真机却会保守塌缩，反而 TD-MPC2 更稳。局限是平台仍是小车单一任务。复现建议先忠实复现 **湿滑赛道 + MPPI reset policy**，重点分析观测延迟、非马尔可夫噪声和 residual 限制探索三件事。
