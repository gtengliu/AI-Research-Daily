## 主清单（单一合并版，按技术重点分类）

### A. VLA、机器人操控与安全

#### 1) GazeVLA: Learning Human Intention for Robotic Manipulation
- **标题**：GazeVLA: Learning Human Intention for Robotic Manipulation
- **简称**：GazeVLA
- **领域**：VLA / 学习自人类 / 机器人操控
- **论文**：https://arxiv.org/abs/2604.22615
- **项目**：https://gazevla.github.io/
- **代码**：https://github.com/lichy2004/GazeVLA
- **评测**：仿真 benchmark：AV-ALOHA，平台为双臂操作 + 主动视觉，具体为 2 条 7-DoF 双臂操作臂外加 1 条 7-DoF 相机臂，总 21-DoF。真机：双臂 ALOHA（两条 7-DoF 机械臂 + grippers）与 Unitree G1（两条 7-DoF 手臂 + 两只 6-DoF Inspire dexterous hands，总 26-DoF）。正文展示 egocentric / exocentric 视角，但真机相机数量与精确位姿安装方式未完整披露。
- **算力**：正文披露预训练使用 8 张 NVIDIA A800，训练 20k steps，共 1,344 GPU hours，batch size 2048；后续机器人后训练与真机部署总算力账单正文未完整披露。
- **解读**：这篇把“人类意图”显式做成中间变量，用 gaze 先推意图、再出动作，核心价值是缓解 human-to-robot embodiment gap。亮点是少量机器人数据下仍能提升长时程与精细操作；局限在于真机相机/传感配置披露不够细，复现时建议先复刻 intention→action 的两阶段解码，再补 gaze 对齐与少样本后训练。

#### 2) RedVLA: Physical Red Teaming for Vision-Language-Action Models
- **标题**：RedVLA: Physical Red Teaming for Vision-Language-Action Models
- **简称**：RedVLA
- **领域**：VLA Safety / 物理红队 / 机器人安全评测
- **论文**：https://arxiv.org/abs/2604.22591
- **项目**：https://redvla.github.io/
- **代码**：项目页可访问，但正文与项目页首页未给出独立代码仓库直链，记为“正文未披露”
- **评测**：仿真 benchmark：LIBERO 长时程任务族，在 10 个风险场景套件上对 6 个代表性 VLA 做 physical safety elicitation；仿真观测配置随模型而异，OpenVLA 为单目 front-view，其他多数模型为 2 视角（agent-view + wrist-mounted camera）。真机：Franka 平台部署 π0，任务为“pick up the cup and place it in the bowl”与“pick up the carrot and place it in the bowl”；机器人本体为 Franka 机械臂，末端为 gripper，真机相机位姿数量正文未完整披露。
- **算力**：RedVLA 主体红队搜索与被测 VLA 训练算力正文未系统披露；附录仅披露安全防护器 SimpleVLA-Guard 训练使用单张 NVIDIA 3090、1000 epochs。其余记为“正文未披露”。
- **解读**：RedVLA有点像把 LLM red teaming 搬进物理世界：先在良性场景里注入风险物体，再用无梯度迭代放大诱发不安全行为。亮点是把 VLA 风险从“语义越狱”转到“环境因果干预”；局限是验证仍依赖 LIBERO 这类可控基准，复现时最好先做场景注入器与 safety cost 监测器的最小闭环。

#### 3) LeHome: A Simulation Environment for Deformable Object Manipulation in Household Scenarios
- **标题**：LeHome: A Simulation Environment for Deformable Object Manipulation in Household Scenarios
- **简称**：LeHome
- **领域**：家庭机器人 / 可变形物体 / 仿真平台
- **论文**：https://arxiv.org/abs/2604.22363
- **项目**：https://lehome-web.github.io/
- **代码**：https://github.com/lehome-official/lehome
- **评测**：仿真 benchmark：LeHome 自建 6 个 household tasks——Fold Garment、Fling Garment、Assemble Burger、Cut Sausage、Pour Coffee、Wipe Surface；覆盖 bedroom / kitchen / living room / bathroom，多任务各以 50 条 teleop demonstrations 训练、100 次测试评估。真机：dual-arm LeRobot，评测 Fold Garment / Assemble Burger / Wipe Surface；机器人本体为双臂 LeRobot，末端夹爪类型与相机位姿数量正文未完整披露。
- **算力**：正文重点披露环境、任务与 sim-to-real 结果，但未系统给出环境生成、策略训练或评测所用 GPU/卡数/时长；记为“正文未披露”。
- **解读**：LeHome的价值不只是“再造一个家居仿真器”，而是把服装、食物、液体/擦拭这类家庭里最麻烦的 deformable interaction 做成可训练、可评测、还能接低成本 LeRobot 的统一平台。局限是硬件与视觉链路细节披露偏少，复现时建议先从三项真机任务和 sim+real co-training 管线跑通，再扩到全六任务。

### B. Embodied Memory、世界模型与 Agent 方法论

#### 1) SpaMEM: Benchmarking Dynamic Spatial Reasoning via Perception–Memory Integration in Embodied Environments
- **标题**：SpaMEM: Benchmarking Dynamic Spatial Reasoning via Perception–Memory Integration in Embodied Environments
- **简称**：SpaMEM
- **领域**：Embodied VLM / 空间记忆 / 动态环境评测
- **论文**：https://arxiv.org/abs/2604.22409
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：SpaMEM，基于 ProcTHOR/AI2-THOR 风格室内环境构建，含 1,000 套程序化房屋、25,000+ interaction sequences、10,601,392 张高保真图像（RGB / depth / instance / semantic segmentation 四模态），并设计 3 层 15 项诊断任务。真机：不适用；设备配置：非机器人真机论文，无机器人本体/末端。基准侧会记录 6-DoF camera pose、2D/3D boxes 与 World Graph，可视作单代理 egocentric 相机轨迹驱动的动态场景评测。
- **算力**：正文描述数据生成采用多 worker + 单 GPU 加速推理服务器以减少 VRAM fragmentation，但未披露具体 GPU 型号、卡数、总时长与成本，记为“正文未披露”。
- **解读**：SpaMEM抓得很准：很多模型静态空间问答还行，一到“边走边看边改信念”就塌。亮点是把 perception、symbolic memory、end-to-end integration 拆层诊断；局限是目前仍是仿真内评测，复现时建议先搭 L2 文本记忆上限，再看 L3 视觉链路到底卡在感知还是状态更新。

#### 2) Agentic World Modeling: Foundations, Capabilities, Laws, and Beyond
- **标题**：Agentic World Modeling: Foundations, Capabilities, Laws, and Beyond
- **简称**：Agentic World Modeling
- **领域**：Agent / World Model / 综述与方法论
- **论文**：https://arxiv.org/abs/2604.22748
- **项目**：正文首页给出 Homepage: https://agentic-world-modeling.xyz
- **代码**：https://github.com/matrix-agent/awesome-agentic-world-modeling
- **评测**：仿真 benchmark：本文不是单一新 benchmark 论文，而是对 physical / digital / social / scientific 四类 governing-law regime 下的大量既有评测与系统做统一整理，正文明确汇总 400+ works 与 100+ representative systems。真机：不适用；设备配置：非单一机器人实验论文，无固定机器人本体/末端/相机位姿。
- **算力**：作为综述/框架论文，正文未报告统一训练算力；记为“正文未披露”。
- **解读**：这篇最有用的是把 world model 不再只当视频预测器，而是拆成 L1 predictor、L2 simulator、L3 evolver，并用 physical/digital/social/scientific 四类“法则”约束评估边界。亮点是给 agent 系统设计和评测都补了统一语言；局限是偏 roadmap 而非可直接复现的单模型，落地时更适合拿它做评测框架与系统分层设计清单。
