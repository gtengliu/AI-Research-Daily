### A. 触觉、灵巧手与真实机器人 VLA

#### 1) T-Rex: Tactile-Reactive Dexterous Manipulation
- **标题**：T-Rex: Tactile-Reactive Dexterous Manipulation
- **简称**：T-Rex
- **领域**：触觉 VLA / 双臂灵巧手 / contact-rich manipulation
- **论文**：https://arxiv.org/abs/2606.17055
- **项目**：https://tactile-rex.github.io/
- **代码**：https://github.com/ZhuoyangLiu2005/T-Rex
- **评测**：仿真 benchmark 正文未披露；真机 benchmark 为 12 个接触丰富灵巧操作任务，每任务 16 trials。设备为固定底座 Dexmate Vega-1 双臂机器人，末端为两只 22-DoF Sharpa Wave 灵巧手；相机为 1 个 ZED X Mini 头部左目 RGB + 2 个 ZED X One S 腕部 RGB，共 3 个视角，并使用每手 5 个指尖触觉传感器。T-Rex 平均成功率 65%，强于 EgoScale 35%、π0.5 17%、Tactile-VLA 15%。
- **算力**：SFT 配置披露为 24 × NVIDIA H100、bf16、DeepSpeed ZeRO-1、per-device batch 16；预训练数据为 22,889 小时人类 egocentric video，触觉中训练数据为 100 小时双臂触觉同步遥操作数据。
- **解读**：T-Rex 的关键是把大规模人类视频预训练、100 小时触觉机器人中训练和少量任务后训练串成闭环，用异步级联去噪同时处理视觉动作和触觉修正。亮点是真机设备与触觉配置披露细；局限是没有独立仿真 benchmark。复现应先对齐 3 视角、10 指触觉和 62 维动作空间。

#### 2) Human Universal Grasping
- **标题**：Human Universal Grasping
- **简称**：HUG
- **领域**：dexterous grasping / human-to-robot transfer / grasp benchmark
- **论文**：https://arxiv.org/abs/2606.17054
- **项目**：正文未披露（摘要称网站释放代码、数据、benchmark、checkpoint 与 demo）
- **代码**：正文未披露
- **评测**：仿真 benchmark 为 HUG-BENCH，90 个未见物体、MuJoCo、MANO right hand，每物体 10 grasp，共 600 val + 300 test；HUG test SR 73.0%，oracle 94.0%。真机包含两套：桌面为 7-DoF xArm + 6-DoF Ability hand + 第三人称 ZED stereo camera；野外为 YOR mobile manipulator + AgileX NERO arm + 20-DoF WUJI hand + Aria Gen 2 头戴视角。30 个 test objects、每方法 300 trials，桌面 HUG 66.7%，野外 62.0%。
- **算力**：2 × RTX 5090，DDP，约 10 小时，100K steps，bf16，batch size 128。
- **解读**：HUG 不用机器人数据训练，而从 1M 人类 egocentric grasp frames 学 MANO 抓取，再 retarget 到不同灵巧手。贡献是 HUG-BENCH 同时覆盖仿真和两类真机；局限是右手、单 grasp、开环执行，接触后滑移仍常见。复现先跑 aria2mesh/aria2mano 与 MuJoCo 资产链。

#### 3) Hierarchical Advantage Weighting for Online RL Fine-Tuning of VLAs from Sparse Episode Outcomes
- **标题**：Hierarchical Advantage Weighting for Online RL Fine-Tuning of VLAs from Sparse Episode Outcomes
- **简称**：HABC
- **领域**：VLA online RL / human intervention / sparse outcome credit assignment
- **论文**：https://arxiv.org/abs/2606.17043
- **项目**：https://acerobotics-vla.github.io/HABC-Website/
- **代码**：正文未披露
- **评测**：仿真 benchmark 正文未披露；真机为 ARX X5 双臂机器人，任务为 Pencil Pouch、Paper Bag、Snack Bag 三个变形物体双臂操作任务，每 checkpoint 50 trials。末端配置正文未明确披露；相机为 1 个顶置 Intel RealSense D455 + 2 个腕部 Intel RealSense D405，共 3 路 RGB。以 π0.5 为 base VLA，最终 HABC+IR 达到 Pencil Pouch 92%、Paper Bag 88%、Snack Bag 38%，高于 SFT 36%/44%/12%。
- **算力**：π0.5 fine-tuning 使用 8 × A800 GPUs。
- **解读**：HABC 把稀疏 episode outcome 拆成 viability 与 efficiency 两个 advantage head，再对自主 rollout 和 intervention 片段软加权。亮点是评估在真实双臂变形物体任务上；局限是末端硬件细节不全，且仍需在线交互。复现要保存 intervention mask、成功/失败 rollout 和轨迹长度。

#### 4) ROVE: Unlocking Human Interventions for Humanoid Manipulation via Reinforcement Learning
- **标题**：ROVE: Unlocking Human Interventions for Humanoid Manipulation via Reinforcement Learning
- **简称**：ROVE
- **领域**：humanoid VLA / intervention learning / offline RL
- **论文**：https://arxiv.org/abs/2606.17011
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark 正文未披露；真机任务为 IRON-R01-1.11 humanoid 上的 Erase the whiteboard 与 Put the bread into the toaster。机器人本体为 IRON-R01-1.11，末端为 dexterous hands，50 维 proprioceptive state/action；相机位姿数量正文未披露，且作者明确指出系统缺少 wrist camera 与 tactile feedback。图中显示 ROVE 在 demonstrations-only、experience learning 与多轮 rollout-intervention 中均优于 SFT/HG-DAgger/Filtered BC/RECAP。
- **算力**：正文未披露。
- **解读**：ROVE 的价值在于把 humanoid 人类接管轨迹分成 rollout、adaptation、recovery，用 optimistic value estimation 避免模仿接管初期的低质量动作。局限是结果主要图示、相机和算力不完整，且未扩展到 loco-manipulation。复现应先定义接管阶段边界和 value 标签。

### B. 几何动作模型、世界模型与数据生成

#### 1) Geometric Action Model for Robot Policy Learning
- **标题**：Geometric Action Model for Robot Policy Learning
- **简称**：GAM
- **领域**：geometry-aware VLA / world-action model / low-latency robot policy
- **论文**：https://arxiv.org/abs/2606.17046
- **项目**：https://cvlab-kaist.github.io/Geometric-Action-Model/
- **代码**：https://github.com/cvlab-kaist/Geometric-Action-Model
- **评测**：仿真 benchmark 覆盖 LIBERO、LIBERO-Plus 与 RoboCasa-Kitchen；LIBERO 每 suite 10 tasks、每 task 50 randomized trials，LIBERO-Plus 按官方扰动设置，RoboCasa-Kitchen 为 24 个厨房任务。真机 benchmark 正文主文称覆盖 real-world benchmarks，但 PDF 可核验的设备细节主要在仿真与输入协议；训练/评测统一使用外部视角 + wrist-mounted camera 两路 RGB，末端/本体配置正文未完整披露。
- **算力**：预训练 64 × NVIDIA GH200、batch 1024、约 96 小时；仿真 fine-tuning 16 × NVIDIA GH200、batch 160、约 48 小时；单 GH200 推理 6.9 ms（CUDA Graphs）。
- **解读**：GAM 把几何 foundation model 拆成浅层 observation encoder 与深层 geometry/action decoder，中间插入 causal future predictor。贡献是 LIBERO-Plus 85.5% 且延迟 6.9 ms；局限是真机设备披露不足。复现重点是 DA3-Giant split layer、两视角输入和 future-depth loss。

#### 2) Qwen-RobotWorld Technical Report: Unifying Embodied World Modeling through Language-Conditioned Video Generation
- **标题**：Qwen-RobotWorld Technical Report: Unifying Embodied World Modeling through Language-Conditioned Video Generation
- **简称**：Qwen-RobotWorld
- **领域**：embodied world model / language-conditioned video generation / synthetic data
- **论文**：https://arxiv.org/abs/2606.17030
- **项目**：https://pre.qwen.ai/blog?id=qwen-robotworld
- **代码**：正文未披露
- **评测**：仿真/生成 benchmark 包括 EWMBench（21 samples / 7 tasks，Overall 4.60）、DreamGen Bench（GR1 Env/Object/Behavior，总分 4.952）、PBench（Overall 0.804）、WorldModelBench（350 instances / 7 domains / 56 subdomains，总分 8.99）和 RoboTwin-IF zero-shot qualitative cases。真机设备不涉及；机器人/场景数据来自 manipulation、driving、indoor navigation 与 human-to-robot transfer，训练数据含 1.6M 个 2-4 同步视角样本。
- **算力**：正文披露 Megatron-LM 混合并行与 selective activation recomputation，但 GPU 型号、数量和总训练时长未披露；模型含 Qwen2.5-VL 7B、VAE 127M、MMDiT 20B。
- **解读**：Qwen-RobotWorld 将自然语言作为统一 action interface，把操作、驾驶、导航和人到机器人迁移合到视频世界模型里。亮点是 8.6M video-text pairs、200M+ frames 和多 benchmark 第一梯队；局限是算力与开源计划不清。复现只能先从评测协议和数据配比入手。

#### 3) R2RDreamer: 3D-aware Data Augmentation for Spatially-generalized 2D Manipulation Policies
- **标题**：R2RDreamer: 3D-aware Data Augmentation for Spatially-generalized 2D Manipulation Policies
- **简称**：R2RDreamer
- **领域**：robot data generation / 3D co-editing / 2D policy augmentation
- **论文**：https://arxiv.org/abs/2606.17040
- **项目**：https://r2rdreamer.github.io/
- **代码**：正文未披露
- **评测**：仿真 benchmark 正文未披露；真机包含 Pot-Food、Hang-Cup、Build-Bridge、Cover-Object 四个空间泛化操作任务，使用真实机器人平台、RGB-D camera 和 parallel-jaw gripper；策略最终只消费 RGB observation。每任务在 held-out object locations 测试，1 Source baseline 为 3.13%，加入 R2RDreamer 后 DP 达 21.9-40.6%，π0 达 21.9-46.9%。
- **算力**：正文未披露。

#### 4) Unified Motion-Action Modeling for Heterogeneous Robot Learning
- **标题**：Unified Motion-Action Modeling for Heterogeneous Robot Learning
- **简称**：UMA
- **领域**：motion-action model / heterogeneous robot learning / few-shot adaptation
- **论文**：https://arxiv.org/abs/2606.16917
- **项目**：https://uma-manipulation.github.io/
- **代码**：正文未披露
- **评测**：仿真 benchmark 为 3 个 simulated robot tasks，各 100 episodes，用于架构 ablation；真机按 DROID setting 做 tabletop manipulation，任务为 Insertion、Sweeping、Folding，每任务 20 trials，随机 scene 与 camera configurations。机器人本体、末端和相机数量正文未在主文明确列出；输入包含当前 observation、3D object motion trajectories 与 robot actions。
- **算力**：正文未披露。
- **解读**：UMA 用 3D object motion trajectory 作为动作与视觉动力学的共享接口，同一个 masked generative model 同时做控制、动力学预测和少样本任务推断。贡献是动作-free 人类视频、真实机器人数据、仿真数据可共训；局限是硬件配置不完整。复现应优先验证 3D keypoint trajectory 提取链。

#### 5) DreamX-World 1.0: A General-Purpose Interactive World Model
- **标题**：DreamX-World 1.0: A General-Purpose Interactive World Model
- **简称**：DreamX-World
- **领域**：interactive world model / camera-controlled video generation / game-real data
- **论文**：https://arxiv.org/abs/2606.16993
- **项目**：https://dreamx-world.github.io/（当前核验为 404；PDF 同时给出 GitHub）
- **代码**：https://github.com/AMAP-ML/DreamX-World

### C. 导航、语义地图与机器人记忆 benchmark

#### 1) SidewalkBench: Benchmarking Visual Navigation on Urban Sidewalks
- **标题**：SidewalkBench: Benchmarking Visual Navigation on Urban Sidewalks
- **简称**：SidewalkBench
- **领域**：visual navigation benchmark / sidewalk robot / social navigation
- **论文**：https://arxiv.org/abs/2606.16953
- **项目**：https://vail-ucla.github.io/SidewalkBench/
- **代码**：https://github.com/VAIL-UCLA/SidewalkBench
- **评测**：仿真 benchmark 基于 NVIDIA Isaac Sim，含 procedurally generated 与 real-world scanned sidewalk scenes；评估 9 个视觉导航模型，330 unit-test scenarios、800 pedestrian-reactive scenarios、105 long-horizon scenarios，总路线 36.5 km。真机设备不涉及；benchmark 面向只需 monocular RGB 的城市人行道导航模型。OpenPilot 在 long-horizon PG 场景最低失败率 1.34/100m；ped-crossing 平均成功率仅 0.01。
- **算力**：正文未披露统一训练/评测 GPU 账单；仿真平台为 GPU-accelerated NVIDIA Isaac Sim。
- **解读**：SidewalkBench 把城市人行道导航拆成结构单测、行人交互和长距离三类，能系统暴露 VLA/导航模型的人类行为理解弱点。局限是主要仍是仿真，没有真机闭环部署。复现建议先跑 Ped-Crossing 与 Gesturing，这两类最能拉开方法差距。

#### 2) Binary Tracking for Spatial QA and Navigation with Open Vision-Language Models
- **标题**：Binary Tracking for Spatial QA and Navigation with Open Vision-Language Models
- **简称**：BinTrack
- **领域**：robot memory / spatial QA / open-source VLM agent / navigation
- **论文**：https://arxiv.org/abs/2606.16902
- **项目**：https://ndb796.github.io/BinaryTracking/
- **代码**：https://github.com/ndb796/BinaryTracking
- **评测**：仿真 benchmark 不涉及；真实 benchmark 包括 SpaceLocQA 270 queries（90 basic / 90 local / 90 global，τ=15m）和新发布 GangnamLoop。GangnamLoop 由 Unitree Go2 四足机器人采集，传感器为 Intel RealSense D455 depth camera、Livox MID-360 LiDAR + built-in IMU，并含人类头戴相机视角；8 round-trip recordings，221m01s，383,800 RGB frames，360 queries。BinTrack 在 SpaceLocQA Overall 67.4%，Global 62.2%，优于开源复现 baseline。
- **算力**：单工作站 2 × NVIDIA RTX 6000 Ada 48GB，无外部 API；7B-class 配置可在 20GB VRAM 内运行。
- **解读**：BinTrack 把“两个锚点之间找目标”简化为沿轨迹的二分搜索，避免开源 VLM 在全局图推理中的多步失稳。贡献是同时给出开源 agent 和真实多次往返户外 benchmark；局限是依赖轨迹时序与锚点识别。复现要先固定 1.5s segment 与多视角 memory。

#### 3) CrossMaps: Confidence-Aware Open-Vocabulary Semantic Mapping for Rover Navigation
- **标题**：CrossMaps: Confidence-Aware Open-Vocabulary Semantic Mapping for Rover Navigation
- **简称**：CrossMaps
- **领域**：open-vocabulary semantic mapping / rover navigation / confidence-aware memory
- **论文**：https://arxiv.org/abs/2606.16935
- **项目**：正文未披露
- **代码**：https://github.com/jniklasklein/crossmaps
- **评测**：仿真 benchmark 正文未披露；真机 proof-of-concept 部署在 Waveshare UGV Jetson Orin rover，配 RGB-D camera 与 NVIDIA Jetson Orin GPU，通过 ROS 2 接收 RGB/depth、camera calibration 与 pose，RTAB-Map 提供定位。相机位姿来自 SLAM pose stream，论文未披露固定相机数量；演示 query 包括 plant、hammer，并发布 rover exploration video。
- **算力**：Jetson Orin-powered UGV + 外部计算机运行 CrossMaps；具体 GPU/帧率正文未披露。
- **解读**：CrossMaps 的核心不是追求大模型，而是给 CLIP 语义地图加入 STM/LTM 和 confidence/coherence 门控，减少视角与光照噪声。贡献是部署链路实际接 ROS 2/RTAB-Map；局限是缺少量化 benchmark。复现适合先验证 STM-to-LTM promotion 阈值和自然语言 heatmap。

#### 4) LOPAL: Local Performance-Aware Active Learning from Imperfect Demonstrations
- **标题**：LOPAL: Local Performance-Aware Active Learning from Imperfect Demonstrations
- **简称**：LOPAL
- **领域**：learning from demonstration / active learning / shared autonomy
- **论文**：https://arxiv.org/abs/2606.16888
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark 为带局部 penalty 的轨迹学习实验，比较 GMR、P-GMR、LP-GMR、LP-GMR+I；真机为 pipe inspection kinesthetic teaching，机器人本体/型号正文片段未明确披露，末端为 end-effector-mounted probe，使用两只 force-torque sensors 判断用户输入与探针接触，视觉/相机位姿不涉及。每种 teaching mode 顺序采 3 条 demonstrations，性能为无 penalty 管道路径百分比；LP-GMR+I 最高提升 27.31%。
- **算力**：正文未披露。
- **解读**：LOPAL 解决的是示范内部质量不均：把局部 performance 编进 GMM/GMR，并在低质量区域通过共享自治主动请求人类修正。亮点是把学习效率和用户负担同时量化；局限是任务较传统、缺少大规模机器人基准。复现应记录 penalty 时间段和人机力输入。
