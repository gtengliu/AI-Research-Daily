### A. 历史感知与歧义消解操控

#### 1) IntentVLA: Short-Horizon Intent Modeling for Aliased Robot Manipulation
- **标题**：IntentVLA: Short-Horizon Intent Modeling for Aliased Robot Manipulation
- **简称**：IntentVLA
- **领域**：VLA / 历史建模 / 歧义消解操控
- **论文**：https://arxiv.org/abs/2605.14712
- **项目**：正文未披露
- **代码**：https://github.com/ZGC-EmbodyAI/IntentVLA （abs 页给出，但发布时仓库返回 404）
- **评测**：仿真 benchmark：**AliasBench（基于 RoboTwin2 的 12 个 ambiguity-aware 任务）**、**SimplerEnv**、**LIBERO**、**RoboCasa-GR1 Tabletop**。正文核验：AliasBench 用 **12 个任务** 专门考察短时意图混叠；SimplerEnv 使用 **4 个 WidowX 任务**；LIBERO 覆盖 **Spatial / Object / Goal / Long** 四个 suite；RoboCasa 使用 **24 个 tabletop tasks**。真机：**无独立真实机器人实验**。设备配置：均为仿真 benchmark，机器人本体分别沿 benchmark 默认设定为 **RoboTwin2 双臂体 / WidowX / LIBERO 默认 Franka 系 / GR1**；末端执行器与相机位姿数量论文未统一表式披露。方法层面正文明确历史分支仅使用 **head-camera** 最近帧，当前 VLA 分支可接收多视角，但统一相机位姿数量 **正文未披露**。
- **算力**：正文核验：AliasBench 训练使用 **16× NVIDIA H100 GPUs，30K steps，batch size 16/GPU，总 batch 256**；标准 benchmark 子节同样注明运行在 **16× NVIDIA H100 GPUs**。统一总 GPU-hours 正文未披露。
- **解读**：这篇抓住了 chunked VLA 的一个真实痛点：当前帧相同，但局部意图可能已在前几帧决定。亮点是用冻结 VGGT 历史编码器抽取短时意图证据，在 AliasBench 上把平均成功率从 9.0% 拉到 45.8%；局限是仍只有 16 帧历史，且全为仿真验证，复现时建议先严格对齐 history window、head-camera 输入与 gated fusion 细节。

### B. 统一理解—想象—行动模型

#### 1) Pelican-Unified 1.0: A Unified Embodied Intelligence Model for Understanding, Reasoning, Imagination and Action
- **标题**：Pelican-Unified 1.0: A Unified Embodied Intelligence Model for Understanding, Reasoning, Imagination and Action
- **简称**：Pelican-Unified 1.0
- **领域**：Embodied foundation model / World-action model / Unified training
- **论文**：https://arxiv.org/abs/2605.15153
- **项目**：正文、abs 页与当前可核验页面均未披露
- **代码**：正文、abs 页与当前可核验页面均未披露
- **评测**：仿真 benchmark：**8 个 VLM benchmark**、**RoboTwin 50-task dual-arm benchmark**、**WorldArena**。正文核验：RoboTwin 平均成功率 **93.5%**，clean / randomized 分别 **93.6% / 93.3%**；WorldArena 的 imagination 评测 EWM Score **66.03**。真机：**UR5e 机械臂**与 **Tienkung humanoid** 工业控制面板任务。设备配置：机器人本体为 **UR5e + Tienkung humanoid**；UR5e 任务正文明确为 **RJ45 插接 + 防水复合任务**，Tienkung 用于面板零样本与组合泛化；末端执行器型号 **正文未披露**；图中出现 “head camera” 输入描述，但真实评测的统一相机位姿数量 **正文未明确披露**。
- **算力**：正文与附录可核验范围内 **未明确披露训练 GPU / GPU 数量 / GPU-hours**，写作中仅说明以 **Qwen3-VL** 初始化统一编码器、以 **Wan2.2** 初始化统一 diffusion decoder。
- **解读**：Pelican 的核心不是再串一个 world model，而是把理解、推理、未来视频生成和动作预测绑进同一训练回路。亮点是单 checkpoint 同时兼顾 VLM、WorldArena 与 RoboTwin；局限是工程细节与算力披露不充分，真实机器人配置也较粗，复现时应先补足数据配比、reasoning trace 监督和 real-world camera setup。

### C. 灵巧手干预纠错与在环后训练

#### 1) Hand-in-the-Loop: Improving Dexterous VLA via Seamless Interventional Correction
- **标题**：Hand-in-the-Loop: Improving Dexterous VLA via Seamless Interventional Correction
- **简称**：HandITL
- **领域**：Dexterous VLA / Human intervention / Interactive imitation learning
- **论文**：https://arxiv.org/abs/2605.15157
- **项目**：https://simpson-li.github.io/Hand-in-the-Loop/
- **代码**：正文、项目页与当前可核验页面均未披露
- **评测**：仿真 benchmark：**无**。真机：围绕 **3 个长程双手灵巧操作任务** 做主评测：**Unscrew Assembly With Electric Drill、Unscrew Assembly By Hand、Pick Up and Place the Parts**；另有接管后操控评测任务 **Pick Up the Drill** 与 **Pick Up and Place the Parts**。设备配置：机器人本体为 **2× Franka FR3**；末端执行器为 **2× Bytedexter V2 21-DoF hands**，组成 **56-DoF bimanual hand-arm system**；视觉为 **3 个外部 RGB-D 相机**，覆盖 **top-down / bottom-up / third-person**，相机位姿数量 **3**。人类接口为 **Meta Quest 3 controllers + Manus Quantum Metagloves + 双踏板**。正文核验：基础遥操作数据集 **20 小时**，后训练新增数据 **1 小时**；评测以 **10 rollouts/model** 统计平均子目标分数。
- **算力**：正文与附录 **未披露训练 GPU / GPU 数量 / GPU-hours**。
- **解读**：HandITL 解决的不是“怎么多采演示”，而是部署中 VLA 出错后如何平滑接管且把纠错轨迹变成更值钱的 on-policy 数据。亮点是把接管边界命令跳变降到近两个数量级，并让 1 小时 intervention 数据优于同预算纯 teleop；局限是真机专用链路较重、缺少算力细节，复现时应优先复刻 relative retargeting、shared-control 接口与 rollout 记录格式。

### D. 轻量空间感知与深度增强

#### 1) Evo-Depth: A Lightweight Depth-Enhanced Vision-Language-Action Model
- **标题**：Evo-Depth: A Lightweight Depth-Enhanced Vision-Language-Action Model
- **简称**：Evo-Depth
- **领域**：VLA / 隐式深度感知 / 轻量部署
- **论文**：https://arxiv.org/abs/2605.14950
- **项目**：正文未披露
- **代码**：https://github.com/MINT-SJTU/Evo-Depth
- **评测**：仿真 benchmark：**Meta-World、VLA-Arena、LIBERO、LIBERO-Plus**。正文核验：Meta-World 总平均 **84.4**；LIBERO 平均成功率 **95.4%**；LIBERO-Plus 平均 **69.6%**。真机：**UFactory xArm6** 上的 **3 个任务**——**Orange Placement、Tennis Deposit、Cup Stacking**。设备配置：机器人本体为 **固定底座 xArm6**；末端执行器正文未单列型号；视觉为 **2 个 Intel RealSense D435i RGB 相机**，其中 **1 个外部视角 + 1 个腕部视角**，相机位姿数量 **2**。正文核验：每个真实任务 **20 trials**，并额外测试 **4 类扰动设置**（背景、未见干扰物、水平位置、高度位置）。
- **算力**：正文/附录核验：训练总体在 **NVIDIA A100 GPUs** 上进行；Meta-World 代表性配置使用 **8× NVIDIA A100**；真机部署效率在 **1× NVIDIA RTX 4090D** 上测得。表中真实部署效率为 **0.9B 参数、3.2GB GPU 显存、12.3Hz 推理频率**。统一总 GPU-hours 正文未披露。
- **解读**：Evo-Depth 用一个 0.13B 的隐式深度编码器和渐进式对齐训练，把空间几何增益注入 0.9B 级 VLA，而不是暴力堆点云或重型 3D 模块。亮点是真机三任务平均成功率做到 90%，同时显存和频率都更友好；局限是仍聚焦桌面单臂操作，复现时建议先从三阶段 progressive alignment 和双视角 D435i 配置入手。

### E. 接触感知状态估计与动态地面运动

#### 1) CoCo-InEKF: State Estimation with Learned Contact Covariances in Dynamic, Contact-Rich Scenarios
- **标题**：CoCo-InEKF: State Estimation with Learned Contact Covariances in Dynamic, Contact-Rich Scenarios
- **简称**：CoCo-InEKF
- **领域**：Legged state estimation / Learned contact covariance / Differentiable filtering
- **论文**：https://arxiv.org/abs/2605.15122
- **项目**：正文未披露
- **代码**：正文、abs 页与当前可核验页面均未披露
- **评测**：仿真 benchmark：基于 **Lima biped robot** 的两类动态运动评测——**dancing motions** 与 **ground motions / falling-style contact-rich motions**。正文核验：ground-motion 训练集由 **81 条 Reallusion 动作序列** 随机拼接生成，仿真以 **600 Hz** 运行；并做不同接触候选点数量与结构的消融。真机：**Lima 双足机器人**，离线评测 **20 条真实 ground-motion 序列**，另有 **10 分钟 / 20 条 messy motion sequences** 的 MoCap 记录用于真实数据分析。设备配置：机器人本体为 **Lima biped**；末端执行器不适用；外感知相机 **无**，方法显式面向 **无 camera / LiDAR** 的惯导与编码器状态估计，相机位姿数量 **0**。
- **算力**：正文核验：模型训练使用 **1× Nvidia RTX 4090 GPU，100k iterations 或最多 5 天**，并行环境数 **E=1280**，history **H=20**，buffer length **L=128**。
- **解读**：CoCo-InEKF 的思路很干净：不把接触当二值开关，而是学习各候选接触点在不同方向上的速度协方差，再把它塞回结构化滤波器。亮点是兼顾可解释滤波结构与端到端训练，在动态接触场景优于多类基线；局限是当前只验证在 Lima 双足平台，复现时要重点对齐候选点采样、BPTT 长度和 slip-aware covariance 设计。

## Agent / OpenClaw 能力增强（固定栏目）

### 今日新增 / 值得关注
- **Pelican-Unified 1.0** 值得持续盯：它把 reasoning / imagination / action 放进同一训练回路，对未来 agent+robot 共用表征很有参考价值。
- **HandITL** 对真实部署尤其重要：说明“部署中纠错数据”比“离线多收演示”更值钱，这对所有具身 agent 的在线改进都很关键。

### 能力提升方法 / 工作流要点
- 今天这批论文对 OpenClaw 类 agent 的直接启发主要有三条：
  1. **IntentVLA** 提醒我们在长链任务中保留短时历史证据，而不是每一步都只看当前快照；
  2. **Pelican-Unified** 强调理解、想象、执行最好共用中间表征，而非事后拼接；
  3. **HandITL / CoCo-InEKF** 都说明“结构化纠错信号”很重要，前者来自人类介入，后者来自接触不确定性建模。

### 机器人 / 设备控制能力相关进展
- 今日主清单中 **HandITL、Evo-Depth、Pelican-Unified** 都涉及真实机器人落地，分别对应 **灵巧双手纠错、轻量深度增强、统一 world-action 训练** 三条路线。
- **CoCo-InEKF** 虽非 VLA，但对双足/动态平台很实用，尤其适合那些相机不可用、接触状态变化激烈的场景。

### 持续监控主题
- 纯 VLA / 统一 world-action model / reasoning-conditioned control
- 灵巧手在环纠错、on-policy intervention data、shared autonomy
- 深度/几何增强但仍可实时部署的轻量 VLA
- 无外感知状态估计、接触建模、动态双足控制
