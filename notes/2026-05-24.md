### A. 交互式 VLA 与指令消歧

#### 1) GesVLA: Gesture-Aware Vision-Language-Action Model Embedded Representations
- **标题**：GesVLA: Gesture-Aware Vision-Language-Action Model Embedded Representations
- **简称**：GesVLA
- **领域**：VLA / Gesture Grounding / Human-Robot Interaction
- **论文**：https://arxiv.org/abs/2605.22812
- **项目**：https://gwxuan.github.io/GesVLA/
- **代码**：https://github.com/GWxuan/GesVLA
- **评测**：仿真 benchmark：**正文未设置公开仿真 benchmark**。真机：**7-DoF ARX5 单臂**，末端为 **gripper（正文未单列型号）**；视觉为 **2 个固定视角 RGB-D 相机（global / right-view）+ 1 个夹爪相机**，相机位姿数量 **3（wrist 1 / head 0 / fixed 2）**。实验分两部分：意图推理在 **300 个半合成 RGB-D 场景训练、88 个真实样本测试**，GesVLA 意图识别 **Acc 94.3% / Progress 97.2%**；操作评测覆盖 **Pick-and-Place Block / Select Jelly / Select Fruit & Vegetable** 三任务，各 **20 trials**，总平均成功率 **80.0%**，对应三类任务均值分别为 **95.0% / 75.0% / 80.0%**。
- **算力**：正文核验到两阶段训练均采用 **AdamW**；训练使用 **4× NVIDIA H20**，推理使用 **1× RTX 4090**。总训练时长、步数正文未披露。
- **解读**：GesVLA把手势直接嵌进 VLA 表征，而不是先转文本再控制，因此对“拿这个、放那里”一类含糊指令更稳。亮点是真实多目标顺序抓放涨幅明显；局限是暂未给公开仿真基准。复现建议先复做手势关键点编码与双 VLM 解耦结构，再测遮挡和多人指向时的退化。

### B. 零样本 humanoid 交互与结构化奖励

#### 1) Imagine2Real: Towards Zero-shot Humanoid-Object Interaction via Video Generative Priors
- **标题**：Imagine2Real: Towards Zero-shot Humanoid-Object Interaction via Video Generative Priors
- **简称**：Imagine2Real
- **领域**：Humanoid-Object Interaction / Motion Tracking / Video Prior
- **论文**：https://arxiv.org/abs/2605.22272
- **项目**：正文未披露
- **代码**：补充材料中声明提供匿名代码与预训练权重，公开 abs / PDF /正文可见链接未给正式仓库
- **评测**：仿真 benchmark：**Isaac Gym 训练 + MuJoCo sim2sim 评测**，围绕 **box carrying / box pushing** 两类 HOI 任务比较带/不带 adaptor 的跟踪与交互能力；正文表 1 可核验到在代表性 HOI 执行中，完整方法成功率从 **0.00%→82.65%**、**29.82%→64.91%**。真机：**Unitree G1 humanoid**，末端为 **双手/全身交互执行**；部署依赖 **外部 motion capture 系统** 完成 base、双手与物体追踪，正文未披露 mocap 相机数量，故相机位姿数量记为 **正文未披露（可确认依赖外部 mocap，非腕/头载视觉主闭环）**。真实任务展示为 **多种箱体搬运** 与 **击打 “Iron Man” 柱体** 的零样本执行。
- **算力**：正文/附录核验到策略在 **Isaac Gym** 中用 **PPO** 训练，采用 **8,192 并行环境**，训练 GPU 为 **1× RTX 4090**；后两阶段向参考关键点注入 **5cm Gaussian noise**。完整墙钟时长正文未披露。
- **解读**：Imagine2Real把视频生成先验压成统一 4D 点轨迹，只跟 base+双手等少量关键点，避开了繁琐的人形-物体重定向。亮点是 zero-shot HOI 真机闭环终于能做出像样交互；局限是仍依赖 mocap 且整体是开环。复现建议先做三点跟踪版，再评估去掉 mocap 后用自跟踪替代的误差上界。

#### 2) Beyond Pixels: Learning Invariant Rewards for Real-World Robotics From a Few Demonstrations
- **标题**：Beyond Pixels: Learning Invariant Rewards for Real-World Robotics From a Few Demonstrations
- **简称**：FLORA
- **领域**：Reward Learning / Real-World RL / Invariant Symbolic Rewards
- **论文**：https://arxiv.org/abs/2605.22123
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**8 个 Meta-World 任务**，并扩展到 **Position OOD / Viewpoint OOD / Object OOD**；正文核验到简单任务上 FLORA 成功率约 **96%**，较最强基线提升 **22%**，困难任务保持 **42%**。真机：**7-DoF Franka Emika research arm**，末端执行器正文未单列型号；视觉来自外部相机但正文未写明具体机位与总数，故相机位姿数量记为 **正文未披露**。真实任务为 **Peg-Insert / Box-Open / Bulb-Unscrew**：20 分钟训练时 FLORA 在前两项均 **20/20**，60 分钟时 **Bulb-Unscrew 达 9/20（45%）**；Box-Open 的 Position / Viewpoint / Object OOD 在 30 分钟时分别做到 **20/20、19/20、15/20、18/20、17/20**。
- **算力**：正文/附录未统一披露训练 GPU 卡型与卡数；可核验训练预算为 **Meta-World 简单任务 100k timesteps、困难任务 200k timesteps**，真实在线 RL 观察窗口为 **20 / 40 / 60 min**。GPU 算力项按要求记为 **正文未披露**。
- **解读**：这篇有价值的点是把奖励从“认像素”改成“认任务不变量”，让少量示教也能蒸出可复用的符号奖励。亮点是 OOD 视角和物体变化下依然稳；局限是硬件与相机细节披露不足。复现建议先挑单任务实现 object-centric 3D point tracking + symbolic potential，再验证奖励排序是否先于 RL 收敛提升。

### C. 可组合控制与恢复数据生成

#### 1) Factored Diffusion Policies: Compositionally Generalized Robot Control with a Single Score Network
- **标题**：Factored Diffusion Policies:Compositionally Generalized Robot Control with a Single Score Network
- **简称**：FDP
- **领域**：Diffusion Policy / Compositional Generalization / Drone Control
- **论文**：https://arxiv.org/abs/2605.22596
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：两部分。其一为 **state-based full multi-gate drone racing**，共 **20 个可行任务 / 104 个 gate**，Factored 模型总通过率 **98/104（94%）**、held-out 组合 **27/30（90%）**，而 12 网络组合基线仅 **1/30（3%）**。其二为 **vision-based single-gate traversal**：机载观测仅 **1 个 320×240 前视 RGB 相机 + noisy gyro**，相机位姿数量 **1（wrist 0 / head 0 / onboard 1）**；在零样本 venue transfer 上从 **25.0% / crash 36.7%** 提升到 **36.7% / crash 15.0%**，within-venue held-out 组合为 **59.4%**。真机：**正文无真实机器人实验**；平台动力学对应 quadrotor，本体为 **四旋翼**，末端 **无**。
- **算力**：正文/附录核验到训练使用 **1× NVIDIA RTX 4080 SUPER 16GB**，**DDPM 100 noise steps、50-step DDIM 推理、batch size 32、1500 epochs**；state-based 模型单模训练约 **30 min**，vision-based 约 **每轮 DAgger 2 h、总计约 50 epochs**。
- **解读**：FDP真正重要的是证明“因子共享参数 + 推理时加性组合”比多模型拼接稳得多，尤其适合组合任务稀疏覆盖场景。亮点是 held-out 组合几乎追平 oracle；局限是仍停留在仿真四旋翼。复现建议优先复做单共享网络+null-token dropout，再查组合收益是否来自参数共享而非更大模型容量。

#### 2) PGDG: Physically Grounded Data Generation for Robust Bimanual Policy Learning from a Single Demonstration
- **标题**：PGDG: Physically Grounded Data Generation for Robust Bimanual Policy Learning from a Single Demonstration
- **简称**：PGDG
- **领域**：Data Generation / Bimanual Manipulation / Sim2Real Imitation
- **论文**：https://arxiv.org/abs/2605.21710
- **项目**：https://cunxid.github.io/PGDG/
- **代码**：项目页当前未给公开代码仓
- **评测**：仿真 benchmark：**Isaac Sim** 上的 **RotateBox-Roll / RotateBox-Pitch / RotateBox-Yaw / BarPass** 四任务，depth-based ACT policy 在 40 episodes 测试下，正文图 6 可核验到 PGDG 成功率约 **0.95 / 0.97 / 0.93 / 0.90**，均明显高于 MimicGen 风格空间重定位。真机：**双臂移动操作平台**，机器人本体为 **pair of 7-DoF arms**，带 **wrist F/T sensors**；末端正文未单列夹爪型号；依赖 **depth observation**，但正文未披露相机数量与机位，故相机位姿数量记为 **正文未披露**。真实部署在同样四个双臂任务上，每任务 **40 trials**，正文给出定性与图示结果，明确指出 **四项任务均优于 MimicGen**，但未在正文打印逐任务精确数值。
- **算力**：正文/附录未披露训练 GPU；可核验策略训练配置为 **ACT：batch 256、50k steps、action chunk 30、20Hz**，**GR00T N1.6 fine-tune：batch 64、30k steps、action chunk 50、20Hz**；输入 backbone 分别为 **ResNet-18 / Eagle-2B**。
- **解读**：PGDG不是简单“从一条 demo 扩增更多轨迹”，而是用物理可行采样、风险状态重标注和多样性筛选，专门补 recovery 数据。亮点是单示教也能做出可靠双臂恢复策略；局限是相机/末端细节披露偏少。复现建议先把 relabel + DPP 两块补齐，不然数据量上去也可能只是重复噪声。

### D. 多模态行为克隆机理分析

#### 1) Understanding Multimodal Failure in Action-Chunking Behavioral Cloning
- **标题**：Understanding Multimodal Failure in Action-Chunking Behavioral Cloning
- **简称**：UMF
- **领域**：Behavioral Cloning / Multimodality / Action Chunking
- **论文**：https://arxiv.org/abs/2605.22493
- **项目**：正文未披露
- **代码**：正文写明 *release the code used in the experiments*，但 abs / PDF / 正文可见链接未给公开仓库
- **评测**：仿真 benchmark：**PushT（state / image）/ Franka Kitchen（state / image）/ UR3 BlockPush**，每模型 **200 rollouts × 3 seeds**。正文表 1 可核验：Act-Flow 在 **PushT-State / PushT-Image / Kitchen-Image** 上分别达 **0.83 / 0.79 / 2.60**，MMD-CWAE-LP 在 **Kitchen-State** 达 **3.16**，而在近单峰的 **UR3 BlockPush** 上确定性 BCAT 反而最好 **1.98 goals**。真机：**正文无真实机器人实验**；涉及的仿真机器人本体包括 **Franka 单臂** 与 **UR3**，末端与相机配置分别随 benchmark 采用环境默认设置，其中 image 版仅明确为 **单视角 RGB 渲染**，相机位姿数量 **1（仿真默认视角）**。
- **算力**：正文/附录核验到所有实验使用 **单张 NVIDIA H100 94GB**；synthetic 任务约 **20–40 min**，仿真 benchmark 约 **10–20 h**。评测以固定 checkpoint 间隔进行，最终结果来自最佳 checkpoint。
- **解读**：这篇最值得记的是一个反直觉点：多模态模型不是越强越好，模式覆盖如果靠“桥接无效动作”实现，闭环反而更差。亮点是把 latent collapse 与 action-space bridge failure 讲清楚了；局限是没有真机验证。复现建议别只看离线似然，要把 rollout 成功率和无效桥接比例一起做诊断。

## Agent / OpenClaw 能力增强（固定栏目）

### 今日新增 / 值得关注
- **GesVLA** 对 OpenClaw 很有启发：多模态指令消歧不一定非得转成纯文本，手势/指向/屏幕框选都可以做成一等条件输入。
- **FLORA** 说明很多 agent 任务也该学“奖励不变量”：比起奖励模型死记界面像素，更应该编码任务结构与阶段目标。
- **UMF** 给了一个很实用的警告：如果 agent 的动作生成只是学到“桥接无效中间态”，离线分数会好看，线上执行会崩。

### 能力提升方法 / 工作流要点
1. **把含糊指令显式接入结构化条件**：手势、框选、对象 ID、坐标点都比事后自然语言改写更稳。
2. **多任务组合优先共享参数，再做因子组合**：FDP 表明“先分开训再拼接”很容易在 held-out 组合上塌掉。
3. **评估模仿学习别只看单步重建误差**：应同时看 rollout 成功率、无效桥接比例与 mode coverage。
4. **小数据真机 RL 要优先抽象不变量**：FLORA 这条路线适合迁移到 OpenClaw 的网页/终端/机器人多环境奖励建模。

### 机器人 / 设备控制能力相关进展
- **Imagine2Real** 给出了一条“视频先验 → 稀疏关键点 → humanoid 执行”的零样本 HOI 路线，适合继续盯人形交互。
- **PGDG** 强调 recovery 数据比 nominal demo 更值钱，未来若 OpenClaw 控实体设备，失败恢复样本应单独建设。
- **GesVLA** 说明多视角固定相机 + 腕部相机的组合，在真实桌面交互里仍然是高性价比配置。
