## 具身操作与 VLA / Diffusion Policy

### 1. VADF: Vision-Adaptive Diffusion Policy Framework for Efficient Robotic Manipulation
- **简称**：VADF
- **领域**：具身操作 / Diffusion Policy / 视觉自适应推理调度
- **论文**：<https://arxiv.org/abs/2604.15938>
- **项目**：正文、abs 页与摘要末尾未给出项目页
- **代码**：正文、abs 页与附录未给出公开代码仓库
- **评测**：仿真覆盖 RoboMimic（Lift、Can、Push-T、BlockPush 等，含 proficient-human/mixed-human 协议）、Kitchen 长程任务与 Adroit dexterous suite（Door / Hammer / Pen）。RoboMimic 专家数据由 6DoF SpaceMouse 在 MuJoCo 中以 50Hz 采集；Adroit 使用 24-DoF Shadow Hand 纯视觉操作。真机部署在 ARX5 机械臂上，末端为原配平行夹爪；相机为 2 个 Intel RealSense D435：腕部 1、前视固定 1，共 2 个位姿；真实任务为抓取易拉罐、放置物体、将杯子插入纸槽三项，每项 15 次试验。
- **算力**：仿真实验使用 2×NVIDIA RTX A6000（48GB）训练；真机推理使用 1×NVIDIA RTX 5880 Ada。主文给出训练区间约 20k–120k gradient steps，batch size 32。
- **解读**：文章把扩散策略的两类痛点拆开处理：训练端用 ALN 做难样本感知重加权，推理端用 HVTS 依据视觉语义动态分配去噪步数。优点是能在不改主干的情况下同时提速训练和在线采样；局限是提升主要建立在 DP 家族上，泛化到更大 VLA 仍待验证。复现建议先完整复现 ARX5 双相机真机链路，再做 ALN 与 HVTS 分别消融。

### 2. AEGIS: Anchor-Enforced Gradient Isolation for Knowledge-Preserving Vision-Language-Action Fine-Tuning
- **简称**：AEGIS
- **领域**：VLA 微调 / 知识保持 / 梯度几何约束
- **论文**：<https://arxiv.org/abs/2604.16067>
- **项目**：正文、abs 页与摘要末尾未给出项目页
- **代码**：正文、abs 页与附录未给出公开代码仓库
- **评测**：核心评测不是闭环机器人成功率，而是“VLA 微调后 VQA 能力是否被破坏”。动作数据来自 LIBERO 台面操作基准合并后的 HDF5 数据，每条样本含 2 路 RGB（agent view + wrist camera）、关节与 gripper state、语言指令，以及 50-step 的 7-DoF 动作 horizon；同时在 OK-VQA 5k OOD 子集上测微调后 VQA 保真。作者未报告真机实验，**真机设备配置：无**；闭环仿真成功率在文中仅作为未来工作讨论，未正式给出。
- **算力**：正文未披露训练 GPU / 集群配置；仅披露主要实验为 1,500 fine-tuning steps，主干为 PaliGemma2-3B-Mix-224，LoRA baseline 采用 r=16、α=32。
- **解读**：这篇不是做更强控制策略，而是解决“把连续动作梯度灌进 VLM 后语义能力被冲垮”的问题。AEGIS 用层级正交投影隔离破坏性梯度方向，在保住 VQA 的同时维持 flow-matching 收敛。局限是尚未给出大规模闭环控制收益。复现建议先严格复现实验中的 LIBERO+OK-VQA 双评测闭环，再决定是否接入真正的 closed-loop VLA 训练。

## 多机器人搜索与空中导航

### 3. Semantic Area Graph Reasoning for Multi-Robot Language-Guided Search
- **简称**：SAGR
- **领域**：多机器人搜索 / 语言引导探索 / 语义拓扑推理
- **论文**：<https://arxiv.org/abs/2604.16263>
- **项目**：正文、abs 页与摘要末尾未给出项目页
- **代码**：正文、abs 页与附录未给出公开代码仓库
- **评测**：仿真基于 Habitat-Matterport3D（HM3D）100 个场景，任务同时覆盖纯 exploration 与 semantic target search；按环境规模分别使用 2/3/4 台机器人。单机器人感知假设为机载单相机，正文显式给出“single onboard camera”，并以最大探测距离与视场角建模。真机实验未做，因此**真机设备配置：无**。
- **算力**：正文未披露本地 GPU/CPU 训练算力；高层推理默认调用 GPT-4o，文中报告每次协同查询约 2.5 秒、prompt 少于 500 tokens。
- **解读**：SAGR 的关键不是让 LLM 直接看原始图像，而是先把未知环境压成语义区域图，再让 LLM 在房间级做分配，底层仍用 frontier/TSP 做几何执行。这样保住了实时性，也显著提升语义搜索效率。局限是依赖上游房间级语义质量，且停留在仿真。复现建议先保证 HM3D 上语义占据栅格和 room-instance 构图稳定，否则高层推理收益会被噪声淹没。

- **评测**：主评测使用作者构建的 AerialVLN-Fine（300 条高质量轨迹，15 个场景，句级对齐标注）与原始 AerialVLN-S-Val（864 条）两套仿真/数据集评测；指标含 SR、SPL、NE、路径长度与 steps。文中另给出初步真机验证：平台为 RoboMaster TT UAV，使用机载前视相机进行第一视角导航，真实指令示例为“Find a chair for me, then land on the table”，17 步到达目标区域；末端执行器：无；相机位姿数量为 1（前视机载相机）。

- **解读**：FineCog-Nav 把 UAV VLN 拆成语言理解、注意、记忆、想象、推理、决策等细粒度模块，核心收益来自模块协作而不是单次长 prompt。AerialVLN-Fine 也补上了句级对齐诊断能力。局限是目前真机只做了小规模可行性演示，抗噪与闭环飞控稳定性仍弱。复现建议先复做 AerialVLN-Fine 标注协议与模块接口，再上真实无人机。

## Embodied Agent / Benchmark

### 5. SocialGrid: A Benchmark for Planning and Social Reasoning in Embodied Multi-Agent Systems
- **简称**：SocialGrid
- **领域**：Embodied Agent / 多智能体社会推理 / 基准设计
- **论文**：<https://arxiv.org/abs/2604.16022>
- **项目**：正文未给出独立项目页
- **代码**：<https://github.com/ml-research/SocialGrid>
- **评测**：基于自建 SocialGrid 网格世界，任务同时考察 task completion、planning success、path optimality 与投票识别 impostor 的社会推理。无辅助导航时，图 2 设置为 7 个 crewmate / episode、每模型 20 个 episode；带 Planning Oracle 时可隔离导航缺陷。代理观测为局部视野网格观测，并带环境可配置房间、门、任务开关等对象。真机实验未做，因此**真机设备配置：无**；仿真 benchmark 亦非机器人本体实验，**机器人本体/末端/相机位姿数量：不适用（网格世界 embodied agent 环境）**。
- **算力**：实验运行在配备 NVIDIA H100 80GB 的集群上，推理使用 vLLM；RL 部分对 Qwen3-4B 采用 PPO + LoRA。
- **解读**：SocialGrid 的价值在于把“空间规划失败”和“社会推理失败”拆开量化，证明现有 LLM agent 即使加了规划 oracle，识别欺骗仍接近随机。它很适合作为 agent 能力诊断基准。局限是环境仍较抽象，离真实机器人社会交互很远。复现建议优先复用其 failure analysis 与 league 评测框架，再把 OpenClaw 式多 agent 行为映射进去做内部基准。
