### A. VLA 策略学习、测试时推理与后训练

#### 1) TempoVLA: Learning Speed-Controllable Vision-Language-Action Policies
- **标题**：TempoVLA: Learning Speed-Controllable Vision-Language-Action Policies
- **简称**：TempoVLA
- **领域**：VLA / speed control / variable-speed execution
- **论文**：https://arxiv.org/abs/2606.06491
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**LIBERO** 四套任务（Spatial / Object / Goal / Long），每套 **10 tasks**，论文写明每套含 **500 human-teleoperated demonstrations**；TempoVLA 在该基准上验证变速训练与测试时速度控制。真机：**7-DoF Franka arm**；末端为 **1-DoF parallel gripper**；相机为 **1 个 primary camera + 1 个 wrist-mounted camera**，共 **2 个机位**；评测 **5 个任务**（4 个 pick-and-place + 1 个 deformable-object），**每任务 50 条 teleoperated trajectories** 训练，**每个命令速度 10 次 rollouts** 评测；1×默认速度成功率 **80%→88%**，接入 GPT-4o 调速后总体平均成功率 **96%**。
- **算力**：正文/附录披露：LIBERO 训练使用 **32× NVIDIA H20 GPU**、**30k iterations**、**batch size 512**；真机训练使用 **16× H20**、**10k iterations**、**batch size 128**。
- **解读**：把动作幅值直接变成可控“速度旋钮”，比单纯提速蒸馏更实用。亮点是真机动态调速确实提升成功率；局限是目前仍依赖分段与外部调速器。复现建议先检查 VSTA 分段和速度落地比值，再做策略对比。

#### 2) MPCoT: Reward-Guided Multi-Path Latent Reasoning for Test-Time Scalable Vision-Language-Action
- **标题**：MPCoT: Reward-Guided Multi-Path Latent Reasoning for Test-Time Scalable Vision-Language-Action
- **简称**：MPCoT
- **领域**：VLA / latent reasoning / test-time scaling
- **论文**：https://arxiv.org/abs/2606.06245
- **项目**：正文未披露
- **代码**：https://github.com/EDGSCOUT/MPCoT
- **评测**：仿真 benchmark：**LIBERO** 与 **CALVIN ABC→D**。LIBERO 上 all-suite 设定平均成功率 **98.9%**、Long **98.9%**；suite-specific 设定平均 **99.0%**、Long **97.8%**。CALVIN 上 **1/2/3/4/5-step SR = 99.8/98.9/96.8/93.7/89.4**，平均成功序列长度 **4.92**。真机：**无真实机器人实验**；机器人本体/末端/相机位姿 **不适用**。
- **算力**：正文未统一披露训练 GPU 型号、卡数与总时长；已披露的是推理延迟：Direct **24ms**，MPCoT `(K=5,M=4)` 为 **38ms**，显著低于显式 Textual CoT 的 **110–160ms**。
- **解读**：核心价值是把“多路径推理”塞进 latent 空间，避免显式 CoT 的 token 时延。局限是仍只在标准 benchmark 里验证。复现建议优先检查 path scorer 的奖励构成，否则很容易只得到更宽但不更准的分支搜索。

#### 3) AffordanceVLA: A Vision-Language-Action Model Empowering Action Generation through Affordance-Aware Understanding
- **标题**：AffordanceVLA: A Vision-Language-Action Model Empowering Action Generation through Affordance-Aware Understanding
- **简称**：AffordanceVLA
- **领域**：VLA / affordance grounding / sim-to-real manipulation
- **论文**：https://arxiv.org/abs/2606.06155
- **项目**：https://skywalker-yqz.github.io/AffordanceVLA/
- **代码**：https://github.com/Skywalker-yqz/AffordanceVLA
- **评测**：仿真 benchmark：**LIBERO** 上平均成功率 **95.8%**（50 rollouts / task），**CALVIN ABC→D** 上 **1/2/3/4/5-step = 96.8/92.0/87.5/80.8/75.9**、平均长度 **4.33**（1000 rollouts）。真机：**7-DoF Franka Emika Panda**；末端为 **DH-Robotics PGC-140 parallel jaw gripper**；相机为 **2 个 Intel RealSense D435**（**1 个固定第三人称 + 1 个 wrist-mounted**），共 **2 个机位**；**每任务 15 trials**。Basic tasks 平均 **88.3%**，Complex tasks（Drawer / Toaster / Clean all the rubbish）平均 **82.9%**。
- **算力**：正文/附录披露：训练使用 **16× NVIDIA H200 GPU（141GB HBM3e / 卡）**；推理部署在 **1× RTX 5090 32GB**。
- **解读**：它把 affordance 从辅助可视化变成策略中间语义层，直接缓解 VLM 语义空间和控制空间错配。局限是 long-horizon 仍未彻底解决。复现建议先保住 affordance 生成与 action expert 的联合训练，否则会退化成普通 VLA。

#### 4) FlowPRO: Reward-Free Reinforced Fine-Tuning of Flow-Matching VLAs via Proximalized Preference Optimization
- **标题**：FlowPRO: Reward-Free Reinforced Fine-Tuning of Flow-Matching VLAs via Proximalized Preference Optimization
- **简称**：FlowPRO
- **领域**：VLA post-training / preference optimization / real-robot fine-tuning
- **论文**：https://arxiv.org/abs/2606.05468
- **项目**：https://wuyeyexvnainai.github.io/flowpro/
- **代码**：正文未披露
- **评测**：仿真 benchmark：**无独立仿真 benchmark**。真机：**Dobot XTrainer bimanual platform**；末端为 **双 6-DoF 机械臂 + 左右 parallel-jaw grippers**；相机为 **1 个 top-down global RGB + 2 个 wrist RGB**，共 **3 个机位**；四个长时程双臂任务 **PACK / CAP / USB / CASE**，**3 个训练种子**，每种子 **100 次 randomized rollouts**。以 PI0.5 为底座时，RPRO 最终成功率达到 **99±1.0 / 99±1.0 / 95±1.6 / 93±1.2**，完成时间分别为 **17s / 23s / 24s / 38s**。
- **算力**：附录披露：所有训练在单机 **4× NVIDIA H20 96GB** 上完成；SFT 阶段 **60,000 steps**、每卡 batch **16**；RPRO 三轮共 **75,000 steps**、全局 batch **20**。
- **解读**：优点是用偏好优化替代脆弱 reward/critic，把真机失败轨迹利用率拉高。局限是目前验证集中在单一双臂平台。复现建议重点检查 proximal regularizer 和 SFT 项共存，否则容易把动作幅值学坏。

### B. 世界模型、语言推理与闭环评估

#### 1) World-Language-Action Model for Unified World Modeling, Language Reasoning, and Action Synthesis
- **标题**：World-Language-Action Model for Unified World Modeling, Language Reasoning, and Action Synthesis
- **简称**：WLA
- **领域**：World-language-action model / long-horizon manipulation / reasoning
- **论文**：https://arxiv.org/abs/2606.05979
- **项目**：正文未披露
- **代码**：https://github.com/SJTU-DENG-Lab/WLA
- **评测**：仿真 benchmark：**RoboTwin 2.0**（50 个双臂任务，按协议 **100 trials / task**）上 Clean / Rand 为 **92.94% / 90.02%**；**LIBERO** 四套任务（每任务 **50 demos、50 trials**）平均 **98.6%**，TTS 后 **98.9%**；**RMBench** 四个 long-horizon bimanual tasks 平均 **56.5%**。真机：**AgilexRobotics Piper dual-arm platform**；末端执行器 **正文未披露**；相机位姿数量 **正文未披露**；四个长时程任务 **Unscrew Cap / Pack Object / Stack Cup / Dispose Trash**，每任务采集 **60 demonstrations**，并在 **Standard / OOD Object / OOD Scenario** 三种设置下各 **10 trials** 评测；WLA-0 的平均成功数为 **7.5 / 5.25 / 4.75**，其中 Dispose Trash 的 standard 设定为 **6/10**。
- **算力**：正文披露推理为 **~40ms / 1× RTX 5090**，真实部署亦在 **单张 RTX 5090** 上同步执行；训练 GPU 型号、卡数与总时长 **正文未完整披露**。
- **解读**：WLA 的关键不是简单拼接 world model 和 VLA，而是把“文本子任务+未来帧+动作”作为统一 next-state 目标。局限是硬件与视觉配置披露不全。复现建议先验证 subtask supervision 的贡献，否则很难复现 RMBench 的大幅领先。

#### 2) PiL-World: A Chunk-Wise World Model for VLA Policy-in-the-Loop Evaluation
- **标题**：PiL-World: A Chunk-Wise World Model for VLA Policy-in-the-Loop Evaluation
- **简称**：PiL-World
- **领域**：World model / VLA evaluation / closed-loop rollout
- **论文**：https://arxiv.org/abs/2606.05773
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**无传统仿真 benchmark**，核心是 imagined-vs-real 闭环评估。真机：论文在 **3 个 real dual-arm manipulation tasks** 上做对齐评测；机器人本体与末端执行器 **正文未披露**；相机为 **1 个 head/top view + 2 个 wrist views**，共 **3 个机位**。训练/测试切分分别为：sorting cubes **100/20 full episodes**、stacking bowls **100/18**、stacking blocks **200/20**；最终形成 **60 / 54 / 40** 个测试 subtasks。相对 Ctrl-World，PiL-World 将 real-imagined success gap 从 **71.8/72.6/45.1** 降到 **15.0/4.2/16.7**，平均 HFR 从 **41.5%** 提升到 **70.1%**。
- **算力**：附录披露：在 RealSource World 上预训练使用 **64× H20 GPU（2 epochs）**，目标任务微调使用 **8× H20（20 epochs）**。
- **解读**：这篇的价值在于把 world model 从“看起来像”推进到“能否替代部分真机闭环评测”。局限是设备披露不充分，且 wrist 近距遮挡仍难。复现建议先守住 action-to-control projection，否则 imagined rollout 很快与真实执行脱钩。

### C. 人形、灵巧手与双手操作

#### 1) LadderMan: Learning Humanoid Perceptive Ladder Climbing
- **标题**：LadderMan: Learning Humanoid Perceptive Ladder Climbing
- **简称**：LadderMan
- **领域**：Humanoid climbing / visuomotor control / sim-to-real
- **论文**：https://arxiv.org/abs/2606.05873
- **项目**：https://ladderman-robot.github.io
- **代码**：正文未披露
- **评测**：仿真 benchmark：在梯子参数网格 **4 个倾角（55/60/65/70°）× 6 个 rung spacing（20–30cm）** 上评测，**100 evaluation episodes / 配置**，在常见范围内成功率超过 **95%**。真机：**29-DoF Unitree G1 humanoid**；末端执行器不适用；感知为 **1 个 onboard Intel RealSense D435i**，共 **1 个机位**；政策运行在 **Jetson Orin**，上梯操作阶段借助 **PICO 4 Ultra** 提供遥操作目标。真实爬梯在 **3 个 ladder 几何** 上进行，关键组件消融为 **10 trials / ladder**，完整方法在 Ladder A/B/C 上成功 **9/10、6/10、7/10**。
- **算力**：附录披露：专家策略训练在 IsaacSim 中使用 **1× NVIDIA L40S GPU**、**4096 parallel envs**、**8,000 epochs**；真实部署时策略在 **Jetson Orin** 上 50Hz 运行，深度 foundation model 在 **RTX 4090** 上实时推理。
- **解读**：亮点是把单条参考动作扩成可泛化梯子策略，并用 foundation stereo 真正打通 sim2real。局限是操作仍偏 ladder-specific。复现建议先把碰撞体和深度清洗做对，否则 climbing 成功率会被接触误差直接击穿。

#### 2) RealDexUMI: A Wearable Universal Manipulation Interface for Dexterous Robot Learning
- **标题**：RealDexUMI: A Wearable Universal Manipulation Interface for Dexterous Robot Learning
- **简称**：RealDexUMI
- **领域**：Dexterous data collection / wearable teleoperation / cross-embodiment learning
- **论文**：https://arxiv.org/abs/2606.06033
- **项目**：https://research.beingbeyond.com/realdexumi
- **代码**：正文未披露
- **评测**：仿真 benchmark：**无仿真 benchmark**。真机：默认评测平台为 **Franka FR3**；末端为 **共享 11-DoF dexterous hand module**，集成 **in-hand camera + fingertip tactile sensors**；相机位姿按主模块为 **1 个 in-hand 机位**。主实验使用 **每任务 200 demonstrations** 训练 ACT，覆盖 **8 个 real-robot tasks**，**每任务 20 real-robot trials**；平均完整任务成功率 **88.75%**。跨机体部署再验证 **Franka FR3 / RealMan RM65 / PND Adam-U** 三种 embodiment，在 Cube / Drawer 两任务上无需重训即可迁移。
- **算力**：正文/附录详细披露了训练步数与策略后端，但**未统一披露训练 GPU 型号、卡数与总时长**；故记为：**正文未披露**。
- **解读**：它补的是“可部署灵巧数据”而不是单纯更像人的示范，因而对接触丰富任务很关键。局限是手部自由度仍有限。复现建议优先保证 action-state correspondence 和 tactile 通道，否则成功率会明显掉到 state-as-action 水平。

#### 3) DexFuture: Hierarchical Future-State Visuomotor Targeting for Bimanual Dexterous Tool Use
- **标题**：DexFuture: Hierarchical Future-State Visuomotor Targeting for Bimanual Dexterous Tool Use
- **简称**：DexFuture
- **领域**：Bimanual dexterous manipulation / future-state targeting / tool use
- **论文**：https://arxiv.org/abs/2606.05699
- **项目**：https://blarklee.github.io/DexFuture_official_website/
- **代码**：正文未披露
- **评测**：仿真 benchmark：基于 **OakInk2** 的双手工具使用任务，覆盖 **cut / pour / wipe / shear / stir** 等 unseen tasks；主表列出 bread-cutting、apple-cutting、pouring、两种 whiteboard wiping、paper shearing 等任务。DexFuture 在这组任务上达到约 **59.69%** 的平均成功率，约为特权 target baseline 的 **90%**，并在 fruit-knife cutting 上达到 **89.79%**、略高于 PhysGraph(GT target) 的 **87.87%**。真机：**无真实机器人实验**；机器人本体/末端/相机位姿 **不适用**。
- **算力**：正文/附录披露：DexFuture 在 **单张 RTX 3090Ti** 上按策略控制率 **60Hz** 执行；对比的 DexWM 规划器在 **8× H100** 支持下、同样单张 **3090Ti** 评测仅约 **0.24Hz**。完整训练算力账单 **正文未完整披露**。
- **解读**：它用分层 future-target 代替高维动作规划，把双手工具任务从“慢 world-model 搜索”拉回实时控制。局限是没有真机验证。复现建议先盯住 future-target 预测误差，难任务里瓶颈明显不在低层 policy 而在目标生成。
