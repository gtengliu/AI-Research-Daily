### A. 空间记忆与场景演化 VLA

#### 1) Spatial Memory for Out-of-Vision Manipulation in Vision-Language-Action
- **标题**：Spatial Memory for Out-of-Vision Manipulation in Vision-Language-Action
- **简称**：SOMA
- **领域**：VLA / Spatial Memory / Out-of-Vision Manipulation
- **论文**：https://arxiv.org/abs/2605.22283
- **项目**：正文未披露
- **代码**：正文注明 *Code will be released soon*
- **评测**：仿真 benchmark：**RoboCasa Tabletop GR1** 与 **SimplerEnv**。正文核验到：RoboCasa Tabletop GR1 按每任务 **30 / 100 / 300 / Full** demonstrations 比较，表 3 中 SOMA 平均成功率最高做到 **52.0%**；SimplerEnv 进一步覆盖 **Bridge**（WidowX：Spoon on Towel / Carrot on Plate / Stack Cube / Eggplant in Basket）与 **Fractal** 套件。真机：自研 humanoid 平台，机器人本体为 **双臂人形底座**，末端为 **两只 adaptive grippers**，双臂为 **2× Realman ZM73 7-DoF**；视觉为 **1 个头部 Intel RealSense D435 RGB-D + 2 个腕部 Intel RealSense D435 RGB-D**，相机位姿数量 **3（wrist 2 / head 1 / third-person 0）**。真实任务共 **5 个 out-of-vision PnP**，覆盖不可见抓取、双物体顺序操作与双臂协同放置。
- **算力**：正文/附录核验到：训练时除 VLM language decoder 外其余组件联合优化，使用 **32× NVIDIA H200**、batch size **60**、训练 **30,000 steps**；推理在 **1× RTX 4090** 服务器执行，机器人机载还部署 **Jetson AGX Orin 64GB**。另给出真实数据规模约 **15.52 小时 expert demonstrations**。
- **解读**：SOMA把“看不见后的空间记忆”做成可检索的多视角记忆图，再让动作生成按指令取回几何证据。亮点是真机 OOV 抓放和仿真都稳；局限是强依赖主动扫视与多视角同步。复现建议先最小化实现记忆写入/检索链路，再测无头部转动时的退化幅度。

#### 2) EvoScene-VLA: Evolving Scene Beliefs Inside the Action Decoder for Chunked Robot Control
- **标题**：EvoScene-VLA: Evolving Scene Beliefs Inside the Action Decoder for Chunked Robot Control
- **简称**：EvoScene-VLA
- **领域**：Chunked VLA / Scene Belief / Bimanual Manipulation
- **论文**：https://arxiv.org/abs/2605.21862
- **项目**：正文未披露
- **代码**：https://github.com/OpenGalaxea/GalaxeaVLA （正文可核验的唯一仓库入口）
- **评测**：仿真 benchmark：**RoboTwin**。正文核验到主评测覆盖 **31 个任务**，在 fixed / randomized 两种设置下，平均成功率从 **87.2% → 89.1%**、**86.1% → 88.5%**。真机：**Galaxea R1-Lite** 双臂平台，末端为双臂夹爪，输入视角为 **1 个 head + 2 个 wrist**，相机位姿数量 **3（wrist 2 / head 1 / third-person 0）**；真实任务聚焦 **Wipe the Mirror / Clean the Sink / Clean the Cutting Board** 三个长时序清洁任务，每任务 **100 次闭环 rollout**，平均成功率 **37.3% → 42.0%**。
- **算力**：正文核验到真实清洁实验在 **1× NVIDIA RTX 4090** 上执行；Galaxea 清洁子集含 **7 个录制 session、439 episodes、48,419 frames、1,756 clips，约 9 小时** 示教。正文/附录未统一披露完整训练 GPU 卡数与总时长。
- **解读**：这篇把“动作执行后场景会变”直接写进 action decoder，让 chunk 内后续动作不再盲信起始帧。亮点是对长时序表面状态变化特别有效；局限是当前真实验证任务面偏窄。复现建议优先做 recurrent scene prefix，再看去掉几何 anchor 后性能掉多少。

### B. 行为表征与视觉原语

#### 1) From Abstraction to Instantiation: Learning Behavioral Representation for Vision-Language-Action Model
- **标题**：From Abstraction to Instantiation: Learning Behavioral Representation for Vision-Language-Action Model
- **简称**：BehaviorVLA
- **领域**：VLA / Behavioral Representation / Sim2Real
- **论文**：https://arxiv.org/abs/2605.22671
- **项目**：https://mrok24.github.io/BehaviorVLA/
- **代码**：正文未披露
- **评测**：仿真 benchmark：**RoboTwin 2.0、LIBERO、CALVIN**。正文核验到主结果分别达到 **58%（RoboTwin 2.0）**、**98%（LIBERO）**、**4.36 Avg.Len（CALVIN）**。真机：**Galaxea R1 Lite**；正文与附录核验到机器人本体为 **23-DoF 双臂移动人形平台**，包含 **2×6-DoF arms + spherical wrists + parallel jaw grippers + 3-DoF torso + 6-DoF omnidirectional base**；视觉明确包含 **wrist-mounted 与 third-person cameras**，但正文未给精确机位总数，故相机位姿数量记为 **正文未披露（可确认含 wrist / third-person 视角）**。真实评测分 **Generalization Tasks** 与 **Long-horizon Tasks**：前者每任务 **100 条示教、50 次 rollout**，后者每任务 **150–200 条示教、50 次 rollout**；正文声明其用 **50% demonstration data** 即可匹配 OpenVLA-OFT 的 sim-to-real 水平。
- **算力**：正文/附录核验到两段训练：VBE 在 **A800 80G** 上训练，batch size **16**、**40 epochs**；整模微调使用 **8× A800 GPUs**、batch size **256**、训练 **30k steps**、learning rate **5e-5**。总墙钟时长正文未披露。
- **解读**：BehaviorVLA想解决的不是单步动作预测，而是把长时序“行为流形”先压成可检索表示，再按当前执行阶段解码。亮点是仿真和 sim2real 都吃到收益；局限是行为库质量很依赖预训练覆盖。复现建议先复做 VBE+PBD 双组件，再检查 behavior memory bank 的召回是否真比纯 action latent 稳。

#### 2) Action with Visual Primitives
- **标题**：Action with Visual Primitives
- **简称**：AVP
- **领域**：VLA / Visual Primitives / Spatial Compositional Generalization
- **论文**：https://arxiv.org/abs/2605.22183
- **项目**：https://kingdroper.github.io/AVP/
- **代码**：项目页标注 **Code (coming soon)**，当前无可核验公开仓库
- **评测**：仿真 benchmark：**正文未设置公开仿真 benchmark**。真机：**AgileX Piper tabletop** 双臂平台，机器人本体为 **2×6-DoF manipulators**，末端为 **parallel grippers**；正文确认使用 **multi-view observations** 并依赖一次性 hand-eye calibration，但未披露精确相机位姿数量，故记为 **正文未披露**。真实任务覆盖 **中国象棋操作、domino placement、通用 pick-and-place**；正文给出在相同条件下整体成功率较 **π0.5 提升 27.61%**，并做了 **未见棋盘转移 / 未见物体零样本泛化**。
- **算力**：正文/附录未披露 GPU 型号与训练时长；仅核验到训练数据规模为 **11.2 小时中国象棋 + 1.7 小时 domino + 1.7 小时通用抓放**，batch size **64**，中国象棋策略训练 **40k steps**，另两类任务各 **10k steps**。
- **解读**：AVP的核心不是再做一个 detector，而是让 VLM 先吐出点/框级 visual primitives，再交给动作专家执行。亮点是空间组合泛化明显；局限是两阶段推理和相机外参漂移都会伤稳定性。复现建议先选单任务做 primitive token 监督，再验证它是否真优于直接语言条件控制。

### C. 运行时安全与多模态操作评测

#### 1) Pre-VLA: Preemptive Runtime Verification for Reliable Vision-Language-Action and World-Model Rollouts
- **标题**：Pre-VLA: Preemptive Runtime Verification for Reliable Vision-Language-Action and World-Model Rollouts
- **简称**：Pre-VLA
- **领域**：VLA Safety / Runtime Verification / World Model Filtering
- **论文**：https://arxiv.org/abs/2605.22446
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**LIBERO-10 / LIBERO-Goal / LIBERO-Object / LIBERO-Spatial** 四套件；正文核验到对 **RynnVLA-002** 加入预验证后，四套件平均闭环成功率从 **30.79% 提升到 37.62%**，并减少任务执行步数。验证集上 Pre-VLA 的动作有效性判别达到 **F1=0.8303、Accuracy=0.9542、False Pass Rate=0.0200**。真机：**正文无真实机器人实验**；论文案例图仅显示 **main + wrist** 两路观测，相机位姿数量可确认 **2（wrist 1 / main 1）**，但仅用于仿真 rollout 展示。
- **算力**：正文/附录核验到训练使用 **single-node DDP**、per-GPU batch size **512**、训练 **10 epochs**，完整训练（含中间评估）约 **8 天 16 小时 43 分钟**；推理实验在 **16× NVIDIA RTX 5090 32GB** 上完成，VLA / Pre-VLA / world model 分卡部署；单 action chunk 平均前向验证耗时 **183.9 ms**。
- **解读**：Pre-VLA像给 VLA 外挂了一层“先审后放”的动作守门员，重点拦截会把 world model 和执行轨迹一起带偏的坏 chunk。亮点是收益直接体现在闭环成功率；局限是目前仍局限于 LIBERO 栈内验证。复现建议先单独复做 validity classifier，再调 resampling 阈值看收益/时延折中。

#### 2) TacO: Benchmarking Tactile Sensors for Object Manipulation
- **标题**：TacO: Benchmarking Tactile Sensors for Object Manipulation
- **简称**：TacO
- **领域**：Tactile Benchmark / Contact-Rich Manipulation / Imitation Learning
- **论文**：https://arxiv.org/abs/2605.21976
- **项目**：正文未披露专属项目页
- **代码**：正文未披露专属代码仓；仅可核验到实验控制库引用 `franky`
- **评测**：仿真 benchmark：**正文未采用仿真 benchmark**，主张全流程 **real-world open-source evaluation**。真机：两地两套 **Franka Panda 7-DOF** 平台。机构 A：Franka + **custom motorized gripper + finray fingers**，配 **Gello** 遥操作，测试 **FSR / FlexiTac / eGain / Daimon / eFlesh**；机构 B：Franka + **standard Franka gripper + finray fingers**，配 **Factr** 遥操作，测试 **contact microphone**。两套系统都带 **1 个 wrist RGB + 1 个 top-down RGB**，相机位姿数量 **2（wrist 1 / top-down 1）**。任务共 **3 类**：**未知质量抓放、遮挡接触几何的插头插入、连续力调制的物体重定向**；共比较 **6 种触觉传感器、4 种模态**。
- **算力**：正文/附录未披露 GPU 型号、卡数、训练时长与显存；仅核验到策略为 transformer-based ACT 风格行为克隆头，action chunk 长度 **64**，部署时以 **32 步 receding horizon** 执行。
- **解读**：TacO的价值不在于再造一个策略，而是第一次把“哪种触觉更适合哪类接触任务”放到同一真实平台里系统比较。亮点是跨机构可复现；局限是暂无高保真仿真对照。复现建议先按其双相机+统一 ACT 管线复刻，再单独替换传感模态做消融。

## Agent / OpenClaw 能力增强（固定栏目）

### 今日新增 / 值得关注
- **Pre-VLA** 很适合映射到 OpenClaw 控真实设备：任何会调用机械臂、电脑或外部执行器的 agent，都不该把“动作有效性判断”混在主策略里，最好单独挂一层预验证器。
- **SOMA / EvoScene-VLA / BehaviorVLA** 共同说明：如果 OpenClaw 未来要稳定控机器人，必须显式保留“记忆中的场景状态”，不能只看当前帧或当前对话上下文。
- **TacO** 提醒一个工程现实：多模态不等于一股脑上更多传感器，真正有用的是按任务匹配触觉模态与策略接口。

### 能力提升方法 / 工作流要点
1. **把“记忆”做成显式中间变量**：机器人侧是 scene belief，agent 侧可以是结构化工作状态；两者都比纯文本上下文更稳。
2. **把“动作前验证”单独模块化**：对 OpenClaw 来说，可迁移成 shell / browser / robot action 的风险过滤器，而不是全交给主模型临场判断。
3. **长时序行为先抽象再解码**：BehaviorVLA 的做法适合借鉴成 agent skill routing：先识别当前阶段，再选动作模板。
4. **真实设备评测必须写清配置**：TacO 这类工作说明，不写机器人本体、末端和机位数量，结论几乎不可复现。

### 机器人 / 设备控制能力相关进展
- **SOMA** 证明主动扫视 + 记忆检索对视野外抓取很关键，适合未来接头部相机或 PTZ 设备。
- **EvoScene-VLA** 对清洁、擦拭、装配这类“动作会持续改写场景”的任务尤其有启发。
- **Pre-VLA** 提供了 world model rollout 的节流思路：先过滤坏动作，再花大算力做前瞻模拟。
