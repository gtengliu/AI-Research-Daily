### A. VLA 表征增强与空间感知

#### 1) GuidedVLA: Specifying Task-Relevant Factors via Plug-and-Play Action Attention Specialization
- **标题**：GuidedVLA: Specifying Task-Relevant Factors via Plug-and-Play Action Attention Specialization
- **简称**：GuidedVLA
- **领域**：Vision-Language-Action / Action Attention / 空间几何感知
- **论文**：https://arxiv.org/abs/2605.12369
- **项目**：https://guidedvla.github.io/project_page/
- **代码**：https://github.com/GuidedVLA/GuidedVLA
- **评测**：仿真 benchmark：**LIBERO-Plus** 与 **RoboTwin 2.0**。正文核验到：GuidedVLA 在 RoboTwin 2.0 八个代表任务的 Agilex Piper 双臂设置上，平均成功率做到 **90.63%**。真机：两套双臂平台共 **6 个任务**。平台 A 为 **ALOHA AgileX 双臂移动操作平台**，末端为双臂夹爪，视觉为 **2 个腕部 Intel Orbbec Dabai 相机 + 1 个第三视角 Orbbec Dabai 相机**，相机位姿数量 **3（wrist 2 / head 0 / third-person 1）**；任务包括分拣果蔬、叠碗、清理桌面。平台 B 为 **PSI-Bot 双臂平台 + RealMan RM63 双臂 + DexHand2 Pro 灵巧手**，视觉为 **head/chest Intel RealSense D435 相机**，正文只明确 **2 个机身视角相机**，相机位姿数量 **2（wrist 0 / head/chest 2）**；任务包括加热套放杯、套杯、铁架加热杯等化学实验室操作。
- **算力**：正文/附录核验到三段训练资源：主训练配置使用 **8× NVIDIA H200**、global batch size **64**、action chunk size **50**；单头/消融模型为 **4× H200**、训练 **30k steps**；真机 6 任务约 **每任务 50 条示教** 的适配实验使用 **2× H200** 训练，并在 **1× RTX 4090** 上评测。总训练时长正文未统一披露。
- **解读**：这篇不是给 VLA 再堆数据，而是给 action decoder 的不同注意力头强行分工：物体 grounding、几何深度、阶段技能各司其职。亮点是仿真和双平台真机都涨点；局限是多依赖额外标注与辅助监督。复现建议先从 object/depth/skill 三类头最小实现切入，再核查跨平台迁移是否仍稳定。

#### 2) X-Imitator: Spatial-Aware Imitation Learning via Bidirectional Action-Pose Interaction
- **标题**：X-Imitator: Spatial-Aware Imitation Learning via Bidirectional Action-Pose Interaction
- **简称**：X-Imitator
- **领域**：Imitation Learning / Spatial Reasoning / Object Pose Interaction
- **论文**：https://arxiv.org/abs/2605.12162
- **项目**：正文未披露
- **代码**：论文注明 *code will be open sourced*，但 abs、PDF、正文超链接当前未给可访问仓库
- **评测**：仿真 benchmark：共 **24 个任务**，覆盖 **Adroit、DexArt、MetaWorld（hard / very hard）与 RoboTwin 2.0**；训练采样设置正文核验为 Adroit/MetaWorld **各任务 10 episodes**、RoboTwin 2.0 **各任务 50 episodes**、DexArt **各任务 100 episodes**。真机：**Flexiv Rizon 4 单臂** + **Dahuan AG-95 gripper**，用 **1 个前视 Intel RealSense D435 RGB-D 相机** 感知，位姿数量 **1（wrist 0 / head 0 / front-view 1）**；通过 **sigma 7** 遥操作采集每任务 **50 条示教**，执行 **Hang Mug、Pour Balls、Pick Place Truck** 三项任务。正文表 3 显示 X-RISE 在 **Hang Mug 最终挂杯成功率从 60% 提升到 80%**，Pour Balls 的入碗球数均值也从 **7.4±3.3** 提升到 **8.9±1.2**。
- **算力**：正文与附录未披露 GPU 型号、GPU 数量、训练时长与显存占用，仅给出部分训练超参：batch size **128**、learning rate **1e-4**、部分模型 horizon **48 / 16**。
- **解读**：X-Imitator 的核心是让动作预测反过来修正 pose 表征，而不是把姿态先验单向喂给策略。亮点是跨 DP3/ACT/RISE 都能接，真机三任务也有稳定增益；局限是训练算力和公开代码都还不透明。复现建议优先复做双向 action-pose 交互层，再看它是否真优于简单的显式姿态条件输入。

### B. 想象驱动动作生成与世界动作建模

#### 1) From Imagined Futures to Executable Actions: Mixture of Latent Actions for Robot Manipulation
- **标题**：From Imagined Futures to Executable Actions: Mixture of Latent Actions for Robot Manipulation
- **简称**：MoLA
- **领域**：World Action Model / Video Policy / Latent Action
- **论文**：https://arxiv.org/abs/2605.12167
- **项目**：https://logosroboticsgroup.github.io/MoLA/
- **代码**：https://github.com/LogosRoboticsGroup/MoLA
- **评测**：仿真 benchmark：**CALVIN、LIBERO、LIBERO-Plus**。正文核验到：LIBERO-Plus 含 **10,030 tasks**，MoLA 对强基线 **OpenVLA-OFT+ 平均成功率绝对领先 13.2%**；同时在 CALVIN 与四个 LIBERO task suites 上也给出系统对比。真机：**Universal Robots UR5e 单臂**，末端为机械夹爪，视觉为 **1 个第三视角 RealSense D455 + 1 个夹爪腕部 RealSense D405**，相机位姿数量 **2（wrist 1 / head 0 / third-person 1）**；共采集 **1,000 条示教轨迹** 覆盖 **5 个操作任务**，每任务评测 **50 trials**，单次 trial 最多 **20 attempts**。
- **算力**：正文/附录核验为三阶段训练：视频生成模型 **8× NVIDIA H100，5 天，500k steps**；混合逆动力学模型 **8× H100，16 小时，batch 256，20 epochs**；MoLA 微调 **8× H100，20 小时，batch 32，20 epochs**。推理延迟表在 **RTX 4090** 上测得，模块总时延正文给出分项统计。
- **解读**：MoLA 把“看未来视频”改成“从未来视频里反推可执行 latent action mixture”，比直接吃预测帧更像真正的世界动作模型。亮点是三大仿真基准和 UR5e 真机都有效；局限是仍受生成未来质量制约。复现建议先按论文三阶段训练拆开做，再重点测 one-step 视频去噪对控制收益是否足够。

#### 2) Premover: Fast Vision-Language-Action Control by Acting Before Instructions Are Complete
- **标题**：Premover: Fast Vision-Language-Action Control by Acting Before Instructions Are Complete
- **简称**：Premover
- **领域**：VLA Inference / Streaming Instruction / Low-Latency Control
- **论文**：https://arxiv.org/abs/2605.12160
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：基于 **π0.5**，在 **LIBERO（Spatial / Object / Goal / LIBERO10）** 与 **VLA-arena Level-1（Safety / Distractor / Extrapolation / Long-horizon，共 55 tasks）** 上测试。正文核验到：在 LIBERO 套件上，平均 wall-clock time 从 **34.0s 降到 29.4s**，降幅 **13.6%**，且成功率与 full-prompt baseline 基本持平（**95.1% vs 95.0%**）；naive premoving 则跌到 **66.4%**。真机：**正文无真实机器人实验**。论文仅在仿真中使用两路视角输入：**1 个固定 agent-mounted 视角 + 1 个 wrist-mounted 视角**，相机位姿数量 **2（wrist 1 / head/agent-mounted 1）**。
- **算力**：正文核验：全部实验在 **8× NVIDIA H200（141GB）** 上完成；每组 projection heads 训练使用 **1 张 GPU**，其余 GPU 并行 rollout 评测。总训练时长正文未披露。
- **解读**：Premover 抓住了 VLA 落地里一个常被忽略的浪费：人还没把指令说完，机器人已经可以先做无害预备动作。亮点是基本不动主干，仅加小头和 readiness gate；局限是暂未做真机闭环。复现建议优先保留 frozen backbone 设定，并仔细检查提前启动是否会引入安全边界问题。

### C. 接触丰富操作与精密装配

#### 1) SI-Diff: A Framework for Learning Search and High-Precision Insertion with a Force-Domain Diffusion Policy
- **标题**：SI-Diff: A Framework for Learning Search and High-Precision Insertion with a Force-Domain Diffusion Policy
- **简称**：SI-Diff
- **领域**：Force-Domain Diffusion Policy / Contact-Rich Assembly / Peg-in-Hole
- **论文**：https://arxiv.org/abs/2605.12247
- **项目**：https://yorklyb.github.io/SI-Diff/
- **代码**：项目页当前未给可核验代码仓链接
- **评测**：仿真 benchmark：正文未采用独立公开仿真 benchmark，主实验围绕 **search + high-precision insertion** 的统一装配设定，与 **TacDiffusion** 等基线比较。真机：**7-DoF Franka Emika 单臂**，末端/腕部为 **集成 wrist tactile / wrench sensing** 的装配执行器；视觉侧正文明确强调方法核心依赖力域观测，未给相机参与主实验配置，可记为相机位姿数量 **0（wrist 0 / head 0）**。训练数据用 **35 mm × 25 mm × 60 mm cuboid peg、0.1 mm clearance** 采集，search teacher policy 获取 **300 samples**，每次初始接触在孔边 **5 mm** 范围内随机化。正文主结论显示相对 SOTA **TacDiffusion**，可将 x-y misalignment tolerance 从 **2 mm 提升到 5 mm**，并具备对未见形状的 zero-shot transfer。
- **算力**：正文核验：训练与评测在 **1× 48GB NVIDIA RTX A6000** 上完成，训练 **2,000 epochs**、batch size **4,096**、diffusion steps **50**，优化器为 Adam，初始学习率 **1e-3**。
- **解读**：SI-Diff 真正有意思的地方是把“搜索”和“精插”两段动作模式塞进同一个力域 diffusion policy，不再靠人工切模式。亮点是 0.1mm 间隙、5mm 初始偏差下仍能统一处理；局限是任务族偏窄、公开代码未见。复现建议先严格复刻力/动作配对采样和 teacher 搜索策略，再谈跨零件泛化。

### D. 具身安全评测与可复用基准

#### 1) SafeManip: A Property-Driven Benchmark for Temporal Safety Evaluation in Robotic Manipulation
- **标题**：SafeManip: A Property-Driven Benchmark for Temporal Safety Evaluation in Robotic Manipulation
- **简称**：SafeManip
- **领域**：Embodied Safety Benchmark / Temporal Logic / Manipulation Evaluation
- **论文**：https://arxiv.org/abs/2605.12386
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：在 **RoboCasa** 中实例化，并覆盖 **RoboCasa365 的 50 个任务**，划分为 **7 个 manipulation task suites**；核心监测 **8 大类时序安全属性**，包括 collision/contact safety、grasp stability、release stability、cross-contamination、action-onset、mechanism recovery、containment、enclosure/access safety。正文比较 **6 个 VLA policy**（含 π0、π0.5、GR00T 及其训练变体），指出任务成功率提升并不可靠地带来更低违规率。真机：**正文无真实机器人实验**；安全谓词依赖 RoboCasa simulator state，机器人本体/末端/相机真实配置 **不适用**。
- **算力**：正文核验：所有实验运行在 **NVIDIA A40 48GB GPU nodes** 上，**每个任务分配 1 张 A40**。总训练/评测时长正文未统一披露。
- **解读**：SafeManip 的价值不是再造一个成功率榜单，而是把“晚放手、脏夹爪碰净区、柜门没全开就伸手”等真实事故模式写成可监控的时序逻辑。亮点是属性模板可复用；局限是当前仍依赖模拟器状态。复现建议优先把 LTLf 模板和 predicate 绑定链路跑通，再考虑迁到真实感知栈。

## Agent / OpenClaw 能力增强（固定栏目）

### 今日新增 / 值得关注
- **SafeManip** 对 agent 控真实设备很有参考价值：它说明“任务成功”不能替代“过程安全”，后续 OpenClaw 若做机器人/机械臂闭环控制，应该把时序安全监控层独立出来。
- **Premover** 提醒一个很实用的 agent 方向：在不改大模型主干的前提下，把“等待用户补完指令”的空窗期变成可控预计算或预动作阶段。
- **GuidedVLA / X-Imitator / MoLA** 共同指向一个趋势：OpenClaw 这类 agent 如果要接真实机器人，单靠语言规划不够，必须把对象定位、几何约束、阶段技能与未来动作表征显式建模。

### 能力提升方法 / 工作流要点
1. **先显式分解，再交给大模型统一**：GuidedVLA 的 attention specialization 说明，给动作头“分工”比一股脑端到端监督更稳定。
2. **把未来想象变成动作空间信号**：MoLA 比“看未来视频再猜动作”更接近可部署方案，值得 agent 工作流借鉴成“预测后先做动作可执行性检查”。
3. **安全监控别混在成功率里**：SafeManip 证明时序违规是独立维度，适合在 OpenClaw 控设备时外接 DFA / rule monitor。
4. **延迟预算要进主指标**：Premover 给出的 wall-clock 指标，比只看离线成功率更像真实系统评价方式。

### 机器人 / 设备控制能力相关进展
- **SI-Diff** 对真实装配 agent 很有启发：接触丰富任务不该拆成“找孔策略 + 插入策略”两套系统，统一力域策略更利于维护与部署。
- **X-Imitator** 说明姿态/空间先验不能只是单向提示；agent 在执行过程中也应允许动作预测反向修正感知假设。
- **GuidedVLA** 的跨平台实验说明，如果未来要做 OpenClaw 到多机器人本体的迁移，结构化视觉归因会比单纯微调更稳。

### 持续监控主题
- Plug-and-play VLA action head specialization
- World Action Model / latent action mixture / video policy
- Contact-rich insertion / force-domain diffusion
- Temporal safety monitoring for embodied agents
- Streaming instruction + low-latency VLA inference
