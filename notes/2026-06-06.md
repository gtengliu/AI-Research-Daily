### A. VLA、世界模型与预训练

#### 1) WorldFly: A World-Model-Based Vision-Language-Action Model for UAV Navigation
- **标题**：WorldFly: A World-Model-Based Vision-Language-Action Model for UAV Navigation
- **简称**：WorldFly
- **领域**：UAV VLA / world model / language-conditioned navigation
- **论文**：https://arxiv.org/abs/2606.06147
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**Urban Canyon Traversal Benchmark**，基于 OpenFly/AirSim 城市场景，正文写明共收集 **4000+ 导航轨迹**，并划分 **TEST-EASY 100 条** 与 **TEST-HARD 100 条** trajectory-instruction pairs；统一使用离散 OpenFly action primitives，单 episode 最多 **100 steps**。WorldFly 在 TEST-EASY 上 **NE/SR/SPL = 7.92m / 87% / 73.25%**，在 TEST-HARD 上 **31.08m / 31% / 27.86%**，相对 OpenFly 在 TEST-HARD 的 SR **16%→31%**。真机：**无真实 UAV 实验**；机器人本体/末端/相机位姿 **不适用**。
- **算力**：正文/附录披露：训练使用 **4× NVIDIA A100 GPU**、**40,000 steps**、**global batch size 128**；推理延迟在 **1× A100** 上测得约 **7.81s/step**，峰值显存约 **14.6GB**。
- **解读**：把未来视觉想象和动作生成做成双分支耦合，比“先世界模型后策略”更紧。亮点是复杂城市场景泛化明显更强；局限是推理频率仍只有约0.5Hz。复现建议先对齐动作离散表与轨迹构造，否则很难复现 TEST-HARD 的增益。

#### 2) DeMaVLA: A Vision-Language-Action Foundation Model for Generalizable Deformable Manipulation
- **标题**：DeMaVLA: A Vision-Language-Action Foundation Model for Generalizable Deformable Manipulation
- **简称**：DeMaVLA
- **领域**：VLA / deformable manipulation / household folding
- **论文**：https://arxiv.org/abs/2605.31286
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**RoboTwin**，覆盖 **50 个双臂操作任务** 的 clean / randomized 两种设定，平均成功率分别为 **88.42% / 86.78%**。真机：**ALOHA-style dual-arm robot**；末端执行器正文未单列型号；视觉输入在模型结构中明确为 **Head-Wrist Image Sequence**，对应 **头部视角 + 双腕视角，共 3 个机位**；真实 benchmark 包含 **shirt / skirt / pant / towel** 四类折叠任务，每任务 **20 次 real-world trials**、使用两组不同物品。多任务单 checkpoint 平均成功率 **92.5%**、平均完成时间 **2:18**，其中 towel 从 π0 的 **55.0%** 提升到 **100.0%**。
- **算力**：正文披露其预训练基于约 **5000 小时真实双臂演示**、多任务后训练数据约 **37 小时真机折叠数据**；但**训练 GPU 型号、卡数与总时长正文未披露**。
- **解读**：重点不只是“做折衣服”，而是证明大规模真实双臂预训练能给可形变任务带来跨品类先验。亮点是 towel 这类高歧义类别收益很大；局限是硬件与末端细节披露不足。复现建议优先保住 RTC 异步执行和 DAgger 纠错环。

#### 3) HARP-VLA: Human-Robot Aligned Representation Learning for Vision-Language-Action Model
- **标题**：HARP-VLA: Human-Robot Aligned Representation Learning for Vision-Language-Action Model
- **简称**：HARP-VLA
- **领域**：VLA / human-video transfer / aligned latent action
- **论文**：https://arxiv.org/abs/2605.31234
- **项目**：正文未披露
- **代码**：https://github.com/anonymity35/HARP-VLA
- **评测**：仿真 benchmark：**RLBench** 18 任务冻结视觉编码器评测，HARP-SRPD 平均成功率 **46.59%**；**CALVIN ABC→D** 上 HARP-VLA 的 **1/2/3/4/5-step = 99.8 / 96.7 / 91.3 / 84.4 / 75.9**，平均成功序列长度 **4.481**。真机：**xArm7**；末端为 **Robotera XHand**；输入为 **third-view + wrist camera，共 2 个机位**；真实操作表按 **4 个任务、每任务 60 trials** 统计，HARP-VLA 平均成功率 **76.3%**，较最强基线 OpenVLA-OFT 的 **60.9%** 提升 **15.4 个百分点**。
- **算力**：正文/附录详细披露了多阶段数据来源与训练流程，但**训练 GPU 型号、卡数与总时长正文未披露**。
- **解读**：这篇真正补的是“人类视频到机器人动作”的表示鸿沟，而不是单纯往 VLA 里混人类视频。亮点是对 CALVIN 和真机都有效；局限是仍依赖成对人机桥接数据。复现建议先检查 SRPD 对齐与 latent action 标签质量，再谈下游增益。

#### 4) Wall-OSS-0.5 Technical Report
- **标题**：Wall-OSS-0.5 Technical Report
- **简称**：Wall-OSS-0.5
- **领域**：open-source VLA / large-scale pretraining / zero-shot robot policy
- **论文**：https://arxiv.org/abs/2605.30877
- **项目**：https://x2robot.com/en/oss
- **代码**：https://github.com/X-Square-Robot/wall-x
- **评测**：仿真 benchmark：正文给出 **LIBERO** 对照实验用于验证 action-space loss，峰值平均成功率 **96.5%**，且在 **20k steps** 时已达 **95.8%**。真机：预训练零样本评测覆盖 **17 个 real-robot tasks**（12 seen + 5 unseen），**每任务 10 条轨迹**，400k checkpoint 的整体平均 task progress **51.1**，其中 **Rope Tightening 82**、**Fruit Sorting 96**、**Block Sorting 100**；下游微调评测覆盖 **15 个 real-robot tasks（10 manipulation + 5 reasoning）**，每任务约 **500 条 demonstration trajectories**。机器人本体与末端按平台而异，正文仅披露为 **tabletop bimanual systems + mobile manipulators**；推理输入为 **three-view image input，共 3 个机位**；微调后平均 task progress **60.5%**，领先 π0.5 的 **43.0%**。
- **算力**：正文未完整披露预训练 GPU 卡数与总时长；已披露部署推理在 **1× RTX 5090**、**三视角输入** 下可达约 **15Hz（448×448）/ 21Hz（224×224）**。
- **解读**：价值在于把“预训练权重”直接当可执行机器人策略来评估，而不是只看 fine-tune 后结果。亮点是零样本就能做出可观行为；局限是平台级硬件细节未完全公开。复现建议先守住动作 tokenizer 与 action-space supervision 的耦合。

### B. 人形运动与全身控制

#### 1) Accelerating and Scaling MPC-Guided Reinforcement Learning for Humanoid Locomotion and Manipulation
- **标题**：Accelerating and Scaling MPC-Guided Reinforcement Learning for Humanoid Locomotion and Manipulation
- **简称**：MPC-RL
- **领域**：humanoid RL / MPC-guided learning / loco-manipulation
- **论文**：https://arxiv.org/abs/2606.05687
- **项目**：正文未披露
- **代码**：https://github.com/junhengl/mpc-rl
- **评测**：仿真 benchmark：**无统一公共 benchmark**，正文评测集中在自建 humanoid velocity tracking、push recovery 与 box-pushing 场景；locomotion episode 最长 **20s / 1000 control steps**，采用 N=10、10Hz 的训练时 MPC 设置，策略训练 **15k iterations、6 小时内完成**。loco-manipulation 部分在随机质量箱体上训练，并展示对 **20kg、1m×1m×1m** 箱体的推进控制。真机：**Westwood Robotics Themis V2 humanoid**；末端执行器不适用；外部相机 **正文未披露/不作为主要观测**；硬件实验验证抗扰动与推车，文中写明可推动 **290kg（约体重 829%）** 载荷，所需推力约 **180N**。
- **算力**：正文披露训练/评测桌面环境为 **1× NVIDIA RTX 5090 32GB** + Intel Ultra 9 285K；未披露更大规模多卡训练。
- **解读**：这篇更像是在回答“训练时引入 MPC 到底值不值”，结论是值，但前提是把批量 GPU MPC 做到够便宜。亮点是真机推车很硬核；局限是缺少统一 benchmark 横比。复现建议先复现 tapered landmark reward，再调 horizon 和 update rate。

#### 2) SSR: Scaling Surefooted and Symmetric Humanoid Traversal to the Open World
- **标题**：SSR: Scaling Surefooted and Symmetric Humanoid Traversal to the Open World
- **简称**：SSR
- **领域**：humanoid traversal / perceptive locomotion / open-world deployment
- **论文**：https://arxiv.org/abs/2605.30770
- **项目**：https://ssr-humanoid.github.io/
- **代码**：正文未披露
- **评测**：仿真 benchmark：在 **Isaac Gym** 中使用 **4096 个 AgiBot X2 humanoids**，每种地形按 **1000 randomized trials** 评测 20s 穿越成功率。真机：主平台为 **AgiBot X2 humanoid**；末端执行器不适用；相机为 **腰部前向 Intel RealSense D435i，共 1 个机位**；机载 **Jetson AGX Orin** 以 **50Hz** 推理。实验按每种地形 **20 trials** 统计：标准设置下上/下楼梯均 **100%（20/20）**，**80cm gap 95%**、**40cm platform 100%**；更难的 **90cm gap 85%**、**45cm platform 95%**，并完成 **1.3km / 40min** 无重置开放环境穿越。
- **算力**：正文/附录披露：训练使用 **1× RTX 4090**、约 **20k iterations**；输入级镜像基线因显存需求单独使用 **48GB RTX 4090**。
- **解读**：SSR 的关键不是单纯更会走，而是把 foothold imagination、等变编码和风格约束绑成一个统一策略。亮点是真机跨超大 gap/高台很强；局限是仍依赖单前向深度视角。复现建议先保证部署一致的深度渲染与镜像结构，否则 sim2real 会明显掉。

### C. 操作系统、工具使用与结构化任务执行

#### 1) GSAM: A Generalizable and Safe Robotic Framework for Articulated Object Manipulation
- **标题**：GSAM: A Generalizable and Safe Robotic Framework for Articulated Object Manipulation
- **简称**：GSAM
- **领域**：articulated object manipulation / safety / VLM-guided planning
- **论文**：https://arxiv.org/abs/2605.30740
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**无独立仿真 benchmark**。真机：**Ranger Mini 3.0 mobile base + Realman 63-F robotic arm**；末端执行器正文未单列型号；相机为 **腕部 Intel RealSense D435i，共 1 个机位**；共进行 **50 次 real-robot trials**，由 **5 类未见 articulated objects × 5 instances × 2 base placements** 组成。以铰链门/抽屉等操作为目标，GSAM 的 **Overall Success Rate 88.0%**，显著高于最优基线 **52.0%**；平均运行时间 **42.1±5.2s**。
- **算力**：正文披露系统端部署使用 **1× NVIDIA RTX 3090 GPU**；训练侧总卡数与时长 **正文未披露**。
- **解读**：它的强点是把感知、常识推理和约束生成串成一个安全闭环，而不是只做单步把手定位。亮点是 OSR 与安全性一起上来；局限是速度仍慢于端到端 imitation。复现建议把坐标变换和约束库先打牢，否则安全优势会很快消失。

#### 2) Any-ttach: Quick End-effector Swapping Enables Manipulation Dexterity with Simplicity
- **标题**：Any-ttach: Quick End-effector Swapping Enables Manipulation Dexterity with Simplicity
- **简称**：Any-ttach
- **领域**：tool use / end-effector swapping / demonstration transfer
- **论文**：https://arxiv.org/abs/2605.30569
- **项目**：https://any-ttach.github.io/
- **代码**：正文未披露
- **评测**：仿真 benchmark：**无仿真 benchmark**。真机：**Franka Research 3 (FR3) 7-DoF**；末端为 **1-DoF parallel gripper + Any-ttach 快换接口/工具坞**；策略观测使用 **workspace RGB image，共 1 个机位**。系统覆盖 **15 个工具**，完整任务包括 Sandwich Assembly 与 Cucumber Preparation。换工具实验中，自主 Any-ttach 的 swapping success 从抓取式换装的 **55.6% 提升到 87.5%**，成功换装时间 **17.00±3.61s**。技能策略统一用 Diffusion Policy、**每个技能 30 demonstrations 训练、15 trials/skill 评测**；三明治相关技能平均成功率从 **44.4% 提升到 71.1%**，并额外打通抓取式基线无法稳定完成的 **peel / cut / spear**。
- **算力**：正文未披露训练 GPU 型号、卡数与总时长，记为 **正文未披露**。
- **解读**：这篇很实用，它用快换末端把“手太复杂”这个问题改写成“接口够不够稳定”。亮点是既提升换装可靠性，也让示教和部署的工具坐标统一；局限是当前仍依赖分技能架构。复现建议优先保证 docking 几何重复精度，再谈策略泛化。
