### A. 具身操作模型与数据体系

#### 1) Learning Versatile Humanoid Manipulation with Touch Dreaming
- **标题**：Learning Versatile Humanoid Manipulation with Touch Dreaming
- **简称**：HTD
- **领域**：人形机器人操作 / 触觉增强模仿学习 / whole-body manipulation
- **论文**：https://arxiv.org/abs/2604.13015
- **项目**：https://humanoid-touch-dream.github.io/
- **代码**：正文未披露
- **评测**：仿真侧先评估其 lower-body controller，对比 **AMO / FALCON**，在 **4096 并行环境、500 timesteps** 下统计速度、躯干高度与姿态跟踪误差。真机侧在 **5 个 contact-rich 任务** 上评估：**Insert-T、Book Organization、Towel Folding、Cat Litter Scooping、Tea Serving**。平台为 **人形机器人本体 + Dex-Hand 全灵巧手**，感知配置可从 Fig.3 核到 **1 个双目头部相机 + 2 个腕部相机（共 3 个相机位姿）**，并带 **手部关节力反馈与分布式触觉传感**；机器人本体具体型号正文未披露。
- **算力**：正文/附录未给出统一 GPU 型号、卡数与训练时长；仅明确 lower-body teacher 在仿真中以 **PPO** 训练、再用 **DAgger** 蒸馏到 student。整体算力 **正文未披露**。
- **解读**：核心贡献是把多视角视觉、手部力觉与触觉潜变量统一进单阶段 Transformer，并用“touch dreaming”预测未来接触信号来稳住接触表征。亮点是真机五类复杂接触任务显著增益；局限是硬件型号与训练算力披露不完整。复现建议先拆开复验 WBC 与触觉 latent 监督两部分增益。

#### 2) XRZero-G0: Pushing the Frontier of Dexterous Robotic Manipulation with Interfaces, Quality and Ratios
- **标题**：XRZero-G0: Pushing the Frontier of Dexterous Robotic Manipulation with Interfaces, Quality and Ratios
- **简称**：XRZero-G0
- **领域**：具身数据采集 / robot-free demonstration / dexterous manipulation
- **论文**：https://arxiv.org/abs/2604.13001
- **项目**：正文未单列项目主页
- **代码**：https://github.com/X-Square-Robot/XRZero-G0
- **评测**：论文主线不在标准仿真 benchmark，而在 **数据采集接口、数据质量闭环与数据混比**。作者构建 **2000 小时 robot-free 数据集**，并比较 **仅真机数据、仅 robot-free 数据、协同训练、少量真机 + 大规模 robot-free（如 10:1）** 等训练范式；正文称闭环流程使 **data validity rate 达到 85%**，并宣称 **10:1** 的混比可取得接近全真机训练的性能、采集成本降到原来的二十分之一。真机部署存在，但本体型号、末端型号、评测任务列表、相机位姿数量与 trial 数在正文披露不够系统；仅可明确其采集端为 **VR 头显 + 顶视相机 + 双专用夹具**，目标是跨 embodiment 零样本迁移。
- **算力**：正文未系统披露模型训练 GPU/时长；数据规模披露为 **2000 小时 robot-free demonstrations**。统一算力信息 **正文未披露**。
- **解读**：价值在于把“采集—质检—训练—自动评测”做成闭环，并首次系统讨论 robot-free 与真机数据的混比规律。问题是 benchmark 与目标机器人细节披露偏弱，难直接横向比较。复现时建议优先验证其 85% 数据有效率与 10:1 mixing ratio 是否稳定复现。

### B. 几何表征驱动的 VLA / 安全评测

#### 3) Robotic Manipulation is Vision-to-Geometry Mapping ($f(v) \rightarrow G$): Vision-Geometry Backbones over Language and Video Models
- **标题**：Robotic Manipulation is Vision-to-Geometry Mapping ($f(v) \rightarrow G$): Vision-Geometry Backbones over Language and Video Models
- **简称**：VGA
- **领域**：VLA / 几何表征学习 / 机器人操作
- **论文**：https://arxiv.org/abs/2604.12908
- **项目**：https://hcplabsysu.github.io/VisionGeometryAction/
- **代码**：正文未披露（项目页当前返回 404）
- **评测**：仿真 benchmark 为 **LIBERO**，正文明确比较 **SpatialVLA、π0.5、GeoVLA、OpenVLA-OFT** 等基线。真机平台为 **Franka Panda**，相机为 **3× RealSense D415：1 个腕部相机 + 2 个固定相机**，其中固定相机分别用于 **ID / OOD** 视角评测；末端执行器型号正文未披露。真实任务含 **Pick Cube、Press Button、Stack Cube** 三项，表 3 给出 ID/OOD 成功率：VGA 平均 **ID 75% / OOD 58%**，均优于 ACT 与 OpenVLA，且 OOD 视角下明显更稳。
- **算力**：正文披露采用 **LoRA** 微调，约 **500M 可训练参数**；全部实验在 **1× NVIDIA A100-SXM4-80GB** 上完成，最长训练约 **60 GPU hours**。附录另披露推理延迟约 **100 ms（约 10 Hz）**。
- **解读**：这篇最值得看的点，是把 VLA 的“语言/视频先验”换成原生 3D 几何 backbone，直接把动作学习约束到可解释空间里。优势是跨视角泛化更强；局限是项目页尚未稳定、末端与数据细节披露不够全。复现建议先重跑 LIBERO，再重点测 OOD 相机位姿泛化。

#### 4) HazardArena: Evaluating Semantic Safety in Vision-Language-Action Models
- **标题**：HazardArena: Evaluating Semantic Safety in Vision-Language-Action Models
- **简称**：HazardArena
- **领域**：VLA 安全评测 / embodied benchmark / semantic safety
- **论文**：https://arxiv.org/abs/2604.12447
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：这是一个 **安全语义 benchmark** 而非真机论文。作者构建 **2000+ assets、40 个 risk-sensitive tasks、7 类真实世界风险语义**，通过 **safe/unsafe twins** 控制几何与动作模板一致、只改变语义风险上下文。训练侧统一用 **600 条 SAFE-only demonstrations（6 个 safe tasks × 100 条）** 微调 **OpenVLA-OFT、VLA-Adapter、NORA、π0**；评测不仅看 success，还看 **attempt / commit / success** 的 stage-wise 指标，用来分离“不会做”与“明知危险还去做”。**无真机评测**，因此机器人本体/末端/相机位姿数量 **不适用**。
- **算力**：附录表 4 明确披露：**OpenVLA-OFT 2×A100、VLA-Adapter 2×A100、NORA 1×A100、π0 4×A800**；训练步数分别为 **10k / 30k / 20k / 20k**。若指单一 benchmark 本身训练算力，则需按被测模型分别看。
- **解读**：贡献不在“更安全的模型”，而在指出现有 VLA 常把“动作成功”误当“语义安全”，并用 twin-task 诊断这个错觉。局限是仍属仿真资产层评测，离真实物理风险还有距离。复现时建议先按其 SAFE-only protocol 重训小模型，再核查 commit-rate 与 unsafe success 的背离。

### C. 强化学习与移动操作

#### 5) FastGrasp: Learning-based Whole-body Control method for Fast Dexterous Grasping with Mobile Manipulators
- **标题**：FastGrasp: Learning-based Whole-body Control method for Fast Dexterous Grasping with Mobile Manipulators
- **简称**：FastGrasp
- **领域**：移动操作 / 灵巧抓取 / sim-to-real / tactile RL
- **论文**：https://arxiv.org/abs/2604.12879
- **项目**：https://taoheng-star.github.io/fastgrasp-page/
- **代码**：https://github.com/taoheng-star/FastGrasp
- **评测**：仿真侧在 **Isaac Sim** 做快速抓取评测，使用 **96 个未见物体（easy 60 / hard 36）**，每个物体 **10×500 grasp trials**，指标为 **Success Rate** 与 **Hand-Object Offset Distance**；作者同时报告 full / partial point cloud 情况。真机侧平台为 **Agilex Bunker Mini 移动底盘 + 6-DoF Dobot CR5 + 16-DoF LeapHand**，末端即 **LeapHand**；视觉为 **1 个安装在末端的 RealSense D435i（1 个相机位姿）**，并在每根手指和手掌布置 **9 个薄膜压力传感器**。真实评测使用 **16 个物体（easy 10 / hard 6）**，每个物体 **10 次连续试验**；作者在高速与低速约束下报告平均成功率约 **32.0% / 34.62%**，消融显示去掉 tactile adaptation、低通滤波或 domain randomization 会明显退化。
- **算力**：正文披露策略训练在 **Intel i9-14900K + NVIDIA GeForce RTX 4090** 上完成，使用 **48 并行环境**；仿真频率 **60 Hz**，策略推理频率 **15 Hz**；真机部署保持 **15 Hz**。其余训练时长正文未披露。
- **解读**：论文把“移动底盘—机械臂—灵巧手—触觉”真正联成一套高速抓取闭环，优点是 sim-to-real 路线清晰、硬件披露也相对完整。短板是真机成功率还不高，说明高速碰撞与滑移仍难。复现建议先在低速约束下重现，再逐步打开速度和 tactile ablation。

#### 6) Whole-Body Mobile Manipulation using Offline Reinforcement Learning on Sub-optimal Controllers
- **标题**：Whole-Body Mobile Manipulation using Offline Reinforcement Learning on Sub-optimal Controllers
- **简称**：WHOLE-MoMa
- **领域**：移动操作 / offline RL / whole-body control
- **论文**：https://arxiv.org/abs/2604.12509
- **项目**：http://sites.google.com/view/whole-moma
- **代码**：正文未披露
- **评测**：仿真侧基于 **TIAGo++ 移动操作平台** 在 Isaac Sim 设计 **Door / Drawer / Cupboard** 三项任务；每任务先由轻量 WBC 生成 **3k trajectories**，最终用 **50 episodes/task** 评估。WHOLE-MoMa 在表 3 中分别达到 **Door 98%、Drawer 80%、Cupboard 78%、平均 100?（文中按分项报告）**，整体优于 WBC、BC、TD3、IDQL、RISE 等基线。真机侧仍是 **TIAGo++ holonomic mobile manipulator**，评测 **Drawer** 与 **Cupboard** 两项，各 **25 trials**；摘要总结为 **bimanual drawer manipulation 80%**、**simultaneous cupboard opening and object placement 68%**。正文未给出统一末端执行器型号和相机位姿数量，故这两项写 **正文未披露**。
- **算力**：论文仅说明训练使用 **FAU/NHR 高性能计算资源**，未披露 GPU 型号、卡数和总时长；统一算力信息 **正文未披露**。
- **解读**：关键 insight 是把“次优 WBC”当成结构先验采样器，再用 offline RL 把碎片化好行为 stitch 成更强策略，避免全程靠昂贵遥操作。亮点是真机零微调直接过渡；局限是视觉配置与算力披露不足。复现建议优先重现 3k-trajectory 数据生成和 Q-chunking 收益。

### D. 人形连续学习

#### 7) Tree Learning: A Multi-Skill Continual Learning Framework for Humanoid Robots
- **标题**：Tree Learning: A Multi-Skill Continual Learning Framework for Humanoid Robots
- **简称**：Tree Learning
- **领域**：人形机器人 / continual RL / multi-skill locomotion
- **论文**：https://arxiv.org/abs/2604.12909
- **项目**：https://yyf-prog.github.io/Tree-learning/
- **代码**：正文未披露
- **评测**：主评测全部在 **Unity 仿真** 完成，机器人为 **Unitree G1 humanoid**。作者从平地行走根技能逐层扩展，覆盖 **11 类动态技能**，包括单腿站立、爬楼、俯卧、跑步、下蹲、踢球、双脚跳、俯卧撑、爬行、起身等，并在 **Super Mario 风格交互场景** 与 **中式园林自主导航场景** 中验证多技能切换与泛化。**无真机实验**，因此机器人本体以外的末端执行器、相机位姿数量 **不适用**。
- **算力**：正文未披露 GPU、训练时长与并行环境规模；统一算力 **正文未披露**。
- **解读**：它的价值不在单一 locomotion 指标，而在用树状参数继承把“增技能”做成低成本持续学习，且宣称 skill retention 达到 100%。局限是验证仍停留在 Unity，没有真实人形平台闭环。复现建议先从根技能到一层分支复现，再看参数继承是否真能避免遗忘。
