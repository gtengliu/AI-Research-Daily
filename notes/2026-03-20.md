## 主清单（按技术重点分类）

### A. 实时 VLA 推理与反应能力

#### 1) FASTER: Rethinking Real-Time Flow VLAs
- **标题**：FASTER: Rethinking Real-Time Flow VLAs
- **简称**：FASTER
- **领域**：实时 VLA / Flow 推理系统 / 闭环控制
- **论文**：https://arxiv.org/abs/2603.19199
- **项目**：https://innovator-zero.github.io/FASTER
- **代码**：https://github.com/innovator-zero/FASTER
- **评测**：仿真 benchmark 为 **LIBERO** 与 **CALVIN ABC→D**；真机为 **AgileX Cobot Magic（Mobile Aloha 风格）+ 4×6-DoF Piper（2 主臂遥操 + 2 从臂执行）**，末端为并联夹爪；相机位姿为 **3 机位**（1 前置 RealSense D455 + 2 腕部 RealSense D435）。
- **算力**：正文披露模型微调在 **8×NVIDIA A800** 完成；部署对比在 **RTX 4090 / RTX 4060** 上进行。
- **解读**：方法把 action chunking、异步执行和流式客户端重排成低时延推理链路，在消费级 GPU 上显著缩短 TTFA。贡献是把“能跑”推进到“更快反应”；局限是主要收益聚焦系统层。复现建议先复刻其三相时延分解，再做任务级误差-时延权衡曲线。

#### 2) Action Draft and Verify: A Self-Verifying Framework for Vision-Language-Action Model
- **标题**：Action Draft and Verify: A Self-Verifying Framework for Vision-Language-Action Model
- **简称**：ADV
- **领域**：VLA 自校验 / 动作候选重排序 / 鲁棒执行
- **论文**：https://arxiv.org/abs/2603.18091
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark 为 **LIBERO** 与 **RoboTwin2.0（Easy/Hard）**；真机任务为未见组合场景（含 pick-place / table-cleaning / hang-cups 等），平台披露为 **xArm 机械臂 + DaHuan 夹爪**；相机位姿数量正文未披露。
- **算力**：正文未给统一 GPU 训练口径（附录仅给训练数据规模与配置表条目）。
- **解读**：ADV 先草拟多条动作轨迹，再由 VLM 验证器打分筛选，缓解单路径解码失误。贡献是在 LIBERO、RoboTwin 与真机都提升成功率；局限是硬件与算力披露不完整。复现建议固定同等 action chunk 与采样预算，单独比较“候选数量-收益”曲线。

### B. 记忆与不确定性增强的 VLA 稳健性

#### 1) MemoAct: Atkinson-Shiffrin-Inspired Memory-Augmented Visuomotor Policy for Robotic Manipulation
- **标题**：MemoAct: Atkinson-Shiffrin-Inspired Memory-Augmented Visuomotor Policy for Robotic Manipulation
- **简称**：MemoAct
- **领域**：记忆增强策略 / 长时程操作 / 视觉运动控制
- **论文**：https://arxiv.org/abs/2603.18494
- **项目**：https://tlf-tlf.github.io/MemoActPage/
- **代码**：正文未披露
- **评测**：仿真 benchmark 为 **MemoryRTBench（基于 RoboTwin2.0）**，并在 **RMBench** 语言无关任务上对比；真机为 **Realman RM65-B + TEK CTAG2F90C 夹爪**，相机位姿 **2 机位**（全局 RealSense D455 + 腕部 RealSense D435）。
- **算力**：正文未披露统一训练算力。
- **解读**：该工作把感官记忆、短时记忆与长期记忆显式拆层，并以门控检索调节扩散动作解码，强化长时依赖。贡献是记忆相关任务优势稳定；局限是算力与跨平台泛化披露有限。复现建议先严格复现 memory bank 更新策略，再做序列长度外推实验。

#### 2) Shifting Uncertainty to Critical Moments: Towards Reliable Uncertainty Quantification for VLA Model
- **标题**：Shifting Uncertainty to Critical Moments: Towards Reliable Uncertainty Quantification for VLA Model
- **简称**：Critical-UQ for VLA
- **领域**：VLA 不确定性估计 / 安全监测 / 失败预警
- **论文**：https://arxiv.org/abs/2603.18342
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark 为 **LIBERO-SPATIAL / OBJECT / GOAL / LIBERO-10**，指标以 rollout-level AUROC 为主；真机设备配置：**本文未开展真机实验（正文未给机器人本体、末端与相机位姿）**。
- **算力**：正文披露 DoF 自适应校准在 **单张 RTX 4090** 上 <5 分钟完成；其余训练算力未系统列出。
- **解读**：方法将不确定性从“全程平均”转向“关键时刻聚焦”，并用 DoF 自适应权重突出高风险轴。贡献是跨 suite 的失败预测显著提升；局限是暂缺真机闭环验证。复现建议先复做 cross-suite transfer，再接入人机接管阈值实验。

### C. 多模态（视觉-触觉）与可解释表征

#### 1) OmniVTA: Visuo-Tactile World Modeling for Contact-Rich Robotic Manipulation
- **标题**：OmniVTA: Visuo-Tactile World Modeling for Contact-Rich Robotic Manipulation
- **简称**：OmniVTA
- **领域**：视觉-触觉世界模型 / 接触密集操作 / 多模态策略
- **论文**：https://arxiv.org/abs/2603.19201
- **项目**：https://mrsecant.github.io/OmniVTA
- **代码**：正文称将公开（当前未给可核验仓库链接）
- **评测**：仿真/离线侧围绕 **OmniViTac benchmark（21,879 轨迹、86 任务、100+ 物体）**；真机为 **UFactory xArm7 + 并联双指夹爪 + 指尖触觉传感器**，相机位姿至少 **2 机位**（腕部 D435 + 第三视角 D435/L515；TacUMI 另有广角视角）。
- **算力**：正文披露 TactileVAE 训练使用 **8×NVIDIA A100（50 epochs）**；并报告组件在 **RTX 4090D** 的推理时间。
- **解读**：OmniVTA 用触觉世界模型预测未来触觉动态，再通过自适应融合驱动策略，缓解视觉遮挡与接触突变。贡献是跨任务接触操作鲁棒性提升；局限是完整开源尚未落地。复现建议先搭建触觉时间同步链路，再逐步加入 LTD 与 RLTC 模块消融。

#### 2) Sparse Autoencoders Reveal Interpretable and Steerable Features in VLA Models
- **标题**：Sparse Autoencoders Reveal Interpretable and Steerable Features in VLA Models
- **简称**：SAE4VLA
- **领域**：VLA 机理可解释性 / 表征分析 / 特征引导
- **论文**：https://arxiv.org/abs/2603.19183
- **项目**：http://drvla.github.io
- **代码**：正文称已提供开源代码基座（以项目页为入口）
- **评测**：仿真 benchmark 为 **LIBERO** 闭环 steering；并使用 **DROID（Franka Panda 实采数据）**做跨数据集表征分析。真机设备配置：**本文未新增独立真机实验平台（机器人本体/末端/相机位姿以所用公开数据口径为准）**。
- **算力**：正文未披露统一 GPU 算力配置。
- **解读**：论文以 SAE 分解 VLA 残差流，得到可解释且可操控的稀疏特征，并验证特征 steering 的因果效应。贡献是把“可解释分析”推进到“可干预控制”；局限是真机闭环证据仍弱。复现建议先复刻 activation 采集与 SAE 训练，再做特征级鲁棒性压力测试。

#### 3) V-Dreamer: Automating Robotic Simulation and Trajectory Synthesis via Video Generation Priors
- **标题**：V-Dreamer: Automating Robotic Simulation and Trajectory Synthesis via Video Generation Priors
- **简称**：V-Dreamer
- **领域**：生成式仿真 / 轨迹合成 / sim2real
- **论文**：https://arxiv.org/abs/2603.18811
- **项目**：https://jia-handsome.github.io/v-Dreamer/
- **代码**：正文未披露
- **评测**：仿真侧基于 7-DoF **Franka** 环境评估生成数据驱动策略；真机部署在 **CobotMagic 左臂（6-DoF Piper）+ 并联夹爪**，相机位姿 **2 机位**（腕部 Orbbec DaBai + 俯视 Orbbec DaBai）。
- **算力**：正文披露合成数据生成在 **8×RTX 4090 工作站**进行（约 500 条轨迹/小时）。
- **解读**：V-Dreamer 将“文本/图像→可物理执行场景+轨迹”自动化，减少手工搭仿真成本并提升零样本迁移。贡献在生成-训练-部署链路打通；局限是代码尚未公开且任务类型仍偏抓取放置。复现建议先做单任务 real2sim 对齐，再扩展到多物体长序列。
