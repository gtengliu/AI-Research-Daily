## 主清单（单一合并版，按技术重点分类）

### A. VLA、机器人操控与部署

#### 1) Learning Human-Intention Priors from Large-Scale Human Demonstrations for Robotic Manipulation
- **标题**：Learning Human-Intention Priors from Large-Scale Human Demonstrations for Robotic Manipulation
- **简称**：MoT-HRA
- **领域**：VLA / 人类先验迁移 / 机器人操控
- **论文**：https://arxiv.org/abs/2604.24681
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：SimplerEnv-WidowX，文中给出 Spoon / Carrot / Stack / Eggplant 四类任务，并明确引入光照、背景纹理与 camera viewpoint 扰动。真机：真实机器人操控任务，正文摘要明确有 real-world manipulation，但在已公开正文片段中未完整给出机器人本体、末端类型与相机位姿数量；可确认其训练同时引入 AgiBot-World 机器人动作数据。设备配置：正文对真机硬件披露不完整。
- **算力**：正文披露训练使用 64× NVIDIA H20，PyTorch FSDP2；每卡 32 samples，全局 batch size 2048，训练 20,000 steps，warmup 1,000 steps。推理延迟/FPS/显存账单正文未系统披露。
- **解读**：这篇把“人类意图”从原始视频里拆出来，先学 3D 轨迹与 MANO 手部运动，再映射到机器人动作，核心是降低 human-to-robot embodiment gap。亮点是大规模 HA-2.2M 人类示范先验；局限是真机硬件细节公开不足，复现时建议先按论文两阶段专家结构复刻，再补真实平台适配。

#### 2) CF-VLA: Efficient Coarse-to-Fine Action Generation for Vision-Language-Action Policies
- **标题**：CF-VLA: Efficient Coarse-to-Fine Action Generation for Vision-Language-Action Policies
- **简称**：CF-VLA
- **领域**：VLA / 动作生成 / 低 NFE 推理
- **论文**：https://arxiv.org/abs/2604.24622
- **项目**：正文未披露
- **代码**：https://github.com/EmbodiedAI-RoboTron/CF-VLA
- **评测**：仿真 benchmark：LIBERO（四个 Franka suites：Spatial / Object / Goal / Long）与 CALVIN 长时程 D→D。真机：5 个代表性任务——抓取、放入盒中、擦桌、倒水、双臂折毛巾。设备配置：正文确认观测字段含 image、wrist_image、state 与 language prompt，说明至少包含 1 路主视角 + 1 路 wrist 相机；双臂折叠任务存在双臂执行，但机器人本体品牌、末端型号与精确相机安装位姿数量正文未完整披露。
- **算力**：正文披露训练使用 8× NVIDIA A100，PyTorch FSDP，2 samples/GPU，全局 batch size 16，学习率 5e-5；LIBERO 统一训练 checkpoint 为 60,000 steps。正文强调 NFE=2 相比 π0.5 的 NFE=10 仍有竞争力，但未统一给显存/FPS总表。
- **解读**：CF-VLA不再从纯高斯噪声硬采动作，而是先做 coarse initialization，再用单步 refinement 修残差，特别适合低 NFE 实时控制。贡献在于把效率和质量前沿同时往前推；局限是硬件与相机布置细节不够全，复现建议先锁定 LIBERO/CALVIN 训练设置，再做 NFE=2 真机迁移。

#### 3) Characterizing Vision-Language-Action Models across XPUs: Constraints and Acceleration for On-Robot Deployment
- **标题**：Characterizing Vision-Language-Action Models across XPUs: Constraints and Acceleration for On-Robot Deployment
- **简称**：VLA-XPU Characterization
- **领域**：VLA 部署 / 异构加速 / 机器人系统
- **论文**：https://arxiv.org/abs/2604.24447
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：LIBERO 四个子套件用于能力分数；并在 OpenVLA/π0 等模型上比较不同硬件平台的控制频率、能耗与成本。真机：Franka 上部署 π0.5-droid，任务为 table-cleaning；正文给出 50 trials 的真实机器人结果。设备配置：机器人本体为 Franka 机械臂，末端与真机相机位姿数量正文未完整披露。硬件平台覆盖 CPU（i7-11700）、GPU（RTX 4090、Jetson Thor、AGX Orin）、NPU（Ascend 310B/310P）与 XPU（Intel B60 Pro）。
- **算力**：本文更偏部署测评而非训练论文。正文/附录披露异构推理硬件规格：RTX 4090 24GB、AGX Orin 64GB、Jetson Thor 128GB、Intel B60 Pro 48GB、Ascend 310B 12GB、Ascend 310P 48GB；并报告 Franka 实机上 DP-Cache 获得 1.28× speedup。统一训练资源正文未披露。
- **解读**：这篇最有价值的是把 VLA 部署问题从“能不能跑”提升成 CET（Cost-Energy-Time）三维选择，并进一步针对 VLM backbone / Action Expert 两阶段瓶颈做加速。亮点是硬件-模型共表征；局限是多数真机末端与视觉链路披露较粗，复现建议先按文中 leaderboards 重建跨平台 profile，再移植优化到自家机器人。

### B. 强化学习基准、Agent 技能与科研助手

#### 1) SpecRLBench: A Benchmark for Generalization in Specification-Guided Reinforcement Learning
- **标题**：SpecRLBench: A Benchmark for Generalization in Specification-Guided Reinforcement Learning
- **简称**：SpecRLBench
- **领域**：强化学习 / 形式化规范 / Benchmark
- **论文**：https://arxiv.org/abs/2604.24729
- **项目**：正文未披露
- **代码**：https://github.com/BU-DEPEND-Lab/SpecRLBench
- **评测**：仿真 benchmark：SpecRLBench 本身，覆盖 19 个环境变体，含 navigation 与 manipulation、单/多智能体、静态/动态环境、不同机器人动力学与观测模态。真机：不适用。设备配置：操控类环境支持 onboard state 或 forward-facing camera 图像输入；非真机论文，无固定机器人本体/末端。
- **算力**：正文重点披露 benchmark 结构、IND/OOD specification 评测协议与比较方法，未系统公开 GPU 型号、卡数、训练总时长或统一 batch 账单，故记为“正文未披露”。
- **解读**：SpecRLBench的意义在于不再只测“会不会解一个规格”，而是专门测 specification-guided RL 对未见公式、动态环境和复杂时序约束的泛化。亮点是把 manipulation 也纳入 LTL 评测；局限是偏仿真、缺少真机闭环，复现时建议先跑 IND/OOD 分拆基线，再加自定义约束生成器。

#### 2) Skill Retrieval Augmentation for Agentic AI
- **标题**：Skill Retrieval Augmentation for Agentic AI
- **简称**：SRA
- **领域**：Agent / Skill Retrieval / 工具调用
- **论文**：https://arxiv.org/abs/2604.24594
- **项目**：https://huggingface.co/datasets/WeihangSu/SRA-Bench
- **代码**：https://github.com/oneal2000/SR-Agents
- **评测**：仿真 benchmark：SRA-Bench，覆盖 skill retrieval、skill incorporation 与 end-task execution 三段评测，包含 5,400 个 capability-intensive 实例、636 个人工构建 gold skills，并混入 web-collected distractor skills。真机：不适用。设备配置：非机器人论文，无机器人本体/末端/相机位姿。
- **算力**：正文主要披露 benchmark 设计、skill corpus 构造与多模型对比，未系统披露训练/推理 GPU、卡数、时长与显存，记为“正文未披露”。
- **解读**：这篇直接命中 OpenClaw/通用 agent 的痛点：技能越多，全文枚举越失效，必须做检索增强。贡献在于把“取技能、塞技能、用技能”拆成可量化链路；局限是当前仍偏离真实长尾工具生态，复现建议优先接入你自己的 skill 库与失败日志做 retrieval 误检分析。

#### 3) NeuroClaw Technical Report
- **标题**：NeuroClaw Technical Report
- **简称**：NeuroClaw
- **领域**：科研 Agent / 多智能体系统 / 可复现研究
- **论文**：https://arxiv.org/abs/2604.24696
- **项目**：https://cuhk-aim-group.github.io/NeuroClaw/index.html
- **代码**：https://github.com/CUHK-AIM-Group/NeuroClaw
- **评测**：仿真 benchmark：NeuroBench，用于衡量 executability、artifact validity 与 reproducibility readiness；正文还将其与通用 agent benchmarks（如 ClawBench）相关工作对照。真机：不适用。设备配置：非机器人论文，无机器人本体/末端/相机位姿；系统面向神经影像 raw data、BIDS 语义、多模态分析流水线。
- **算力**：正文强调环境管理、Docker、GPU configuration 与单 GPU/多工具科研执行支持，但未统一披露模型训练或系统评测所用 GPU 型号、卡数、时长，记为“正文未披露”。
- **解读**：NeuroClaw的强点不是“再做一个聊天科研助手”，而是把 raw-data-aware orchestration、分层 skill/agent、执行校验和审计链放进闭环里。亮点是可执行与可复现优先；局限是领域专用性较强，复现建议先借鉴其三层 skill/agent 分工和 post-execution verification 设计到通用科研工作流。
