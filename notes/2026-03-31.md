### A. VLA 视觉利用与流式执行

#### 1) FocusVLA: Focused Visual Utilization for Vision-Language-Action Models
- **标题**：FocusVLA: Focused Visual Utilization for Vision-Language-Action Models
- **简称**：FocusVLA
- **领域**：VLA / 视觉-动作对齐 / Manipulation
- **论文**：https://arxiv.org/abs/2603.28740
- **项目**：正文未披露独立项目页
- **代码**：正文说明“代码将公开”，但未给出仓库地址
- **评测**：仿真 benchmark：LIBERO 与 RoboTwin。LIBERO 单模型设置在 Spatial/Object/Goal/Long 四个 suite 上分别为 99.2/98.4/97.0/98.8，平均 98.4；多权重设置下在四个 suite 上也保持 99% 左右。RoboTwin 采用 Easy/Hard 设定，Easy task 每项 100 trials、Hard task 每项 300 trials。真机：Realman 双臂平台，3 个任务——Grasp Fruit、Stack Cups、Place Left Block；每任务仅 50 episodes 训练，每个 variation 做 25 trials，总计每任务 100 次试验。设备配置：Realman 平台，双臂操作；每臂 7-DoF manipulator + 1-DoF gripper；相机为 1 个 top camera + 1 个 wrist camera，共 2 个位姿。
- **算力**：正文给出 LIBERO 使用 4× NVIDIA A100、总 batch size 64；单模型训练 100k steps，多权重训练 10k–80k steps 不等。RoboTwin 使用 8× A100、总 batch size 64，Easy 训练 20k steps、Hard 训练 100k steps。
- **解读**：FocusVLA的核心不是换更强视觉 backbone，而是强迫策略真正“看见”与任务相关的局部细节：用 Modality Cascaded Attention 切断 shortcut，再用 Focus Attention 动态筛选关键 patch。优点是精细操控与收敛速度都明显提升；局限是仍建基于自回归 VLA 框架。复现时应优先核查 patch 选择门控、visual token 数量与真实相机视角的一致性。

#### 2) StreamingVLA: Streaming Vision-Language-Action Model with Action Flow Matching and Adaptive Early Observation
- **标题**：StreamingVLA: Streaming Vision-Language-Action Model with Action Flow Matching and Adaptive Early Observation
- **简称**：StreamingVLA
- **领域**：VLA / 实时部署 / Action Flow Matching
- **论文**：https://arxiv.org/abs/2603.28565
- **项目**：正文未披露独立项目页
- **代码**：正文未披露独立代码仓库
- **评测**：仿真 benchmark：基于 π0.5-Libero，在 LIBERO Spatial/Object/Goal/Long 四个 suite 上报告 96.6/95.4/91.0/94.9，平均 94.9；同时报告速度与流畅性指标，最佳配置实现 31.625（2.36×）的 latency speedup 与 36.0（6.45×）的 halting reduction。真机：Franka Panda 单臂桌面 pick-and-place 验证，用于展示部署效率与流式执行可行性。设备配置：机器人本体为 Franka Panda；末端执行器正文未单列型号；相机为 1 个固定 RGB camera，共 1 个位姿。
- **算力**：正文披露训练使用 4× NVIDIA A800-SXM4-80GB；主模型训练 60,000 iterations、学习率 1e-5，耗时约 10–20 GPU hours；自适应早观测预测器训练 54,000 iterations、初始学习率 1e-4。
- **解读**：StreamingVLA把 VLA 的“看—算—动”串行流水线改造成可重叠的流式流程：一边执行动作，一边继续生成后续动作并提前观察下一帧，关键在 action flow matching 与 adaptive early observation 的配合。它对部署延迟很有现实价值，但真机验证仍偏轻量。复现时建议先复现实验中的 latency/halting 计量，再对比不同 overlap 策略是否真能保性能。

### B. 以视频语言推理直接做机器人奖励

#### 1) SOLE-R1: Video-Language Reasoning as the Sole Reward for On-Robot Reinforcement Learning
- **标题**：SOLE-R1: Video-Language Reasoning as the Sole Reward for On-Robot Reinforcement Learning
- **简称**：SOLE-R1
- **领域**：Robot RL / Video-Language Reasoning / Reward Modeling
- **论文**：https://arxiv.org/abs/2603.28730
- **项目**：正文未给出项目页/Homepage
- **代码**：正文声明将发布模型 checkpoints、完整训练数据与在线 RL 代码，但未给出公开地址
- **评测**：仿真 benchmark：RoboSuite、ManiSkill、Meta-World、LIBERO 四套环境；论文汇总 40 个仿真任务，并额外给出 10 个简单 Meta-World 任务的 prompt 变体比较。整体上，含真机在内共评测 50 个任务，声明在 24 个未见任务上成功完成从零开始的 online RL。真机：桌面 Franka FR3 平台，按单臂独立 RL 运行；观测使用 wrist + shoulder 两路 RGB 与本体状态。设备配置：工作站上安装一对 Franka FR3，但实验按单臂独立执行；末端为 Robotiq 2F-85 gripper（带 Fin-Ray compliant fingers）；相机为每臂 1 个 wrist-mounted RealSense D405 + 1 个 over-the-shoulder RGB，相当于单臂 2 个位姿（整套工位总共 4 个常用位姿）。
- **算力**：正文/附录给出奖励标注服务器可自托管在单张 H100 上；推理速度测试中，SOLE-R1 在单 H100 上对 2 段、每段 20 帧视频做奖励标注，平均 40 秒。完整训练总 GPU 卡数、训练时长与总算力账单正文未披露。
- **解读**：SOLE-R1最激进的地方在于把“视频语言推理器”直接当作 online RL 的唯一奖励源，不再依赖手工 reward、success detector 或演示。贡献在于把时序 CoT 与 progress supervision 结合得足够稳定；局限是奖励推理仍慢，且真实部署依赖高质量多视角视频。复现时应优先重做 reasoning data synthesis 与 reward hacking 对抗测试，否则模型很容易只学会“会说”而不是“会判”。

### C. 真实世界通用操控评测基建

#### 1) ManipArena: Comprehensive Real-world Evaluation of Reasoning-Oriented Generalist Robot Manipulation
- **标题**：ManipArena: Comprehensive Real-world Evaluation of Reasoning-Oriented Generalist Robot Manipulation
- **简称**：ManipArena
- **领域**：Embodied Benchmark / Real-world Evaluation / Generalist Manipulation
- **论文**：https://arxiv.org/abs/2603.28545
- **项目**：https://maniparena.x2robot.com/
- **代码**：https://github.com/maniparena/maniparena-repo
- **评测**：仿真 benchmark：IsaacLab + IsaacLab-Arena 构建的 real-to-sim 对应环境，结合 3D Gaussian Splatting 复原真实工位，并支持 GPU 并行评测。数据/任务规模：20 个任务、10,812 条 expert trajectories，覆盖 execution reasoning、semantic reasoning 与 mobile manipulation。真机：两类平台——(1) 桌面双臂平台：4 套 bimanual 6-DOF follower arms 设备；(2) 移动平台：1 台 Quanta X1，带 mecanum-wheel 全向底盘、升降柱与 bimanual ARX arms。设备配置：桌面平台为双臂 6-DOF follower arms，移动平台为 Quanta X1 + 双臂 ARX；末端执行器正文未单列夹爪型号；相机统一为 3 个 RGB 位姿（face/front 全局视角 + left wrist + right wrist），20 fps、640×480。
- **算力**：正文明确仿真端采用 GPU-accelerated parallel environments，但未披露训练/评测所用 GPU 型号、卡数与总时长；属于“正文未披露具体算力账单”。
- **解读**：ManipArena的价值不在提出新策略，而在把“真实世界评测”做成可提交、可复现、可对齐的统一基建：单模型 HTTP endpoint、双臂+移动混合任务、真实工位到 IsaacLab 的数字孪生闭环都很实用。局限是目前更像 challenge/benchmark 基础设施。复现时建议先打通其数据协议、三路相机与 56D/62D 状态接口，再上复杂 VLA 或 world model。
