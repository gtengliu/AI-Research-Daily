### A. VLA 后训练、长时序学习与失败机理

#### 1) Simple Recipe Works: Vision-Language-Action Models are Natural Continual Learners with Reinforcement Learning
- **标题**：Simple Recipe Works: Vision-Language-Action Models are Natural Continual Learners with Reinforcement Learning
- **简称**：Simple Recipe Works
- **领域**：VLA / Continual RL / 后训练
- **论文**：https://arxiv.org/abs/2603.11653
- **项目**：正文未披露
- **代码**：http://github.com/UT-Austin-RobIn/continual-vla-rl
- **评测**：仿真 benchmark：5 个 lifelong RL 基准，正文明确包含 LIBERO-Object、LIBERO-Spatial、LIBERO-Long-Horizon 等；输入为 camera images，输出为末端位姿与夹爪控制。真机：正文未报告真机实验。设备配置：不适用。
- **算力**：正文给出方法、LoRA 持续后训练设定与多模型比较，但未统一披露 GPU 型号、卡数与总训练时长；推理开销也未统一给出。披露状态：正文未披露。
- **解读**：作者发现 VLA 做持续强化学习时，简单的顺序微调加 LoRA 就很强，遗忘并没有想象中严重。价值在于给后训练提供了低复杂度强基线；局限是真机闭环与算力账单不足。复现建议先在 LIBERO 连续任务序列上复刻 Seq. FT+LoRA，再对比 ER/EWC 类方法。

#### 2) Chunk-Boundary Artifact in Action-Chunked Generative Policies: A Noise-Sensitive Failure Mechanism
- **标题**：Chunk-Boundary Artifact in Action-Chunked Generative Policies: A Noise-Sensitive Failure Mechanism
- **简称**：Chunk-Boundary Artifact
- **领域**：生成式机器人策略 / 失败分析
- **论文**：https://arxiv.org/abs/2603.11642
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：基于 LIBERO Goal 与 LIBERO-10，对 OpenPI 冻结 checkpoint（pi0.5-libero、pi0-libero）做 70 rollout/任务与多上下文 steering 分析。真机：正文未报告真机实验。设备配置：不适用。
- **算力**：正文聚焦机制分析与干预，不给统一训练 GPU 账单；推理/干预计算开销也未汇总为统一时延或显存表。披露状态：正文未披露。
- **解读**：这篇把 action chunk 边界处的抖动/jerk 量化成可观测失效征兆，并证明它和失败显著相关。意义是给生成式策略提供了可诊断、可干预的机理视角；局限是真机证据缺失。复现建议先按文中 jerk 指标重建分析管线，再接入自己的 chunked policy 做 rollout 体检。

### B. 具身规划与 Agent 框架

#### 1) SVLL: Staged Vision-Language Learning for Physically Grounded Embodied Task Planning
- **标题**：SVLL: Staged Vision-Language Learning for Physically Grounded Embodied Task Planning
- **简称**：SVLL
- **领域**：Embodied Planning / VLM 任务规划
- **论文**：https://arxiv.org/abs/2603.11563
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：交互式 AI2-THOR embodied planning benchmark，覆盖 5 类任务。真机：6-DoF 机械臂零样本部署，正文明确评测 9 个真实长时序任务。设备配置：机器人本体为 6-DoF robotic manipulator；末端为 parallel-jaw gripper；相机为 wrist-mounted RGB camera（wrist 位姿=1）；head 相机不适用。
- **算力**：训练资源：8× NVIDIA A100 80GB；Stage 1 与 Stage 2 各约 25 小时，Stage 3 LoRA + B-DPO 约 4 小时。推理开销：正文未统一披露时延/FPS/显存。披露状态：正文披露较完整训练资源，推理账单未完整披露。
- **解读**：SVLL 把“先学物理 grounding、再学全上下文、最后偏好优化”拆成三阶段，明显针对长时序规划里常见的 shortcut 动作问题。亮点是仿真和真机都做了闭环验证；局限是机器人硬件细节仍偏粗。复现建议先在 AI2-THOR 复刻三阶段训练，再把 wrist 单视角和约束违例指标迁到实机。

#### 2) RoboClaw: An Agentic Framework for Scalable Long-Horizon Robotic Tasks
- **标题**：RoboClaw: An Agentic Framework for Scalable Long-Horizon Robotic Tasks
- **简称**：RoboClaw
- **领域**：Agentic Robotics / 长时序操作框架
- **论文**：https://arxiv.org/abs/2603.11558
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：正文结合 RoboCasa、MimicGen、GenH2RSim、Robotwin 2.0 等相关体系讨论，并在长时序任务中做对比。真机：real-world manipulation tasks，正文报告在长时序任务上相对基线成功率提升 16.67%。设备配置：正文强调统一 agent 框架与技能编排，但未在主文统一披露机器人本体、末端和相机位姿数量。
- **算力**：正文围绕框架、数据闭环和任务执行机制展开，未统一披露训练 GPU/卡数/时长，也未给统一推理时延/FPS。披露状态：正文未披露。
- **解读**：RoboClaw 的核心不是新 policy，而是把采集、学习、部署和失败恢复放进同一个 agent 语义闭环里，减少长时序执行中的语义漂移。局限是硬件与算力账单不够透明。复现建议先做小规模技能库 + 状态监控 + retry/recovery 三件套，再逐步接入自动数据回流。

### C. 感知世界模型与机器人硬件

#### 1) O3N: Omnidirectional Open-Vocabulary Occupancy Prediction
- **标题**：O3N: Omnidirectional Open-Vocabulary Occupancy Prediction
- **简称**：O3N
- **领域**：开放词汇占据预测 / 全景机器人感知
- **论文**：https://arxiv.org/abs/2603.12144
- **项目**：正文未披露
- **代码**：https://github.com/MengfeiD/O3N
- **评测**：仿真 benchmark：Human360Occ（H3O，CARLA-based human-ego 360° occupancy）。真机/真实数据 benchmark：QuadOcc（real-world first-person 360° occupancy，四足机器人校园场景）。设备配置：机器人本体为 quadruped platform（QuadOcc 数据集）；感知为 360° 全景视觉相机，正文未给 head/wrist 机位定义与数量；末端执行器不适用。
- **算力**：训练资源：4× NVIDIA RTX 3090，25 epochs，总 batch size 4。推理开销：正文未统一披露时延/FPS/显存。披露状态：正文披露训练硬件，推理账单未完整披露。
- **解读**：O3N 把开放词汇语义、全景视觉和 occupancy 统一起来，适合机器人在陌生场景里做更可扩展的 3D 语义建模。局限是更偏感知基座，尚未直接闭环到操控策略。复现建议先在 QuadOcc/H3O 跑通相同体素参数化，再评估开放类扩展对导航或探索模块的增益。

#### 2) CRAFT: A Tendon-Driven Hand with Hybrid Hard-Soft Compliance
- **标题**：CRAFT: A Tendon-Driven Hand with Hybrid Hard-Soft Compliance
- **简称**：CRAFT
- **领域**：灵巧手硬件 / 遥操作 / 机器人学习平台
- **论文**：https://arxiv.org/abs/2603.12120
- **项目**：http://craft-hand.github.io/
- **代码**：项目页声明将开源，正文未给独立代码仓库
- **评测**：仿真 benchmark：MuJoCo simulation integration，用于 policy learning 支持。真机：与 LEAP Hand 做结构/抓持/能耗对比，并进行 5 类 teleoperation 实物任务用户研究。设备配置：机器人本体为手/臂一体遥操作系统；末端为 CRAFT tendon-driven hand；相机为单 RGB camera 做 whole-arm teleoperation（固定机位=1）；无 wrist-mounted 传感器或手套。
- **算力**：正文重点在硬件与仿真/遥操作接口，未统一披露 GPU 型号、卡数与训练时长；推理开销同样未统一给出。披露状态：正文未披露。
- **解读**：CRAFT 的亮点是用混合软硬顺应性把“低成本、易遥操作、可学习”三件事同时兼顾，而且整手成本压到 600 美元级。局限是更偏平台而非统一策略基准。复现建议优先复刻单相机遥操作标定与 MuJoCo 接口，再验证顺应性对示教质量和 RL 稳定性的帮助。
