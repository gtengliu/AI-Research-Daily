### A. VLA 可控性与表征约束

#### 1) GuidedVLA: Specifying Task-Relevant Factors via Plug-and-Play Action Attention Specialization
- **标题**：GuidedVLA: Specifying Task-Relevant Factors via Plug-and-Play Action Attention Specialization
- **简称**：GuidedVLA
- **领域**：Vision-Language-Action / Attention specialization / Generalization
- **论文**：https://arxiv.org/abs/2605.12369
- **项目**：https://guidedvla.github.io/project_page/
- **代码**：正文、abs 页与当前项目页均未给出可稳定核验的代码仓库，记为“正文未披露”
- **评测**：仿真 benchmark：**LIBERO-Plus** 与 **RoboTwin 2.0**。正文核验为 **RoboTwin 2.0 的 8 个代表任务**（Adjust Bottle、Beat Hammer Block、Click Bell、Dump Bin BigBin、Moving PlayingCard away、Lift Pot、Place Burger Fries、Place Can Basket），并在随机未见设置下评估。真机：**2 套双臂平台、6 个任务**。平台 A 为 **ALOHA AgileX 双臂系统**，配 **2 个腕部 Intel Orbbec Dabai 相机 + 1 个第三视角 Orbbec Dabai 相机**，共 **3 个相机位姿**，任务为 fruits/vegetables 分类放置、stack bowls、clean tabletop。平台 B 为 **PSI-Bot RealMan 双臂平台**，正文明确使用 **Intel RealSense D435** 视觉，任务为 place beaker in heating mantle、stack beakers、heat beaker；平台 B 的末端执行器形式与相机位姿数量正文未统一表格式披露。真机每任务每模型 **20 trials**。
- **算力**：附录明确训练采用 **8× NVIDIA H200 GPUs**（global batch size 64，action chunk size 50，mixed precision），评测在 **1× NVIDIA RTX 4090** 上完成；统一总 GPU-hours 正文未披露。
- **解读**：这篇把 VLA 常见的“隐式学会一切”改成显式头部专门化：对象、几何、技能三类因素分别引导 action decoder 注意力。亮点是对 π0 在鲁棒与真机泛化上都有稳定增益；局限是对额外因子标注流水线有依赖，复现时先对齐对象掩码、技能软标签与深度分支注入位置。

#### 2) See What Matters: Differentiable Grid Sample Pruning for Generalizable Vision-Language-Action Model
- **标题**：See What Matters: Differentiable Grid Sample Pruning for Generalizable Vision-Language-Action Model
- **简称**：GridS
- **领域**：Efficient VLA / Visual token pruning / Generalizable manipulation
- **论文**：https://arxiv.org/abs/2605.11817
- **项目**：正文未披露
- **代码**：https://github.com/Fediory/Grid-Sampler
- **评测**：仿真 benchmark：**LIBERO**，并在正文中说明扩展评测覆盖 **LIBERO-PLUS** 与 **RoboTwin**（细节在附录）。真机：**LeRobot SO-100 单臂平台**，评测 **3 个语言条件任务**：Pick & Place、Stack Cubes、Transfer Pen，并额外构造 **21 个 OOD 场景**。设备配置：机器人本体为 **SO-100 follower arm**；视觉为 **1 个固定 Intel RealSense D435 第三视角 + 1 个腕部 Intel RealSense D405**，共 **2 个相机位姿**；策略仅使用 **RGB**。末端执行器为 SO-100 原生夹爪，正文未单列型号。
- **算力**：正文明确实验运行于 **1× NVIDIA RTX 3090（24GB）** 工作站；真实机器人策略训练 batch size 16、**50,000 optimization steps**。统一总 GPU-hours 正文未披露。
- **解读**：GridS 的核心不是简单删 token，而是用可微网格重采样保留最小必要视觉几何，因此在极低视觉预算下还能守住甚至提升成功率。亮点是兼顾速度、FLOPs 和 OOD 泛化；局限是当前主验证仍依赖特定 backbone，复现时要先校准采样 token 数 K、插值位置与多视角对齐方式。

### B. 流式交互与低时延 VLA

#### 1) Premover: Fast Vision-Language-Action Control by Acting Before Instructions Are Complete
- **标题**：Premover: Fast Vision-Language-Action Control by Acting Before Instructions Are Complete
- **简称**：Premover
- **领域**：Streaming VLA / Early action initiation / Human-robot interaction latency
- **论文**：https://arxiv.org/abs/2605.12160
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**LIBERO** 四个 suite（Spatial / Object / Goal / LIBERO10）与 **VLA-arena Level-1**（Extrapolation / Distractor / Safe / Long-horizon）。正文核验：LIBERO 评测 **每 suite 350 rollouts，共 1400**；VLA-arena 覆盖 **55 个任务、11 个 suite、每 suite 5 个任务**，并使用 episodes 15–49 做评测。真机：**无独立真实机器人实验**。设备配置：两套仿真 benchmark 都提供 **2 个相机位姿**，即 **固定 agent-mounted 视角 + wrist-mounted 视角**；机器人本体与末端执行器按仿真环境定义，不适用真机配置描述。
- **算力**：附录明确全部实验使用 **8× NVIDIA H200（141GB）**；每组 projection heads 在 **单卡** 上训练，评测 rollout 并行跑在其余 GPU 上。总 GPU-hours 正文未披露。
- **解读**：Premover 的贡献是把“用户还没把指令说完/打完”的等待时间利用起来，用 focus map 和 readiness gate 决定何时提前启动动作。亮点是能显著缩短端到端交互时延且基本不掉点；局限是目前依赖可得分割监督与按 benchmark 单独训练，复现时要重点核对 streaming prefix 构造、阈值校准和双视角聚合。

### C. 想象驱动的动作生成与 latent action

#### 1) From Imagined Futures to Executable Actions: Mixture of Latent Actions for Robot Manipulation
- **标题**：From Imagined Futures to Executable Actions: Mixture of Latent Actions for Robot Manipulation
- **简称**：MoLA
- **领域**：Robot manipulation / World model / Latent action / Video imagination
- **论文**：https://arxiv.org/abs/2605.12167
- **项目**：https://logosroboticsgroup.github.io/MoLA
- **代码**：正文写明“Code will be released”，当前正文、abs 页与 Homepage 未给出可核验仓库链接，记为“正文未披露”
- **评测**：仿真 benchmark：**CALVIN ABC-D、LIBERO、LIBERO-Plus**。正文核验：LIBERO 在四个 lifelong learning suites 上汇报结果；LIBERO-Plus 覆盖 **10,030 tasks** 的扰动扩展设置。真机：**Universal Robots UR5e 单臂平台**，使用 **5 个 manipulation tasks**，每任务 **50 trials**，每 trial 最多 **20 attempts**。设备配置：机器人本体为 **UR5e**；视觉为 **1 个第三视角 RealSense D455 + 1 个夹爪端 RealSense D405**，共 **2 个相机位姿**；末端执行器正文仅说明 gripper-mounted camera，夹爪型号正文未披露。
- **算力**：附录披露两阶段主要训练资源：视频生成模型微调使用 **8× NVIDIA H100，5 天**；mixture of inverse dynamics models 预训练使用 **8× NVIDIA H100，16 小时**。推理开销测量在 **1× NVIDIA GeForce RTX 4090** 上完成。统一总 GPU-hours 正文未披露。
- **解读**：MoLA 不是直接拿想象视频喂策略，而是先把 imagined future 转成面向控制的 latent actions，再交给混合逆动力学模型执行。亮点是把“看起来像未来”变成“可执行的未来”，在仿真和 UR5e 上都提升泛化；局限是链路更长、组件耦合更强，复现时要严格对齐 video model 微调数据、latent codebook 与多视角未来帧对齐。

### D. 接触丰富操控与高精度装配

#### 1) SI-Diff: A Framework for Learning Search and High-Precision Insertion with a Force-Domain Diffusion Policy
- **标题**：SI-Diff: A Framework for Learning Search and High-Precision Insertion with a Force-Domain Diffusion Policy
- **简称**：SI-Diff
- **领域**：Contact-rich manipulation / Assembly / Force-domain diffusion policy
- **论文**：https://arxiv.org/abs/2605.12247
- **项目**：https://yorklyb.github.io/SI-Diff/
- **代码**：正文、abs 页与当前项目页均未给出可稳定核验的公开代码仓库；记为“正文未披露”
- **评测**：仿真 benchmark：**无**。正文以真实装配与跨形状转移为主。真机：**7-DoF Franka Emika 单臂平台**，使用 **腕部触觉/力觉传感** 收集与执行；训练数据包括 **300 条 search 成功样本** 与来自 TacDiffusion 的 **1,500 条 insertion demonstrations**。测试覆盖 **5 种未见形状**（六棱柱、五棱柱、三棱柱、圆柱、USB-A），并包含 **0.1 mm / 0.2 mm clearance** 等高精度设置。设备配置：机器人本体为 **7-DoF Franka Emika**；末端为 peg-in-hole 装配端；感知核心是 **集成于 wrist 的 tactile/wrench sensing**；正文未采用相机作为主评测传感，因此相机位姿数量记为 **0（正文主实验未用视觉作为策略输入）**。
- **算力**：正文明确框架以 PyTorch 实现，训练与评测使用 **1× 48GB NVIDIA RTX A6000**；训练 **2,000 epochs**、batch size **4,096**、diffusion steps **50**。总 GPU-hours 正文未披露。
- **解读**：SI-Diff 的价值在于把 search 和 insertion 两类动作模式合进同一 force-domain diffusion policy，不再在搜索/插入间切模型切权重。亮点是对极小间隙和未见截面形状都有迁移性；局限是验证任务族仍集中在 peg-in-hole 变体，复现时要优先对齐力反馈维度、BBS 采样与 1kHz 控制回路。
