### A. 在线适应与闭环 VLA

#### 1) Agentic-VLA: Efficient Online Adaptation for Vision-Language-Action Models
- **标题**：Agentic-VLA: Efficient Online Adaptation for Vision-Language-Action Models
- **简称**：Agentic-VLA
- **领域**：VLA / Online Adaptation / RL Fine-tuning
- **论文**：https://arxiv.org/abs/2605.22896
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：主评测为 **LIBERO** 四套件（Spatial / Object / Goal / Long），每任务 **50 expert demonstrations**；正文核验到相对现有在线适应方法，**Long-horizon 提升 +12.3%**、**1-shot 提升 +28.5%**、跨任务迁移从 **0% 提到 31.2%**，并且达到 **2.4× 更快收敛**。补充评测为 **RoboTwin 2.0** 代表性双臂任务子集，平台为 **dual-arm Aloha AgileX**，每任务 **50 clean demonstrations、100 rollouts**，Hard 设置含 clutter / lighting / texture / height 随机化。真机：**正文无真实机器人实验**；真实部署仅作为未来工作讨论。机器人本体/末端/相机位姿数量按正文可确认到 **仿真为双臂 Aloha 平台，更多硬件细节正文未披露**。
- **算力**：正文核验到在线优化使用 **GRPO**，学习率 **1e-5**、**batch size 32**、group size **8**；ARS 使用 **Llama-3-8B** 做任务分解，全部实验使用 **4× NVIDIA A100 GPUs**。总训练时长正文未披露。
- **解读**：这篇把“会执行”的 VLA 往“会边干边学”推进了一步，关键不是单纯加 RL，而是把奖励重写、语言引导探索和经验记忆串成闭环。亮点是长程任务和跨任务迁移都明显起量；局限是真机还没落地。复现建议先单独复做 reward synthesis，再叠 memory 模块验证增益来源。

#### 2) $\pi_0$-EqM: Equilibrium Matching for Closed-Loop Vision-Language-Action Control
- **标题**：$\pi_0$-EqM: Equilibrium Matching for Closed-Loop Vision-Language-Action Control
- **简称**：π0-EqM
- **领域**：VLA / Action Decoder / Iterative Inference
- **论文**：https://arxiv.org/abs/2605.23128
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：主评测为 **19-task RoboTwin benchmark**，每任务 **500 demonstrations（100 clean + 400 randomized）**，总计 **9,500**；在与原 π0 **matched 300-step inference budget** 下，平均成功率由 **40.4% 提升到 50.2%**，19 个任务中 **12 提升 / 5 下降 / 2 持平**。附加评测为 **LIBERO** 四套件，正文核验到 **LIBERO-Spatial 96.8%→97.2%**、**LIBERO-10 85.2%→87.0%**，四套件均值 **94.15%→94.35%**。真机：**正文无真实机器人实验**。RoboTwin 所对应双臂平台、末端和相机机位在本文中未展开，故机器人本体/末端/相机位姿数量记为 **正文未披露**。
- **算力**：正文/附录未披露训练 GPU 卡型与卡数；可核验推理预算为 **300 iterative decoding steps**，并围绕 residual threshold 做自适应停止分析。算力项按要求记为 **正文未披露**。
- **解读**：π0-EqM最有意思的点，是把 action decoder 从“按时间步采样”改成“求平衡点”，于是停止时机本身成了策略设计的一部分。亮点是同等 300 步预算下 RoboTwin 提升很实；局限是增益仍主要停留在仿真。复现建议重点盯 residual 与真实成功率脱钩的 stationarity–executability gap，而不只看数值收敛。

### B. 几何一致世界模型与动作抽取

#### 1) GEM-4D: Geometry-Enhanced Video World Models for Robot Manipulation
- **标题**：GEM-4D: Geometry-Enhanced Video World Models for Robot Manipulation
- **简称**：GEM-4D
- **领域**：Video World Model / Geometry Consistency / Manipulation
- **论文**：https://arxiv.org/abs/2605.22882
- **项目**：https://anonymous-submission-20.github.io/gem.github.io/
- **代码**：项目页当前仅给匿名展示页，公开 abs / PDF / README Homepage 未给正式代码仓库
- **评测**：仿真 benchmark：**RLBench** 上执行由生成视频反推出的轨迹，正文表 2 核验到不同任务成功率落在 **63%–82%**；4D 场景预测评测还比较 **RGB 重建、深度重建、点跟踪/对应关系质量**。真实 benchmark：基于 **Droid dataset** 的 **400 unseen real-world samples** 做真实域评测，成功率由最强基线 **61% 提升到 81%**；文中同时给出 **UF robot arm** 的真实场景定性执行图。真机：可确认存在 **UF robot arm** 定性执行，但机器人本体具体型号、末端型号未披露；输入设定为 **single-view RGB-only**，相机位姿数量可核验为 **1（固定/单视角；wrist/head 未披露）**。
- **算力**：正文/附录未披露训练 GPU 型号、卡数与总时长；仅披露输入设定与评测样本规模。算力项按要求记为 **正文未披露**。
- **解读**：GEM-4D的贡献不只是视频更好看，而是把几何一致性硬塞进中间表征，让生成视频终于能稳定反推出可执行 6-DoF 动作。亮点是真实域成功率直接从 61% 拉到 81%；局限是硬件与训练资源披露偏少。复现建议先验证 correspondence 质量提升是否真的对应动作抽取成功率提升，而非仅改善视觉指标。

### C. 触觉安全控制与农业真机闭环

#### 1) TactileReflex: Noise-Statistics-Driven Vision-Tactile Reflex Control for Force-Sensitive Manipulation
- **标题**：TactileReflex: Noise-Statistics-Driven Vision-Tactile Reflex Control for Force-Sensitive Manipulation
- **简称**：TactileReflex
- **领域**：Vision-Tactile Control / Force-Sensitive Manipulation / Reflex Layer
- **论文**：https://arxiv.org/abs/2605.23568
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**正文未设置公开仿真 benchmark**。真机：单臂平台，末端为 **Piper parallel gripper**，夹爪两侧安装 **2 个 MC-Tac 视觉触觉传感器**，每个传感器输出 **640×480 tactile images**；相机位姿数量 **2（末端 fingertip tactile cameras 2）**。正文核验到三类真实实验：其一是静态抗形变消融，完整三通道系统 **5/5 成功**、其余配置至多 **1/5**；其二是动态倒水任务，在 **两档水量** 下固定力度基线 **0/10** 全失败，而 **TactileReflex 9/10 成功**；其三给出 slip fraction 随装水量提升而上升的量化分析。
- **算力**：正文未披露训练 GPU；系统以 **单 Python 进程** 在 **standard workstation** 运行，控制频率约 **12 Hz**。若按训练/推理算力口径，记为 **正文未披露**。
- **解读**：这篇的价值在于把触觉从“做分类”改成“做 reflex”，直接闭环调夹持力，特别适合脆弱物体和带液体容器。亮点是无需外部力标定，靠噪声统计就能自校阈值；局限是仍是夹爪级安全层，还没扩到整臂策略。复现建议先复现 static hold/unload 校准流程，否则阈值迁移会很不稳。

#### 2) Robotic Strawberry Harvesting with Robust Vision and Deep Reinforcement Learning based Sim-to-Real Control
- **标题**：Robotic Strawberry Harvesting with Robust Vision and Deep Reinforcement Learning based Sim-to-Real Control
- **简称**：Strawberry-Harvest DRL
- **领域**：Agricultural Robotics / Sim-to-Real RL / Harvesting
- **论文**：https://arxiv.org/abs/2605.23863
- **项目**：正文未披露
- **代码**：正文仅可核验到引用 Ultralytics 仓库，未给本工作独立代码仓
- **评测**：仿真 benchmark：在 **Isaac Sim/Lab** 中训练 **UR10e** reaching controller，与 IK-based MoveIt 基线比较运动质量； reaching 成功率定义采用 **2 cm tolerance**。真机：移动底盘为 **RB-VOGUI mobile robot**，机械臂为 **UR10e 单臂**，末端为 **gripper（正文未单列型号）**；视觉使用 **1 个 RGB-D 相机** 获取温室果实场景，相机位姿数量 **1（固定外部 RGB-D）**。正文核验到温室真实试验共采收 **281 颗草莓**，达到 **96.6% reaching success / 91.3% grasp-and-pull success / 84.3% overall harvesting success**。
- **算力**：正文核验到视觉训练运行在一台含 **NVIDIA RTX A6000 48GB** 的工作站上；输入分辨率 **640×640**、**batch size 16**、最多 **200 epochs**、early stopping patience **15**、优化器 **AdamW**、初始学习率 **2e-4**。PPO 在 Isaac Sim/Lab 中训练，但正文未单列额外 GPU 时长与卡数。
- **解读**：这篇不算“范式创新”，但工程闭环做得很完整：分割、3D 点估计、PPO reaching、ROS 执行串成能在温室里稳定摘果的系统。亮点是真实采收 281 颗且整体成功率过 84%；局限是末端与感知冗余设计披露不细。复现建议优先复做 target smoothing 和 reaching policy 的随机化范围，不然 sim-to-real 往往卡在末端对位。

### D. 隐式意图导航基准

#### 1) IntentionNav: A Benchmark for Intent-Driven Object Navigation from Implicit Human Instruction
- **标题**：IntentionNav: A Benchmark for Intent-Driven Object Navigation from Implicit Human Instruction
- **简称**：IntentionNav
- **领域**：Embodied Navigation / Benchmark / Intent Grounding
- **论文**：https://arxiv.org/abs/2605.23187
- **项目**：正文未披露
- **代码**：论文称提交的 artifact 含评测脚本、prompt、配置与复现实验工具，但公开 abs / PDF / 正文可见链接未给正式仓库
- **评测**：仿真 benchmark：**IntentionNav** 基准包含 **500 intents、176 个 Isaac Sim 场景、64 个目标类别**，每条 intent 重写为 **4 种 instruction style**，形成 **2000 controlled episodes**；输入为 **RGB-D observations + pose**。正文核验到在统一 active-navigation agent 下，三种 VLM 后端平均 **意图识别 48.3%**、进入目标 **2m 邻域 68.7%**、终止成功 **24.9%**、grounded 1m success **5.5%**；event-script intent 成功率 **28.7%**，physical-state / affordance 分别 **19.2% / 18.5%**。真机：**无真实机器人实验**；相机位姿数量按基准输入仅可确认 **仿真 RGB-D 视角，具体机位数正文未披露**。
- **算力**：正文核验到三模型共享 **2000-episode** 评测总共约 **150 GPU-hours on an RTX 5090**；由于外部 VLM API 延迟，墙钟时间存在波动。训练算力正文未披露。
- **解读**：IntentionNav把“帮我找一下那个适合切面包的东西”这类含糊请求正式做成了 benchmark，难点不在走路，而在目标推断和最后一米确认。亮点是把 style 和 cue type 解耦得很干净；局限是目前仍停留在仿真场景。复现建议别只报 SR，要把 intent match、2m 邻域命中和 grounded success 分层看。

### E. 人形全身跟踪跨本体迁移

#### 1) Any2Any: Efficient Cross-Embodiment Transfer for Humanoid Whole-Body Tracking
- **标题**：Any2Any: Efficient Cross-Embodiment Transfer for Humanoid Whole-Body Tracking
- **简称**：Any2Any
- **领域**：Humanoid / Cross-Embodiment Transfer / Whole-Body Tracking
- **论文**：https://arxiv.org/abs/2605.23733
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：在 **MuJoCo** 部署下构建四种人形本体的 cross-embodiment benchmark，覆盖 **LimX Oli（31-DoF）/ LimX Luna（27-DoF）/ Unitree G1 / Unitree H1**，评测 **5 个 source→target transfer 实例**，指标包括 **tracking success rate、MPJPE、base position/orientation error、动作速度与加速度**。正文核验到：从大规模预训练 WBT（如 Sonic）迁移到新本体时，Any2Any 仅用约 **1% 原始 compute/data** 即可完成适配；示意对比给出预训练 **~9000 GPU-hours + ~700h 数据**，而如 **Sonic2Oli** 级别迁移约 **~80 GPU-hours + ~8h 数据**。真机：**正文无真实机器人实验**；相机位姿数量 **无外部视觉主输入，正文未披露相机配置**。
- **算力**：正文核验到目标侧适配实验使用 **4× NVIDIA A100 GPUs**；不同采样率对比中，**16K/32K/64K** 分别对应 **1/2/4 张 RTX 4090**，**80K** 对应 **4 张 A100**。原始预训练成本示意为 **~9000 GPU-hours**。
- **解读**：Any2Any最重要的不是“又一个 humanoid tracker”，而是证明现成 WBT 专家模型可以像软件组件一样跨本体复用。亮点是把迁移成本压到 1% 量级；局限是验证仍在 MuJoCo，没有真机闭环。复现建议先分开看 kinematic alignment 和 PEFT 贡献，不然很难判断收益到底来自结构对齐还是额外参数容量。

## Agent / OpenClaw 能力增强（固定栏目）

### 今日新增 / 值得关注
- **Agentic-VLA** 和 **π0-EqM** 一起说明：VLA 的改进点正从“更大 backbone”转向“部署时如何继续推理、继续适配”。
- **GEM-4D** 提醒一个关键趋势：世界模型若没有几何一致性，视频再逼真也未必能抽出可执行动作。
- **TactileReflex** 很适合当 OpenClaw 未来接实体设备时的底层安全层：高层策略未必可靠，但夹爪 reflex 可以先把风险挡住。

### 能力提升方法 / 工作流要点
1. **在线适配要模块化分解**：奖励重写、探索反馈、经验记忆分开做，才能看清哪一环真正带来收益。
2. **世界模型评估别只看 FVD/PSNR**：必须再看 correspondence、动作抽取成功率和真实执行闭环。
3. **安全敏感操作优先加低层 reflex**：触觉/力控层先兜底，再让 VLA 或规划器往上叠。
4. **人形迁移要先做语义对齐再谈大规模预训练复用**：Any2Any 说明 observation/action semantics 不对齐时，算力堆再多也浪费。

### 机器人 / 设备控制能力相关进展
- **Strawberry-Harvest DRL** 是典型值得学的工程闭环：感知、3D 点位、控制、执行指标都能对上。
- **IntentionNav** 提供了一个很适合 agent/robot 共用的诊断思路：把目标推断、近邻到达、终止确认拆开评估。
- **TactileReflex** 展示了视觉触觉并不一定要上大模型，轻量 reflex 在特定安全任务上更实用。

### 持续监控主题
- Online-adaptive VLA with reward synthesis / adaptive decoding
- Geometry-consistent video world models for executable manipulation
- Vision-tactile reflex layers for fragile-object handling
- Intent-driven embodied navigation from implicit instructions
- Cross-embodiment reuse of humanoid whole-body trackers
