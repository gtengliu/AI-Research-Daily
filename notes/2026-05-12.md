### A. Generative VLA：测试时记忆、检索与闭环稳定性

#### 1) Retrieve-then-Steer: Online Success Memory for Test-Time Adaptation of Generative VLAs
- **标题**：Retrieve-then-Steer: Online Success Memory for Test-Time Adaptation of Generative VLAs
- **简称**：Retrieve-then-Steer
- **领域**：Generative VLA / Test-time adaptation / Online success memory
- **论文**：https://arxiv.org/abs/2605.10094
- **项目**：正文未披露
- **代码**：https://anonymous.4open.science/r/
- **评测**：仿真 benchmark：**LIBERO-10**，以及 **SimplerEnv 4 个任务**（Pick Coke Can、Move Near、Open/Close Drawer、Open Top Drawer and Place Apple）。真机：**2 套双臂平台**，分别为 **OpenArm-based dual-arm system** 与 **ALOHA-PiPER**；共评测 **4 个任务**：OpenArm 上的 bowl stacking、cube handoff、sequential test-tube placement，以及 ALOHA-PiPER 上的 bimanual T-shirt folding。设备配置：OpenArm 为 **2×7-DoF humanoid arms + parallel grippers**；ALOHA-PiPER 为 **2×6-DoF AgileX PiPER arms + two-finger grippers**；两平台均使用 **2 个腕部相机 + 1 个第三视角相机**，共 **3 个相机位姿**。
- **算力**：正文与附录主要披露模型结构、训练/测试流程和真机协议，**统一 GPU 型号、卡数与总 GPU-hours 正文未披露**，记为“正文未披露”。
- **解读**：它把 VLA 部署从“一次性 zero-shot”改成“持续在线运行”，用历史成功轨迹做检索先验，再按检索置信度引导当前采样。亮点是 LIBERO、SimplerEnv 和双臂真机都能稳步增益；局限是依赖环境相关性和成功记忆质量，复现时先对齐 memory 更新间隔、critic 打分与检索退避机制。

### B. 生成式机器人策略：流匹配、latent imagination 与测试时重排

#### 1) Data-Asymmetric Latent Imagination and Reranking for 3D Robotic Imitation Learning
- **标题**：Data-Asymmetric Latent Imagination and Reranking for 3D Robotic Imitation Learning
- **简称**：DALI-R
- **领域**：3D imitation learning / Latent world model / Test-time reranking
- **论文**：https://arxiv.org/abs/2605.10166
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**Adroit** 与 **MetaWorld**。正文表 1 核验为 **Adroit 3 个任务（Door / Pen / Hammer）+ MetaWorld 5 个任务（Disassemble / Shelf Place / Stick Pull / Pick Place / Wall）**，并统一采用 **每任务 100 evaluation episodes、3 个随机种子**。真机：无独立真实机器人实验。设备配置：均为仿真环境，机器人本体、末端执行器与相机位姿数量不适用。
- **算力**：附录明确 **全部实验运行于单张 NVIDIA RTX 3090（24GB）**；训练与评测均在该卡完成。总 GPU-hours 正文未披露。
- **解读**：DALI-R 的价值在于把“低质量轨迹”从噪声变成测试时选择信号：先用 latent world model 想象后果，再用 task completion scorer 重排候选动作块。亮点是无需新增高质量示范也能提升 3D policy；局限是验证仍集中于仿真，复现时要严控 candidate 数量、重排时延与 latent rollout 长度。

#### 2) Guided Streaming Stochastic Interpolant Policy
- **标题**：Guided Streaming Stochastic Interpolant Policy
- **简称**：SSIP
- **领域**：Generative robot policy / Streaming control / Inference-time guidance
- **论文**：https://arxiv.org/abs/2605.10051
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**Push-T**（含 static obstacle 与 dynamic obstacle，两类动态障碍又细分 **Intercept / Oscillate / Chase** 三种模式）以及 **Robomimic 3 个任务（Lift / Square / Can）**。真机：**单臂 Franka Panda**，评测 **2 类任务**：moving-obstacle avoidance 与 position-preference grasping。设备配置：机器人本体为 Franka Panda；末端执行器类型、真机相机位姿数量正文未单独披露。文中强调面向 **20 Hz 以上** 实时控制。
- **算力**：正文与附录明确实验和时延测量运行于 **AMD EPYC 7543 + 单张 NVIDIA RTX A5000** 工作站；总 GPU-hours 正文未披露。
- **解读**：这篇的关键不只是“给生成策略加引导”，而是把引导做成适合高频闭环的 streaming 版本，兼容 repulsion、STEG 和 critic guidance。亮点是动态障碍场景下明显优于 chunk-based diffusion/flow baselines；局限是对已知约束形式更友好，复现时应优先核对 control frequency、small-K/large-Δt 设定与真机安全边界。

### C. 泛化操控：异构对象交互与多基础模型路由

#### 1) HeteroGenManip: Generalizable Manipulation For Heterogeneous Object Interactions
- **标题**：HeteroGenManip: Generalizable Manipulation For Heterogeneous Object Interactions
- **简称**：HeteroGenManip
- **领域**：Generalizable manipulation / Cross-type interaction / Foundation-model-guided policy
- **论文**：https://arxiv.org/abs/2605.10201
- **项目**：https://yzmyalier.github.io/HeteroGenManip/
- **代码**：正文、abs 页与当前 Homepage 均未给出可用代码仓库；记为“正文未披露”
- **评测**：仿真 benchmark：正文明确覆盖 **RoboTwin、DexGarmentLab** 以及作者构造的 **different-type interaction tasks**，用于验证 rigid / deformable / articulated objects 的跨类型交互；但统一任务总数正文未集中披露。真机：**4 个 real-world tasks**。设备配置：**ARX-X7s humanoid robot**，带 **两条 7-DoF 机械臂 + 两个 grippers**；视觉为 **1 个 Intel RealSense L515 depth camera**，即 **1 个相机位姿**；每任务 **25 条示教**，train/test 各自 **15 trials**。
- **算力**：附录明确作者模型与基线训练均使用 **单张 NVIDIA GeForce RTX 4090（24GB）**；总 GPU-hours 正文未披露。
- **解读**：HeteroGenManip 把“抓哪里”和“后续如何交互”显式拆开：先用 foundation correspondence 找接触点，再用多基础模型路由的扩散策略处理后续跨类型轨迹。亮点是对异构对象交互比单模型路线更稳；局限是项目页尚未提供可直接复现实装代码，落地时先检查 grasp anchoring 与 category routing 的耦合误差。

### D. 视觉输入带宽：立体感知对操控策略与 VLA 的收益

#### 1) StereoPolicy: Improving Robotic Manipulation Policies via Stereo Perception
- **标题**：StereoPolicy: Improving Robotic Manipulation Policies via Stereo Perception
- **简称**：StereoPolicy
- **领域**：Robot perception / Stereo visuomotor policy / VLA adaptation
- **论文**：https://arxiv.org/abs/2605.09989
- **项目**：https://stereopolicy.github.io/
- **代码**：正文、abs 页与 Homepage 当前未给出可稳定核验的代码仓库链接；记为“正文未披露”
- **评测**：仿真 benchmark：**RoboMimic 3 个任务、RoboCasa 24 个任务、OmniGibson 4 个任务**。真机：**2 类机器人 embodiment**——**tabletop Franka arm** 与 **Galaxea R1 Pro 双臂移动操作平台**；共 **7 个任务**，其中桌面任务 **5 个**（Banana PnP、Toast Insert、Cup Hang、Steel Cup Hang、Glass Cup Hang），移动操作任务 **2 个**（PnP Toast、Turn on Radio）。设备配置：Franka 真机正文明确为 **7-DoF Franka Emika arm**，视觉为 **1 个外部 ZED Mini + 1 个腕部 ZED Mini**，两套均为 stereo，对应 **2 个相机位姿**；Galaxea R1 Pro 的末端执行器与相机位姿数量正文未统一表格式披露。
- **算力**：附录明确 **StereoPolicy-VLA 微调使用 8× H100 GPUs**；在 RoboCasa 上以 **GR00T-N1.5** 为预训练检查点训练 **60K steps**，π0.5 相关设定训练 **80K steps**。统一总 GPU-hours 正文未披露。
- **解读**：这篇最有意思的地方是它不走显式 3D 重建，而是直接把 stereo pair 喂进 transformer 融合几何线索，因此兼顾精度和时延。亮点是同时提升 diffusion policy 与 VLA；局限是收益依赖视角重叠与基线长度，复现时要重点校准 stereo baseline、腕视角域偏移和外参稳定性。
