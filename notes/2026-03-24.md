### A. 纯 VLA / 推理与接口设计

#### 1) DualCoT-VLA: Visual-Linguistic Chain of Thought via Parallel Reasoning for Vision-Language-Action Models
- **标题**：DualCoT-VLA: Visual-Linguistic Chain of Thought via Parallel Reasoning for Vision-Language-Action Models
- **简称**：DualCoT-VLA
- **领域**：纯VLA / 并行推理
- **论文**：https://arxiv.org/abs/2603.22280
- **项目**：https://livfour.github.io/DualCoT-VLA/
- **代码**：https://github.com/Livfour/DualCoT-VLA
- **评测**：仿真 benchmark：LIBERO、RoboCasa GR1。真机：基于 Mobile ALOHA 设计的 AgileX Cobot 双臂平台。设备配置：双 7-DoF 机械臂 + parallel grippers；机载 RGB 相机为 front-facing + wrist-mounted，正文明确为前视与腕视，但未给总位姿数拆分。
- **算力**：训练资源：正文披露“all training is conducted on NVIDIA H100 GPUs”，但未统一给出卡数、总时长与总训练步数。推理开销：方法强调将串行 CoT 改为单步并行 reasoning query，正文突出高频驱动动作专家，但未给统一 latency/FPS/显存表。披露状态：正文部分披露，附录未补全完整账单。
- **解读**：核心贡献是把视觉 CoT 与语言 CoT 并行化，避免显式自回归思维链拖慢控制闭环。适合多步任务与细粒度操作，但复现时要重点核查 reasoning token 与 action expert 的接口细节，且真实部署延迟账单仍需作者后续补足。

#### 2) VP-VLA: Visual Prompting as an Interface for Vision-Language-Action Models
- **标题**：VP-VLA: Visual Prompting as an Interface for Vision-Language-Action Models
- **简称**：VP-VLA
- **领域**：纯VLA / 视觉提示接口
- **论文**：https://arxiv.org/abs/2603.22003
- **项目**：https://visualprompt-vla.github.io/
- **代码**：正文未披露
- **评测**：仿真 benchmark：RoboCasa-GR1-Tabletop、SimplerEnv（WidowX 任务集）。真机：Franka Research 3 桌面操作。设备配置：Franka Research 3 7-DoF + 末端夹爪；2 路 RGB 输入，其中 1 路固定 third-person、1 路末端 first-person/wrist camera；总相机位姿数=2。
- **算力**：训练资源：正文披露基于 starVLA 框架训练，使用 8 GPUs、total batch size 256、action chunk size 16；GPU 型号与总训练时长正文未披露。推理开销：正文未给统一 latency/FPS/显存。披露状态：正文披露部分训练超参，完整算力账单未公开。
- **解读**：这篇把视觉 prompt 当成 VLA 的显式空间接口，用目标框和十字准星替代含糊文本指代，明显更贴近真实操控。优点是工程改造成本低；局限是仍依赖额外视觉提示器质量，复现时建议先单独验证 prompt 生成稳定性再接控制头。

### B. 灵巧操作 / 泛化 VLA

#### 1) UniDex: A Robot Foundation Suite for Universal Dexterous Hand Control from Egocentric Human Videos
- **标题**：UniDex: A Robot Foundation Suite for Universal Dexterous Hand Control from Egocentric Human Videos
- **简称**：UniDex
- **领域**：灵巧操作 / 通用灵巧手 VLA
- **论文**：https://arxiv.org/abs/2603.22264
- **项目**：https://unidex-ai.github.io/
- **代码**：https://github.com/unidex-ai/UniDex
- **评测**：仿真 benchmark：正文主打真实机器人 benchmark 与对象/手型泛化，对统一公开仿真 benchmark 名称披露不充分。真机：7-DoF Franka 平台多种灵巧手工具操作。设备配置：Franka Panda + 三类末端（Inspire Hand 6 active/12 full DoFs、Wuji Hand 20 active DoFs、Oymotion Hand 6 active/11 full DoFs）；1 路 Intel RealSense L515 ego RGB-D；总相机位姿数=1。
- **算力**：训练资源：附录披露预训练使用 8×NVIDIA H800、total batch size 128、3 epochs（约 30k steps）、约 24 小时。推理开销：正文/附录未给统一 latency/FPS/显存。披露状态：训练账单较完整，推理侧未系统披露。
- **解读**：价值在于把“人类第一视角视频 → 多手型统一动作空间 → 真实灵巧手控制”串成一套 foundation suite，解决跨手型复用难题。亮点是 FAAS 统一动作表示；局限是公开 benchmark 对齐还不够清晰，复现时应优先验证手型映射与 RGB-D 几何对齐链路。

### C. World Action Model / 鲁棒性分析

#### 1) Do World Action Models Generalize Better than VLAs? A Robustness Study
- **标题**：Do World Action Models Generalize Better than VLAs? A Robustness Study
- **简称**：WAM-vs-VLA-Robustness
- **领域**：World Action Model / 鲁棒性评测
- **论文**：https://arxiv.org/abs/2603.22078
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：LIBERO-Plus、RoboTwin 2.0-Plus。真机：无新增真机实验，主体为受控扰动鲁棒性评测。设备配置：LIBERO-Plus 使用 7-DoF Franka Panda + 2 相机（1 third-person + 1 wrist）；RoboTwin 2.0-Plus 使用 Aloha-Agilex 双臂系统 + 3 相机（1 head-mounted third-person + 2 wrist-mounted）；机器人本体、末端与相机位姿数量均由正文给出。
- **算力**：训练资源：本文为对比评测研究，不新增统一训练账单；对被评测模型仅做文中架构/资源讨论。推理开销：正文指出 DreamZero warm-up 超过 15 分钟，因此未纳入量化评测；其余模型未给统一 FPS/显存横表。披露状态：本文以 benchmark 对比为主，正文未披露自有完整算力账单。
- **解读**：这篇不是提新模型，而是认真回答“WAM 到底是不是比 VLA 更抗扰”。结论偏克制：WAM 在噪声/光照/布局扰动上更稳，但语言与机器人形态变化仍难。对选型很有参考价值，复现时建议优先复刻其 perturbation protocol，而不只是抄最终成功率。

### D. Sim2Real / 机器人控制与建模

#### 1) Sim-to-Real of Humanoid Locomotion Policies via Joint Torque Space Perturbation Injection
- **标题**：Sim-to-Real of Humanoid Locomotion Policies via Joint Torque Space Perturbation Injection
- **简称**：JTPI-Sim2Real
- **领域**：Sim2Real / 人形 locomotion RL
- **论文**：https://arxiv.org/abs/2603.21853
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：Isaac Gym 大规模并行 locomotion 训练与 sim-to-real 验证。真机：正文明确有人形机器人真实部署，但机器人本体型号、末端配置、外设相机位姿数量在当前版本正文中披露不完整。设备配置：机器人为 humanoid locomotion 平台；末端与相机细节正文未完整披露。
- **算力**：训练资源：正文披露使用 Isaac Gym，并行 rollout 4,096 agents、每次 policy update 收集 98,304 samples、总计 10,000 policy updates；GPU 型号正文未披露。推理开销：正文未给统一 latency/FPS/显存。披露状态：训练规模披露较清楚，硬件账单不完整。
- **解读**：创新点是把扰动从常见参数随机化推进到“关节力矩空间、且状态相关”的注入方式，直接扩大现实落差覆盖面。对人形落地有实用价值，但论文当前硬件披露偏少；复现建议先在同等 rollout 配置下验证鲁棒增益，再迁移真机。

#### 2) RAFL: Generalizable Sim-to-Real of Soft Robots with Residual Acceleration Field Learning
- **标题**：RAFL: Generalizable Sim-to-Real of Soft Robots with Residual Acceleration Field Learning
- **简称**：RAFL
- **领域**：Sim2Real / 软体机器人建模
- **论文**：https://arxiv.org/abs/2603.22039
- **项目**：https://generalroboticslab.com/RAFL
- **代码**：https://github.com/generalroboticslab/RAFL
- **评测**：仿真 benchmark：正文以 sim-to-sim 与 sim-to-real 的 unseen morphology 误差对比为主，基于可微软体仿真而非通用公开 benchmark。真机：真实软体机器人 marker 轨迹校准实验。设备配置：正文明确依赖 sparse marker observations，但机器人本体、末端形式、相机/追踪位姿数量在主文中未系统披露。
- **算力**：训练资源：正文强调端到端通过 differentiable simulator 训练 residual acceleration field，但 GPU 型号、卡数、总时长正文未披露。推理开销：正文未给 latency/FPS/显存。披露状态：正文未披露统一算力账单。
- **解读**：RAFL 的思路很干净：保留可微基础物理模型，再学一个对真实残差负责的加速度场，且对网格拓扑变化有泛化。适合软体机器人这种模型误差重灾区；局限是硬件测量链信息不足，复现时要优先搞定 marker 观测与 FEM 网格一致性。

### E. 双臂协作 / 任务级 RL

#### 1) BiPreManip: Learning Affordance-Based Bimanual Preparatory Manipulation through Anticipatory Collaboration
- **标题**：BiPreManip: Learning Affordance-Based Bimanual Preparatory Manipulation through Anticipatory Collaboration
- **简称**：BiPreManip
- **领域**：双臂操作 / 预备操作学习
- **论文**：https://arxiv.org/abs/2603.21679
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：SAPIEN 双臂操作环境，基于 Franka Panda 双臂模拟多类别物体的 preparatory manipulation。真机：ARX-X7s 双臂平台实测，并补充 robot-human handover。设备配置：ARX-X7s dual-arm + 双臂末端执行器；1 路 Intel RealSense L515 采集 partial 3D point cloud；总相机位姿数=1。
- **算力**：训练资源：正文披露在单张 NVIDIA V100 上训练约 24 小时收敛。推理开销：正文披露推理显存约 1,166 MB，并给出平均模型推理时间（不含物理仿真）约 0.04 秒。披露状态：训练与推理账单相对完整。
- **解读**：亮点是把“辅助另一只手更容易完成主任务”的 preparatory manipulation 单独建模，抓住了双臂协作里最常被忽略的前置动作。实用性强，但依赖点云 affordance 学习；复现时建议先复刻失败样本与成功样本成对采集流程，否则泛化会掉得很快。

#### 2) Closed-Loop Verbal Reinforcement Learning for Task-Level Robotic Planning
- **标题**：Closed-Loop Verbal Reinforcement Learning for Task-Level Robotic Planning
- **简称**：VRL-Planning
- **领域**：任务级 RL / 语言反馈规划
- **论文**：https://arxiv.org/abs/2603.22169
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：不依赖仿真训练主线，核心是物理机器人闭环策略改进。真机：physical robotic systems 上的任务级规划与执行失败后 verbal feedback 修正。设备配置：正文明确使用物理机器人平台与 Behavior Tree 执行闭环，但机器人本体、末端执行器、相机位姿数量在主文披露有限。
- **算力**：训练资源：该框架侧重在真实环境中用 verbal critic 迭代修正策略，正文未披露 GPU 型号、卡数、训练时长。推理开销：正文未给统一 latency/FPS/显存，重点在 explainable policy improvement。披露状态：正文未披露完整算力账单。
- **解读**：这条路线把 RL 从低层连续控制搬到任务级行为树优化，用自然语言 critic 做闭环纠错，优点是可解释、免梯度、能直接上真机。局限是依赖 critic 质量与提示工程；复现时应把 BT 语法约束和失败日志结构化做扎实。
