### A. 长时序历史建模与任务进度跟踪

#### 1) DSSP: Diffusion State Space Policy with Full-History Encoding
- **标题**：DSSP: Diffusion State Space Policy with Full-History Encoding
- **简称**：DSSP
- **领域**：Diffusion Policy / 长时序历史建模 / 操作策略
- **论文**：https://arxiv.org/abs/2605.14598
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**RoboTwin 2.0（50 个双臂任务）**、**MetaWorld（34 个单臂桌面任务）**、**Adroit（3 个灵巧手 in-hand 任务）**。正文核验：并专门按 episode 长度把 RoboTwin 划分为 short / mid / long horizon 做分析。真机：**AgileX 平台**上的 **3 个桌面长时序任务**——**Put Bottles、Object Swap、Morse Tapping**。设备配置：机器人本体为 **AgileX 桌面操作平台**；末端执行器正文未统一表式披露；感知为 **1 个固定 Intel RealSense L515**，用于 point-cloud observation；相机位姿数量 **1**，无 wrist/head 相机。
- **算力**：正文核验：所有模型均在 **NVIDIA RTX 4090 GPUs** 上训练；RoboTwin 与真机使用 **每任务 50 条 demonstrations**，MetaWorld/Adroit 使用 **每任务 10 条 demonstrations**。附录给出分 benchmark 超参表，但**统一 GPU 数量 / 总 GPU-hours 正文未披露**。推理侧正文给出历史编码成本：相较 Transformer，full-history Mamba 编码器在六任务分析子集上将 p95 编码延迟从 **3.61 ms** 降至 **1.97 ms**，峰值显存从 **586.2 MB** 降至 **238.5 MB**。
- **解读**：DSSP把“看全历史”做成紧凑的 state-space history encoder，核心价值是解决视觉别名与任务进度丢失。亮点是真机三项长程任务也显著受益；局限是仍依赖固定相机点云、任务覆盖偏桌面操作。复现时建议优先对齐 history encoder、dynamics-aware 辅助损失与 RoboTwin 长时序子集设置。

### B. 语言条件人形控制与前瞻式动作意图

#### 1) Before the Body Moves: Learning Anticipatory Joint Intent for Language-Conditioned Humanoid Control
- **标题**：Before the Body Moves: Learning Anticipatory Joint Intent for Language-Conditioned Humanoid Control
- **简称**：DAJI
- **领域**：Humanoid control / Language-conditioned control / Anticipatory latent action
- **论文**：https://arxiv.org/abs/2605.14417
- **项目**：https://hxxxz0.github.io/DAJI_PAGE/
- **代码**：正文、abs 页与项目页当前均未披露
- **评测**：仿真 benchmark：**HumanML3D-style robot motion benchmark** 与 **BABEL** 长时程 streaming instruction following。正文核验：主结果给出 **94.42% rollout success**（HumanML3D-style），并报告 **0.152 subsequence FID**（BABEL）。真机：**无实体机器人定量实验**。设备配置：论文评测运行在 **MuJoCo humanoid simulation** 与“robot-motion execution pipeline”；不涉及真实机器人本体、末端执行器和相机位姿，故真机配置 **不适用**。
- **算力**：正文/附录核验：**Teacher tracker 与 DAJI-Act** 训练使用 **8× NVIDIA A100 GPUs**，**DAJI-Flow generator** 训练使用 **4× NVIDIA A100 GPUs**；控制频率 **50 Hz**。表 1 给出部署延迟：**DAJI-Act 64D MLP 学生在 CPU 上 2.6 ms，成功率 91.8%，跌倒率 7.2%**；教师为 **1.73 ms（GPU）**。附录给出 generator **max training steps = 200,000**，但统一 GPU-hours 未披露。
- **解读**：DAJI的关键不是把语言直接翻成关节轨迹，而是先学一个“前瞻式 joint intent”接口，把接触变化、支撑转移和重心准备编码进去。亮点是生成与可执行性一起优化；局限是仍停留在仿真与导出验证。复现时建议先复刻 DAJI-Act 的 in-loop distillation 和 64D latent 接口，再接生成器。

### C. Flow-Matching / 生成式机器人策略

#### 1) WarmPrior: Straightening Flow-Matching Policies with Temporal Priors
- **标题**：WarmPrior: Straightening Flow-Matching Policies with Temporal Priors
- **简称**：WarmPrior
- **领域**：Flow-matching policy / 生成式机器人控制 / Temporal prior
- **论文**：https://arxiv.org/abs/2605.13959
- **项目**：https://sinnnj.github.io/WarmPrior/
- **代码**：正文、abs 页与项目页当前均未披露
- **评测**：仿真 benchmark：**Robomimic**（state 与 image 设定）和 **MimicGen**。正文核验：Robomimic 选取 **5 个 state tasks**，MimicGen 选取 **3 个 image tasks**；论文强调在低 NFE 和更难任务上优势更明显。真机：**Franka Research 3** 上 **4 个桌面任务**——**Food Waste Disposal、Cup Stacking、Block Stacking、Cable Insertion**。设备配置：机器人本体为 **Franka Research 3**；末端执行器正文未单列型号；真实视觉为 **3 个 RGB 相机（2 个 third-person view + 1 个 wrist）**，相机位姿数量 **3**。附录另报告对 **Block Throwing / Towel Folding** 的 RTC 对比实验，使用相同 Franka 平台、**20 trials/seed**。
- **算力**：正文/附录核验：仿真实验训练 **200k iterations**，batch size 为 **1024（state）/ 256（image）**；真机实验每任务 **30 条 demonstrations**、训练 **30k steps**、batch size **256**。表 3 显示 real-robot backbone 为 **GR00T N1.5-3B** 风格设置，仅更新 **action-head projector + DiT**；**GPU 型号/数量正文未明确披露**。评测次数：主真实机器人实验为 **50 trials per seed，3 个训练 seeds**。
- **解读**：WarmPrior很聪明的一点是没去改 action head 目标，而是改生成源分布：把前一段动作“加热”成先验，减少流场弯折。亮点是在仿真与 Franka 真机上都稳定涨点；局限是对快速重规划场景未必总优。复现时建议先从固定 σ 的 WP-Past 做起，并严格对齐 chunk 长度、NFE 与相机布局。

### D. 触觉具身感知与跨模态对齐

#### 1) Let Robots Feel Your Touch: Visuo-Tactile Cortical Alignment for Embodied Mirror Resonance
- **标题**：Let Robots Feel Your Touch: Visuo-Tactile Cortical Alignment for Embodied Mirror Resonance
- **简称**：Mirror-Touch-Net / AMTNet
- **领域**：Visuo-tactile learning / Embodied perception / Human-robot interaction
- **论文**：https://arxiv.org/abs/2605.14571
- **项目**：正文未披露
- **代码**：https://github.com/fun0515/Mirror-Touch-Net
- **评测**：仿真 benchmark：**无标准仿真 benchmark**。真机：围绕“robotic mirror touch”设置做真实系统实验。设备配置：机器人本体为 **一只拟人化机器人手**；末端执行器即 **机器人手本体**；触觉硬件为 **11 个电磁触觉传感器、总计 1,140 个 taxels**；视觉为 **1 个单目 RGB 相机** 观察触碰事件，相机位姿数量 **1**；无 wrist/head 相机体系。正文核验：AMTNet 可从 **30 Hz** 的人手被触图像预测机器人手上的对应触觉并诱发相应手指反应；物理触发阈值实验中使用 **0.2 N** 作为任一手指最大预测接触力阈值。
- **算力**：正文与补充材料聚焦模型与硬件原型，**训练 GPU / GPU 数量 / GPU-hours 正文未披露**。
- **解读**：这篇的亮点是把“看见别人被碰”映射成机器人自己的触觉表征，不只是多模态融合，而是借 cortical alignment 做可解释跨域迁移。局限也明显：任务仍偏感知演示，尚未闭环到复杂操作。复现时建议先搭好高分辨率触觉手、单目相机和 human-robot paired pose 数据采集链路。

### E. 主动感知与低成本具身视觉

#### 1) Behavior Cloning for Active Perception with Low-Resolution Egocentric Vision
- **标题**：Behavior Cloning for Active Perception with Low-Resolution Egocentric Vision
- **简称**：BC for Low-Res Active Perception
- **领域**：Active perception / Behavior cloning / Egocentric robot vision
- **论文**：https://arxiv.org/abs/2605.14106
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**无**。真机：单一真实 object-finding 任务。设备配置：机器人本体为 **桌面安装的 Lynxmotion AL5D 六自由度机械臂**；末端执行器为 **并联夹爪**；视觉为 **1 个 wrist-mounted 单目 RGB 相机**，分辨率 **64×64**、频率 **10 Hz**；相机位姿数量 **1**。任务要求机器人在目标植物初始仅部分可见时，主动移动腕部视角，使其居中并最终闭合夹爪完成任务。论文报告训练/测试在 **2、4、8 条 demonstrations** 下的 closed-loop 成功表现，并指出 **8 条 demonstrations** 时两种动作表示均可完成任务。
- **算力**：正文与附录 **未披露训练 GPU / GPU 数量 / GPU-hours**；从方法规模看为轻量端到端视觉编码器 + 时间控制器，但仅能确认控制输入输出与数据规模，无法据正文补齐硬件账单。
- **解读**：这篇胜在把主动感知压到一个极简、可复现的真实平台：低清腕部视角、少量示教、闭环 BC。亮点是说明“为了看清而动”不一定需要重型规划器；局限是任务结构化、背景单一。复现时建议严格保持相机帧率、64×64 输入和相对关节增量动作表示。

## Agent / OpenClaw 能力增强（固定栏目）

### 今日新增 / 值得关注
- **DSSP** 对 agent 控机器人很有启发：很多失败不是不会抓，而是“忘了做到哪一步”。历史压缩器比一味堆上下文更实用。
- **DAJI** 值得持续盯：它把语言到控制之间的接口改成“joint intent latent”，这对未来 agent 直接驱动人形或复杂全身设备很关键。
- **WarmPrior** 说明生成式策略还有很大“先验工程”空间，不一定非得改大模型本体。

### 能力提升方法 / 工作流要点
- 给 OpenClaw 类 agent 设计机器人控制链路时，今天这批论文给出三个直接建议：
  1. **历史不要只堆 token**，要做任务相关压缩（DSSP）；
  2. **控制接口不要只输出短视动作**，最好显式编码未来接触/支撑意图（DAJI）；
  3. **生成式策略的源分布可以工程化**，先验设计本身就是性能杠杆（WarmPrior）。

### 机器人 / 设备控制能力相关进展
- **Mirror-Touch-Net / AMTNet** 提醒我们：对真实机器人来说，视觉-触觉对齐不只是“多一个模态”，还可能成为预判接触与共享注意的基础能力。
- **Low-Res Active Perception** 则反过来说明，很多场景下先把主动视角调整做好，可能比盲目上更高分辨率、更大 backbone 更划算。

### 持续监控主题
- 长时序操控中的历史压缩、任务进度建模、反视觉别名
- 语言条件人形控制中的 latent intent / anticipatory action interface
- flow-matching / diffusion policy 的 source prior、chunking 与实时性折中
- 视觉-触觉对齐、主动感知、小样本真实机器人闭环学习
