### A. 机器人操控、世界模型与物理规划

#### 1) STARRY: Spatial-Temporal Action-Centric World Modeling for Robotic Manipulation
- **标题**：STARRY: Spatial-Temporal Action-Centric World Modeling for Robotic Manipulation
- **简称**：STARRY
- **领域**：双臂操作 / World Model / Action Generation
- **论文**：https://arxiv.org/abs/2604.26848
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：RoboTwin 2.0，覆盖 50 个双臂操作任务，在 Clean / Randomized 两种设置下联合训练并报告平均成功率，STARRY 达到 93.82% / 93.30%。真机：ARX R5 双臂机器人平台；相机位姿数量为 3（1×头部相机 + 2×腕部相机），末端执行器正文未细列；任务为 Hand Over Vegetables、Tidy Up Room、Wash Baby Bottle 三组两阶段任务，每任务 50 条真机示教训练、每方法 20 次 rollout 评估。
- **算力**：正文/附录披露微调与评测使用配备 8× NVIDIA Tesla A100-80G 的 GPU worker，单 worker 112 vCPU、1960 GiB 内存、80 Gbit/s 网络；典型训练约 1 周。大规模预训练使用同型 A100 节点并按阶段扩展，但未给出总卡时。
- **解读**：STARRY把未来时空 latent 与动作联合去噪，再用 GASAM 把预测深度和末端几何显式注入动作注意力，补足 reactive VLA 对接触关系建模不足。亮点是 RoboTwin 与真机双验证都强；局限是成本高、真机任务仍偏有限，复现宜先还原 3 相机 RGB-D 输入与两阶段训练流程。

#### 2) Bi-Level Optimization for Contact and Motion Planning in Rope-Assisted Legged Robots
- **标题**：Bi-Level Optimization for Contact and Motion Planning in Rope-Assisted Legged Robots
- **简称**：ALPINE Bi-Level Planner
- **领域**：攀爬机器人 / 接触规划 / 最优控制
- **论文**：https://arxiv.org/abs/2604.26910
- **项目**：正文未单列项目页
- **代码**：https://anonymous.4open.science/r/climbing_robots2-3D28 （论文正文给出匿名代码仓；PDF 文本中空格断裂，按链接规范还原）
- **评测**：仿真 benchmark：三类地形用例——半球障碍、Multi-Bulge 地形、Rocky terrain Gazebo simulation；在第三类中结合 MPC 与 ODE 物理引擎跟踪优化结果，地图大小 10×20 m、分辨率 0.5 m、路径间距 1 m。真机：ALPINE 新型绳索辅助攀爬机器人原型；机器人本体包含单条可伸缩腿、双锚点绳索与背部螺旋桨，末端为单足接触点；相机位姿数量正文未披露，实验主证据仍以动态仿真为主。
- **算力**：正文披露实验运行在 20-core Intel Core 7 笔记本 CPU（最高 4.9 GHz）；内层 OCP 用 C++ 实现、外层 CEM 用 Python 多线程并行。GPU/训练卡数不适用，统一总耗时未系统给出。
- **解读**：这篇把“落脚区域选择+绳力/腿力/落点优化”拆成外层 CEM 与内层非线性 OCP，适合强非凸攀爬场景。贡献在于把接触离散决策和动力学可行性耦合起来；局限是真机量化结果仍少，复现时建议先从三类高度图用例和并行 CEM 初始化策略入手。

#### 3) Safe Navigation using Neural Radiance Fields via Reachable Sets
- **标题**：Safe Navigation using Neural Radiance Fields via Reachable Sets
- **简称**：NeRF-ReachNav
- **领域**：安全导航 / NeRF / 可达集规划
- **论文**：https://arxiv.org/abs/2604.26899
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：两类路径规划仿真，一类为含多个障碍物的 3D 场景，另一类为 10 个随机障碍物场景；用 NeRF 重建障碍物/机器人几何，再结合 Minkowski sum 与 reachable tube 做约束最优控制。真机：正文未做实体机器人实验。设备配置：无真机机器人本体/末端/相机位姿数量披露。
- **算力**：正文披露 NeRF 训练与规划实验使用 16-core、2.3 GHz 处理器；NeRF 训练耗时约 8 分钟。GPU、显存、总卡时未披露。
- **解读**：作者把 NeRF 从“可看场景”推进到“可算安全约束”，核心是把体渲染几何转成凸包并嵌入可达集优化。亮点是表示统一；局限是仍停留在小规模仿真、实时性和噪声鲁棒性未充分证明，复现时应优先验证凸包近似与 reachable-tube 约束是否稳定。

### B. 长程导航与社会化机器人

#### 1) Walk With Me: Long-Horizon Social Navigation for Human-Centric Outdoor Assistance
- **标题**：Walk With Me: Long-Horizon Social Navigation for Human-Centric Outdoor Assistance
- **简称**：Walk With Me
- **领域**：户外社会导航 / 高层意图落地 / VLM+VLA
- **论文**：https://arxiv.org/abs/2604.26839
- **项目**：https://linglingxiansen.github.io/Walk-with-Me/
- **代码**：正文未披露
- **评测**：仿真 benchmark：正文未设置仿真 benchmark，核心评估完全基于真实户外场景。真机：Athena 2.0 Pro AGV 轮式机器人 + 前向 1× Intel RealSense D455 RGB-D 相机 + GPS + 机载 SLAM；共 1 个相机位姿。主实验覆盖 last-mile delivery（Milk tea to Building B、Package to Building A）与 blind guidance（Go for a walk、Go shopping）四个场景，各 5 次独立试验，共 20 次真机 trial，整体成功率 60%。
- **算力**：正文披露低层 VLA 与开源高层模型部署在配备 NVIDIA H20 GPU 的远程服务器；闭源高层 VLM 通过官方 API 调用。未披露 GPU 数量、显存占用与累计推理成本。
- **解读**：这套系统把“抽象意图→POI 落点→粗路由→观察感知路由→低层 VLA 执行”串成完整户外闭环，避免依赖 HD map。亮点是公里级真实任务；局限是成功率仍只有 60%，且主要验证单相机轮式平台，复现时先固定地图 API 与 stop-and-wait 安全策略最稳。

#### 2) Three-Step Nav: A Hierarchical Global-Local Planner for Zero-Shot Vision-and-Language Navigation
- **标题**：Three-Step Nav: A Hierarchical Global-Local Planner for Zero-Shot Vision-and-Language Navigation
- **简称**：Three-Step Nav
- **领域**：VLN / 零样本导航 / 分层规划
- **论文**：https://arxiv.org/abs/2604.26946
- **项目**：正文未单列项目页
- **代码**：https://github.com/ZoeyZheng0/3-step-Nav
- **评测**：仿真 benchmark：R2R-CE 与 RxR-CE 两个连续体 VLN benchmark，基于 Habitat 模拟器与全景 RGB-D 候选视点；在 R2R-CE val-unseen 上达到 57.70 nDTW、5.87 NE，并在 RxR-CE 上取得同类 zero-shot SOTA。真机：正文未进行实体机器人实验。设备配置：无真机机器人本体/末端/相机位姿数量；仿真输入为每步 12 张 RGB 图像与对应深度视图。
- **算力**：正文披露核心实验调用 gpt-5-2025-08-07 API 于 Habitat 模拟环境中完成；未披露 GPU/TPU 卡数、推理 token 成本与总时长，记为“正文未披露”。
- **解读**：Three-Step Nav 的“look forward / look now / look backward”三视角协议，本质是在 zero-shot VLN 中补上全局地标规划、局部对齐和停前审计三层纠偏。优点是无需训练即可插入现有流水线；局限是尚未落到真实机器人，复现应先把 waypoint proposal 和 backward audit 独立做消融。

### C. Agent 系统与训练基础设施

#### 1) ClawGym: A Scalable Framework for Building Effective Claw Agents
- **标题**：ClawGym: A Scalable Framework for Building Effective Claw Agents
- **简称**：ClawGym
- **领域**：Claw-style Agent / 数据合成 / Benchmark / Agent Training
- **论文**：https://arxiv.org/abs/2604.26904
- **项目**：https://clawhub.ai/
- **代码**：https://github.com/ClawGym
- **评测**：仿真 benchmark：ClawGym-Bench（200 个 workspace-grounded 实例，六类任务）与 PinchBench（2026-04-10 发布版本中保留 30 个非多模态任务）；论文还报告 50 题重复评测稳定性分析与 RL 训练曲线。真机：不适用。设备配置：非机器人论文，无机器人本体、末端执行器与相机位姿。
- **算力**：正文给出 RL 训练超参：以 2,000 个任务、每 prompt 8 次 rollout、100 training steps、最大 64K token 响应长度进行轻量 RL；但未系统披露 GPU 型号、卡数、显存与总训练时长，记为“正文未披露”。
- **解读**：ClawGym补的不是单一 agent trick，而是 Claw 环境里的全链路开发底座：13.5K 合成任务、24.5K 过滤轨迹、200 题评测集和可并行 sandbox RL。贡献很实用；局限是不少结果依赖合成分布，复现时最好先检查 verifier 质量，再谈模型规模提升。
