## 主清单（单一合并版，按技术重点分类）

### A. VLA / 4D 感知推理 / Offline-to-Online RL

#### 1) ConsisVLA-4D: Advancing Spatiotemporal Consistency in Efficient 3D-Perception and 4D-Reasoning for Robotic Manipulation
- **标题**：ConsisVLA-4D: Advancing Spatiotemporal Consistency in Efficient 3D-Perception and 4D-Reasoning for Robotic Manipulation
- **简称**：ConsisVLA-4D
- **领域**：VLA / 3D感知 / 4D推理 / 双臂操作
- **论文**：https://arxiv.org/abs/2605.05126
- **项目**：正文未披露
- **代码**：https://github.com/iLearn-Lab/CVPR26-ConsisVLA-4D （PDF 首页给出 `JiuTian-VL/ConsisVLA-4D`，GitHub 实际跳转并归档到 `iLearn-Lab/CVPR26-ConsisVLA-4D`）
- **评测**：仿真 benchmark：LIBERO 四个 suites、ManiSkill2 三个 pick-and-place 任务、RoboTwin 2.0 的 ALOHA 风格双臂任务。真机：AgileX Cobot Magic 与 Galaxea R1 Lite 两个平台，覆盖 Microwave Operation、Banana Peeling、Drawer Arrangement、T-shirt Folding。设备配置：AgileX Cobot Magic 为移动底盘 + 多臂协作系统 + RGB-D；Galaxea R1 Lite 为移动底盘 + 双臂夹爪。策略层正文重点展示 Main View + Wrist View 两视角；平台级还带双目、单目深度与 LiDAR，但正文未完全逐一说明哪些全部进入策略。
- **算力**：正文/附录披露训练使用 4×A800 GPUs；真机推理在单张 RTX 5090。训练步数：仿真 120K steps、batch size 64；真机 80K steps、batch size 32；每 10K steps 做 checkpoint 评估。正文给出 latency / throughput / 训练成本对比。
- **解读**：这篇不是简单给 VLA 加 3D token，而是把跨视角对象对齐、跨物体几何关系和跨时间场景推理串成闭环，因此在 LIBERO 和双臂真机上都吃到收益。局限是平台传感器与策略输入边界写得不够统一，复现时建议先还原 Main+Wrist 两视角版本，再逐步加深度/多视角模块。

#### 2) When Life Gives You BC, Make Q-functions: Extracting Q-values from Behavior Cloning for On-Robot Reinforcement Learning
- **标题**：When Life Gives You BC, Make Q-functions: Extracting Q-values from Behavior Cloning for On-Robot Reinforcement Learning
- **简称**：Q2RL
- **领域**：Offline-to-Online RL / Behavior Cloning / On-Robot RL
- **论文**：https://arxiv.org/abs/2605.05172
- **项目**：http://q2rl.rai-inst.com/
- **代码**：正文未披露（论文首页与项目页可核验到项目站，但未给稳定代码仓直链）
- **评测**：仿真 benchmark：D4RL（Adroit Pen、Adroit Door、Kitchen）与 robomimic（Lift、Can、Square，含 state/image 两种设定）。真机：Peg Insertion、Plumbing Pipe Assembly、Kitting with Task Distribution Shift。设备配置：桌面 Franka FR3 / Panda 系机械臂 + Robotiq 2F-85 夹爪（附 3D 打印 finray 指尖）；2 路 Intel RealSense D405，分别为 workspace view 与 wrist view；另有 wrist 力矩传感器但未用于策略。
- **算力**：正文披露 BC 策略训练使用单张 V100 GPU；真机在线训练每个任务最多约 2.5 小时，对应不超过 170k gradient steps 与 80k environment steps；系统平均每小时约执行 13k RL actions、44k learner steps。更完整的统一显存/功耗账单正文未披露。
- **解读**：Q2RL 的巧劲在于先从 BC 提炼 Q，再用 Q-gating 让 BC 和 RL 分工：稳定动作继续交给 BC，接触密集或分布外段落再让 RL 接管。优势是上机样本效率高；局限是仍需人工 reset/成功判定，复现时应优先还原双 D405 输入、稀疏奖励和 replay seeding 细节。

### B. 触觉建模 / Differentiable Simulation

#### 1) Reduced-order Neural Modeling with Differentiable Simulation for High-Detail Tactile Perception
- **标题**：Reduced-order Neural Modeling with Differentiable Simulation for High-Detail Tactile Perception
- **简称**：Tac2Detail
- **领域**：触觉仿真 / Differentiable Simulation / 触觉感知
- **论文**：https://arxiv.org/abs/2605.05053
- **项目**：https://tac2detail.github.io （发布时返回 404）
- **代码**：正文未披露
- **评测**：仿真 benchmark：与 Tacchi、TacIPC 对比高分辨率触觉表面重建与几何误差。真机：UR5e + GelSight Mini 光学触觉传感器，8 个 FDM 打印复杂表面物体、每个物体 3×3 接触位置、压入深度 0–0.5 mm 形成真实 benchmark。设备配置：UR5e 机械臂；末端为 GelSight Mini 触觉传感器；相机位姿数量为 1 个传感器内置相机（位于弹性体下方），正文未涉及外部 wrist/head 相机阵列。
- **算力**：正文披露训练数据来自高/低分辨率 MPM 配对仿真，约 6k 训练对；Adam 训练 3k epochs，batch size 32，在单张 RTX 3090 上每 epoch 约 7 秒、总收敛约 12 小时。推理侧单张 RTX 3090 上全细节重建约 1.20 分钟，比 TacIPC 快约 3×。
- **解读**：这篇把“粗粒度物理 + 神经超分重建”组合起来，击中了高保真触觉模拟太慢的核心痛点。亮点是物理一致性和可微性兼得；局限是真机验证仍偏离线重建而非闭环操控，复现时建议先严格重建 GelSight 数据采集与 latent 优化，再谈策略耦合。

### C. 纯RL / 特种运动控制

#### 1) LineRides: Line-Guided Reinforcement Learning for Bicycle Robot Stunts
- **标题**：LineRides: Line-Guided Reinforcement Learning for Bicycle Robot Stunts
- **简称**：LineRides
- **领域**：纯RL / 自行车机器人 / 特技控制
- **论文**：https://arxiv.org/abs/2605.05110
- **项目**：https://seungeunrho.github.io/projects/LineRides/
- **代码**：正文未披露
- **评测**：仿真 benchmark：IsaacLab 中训练并评估 MiniHop、LargeHop、ThreePointsTurn、DriftTurn、Backflip 五类动作；DriftTurn 和 Backflip 仅在仿真验证。真机：Ultra Mobility Vehicle（UMV）平台上验证 MiniHop、LargeHop、ThreePointsTurn，并报告重复执行 100% 成功率。设备配置：UMV 为自研双轮“儿童自行车”式平台，4 个执行关节（upper-body、lower-body、fork、rear wheel）；无机械臂末端；正文未披露用于策略输入的外部相机，属于以本体状态为主的控制设定，相机位姿数量可记为正文未披露/不适用。
- **算力**：正文披露训练在 IsaacLab 高吞吐 GPU 仿真器完成，不同任务在单张 NVIDIA L40 GPU 上约 12–24 小时。统一训练步数、显存曲线正文未披露。
- **解读**：LineRides 的贡献是把“用户画一条线”变成 stunt reward 规范，而不是依赖演示轨迹或手搓复杂奖励。对奇特平台很有吸引力；局限是目前主要靠几何线索和稀疏关键姿态，复现时应先从 MiniHop/ThreePointsTurn 这类低风险技能验证奖励设计，再尝试更激进动作。
