### A. 低时延推理与在线控制

#### 1) Realtime-VLA FLASH: Speculative Inference Framework for Diffusion-based VLAs
- **标题**：Realtime-VLA FLASH: Speculative Inference Framework for Diffusion-based VLAs
- **简称**：FLASH
- **领域**：Diffusion-based VLA / Speculative inference / Low-latency control
- **论文**：https://arxiv.org/abs/2605.13778
- **项目**：https://dexmal.github.io/realtime-vla-flash
- **代码**：https://github.com/dexmal/realtime-vla-flash
- **评测**：仿真 benchmark：**LIBERO** 四个 suite（Spatial / Object / Goal / LIBERO-10），正文核验为 **每个 suite 10 个任务、每任务 50 trials**。真机：**单臂 UR5** 传送带分拣。设备配置：机器人本体为 **UR5**；末端执行器正文未单列型号；视觉为 **2 个 Intel RealSense D435i** 多视角观测，相机位姿数量 **2**。真实数据收集 **200 demonstrations**，每个真实条件 **10 trials**。
- **算力**：正文核验在线推理运行于 **1× NVIDIA RTX 4090D**。附录披露：LIBERO 草稿模型训练使用 **4× NVIDIA RTX 4090D，单个 draft model 约 6 小时**；真机 LoRA 微调与草稿训练使用 **8× NVIDIA H20，分别约 12 小时与 2 小时**。统一总 GPU-hours 正文未披露。
- **解读**：这篇把 dVLA 的慢推理问题改写成“草稿-校验”式 speculative inference，并用 phase-aware fallback 防止精细阶段误差累积。亮点是几乎不改原策略就把任务级延迟降到 19.1ms；局限是阈值与 fallback 规则仍偏启发式，复现时要优先对齐 acceptance 阈值、回退时机与 action chunk 配置。

### B. 训练效率与数据压缩

#### 1) FrameSkip: Learning from Fewer but More Informative Frames in VLA Training
- **标题**：FrameSkip: Learning from Fewer but More Informative Frames in VLA Training
- **简称**：FrameSkip
- **领域**：VLA training efficiency / Keyframe selection / Data-efficient learning
- **论文**：https://arxiv.org/abs/2605.13757
- **项目**：正文未披露
- **代码**：https://github.com/ZGC-EmbodyAI/FrameSkip
- **评测**：仿真 benchmark：**RoboCasa-GR1、SimplerEnv、LIBERO**。正文核验：RoboCasa-GR1 使用 **24 个 tabletop tasks** 与 **24K GR1 teleoperation simulation demonstrations**；SimplerEnv 使用 **4 个 held-out WidowX 操作任务**，训练数据来自 **BridgeV2**；LIBERO 覆盖 **Spatial / Object / Goal / Long** 四个 suite。真机：**无独立真实机器人实验**。设备配置：均为仿真/real-to-sim benchmark，机器人本体分别依 benchmark 为 **GR1 / WidowX / Franka 仿真体**；末端执行器与相机位姿数量沿 benchmark 默认设定，论文未统一表格式披露。
- **算力**：正文明确训练使用 **8× NVIDIA H100 GPUs**（DeepSpeed ZeRO2）。各 benchmark 训练步数正文披露为：RoboCasa-GR1 **100K steps**、SimplerEnv **60K steps**、LIBERO **30K steps**；统一总 GPU-hours 正文未披露。
- **解读**：FrameSkip 不改 VLA 主体，只在 dataloader 侧按动作变化、视觉-动作一致性、任务进度先验和夹爪切换去挑关键帧，属于很“工程友好”的加速思路。亮点是跨三个主流 benchmark 都涨点；局限是真机闭环尚未验证，复现时建议先复刻它的 progress prior 拟合与 gripper-transition 保留规则。

### C. 动作建模与表示学习

#### 1) AttenA+: Rectifying Action Inequality in Robotic Foundation Models
- **标题**：AttenA+: Rectifying Action Inequality in Robotic Foundation Models
- **简称**：AttenA+
- **领域**：Robotic foundation model / Action reweighting / Precision-critical manipulation
- **论文**：https://arxiv.org/abs/2605.13548
- **项目**：正文未披露
- **代码**：正文、abs 页与当前可核验页面均未披露
- **评测**：仿真 benchmark：**LIBERO** 四类任务与 **RoboTwin 2.0**。正文核验：RoboTwin 2.0 覆盖 **50 个双臂操作任务**，含 clean / randomized 设置。真机：**Franka Panda 单臂平台**，共 **4 类任务**（drawer opening、pick-and-place、multi-objects、两阶段 sequential placement）。设备配置：机器人本体为 **Franka Panda**；末端执行器正文未单列型号；相机系统正文未给出统一配置表，**相机位姿数量正文未披露**。示教数据：简单任务各 **50 trajectories**，较难任务各 **100 trajectories**；每任务测试 **50 trials**。
- **算力**：正文核验真机微调遵循 OpenVLA-OFT recipe，使用 **2× NVIDIA H800 GPUs**；测试部署在 **1× RTX 4090**。附录披露：LIBERO 单卡 H800 训练约 **35 小时**；RoboTwin 使用 **2× H800，约 4 天**。统一总 GPU-hours 正文未披露。
- **解读**：AttenA+ 的核心观点很直接：机器人动作并不“步步等价”，低速、精细、接触敏感阶段更该被放大监督。亮点是对 OpenVLA-OFT 和 FastWAM 两条线都能稳健增益；局限是重要性近似仍主要依赖速度代理，复现时需重点校准 weighting map、clip 上限与不同任务族的速度分布。

#### 2) RotVLA: Rotational Latent Action for Vision-Language-Action Model
- **标题**：RotVLA: Rotational Latent Action for Vision-Language-Action Model
- **简称**：RotVLA
- **领域**：VLA / Latent action / SO(n) action representation
- **论文**：https://arxiv.org/abs/2605.13403
- **项目**：https://qiweili00.github.io/rotvla-page/
- **代码**：https://github.com/qiweili00/RotVLA
- **评测**：仿真 benchmark：**LIBERO** 与 **RoboTwin 2.0**。正文核验：LIBERO 四个 suite，每个 **10 个任务**，联合训练单一多任务模型；RoboTwin 2.0 覆盖 **50 个双臂协作任务**，每任务含 **50 clean demonstrations + 500 randomized demonstrations**，评测 **100 rollouts/task**。真机：**双臂 ARX R5 平台**，共 **3 个任务**：两类单臂任务（pick-and-place、另一个单臂操作任务）与 **1 个双臂三杯堆叠**。设备配置：机器人本体为 **双臂 ARX R5**；末端执行器正文未单列型号；视觉为 **3 个 RealSense D405**，其中 **1 个头部 + 2 个腕部**，相机位姿数量 **3**。每个真机任务 **100 demonstrations**。
- **算力**：正文明确预训练使用 **8× NVIDIA H200 GPUs，50 小时**；附录披露下游训练步数为 **LIBERO 80K、RoboTwin 120K、真机 40K**。统一总 GPU-hours 正文未披露。
- **解读**：RotVLA 把 latent action 约束到旋转群结构里，用可组合的 SO(n) 表示替代更随意的连续潜变量，试图让动作组合更贴近真实控制几何。亮点是 1.7B 规模下在 LIBERO/RoboTwin 都很强；局限是表示假设较强、训练链路更复杂，复现时要先对齐 orthogonal projection、triplet frame 设计和 flow-matching action expert。

### D. 交互式具身推理与可引导控制

#### 1) Guide, Think, Act: Interactive Embodied Reasoning in Vision-Language-Action Models
- **标题**：Guide, Think, Act: Interactive Embodied Reasoning in Vision-Language-Action Models
- **简称**：GTA-VLA
- **领域**：Interactive embodied reasoning / Spatial CoT / Guided VLA
- **论文**：https://arxiv.org/abs/2605.13632
- **项目**：https://signalispupupu.github.io/GTA-VLA_ProjPage/
- **代码**：https://github.com/FutianLabs/GTA-VLA
- **评测**：仿真 benchmark：**LIBERO、SimplerEnv（Bridge / WidowX）**，并新增 **SimplerEnv-Plus** 用于 OOD 评测。正文核验：SimplerEnv-Plus 系统扰动涵盖 **camera variation、lighting changes、unseen objects、language perturbations**。真机：**Agile Piper 单臂平台**，评测 **4 个真实抓取任务**，按“是否见过目标 / 是否存在多个候选物体”两轴组织。设备配置：机器人本体为 **Agile Piper**；末端执行器正文未单列型号；视觉为 **1 个主视角 RealSense D435 + 1 个腕部 RealSense D435**，相机位姿数量 **2**。
- **算力**：正文/附录核验：预训练使用 **48× NVIDIA H800 GPUs**，各类微调使用 **16× NVIDIA H800 GPUs**；实验评测在 **NVIDIA L20** 上完成。真机推理工作站为 **2× NVIDIA RTX 5090**，其中 VLM 分支与动作头分开部署。统一总 GPU-hours 正文未披露。
- **解读**：GTA-VLA 把“看图直接动手”改成“先做空间-视觉 CoT，再走快控制分支”，并允许人类在失败时给点、框、轨迹引导。亮点是 OOD 与歧义场景提升明显，且真机能落地；局限是链路更长、标注与自动生成中间监督较重，复现时应先固定 Spatial-CoT 序列化格式和主/腕视角融合方式。
