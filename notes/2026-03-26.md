### A. 主动协作与知识增强操作

#### 1) Event-Driven Proactive Assistive Manipulation with Grounded Vision-Language Planning
- **标题**：Event-Driven Proactive Assistive Manipulation with Grounded Vision-Language Planning
- **简称**：Event-Driven Assistive Manipulation
- **领域**：主动协作操作 / Grounded VLM Planning
- **论文**：https://arxiv.org/abs/2603.23950
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：无。真机：桌面数字块协作任务，40 次试验（可解 20、不可解 20），比较 proposed / post-only / always-on / request-driven。设备配置：Yahboom JetCobot 平台，含 Elephant MyCobot 280 for Arduino 机械臂；外置 1 路 Intel RealSense D435i RGB-D 相机俯视桌面；末端执行器为 MyCobot 抓取末端；wrist/head 相机位姿数量正文未披露。
- **算力**：正文披露推理端使用 RTX 6000 Ada GPU 做 VLM reasoning；事件前后处理运行在 Jetson Orin 8GB 机器人端。训练资源正文未披露。
- **解读**：亮点是把“等人下指令”改成“看见状态变化就主动补下一步”，并用 pre/post 快照约束规划输出。适合协作补位类任务，但当前任务仍较结构化；复现时应先保证事件检测稳定和对象 ID 绑定可靠，否则高层规划很容易漂移。

#### 2) Knowledge-Guided Manipulation Using Multi-Task Reinforcement Learning
- **标题**：Knowledge-Guided Manipulation Using Multi-Task Reinforcement Learning
- **简称**：KG-M3PO
- **领域**：多任务操作 / 知识图谱强化学习
- **论文**：https://arxiv.org/abs/2603.24083
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：基于 NVIDIA Isaac Lab / Isaac Sim 自建多任务操作基准，覆盖 Franka Panda 与 UR5 两种平台，含 fully observable 与 partially observable 任务，重点场景包括 pick、pick-place、open-cabinet、被遮挡取物等。真机：无。设备配置：Franka 7-DoF + 原生 parallel-jaw gripper；UR5 6-DoF + Robotiq 2F 系列 parallel-jaw gripper；1 路 wrist-mounted 64×64 pinhole camera 作为相机输入，head/外置相机不适用。
- **算力**：正文披露在单机 NVIDIA RTX 5070 Ti（16GB VRAM）+ 64GB RAM 上运行 1024 并行环境；表中给出 400M steps 训练成本，M3PO 为 52,734s / 0.610 GPU-day，KG-M3PO 为 68,555s / 0.793 GPU-day。未披露更细训练批大小与多卡设置。
- **解读**：这篇的关键不是再堆一个世界模型，而是把动态 3D scene graph 直接接进 RL 计算图，让关系表示受控制目标反向塑形。对遮挡和容器内目标尤其有效；局限是目前仍停留在仿真。复现建议先把图更新频率、检测噪声和多任务共享表示拆开消融。

### B. 人形与具身控制

#### 1) SafeFlow: Real-Time Text-Driven Humanoid Whole-Body Control via Physics-Guided Rectified Flow and Selective Safety Gating
- **标题**：SafeFlow: Real-Time Text-Driven Humanoid Whole-Body Control via Physics-Guided Rectified Flow and Selective Safety Gating
- **简称**：SafeFlow
- **领域**：Humanoid Control / Text-to-Motion / 安全控制
- **论文**：https://arxiv.org/abs/2603.23983
- **项目**：https://hanbyelcho.info/safeflow/
- **代码**：正文未披露
- **评测**：仿真 benchmark：在 MuJoCo 做系统级验证，并以 BABEL validation prompts 评估可执行性、碰撞率、关节越界率与追踪成功率；低层 controller 训练在 Isaac Lab。真机：Unitree G1 长时文本驱动动作序列部署。设备配置：Unitree G1 humanoid；末端为双臂/全身关节执行，不是抓取型末端任务；相机不作为主评测输入，camera 位姿数量不适用。
- **算力**：正文披露高层生成器训练 200k iterations，reflow 蒸馏再 200k iterations；推理延迟评测在单张 NVIDIA RTX A6000 上完成，完整 guarded generator 为 14.78 ms（约 67.7 Hz）；低层 tracking controller 运行在机载 Jetson Orin，50 Hz。GPU 数量正文未披露。
- **解读**：SafeFlow把“可说人话控制人形”真正往可部署方向推了一步：生成阶段加物理引导，执行阶段再叠三层安全门。优点是实时性和安全性都兼顾；局限是动作域仍依赖 BABEL 重定向。复现时别只看生成器，安全门阈值和 controller 一致性同样关键。

### C. 移动操作与搜索规划

#### 1) Object Search in Partially-Known Environments via LLM-informed Model-based Planning and Prompt Selection
- **标题**：Object Search in Partially-Known Environments via LLM-informed Model-based Planning and Prompt Selection
- **简称**：LLM-informed Object Search
- **领域**：目标搜索 / LLM 规划 / 移动机器人
- **论文**：https://arxiv.org/abs/2603.23800
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：150 个 ProcTHOR 地图中的部分已知环境目标搜索，对比 LLM+MODEL、LLM-DIRECT 与不同 prompt/LLM 选择策略。真机：LoCoBot 公寓搜索实验，场景含 kitchen、dining room、living room、bedroom 共 9 个可搜索容器，5 个目标物体试验（blanket、cell phone、remote、pillow、wallet）。设备配置：LoCoBot 移动机器人；末端抓取不在主任务闭环中；相机与传感器安装位姿正文未细列。
- **算力**：正文未披露训练 GPU/时长；仅披露部署侧使用 GPT-5 Mini、Gemini 2.5 Flash 等 LLM，并给出 token 成本统计。属于规划与选择框架论文，统一算力账单正文未披露。
- **解读**：作者没让 LLM 直接拍脑袋选下一步，而是让 LLM 提供“位置找到目标的概率先验”，再交给模型规划器算长期代价，这一点很务实。局限是硬件与感知栈写得不细；复现时建议先复刻其高层 action abstraction 和 replay-based prompt selection，再换自家机器人底盘。

### D. 农业机器人与可达性建模

#### 1) Learning What Can Be Picked: Active Reachability Estimation for Efficient Robotic Fruit Harvesting
- **标题**：Learning What Can Be Picked: Active Reachability Estimation for Efficient Robotic Fruit Harvesting
- **简称**：Active Reachability for Harvesting
- **领域**：农业机器人 / 可达性预测
- **论文**：https://arxiv.org/abs/2603.23679
- **项目**：正文未披露
- **代码**：https://github.com/wsu-cyber-security-lab-ai/active-learning
- **评测**：仿真 benchmark：无。真机/实景数据：华盛顿州商业果园 RGB-D 采集与机器人苹果采摘平台可达性评估。设备配置：5-DoF 采摘机械臂（2 个 prismatic + 3 个 revolute joints）；末端为固定俯仰的采摘末端，用于抓取并拉拽果实；1 路 Kinect v2 采集 RGB-D，外置视角 1 个，wrist/head 相机不适用。数据集 3,480 对同步 RGB-depth，工作子集 2,519 对。
- **算力**：正文未披露 GPU 型号/卡数/训练时长；披露主动学习设置为初始标注集 |L|=800、未标注池约 24,000、每轮 batch query 50/100，并在多轮增量训练下评估。严格意义上的训练算力正文未披露。
- **解读**：这篇很接地气：不再对每个苹果都跑一次 IK，而是先学“这个果能不能摘”，把昂贵几何求解后移。贡献主要在系统吞吐而非算法新奇性；局限是仍偏特定平台。复现时建议优先校准相机外参与 reachability 标注协议，否则主动学习收益会被噪声吃掉。

### E. 科学实验机器人平台

#### 1) AgentChemist: A Multi-Agent Experimental Robotic Platform Integrating Chemical Perception and Precise Control
- **标题**：AgentChemist: A Multi-Agent Experimental Robotic Platform Integrating Chemical Perception and Precise Control
- **简称**：AgentChemist
- **领域**：实验室机器人 / 多智能体具身系统
- **论文**：https://arxiv.org/abs/2603.23886
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：无。真机：酸碱滴定、络合滴定、固体称量与溶解等实验流程。设备配置：AgileX Mobile ALOHA 移动双臂平台；双 7-DoF Piper Standard 机械臂 + gripper；3 路 Orbbec Dabai DC1 深度相机（左腕、右腕、前视）+ 2 个集成在 gripper 内的接触麦克风；并接入 pH、温度、电导等化学传感器。相机位姿数量=3。
- **算力**：正文披露策略/多模态模型训练使用 8× NVIDIA H20 GPUs，global batch size 1024，200,000 iterations，bfloat16 mixed precision，训练约 4 天。推理 latency/FPS/显存正文未给统一账单。
- **解读**：AgentChemist真正有意思的地方在于把化学实验建成 FSM 驱动的多智能体闭环：规划、视觉监督、音频监督、动作执行、总结报告串起来了。它比“机械臂自动移液”更接近实验助手。局限是工程耦合较重；复现时要先打通多模态事件对齐和异常回退机制。
