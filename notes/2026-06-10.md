### A. VLA 推理加速与人机协同控制

#### 1) SAFE-Pruner: Semantic Attention-Guided Future-Aware Token Pruning for Efficient Vision-Language-Action Manipulation
- **标题**：SAFE-Pruner: Semantic Attention-Guided Future-Aware Token Pruning for Efficient Vision-Language-Action Manipulation
- **简称**：SAFE-Pruner
- **领域**：VLA / token pruning / real-time manipulation
- **论文**：https://arxiv.org/abs/2605.29662
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**LIBERO** 四个 suite、共 **40 tasks / 2000 evaluation episodes**；另在 **SIMPLER** 的 **Visual Matching + Variant Aggregation** 两设定、各含 **4 个 manipulation tasks** 上评测。真机：**Astribot S1 双臂**，任务为 **Pick and Place / Throw Basketball / Pack Doll**，每个 task-specific 模型 **20 trials**。设备配置：**Astribot S1 dual-arm，14-DoF arms + grippers**；**1 个 head RGB（Orbbec Femto Bolt）+ 2 个 wrist RGB（文中图示 Intel RealSense D401）**，共 **3 个机位**。
- **算力**：训练/评测资源：除真机外，正文披露实验统一在 **单张 NVIDIA RTX 4090 24GB** 上完成；真机对比表中的 VLM backbone latency 额外在 **RTX 3090** 上测得。部署结果：真机表中 SAFE-Pruner 将 π0.5 延迟从 **80.63 ms** 降到 **43.72 ms**，FLOPs 从 **2.264T** 降到 **1.543T**。披露状态：正文给出评测 GPU 与延迟/FLOPs，但未给总训练时长。
- **解读**：核心价值是把“未来层注意力”拉进裁剪决策，避免浅层过早删掉后续动作真正依赖的视觉 token。优点是仿真与真机都能换来近 1.9× 加速且成功率损失很小；局限是仍属推理侧补丁。复现时先对齐 π0.5/OpenVLA-OFT 的 saliency 抽取层位与子任务切分策略。

#### 2) Flow Control: Steering Vision-Language-Action Models with Simple Real-Time Inputs
- **标题**：Flow Control: Steering Vision-Language-Action Models with Simple Real-Time Inputs
- **简称**：Flow Control
- **领域**：VLA / shared autonomy / real-time steering
- **论文**：https://arxiv.org/abs/2606.10180
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**正文未设置标准公开仿真 benchmark**。真机：基于 **π0.5-DROID** 在 **Franka Panda** 上完成 **Two-block pick-and-place / Five-block pick-and-place / Marker-in-Bowl / Cup-Stacking** 四类任务；用户研究包含 **16 名参与者**，每人对 Marker-in-Bowl 与 Cup-Stacking 各做 **10 trials**，另可有最多 5 次练习。关键结果：Marker-in-Bowl 成功率 **53.3% → 99.4%**，Cup-Stacking **48.0% → 87.7%**。设备配置：**Franka Panda 单臂 + gripper**；**1 个 scene camera（ZED）+ 1 个 wrist camera（ZED Mini）**，共 **2 个机位**。
- **算力**：推理资源：正文披露所有推理均在 **单张 NVIDIA GeForce RTX 5090** 上完成。训练资源：主方法不需重训；附加微调实验使用 **60 条成功的 flow-control trials**，GPU 卡数与总时长 **正文未披露**。披露状态：部署 GPU 清楚，训练账单有限。
- **解读**：这篇最实用的点是把键盘等粗粒度输入直接变成 VLA 的“同分布可控初值”，因此不改模型也能实时纠偏。亮点是共享自治效果很硬；局限是目前验证任务仍偏桌面单臂。复现时优先保住 flow-matching 初值注入，而不是先急着做复杂 UI。

### B. 泛化操作策略与数据采集基础设施

#### 1) GHOST: Hierarchical Sub-Goal Policies for Generalizing Robot Manipulation
- **标题**：GHOST: Hierarchical Sub-Goal Policies for Generalizing Robot Manipulation
- **简称**：GHOST
- **领域**：hierarchical policy / manipulation generalization / human-demo transfer
- **论文**：https://arxiv.org/abs/2606.10025
- **项目**：https://ghost-human-demo.github.io/
- **代码**：https://github.com/r-pad/lfd3d
- **评测**：仿真 benchmark：**正文未设置标准公开仿真 benchmark**。真机：基于 **ALOHA** 右侧 leader-follower 单臂链路评测 **pick-and-place / fold-onesie / fold-towel / hammer-pin** 等任务，所有实验 **每方法 30 trials**。数据规模：如 **mug/plate pick-place** 多为 **20 robot demos**，**fold-onesie 33 robot demos**，**fold-towel 50 human demos**。结果上，pick-and-place OOD 项中 GHOST 达 **98.3%**，fold-onesie 最终成功率 **93.3%**，fold-towel 组合泛化 **36.7%**。设备配置：**ALOHA 平台，WidowX leader + ViperX follower，但仅右侧一对处于工作状态；末端 gripper 宽度纳入动作空间**；**2 个 Azure Kinect workspace cameras + 1 个 Intel RealSense wrist camera**，共 **3 个机位**。
- **算力**：训练资源：正文披露低层策略 **100 epochs / batch size 128**，高层策略 **batch size 4**，但 **GPU 型号、卡数与总训练时长正文未披露**。推理资源：附录给出在 **单张 RTX 4090** 上的 wall-clock 推理时间，GHOST 为 **149.3 ± 2.1 ms/step**。披露状态：训练超参较完整，硬件账单不完整。
- **解读**：GHOST 的关键不是再堆一个 planner，而是把高层子目标落成 3D 末端位姿，再投影成低层能吃的热图接口，所以人类视频也能直接拿来补高层泛化。亮点是 onesie/towel 这类跨对象、跨上下文任务明显受益；局限是速度慢于扁平 DP。复现时先把高层 3D goal→2D heatmap 接口做准。

#### 2) YUBI: Yielding Universal Bidigital Interface for Bimanual Dexterous Manipulation at Scale
- **标题**：YUBI: Yielding Universal Bidigital Interface for Bimanual Dexterous Manipulation at Scale
- **简称**：YUBI
- **领域**：bimanual dexterous manipulation / data collection / embodiment transfer
- **论文**：https://arxiv.org/abs/2606.10244
- **项目**：https://yubi.airoa.io/
- **代码**：软件：https://github.com/airoa-org/yubi-sw ；硬件：https://github.com/toyota/yubi-hw
- **评测**：仿真 benchmark：**正文未设置标准公开仿真 benchmark**。真机：一方面做人工操作评测——六规格螺母精细抓取与 **5 个效率任务**（每条件 **5 trials**）；另一方面做机器人部署——**π0.5-based multi-task policy** 在 **Bimanual UR / Bimanual Franka / ELEY** 三平台、**6 个任务**上各 **20 rollouts/task**，结果为 **Ball in basket 20/20、Stack cup pyramid 13/20、Unfold glasses 9/20、Pick&Place socks 18/20、Tape in box 18/20、Cup placement 11/20**。设备配置：数据采集阶段为 **双 YUBI 手持夹爪**；每手 **1 个 wrist camera**，外加 **1 个桌面顶视 RealSense D435**，共 **3 个机位**；部署阶段策略只使用 **2 个 wrist RGB 机位**，末端统一为 **YUBI gripper**。
- **算力**：训练资源：正文/附录披露 **UR 任务 30K iterations, batch size 256, action chunk 16**；**Franka 与 ELEY 任务 150K iterations, batch size 64, action chunk 32**；**GPU 型号、卡数与总时长正文未披露**。推理资源：正文仅给出 wrist RGB 30 Hz 输入、部署控制示例 **10 Hz**，未给 GPU 型号。披露状态：训练步数充分，硬件账单缺失。
- **解读**：YUBI 真正有分量的是把“可规模化采集”与“末端统一可迁移”绑在一起：同一夹爪和轨迹空间直接跨 UR、Franka、ELEY 落地。亮点是 8434 小时、120 万 episode 的量级已经像机器人版数据工厂；局限是当前部署仍主要依赖 wrist-only 观测。复现时先复刻夹爪与同步采集链路，再考虑模型规模化。

### C. 仿真可信评测与具身安全鲁棒性

#### 1) A Practical Recipe Towards Improving Sim-and-Real Correlation for VLA Evaluation
- **标题**：A Practical Recipe Towards Improving Sim-and-Real Correlation for VLA Evaluation
- **简称**：Sim-Real Correlation for VLA Eval
- **领域**：VLA evaluation / sim-and-real correlation / benchmark methodology
- **论文**：https://arxiv.org/abs/2606.10366
- **项目**：正文未披露
- **代码**：正文未披露（正文说明将连同 teleoperation code 一并释放）
- **评测**：仿真 benchmark：系统比较 **VLA-Arena（MuJoCo）/ SIMPLER（Sapien）/ REALM（Isaac-Sim）** 三类平台，在 **9 个对齐任务**、**vision / layout / language / behavior** 四类扰动上评估 **π0、π0-FAST、π0.5、GR00T N1.6、GR00T N1.7**。单项协议为 **20 simulated rollouts** 与 **5 real-world rollouts** / policy / task / perturbation，合计 **11,800 simulated rollouts + 1,115 real-world rollouts**。真机：采用 **DROID embodiment** 的同一九任务套件。设备配置：正文附录披露 **VIDEO_KEYS = ["image", "wrist_image"]**，即 **1 个主视角 + 1 个 wrist 视角，共 2 个机位**；状态/动作为 **joint_position + gripper**。关键结果：零样本下代理相关性 **Spearman 0.700**，在 **REALM** 做轻量 post-training 后升至 **0.875**，Proxy MMRV **0.030 → 0.015**。
- **算力**：训练资源：附录披露基线统一采用 **DROID/New embodiment、10,000 training steps、batch size 32、action horizon 16**；**GPU 型号、卡数、总训练时长正文未披露**。部署资源：正文未给实时推理显卡。披露状态：训练步数与 batch 清楚，但硬件账单不足。
- **解读**：这篇最重要的结论不是“哪个模拟器最好”，而是评测要同时保住模型排序和扰动脆弱性图谱，且少量目标分布后训练比纯零样本更像真实世界。优点是给出了可执行的 proxy 指标体系；局限是当前真机只覆盖固定 DROID 桌面具身。复现时先严格对齐九任务与四类扰动，不要只比平均成功率。

#### 2) Test-time Adversarial Takeover: A Real-time Hijacking Interface against Robotic Diffusion Policies
- **标题**：Test-time Adversarial Takeover: A Real-time Hijacking Interface against Robotic Diffusion Policies
- **简称**：TAKO
- **领域**：robot safety / adversarial attack / diffusion policy robustness
- **论文**：https://arxiv.org/abs/2606.10371
- **项目**：https://tako-attack.github.io/
- **代码**：正文说明将于录用后公开 patch optimization / attack / evaluation scripts，当前未给仓库链接
- **评测**：仿真 benchmark：**PushT（2D manipulation）/ DeliverDrone（Gazebo Harmonic aerial delivery）/ SimNav（AI2-THOR indoor navigation）**；另有真机 **RealNav**。人工 takeover 协议中 **4 名研究者**分别对每任务做 **10 trials/operator**，即 **40 trials/task**。正文报告跨 **4 tasks × 2 encoders × 3 generative inference families**，攻击者在所有评测设定上都达到 **100% takeover success**。真机：**RealNav** 采用室内走廊导航。设备配置：**iRobot Create 3 chassis + Raspberry Pi 4B companion computer**；策略只消费 **1 个前向 170° USB 广角相机**，安装在距底盘约 **42 cm 高、15 cm 后移**的位置，共 **1 个机位**；RPLIDAR 仅接急停，不输入策略。
- **算力**：训练/攻击资源：单个 universal patch 在 **单张 NVIDIA A40** 上约 **10 分钟**完成（**500 Adam steps, batch size 8**）；RealNav 真机推理运行在独立移动工作站 **RTX 4090** 上，控制频率 **8 Hz**。总实验算力合计约 **60 A40-hours + 10 RTX 4090-hours**（正文说明完整项目总算力约为该值 3 倍含废弃实验）。
- **解读**：TAKO 把“扰乱机器人”升级成“接管机器人”，说明扩散策略的视觉条件通道本身就是实时操纵接口。亮点是跨操作空间、跨推理家族、跨真机都能复现 takeover；局限是默认白盒与可见贴片前提较强。复现时先做离线 directional patch vocabulary，再上在线人控切换接口。
