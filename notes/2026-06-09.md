### A. World Action Model、时序记忆与长程执行

#### 1) MotionWAM: Towards Foundation World Action Models for Real-Time Humanoid Loco-Manipulation
- **标题**：MotionWAM: Towards Foundation World Action Models for Real-Time Humanoid Loco-Manipulation
- **简称**：MotionWAM
- **领域**：World Action Model / humanoid loco-manipulation
- **论文**：https://arxiv.org/abs/2606.09215
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：正文未设置标准公开仿真 benchmark，重点评测为真机。真机：**Unitree G1** 上 9 个 whole-body loco-manipulation 任务、每任务 **20 trials**；总体成功率 **76.1%**，较最强 VLA 基线 **43.9%** 高 **32.2 pct**。设备配置：**Unitree G1 humanoid + dual ALOHA2 grippers**；**1 个 head-mounted Intel RealSense D435i RGB**；无 wrist/外置相机输入。
- **算力**：训练资源：正文披露主训练阶段为 **40,000 steps / 8× NVIDIA A100**。推理开销：单张 **NVIDIA A100** 上 chunk 级闭环频率 **4.9 Hz**；真机部署为单台 **NVIDIA RTX 4090** 工作站作 policy server。披露状态：正文给出训练步数、GPU 卡数与部署 GPU，未给总训练时长。
- **解读**：把 WAM 从桌面操作推到人形全身动作是这篇的核心价值：统一 motion latent 直接覆盖腿、腰、手协同，连踢球和推车这类足部参与任务都能进同一策略。局限是强依赖单头部视角，物体出视野就会掉稳；复现时先还原三阶段 egocentric→embodiment 训练配方。

#### 2) AHA-WAM: Asynchronous Horizon-Adaptive World-Action Modeling with Observation-Guided Context Routing
- **标题**：AHA-WAM: Asynchronous Horizon-Adaptive World-Action Modeling with Observation-Guided Context Routing
- **简称**：AHA-WAM
- **领域**：World Action Model / asynchronous planning-execution
- **论文**：https://arxiv.org/abs/2606.09811
- **项目**：https://serene-sivy.github.io/aha-wam/
- **代码**：项目页显示 **Code / Coming soon**，正文未给公开仓库
- **评测**：仿真 benchmark：**RoboTwin 2.0**，覆盖 **50 个双臂任务**；clean 设定每任务 **50 demos**、randomized 设定每任务 **500 demos**，每任务 **100 trials**，平均成功率 **92.80%**。真机：**bimanual AgileX Piper** 平台，4 个任务（Fold Towel / Organize Desktop / Prepare Soy Milk / Store Plate），每任务约 **120 episodes** 数据。设备配置：真机输入仅 **1 个 ego/head-view RGB 相机**；机器人本体为 **AgileX Piper 双臂**；末端执行器类型正文未单列。补充：RoboTwin 训练配置使用 **head + left wrist + right wrist 共 3 视角**。
- **算力**：训练资源：正文披露预训练使用 **RoboCOIN subset 24,600 trajectories / 约 165 小时 robot data**，任务训练默认 **global batch size 512 / 5 epochs**；训练 GPU 型号与卡数 **正文未披露**。推理开销：单张 **RTX 5090D** 上 AHA-WAM 延迟 **41.37 ms / 24.17 Hz**，AHA-WAM-Flash 为 **17.56 ms / 56.95 Hz**。披露状态：正文充分披露推理延迟，但训练硬件账单不完整。
- **解读**：亮点是把“慢速世界规划器 + 快速动作执行器”真正异步化，避免每次控制都重跑视频分支，因而把 WAM 从会想象拉到能闭环。局限是训练硬件不透明、真机末端细节不足；复现时优先保住 OVCR 路由与 planner-executor 相位错位训练。

#### 3) MemoryVLA++: Temporal Modeling via Memory and Imagination in Vision-Language-Action Models
- **标题**：MemoryVLA++: Temporal Modeling via Memory and Imagination in Vision-Language-Action Models
- **简称**：MemoryVLA++
- **领域**：VLA / temporal memory & imagination
- **论文**：https://arxiv.org/abs/2606.09827
- **项目**：https://shihao1895.github.io/MemoryVLA-PP-Web/
- **代码**：https://github.com/shihao1895/MemoryVLA
- **评测**：仿真 benchmark：**LIBERO 98.4%**、**SimplerEnv 74.0%**、**Mikasa-Robo 44.4%**、**CALVIN 4.29**，并在 **LIBERO-Plus** 做鲁棒性评测。真机：覆盖 **Franka / WidowX / Dual-ARX5** 三个平台、近 **200 个任务变体**。设备配置：Franka 与 WidowX 使用 **3 个固定 Intel RealSense D435 RGB 视角**；Dual-ARX5 使用 **固定 RGB 相机 + 1 个 wrist RGB**；各平台末端执行器类型正文未逐项展开。
- **算力**：训练资源：正文披露使用 **8× NVIDIA A100 或 H20**，global batch **208–256**；BridgeData-v2 设定下策略训练 **50k steps**、对应 world model **40k steps**。推理开销：表中给出 **RTX 4090 / HGX H20** 延迟与显存；MemoryVLA++ 在 **RTX 4090** 上延迟 **0.241 s**、显存 **21.7 GB**。披露状态：正文与附录对训练卡数、步数和推理显存披露较完整。
- **解读**：它把“记住过去”和“想象未来”都做进动作专家里，不再只靠当前帧拍脑袋，因此对长时依赖和遮挡恢复更稳。亮点是跨 LIBERO、SimplerEnv、真实多平台都有效；局限是开销明显抬升。复现时先守住 memory bank 与 imagination 分支，再考虑替换 backbone。

### B. VLA 失效恢复、轻量执行与接触强化

#### 1) ProbeAct: Probe-Guided Training-Free Failure Recovery in Vision-Language-Action Models
- **标题**：ProbeAct: Probe-Guided Training-Free Failure Recovery in Vision-Language-Action Models
- **简称**：ProbeAct
- **领域**：VLA / failure recovery / training-free intervention
- **论文**：https://arxiv.org/abs/2606.09740
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**LIBERO-plus**，覆盖 **Camera / Robot / Language / Light / Background / Noise / Layout** 等 7 类扰动。正文报告在 **OpenVLA-OFT** 上总成功率由 **69.6% 提升到 74.1%**。真机：正文未报告真机实验。设备配置：不适用。
- **算力**：训练资源：正文披露 probe 训练收集 **50,000 组（hidden-state, object-position）** 配对数据；GPU 型号、卡数与总时长 **正文未披露**。推理开销：主打 training-free intervention loop，但正文未统一给出 latency / FPS / 显存表。披露状态：事实闭环充分，但算力账单较少。
- **解读**：这篇有意思的地方在于不改 VLA 权重，而是从隐藏状态读出几何位置，再用极小的运动学状态机把策略从失败轨道拉回来。优点是后装成本低、能直接接在现有 OpenVLA 类系统后面；局限是只在仿真扰动验证。复现时先复现 object probe 与 phase state machine。

#### 2) CT-VAM: A Cerebello-Thalamic-Inspired Vision-Action Model for Efficient Visuomotor Control
- **标题**：CT-VAM: A Cerebello-Thalamic-Inspired Vision-Action Model for Efficient Visuomotor Control
- **简称**：CT-VAM
- **领域**：vision-action model / efficient low-level control
- **论文**：https://arxiv.org/abs/2606.09572
- **项目**：https://embodied-ai-research.github.io/ct-vam/
- **代码**：项目页显示 **Code (coming soon)**
- **评测**：仿真 benchmark：**LIBERO** 四个 suite、每任务 **50 rollouts**，平均成功率 **82.1%**。真机：**OpenArm** 平台，任务包括 **Ball Pouring** 与 **Box Opening and Placement**。设备配置：**OpenArm 左臂 7-DoF + gripper**；**1 个第三视角 Intel RealSense D435 + 1 个 wrist Intel RealSense D405，共 2 个机位**；每任务 **30 demonstrations**，Ball Pouring 在不同部署平台上各评 **20 trials**。
- **算力**：训练资源：正文强调模型仅 **68M 参数**，但训练 GPU 型号/卡数/总时长 **正文未系统披露**。推理开销：正文给出 **RTX 4080** 与 **Jetson Orin NX** 部署对比；CT-VAM 支持 Jetson 机载部署，表中 **Jetson Orin NX** 平均推理时间约 **200.60 ms**。披露状态：部署算力披露清楚，训练算力不足。
- **解读**：CT-VAM 的取舍很明确：把语言从高频控制环里拿掉，保留一个轻量 dual-view 视觉动作器去做真正的低延迟执行。优点是 68M 参数就能打到不错的 LIBERO 和真机表现；局限是高层语义仍要外接。复现时先做 TARS 路由和 FCI 异步 chunk 执行。

#### 3) TORL-VLA: Tactile Guided Online Reinforcement Learning for Contact-Rich Manipulation
- **标题**：TORL-VLA: Tactile Guided Online Reinforcement Learning for Contact-Rich Manipulation
- **简称**：TORL-VLA
- **领域**：tactile VLA / online RL / contact-rich manipulation
- **论文**：https://arxiv.org/abs/2606.09337
- **项目**：正文未披露
- **代码**：正文注明 **code will be released for reference**，当前未公开链接
- **评测**：仿真 benchmark：正文未设置标准公开仿真 benchmark。真机：自建 **latch-box** benchmark，含 **Coffee Cup / Latch / Egg** 三个 contact-rich subtasks；subtask 与 full-task 均为 **30 autonomous trials**。设备配置：**Piper robotic arm + Agile Robots Pika gripper**；视觉为 **1 个全局 Intel RealSense D435i + gripper 内置 fisheye + 1 个 wrist Intel RealSense D405，共 3 个机位**；另有 **2 个 Tacta fingertip tactile sensors**。结果上，TORL-VLA 的 full-task success 为 **30/30**，高于 π0.5 的 **12/30**。
- **算力**：训练资源：正文未披露离线训练 GPU 型号/卡数/总时长。推理开销：部署延迟在单张 **RTX 4090** 上评测，端到端 TORL-VLA query 平均 **103.0 ms**；观测与策略查询频率为 **20 Hz**。披露状态：部署延迟披露明确，训练账单缺失。
- **解读**：把触觉先转成 wrench，再用在线 actor-critic 只修局部接触瓶颈，这条路线比“把所有触觉都塞进大模型重学一遍”更务实。亮点是 full-task 30/30 很硬；局限是任务族还集中在单一 latch-box 场景。复现时重点是 stage estimator、intervention-censored critic 和 tactile-to-wrench 标定。
