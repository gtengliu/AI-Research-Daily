### A. World Action Model / VLA 适配与执行

#### 1) CKT-WAM: Parameter-Efficient Context Knowledge Transfer Between World Action Models
- **标题**：CKT-WAM: Parameter-Efficient Context Knowledge Transfer Between World Action Models
- **简称**：CKT-WAM
- **领域**：World Action Model / 参数高效迁移 / 长时程操控
- **论文**：https://arxiv.org/abs/2605.06247
- **项目**：https://github.com/YuhuaJiang2002/CKT-WAM
- **代码**：https://github.com/YuhuaJiang2002/CKT-WAM
- **评测**：仿真 benchmark：LIBERO-Plus，训练集使用 LIBERO 四个 suites 的合并训练集，并在 camera / robot / language / lighting / layout / background / noise 七个扰动维度做 zero-shot 评测。真机：4 个长时程多阶段任务——clothes folding、fruit sorting、cube catching、unmanned retail；每任务 45 次真实试验。设备配置：正文确认是连续视觉反馈下的真实闭环操控，但机器人本体、末端执行器型号与相机安装细节正文未逐项披露；相机位姿数量正文未披露。
- **算力**：正文/附录披露所有模型在 8×NVIDIA A800 上以 distributed bf16 训练；仅训练 CKT 模块 187.4M 参数，占总参数 1.17%。统一 wall-clock 或总 GPU-hours 正文未披露。
- **解读**：这篇不是再训一个更大的 WAM，而是把“教师 WAM 的上下文”压缩后注入学生模型，用广义适配器+稀疏专长路由做轻量迁移。亮点是 1.17% 可训练参数就把 LIBERO-Plus 和真机长时任务都推上去；局限是硬件与相机配置披露不足，复现时优先按其公开仓库核对 teacher/student checkpoint 和 LIBERO-Plus 评测协议。

#### 2) When to Trust Imagination: Adaptive Action Execution for World Action Models
- **标题**：When to Trust Imagination: Adaptive Action Execution for World Action Models
- **简称**：FFDC-WAM
- **领域**：World Action Model / 自适应执行 / 在线校验
- **论文**：https://arxiv.org/abs/2605.06222
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：RoboTwin，覆盖 50 个 manipulation tasks；每任务执行 100 次，并含 clean / random 两种设定，random 额外引入背景、桌面杂乱度、高度扰动与光照变化。真机：2 个 pick-and-place 任务（pick banana and place、pick carrot and place in wooden box）。设备配置：Astribot S1，34 DoF；论文明确给出机器人本体，但末端执行器型号、相机型号与安装位姿数量正文未披露。
- **算力**：正文披露 FFDC-WAM backbone 训练使用 4×A100-80GB，verifier 训练使用 1×A100，评测使用单张 A100。统一训练步数对应的总 GPU-hours 正文未披露。
- **解读**：它的关键想法很实用：不是默认相信 WAM 预测出来的长 chunk，而是让模型持续比较“想象未来”和“真实观测”是否一致，再决定继续执行还是重规划。优点是显著减少 forward pass 并提升真机鲁棒性；局限是 verifier 仍是二分类，复现时建议先把 RoboTwin 的失配数据构造和阈值 τ 调稳。

#### 3) VLA-GSE: Boosting Parameter-Efficient Fine-Tuning in VLA with Generalized and Specialized Experts
- **标题**：VLA-GSE: Boosting Parameter-Efficient Fine-Tuning in VLA with Generalized and Specialized Experts
- **简称**：VLA-GSE
- **领域**：VLA / 参数高效微调 / 泛化操控
- **论文**：https://arxiv.org/abs/2605.06175
- **项目**：https://github.com/YuhuaJiang2002/VLA-GSE
- **代码**：https://github.com/YuhuaJiang2002/VLA-GSE
- **评测**：仿真 benchmark：LIBERO-Plus，训练使用 LIBERO 四个 suites 合并集，并按七个扰动维度做 zero-shot 泛化评测。真机：AgileX PiPER 机器人上的 4 个任务——pour water into bowl、pick spoon into plate、put carrot into bowl、take pen out of pen container；每个任务在 lighting / language / clutter layout / background tablecloth 四类分布偏移下各收集 15 个测试样本，共 60 次试验。设备配置：AgileX PiPER + fixed front-view camera + wrist camera；机器人末端执行器型号正文未披露；相机位姿数量可核验为 2。
- **算力**：正文披露从 Qwen3-VL-4B-Instruct 初始化，训练 80,000 steps，batch size 为 16/GPU，8×NVIDIA A100，总有效 batch size 128；可训练参数 114.04M，占 2.51%。统一 wall-clock 与总 GPU-hours 正文未披露。
- **解读**：VLA-GSE本质是在 LoRA 式 PEFT 上加一条“通用专家 + 路由专长专家”分解路径，解决 PEFT 保知识但动作适配不够强的问题。优势是仿真与真机 OOD 都稳；局限是仍绑定单一 VLM 骨干与有限真实任务，复现时建议先复刻其 expert routing 与梯度 scale balancing，再看是否真优于 plain LoRA。

### B. 机器人世界模型 / 视频世界模型

#### 1) EA-WM: Event-Aware Generative World Model with Structured Kinematic-to-Visual Action Fields
- **标题**：EA-WM: Event-Aware Generative World Model with Structured Kinematic-to-Visual Action Fields
- **简称**：EA-WM
- **领域**：视频世界模型 / 机器人生成模型 / 动作条件建模
- **论文**：https://arxiv.org/abs/2605.06192
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：WorldArena，沿用其基于 RoboTwin 数据与分割流程的 Physics Adherence、3D Accuracy、Controllability 三大维度评测，并汇总为 P3CScore。真机：无独立真实机器人闭环实验。设备配置：模型侧显式使用双臂机器人动作、运动学状态、camera intrinsic/extrinsic 构建 KVAF，但具体真实机器人本体、末端执行器与相机位姿数量未在正文作为单独真机系统披露。
- **算力**：正文披露两阶段训练，LoRA rank=32，统一学习率 8e-5；训练使用 32×H100 GPUs，batch size 32。总训练时长正文未披露。
- **解读**：EA-WM 的重点不是把 action token 再喂厚一点，而是直接把运动学和动作抬到相机视角里，生成结构化 KVAF，再让视频流和事件流双向融合。亮点是明显提升 interaction / trajectory / depth 指标；局限是仍停在生成质量评测，复现时要特别核对相机投影、关键点渲染和事件监督三条链路是否对齐。

#### 2) Reconstruction or Semantics? What Makes a Latent Space Useful for Robotic World Models
- **标题**：Reconstruction or Semantics? What Makes a Latent Space Useful for Robotic World Models
- **简称**：Semantic-WM Study
- **领域**：机器人世界模型 / latent space 设计 / policy evaluation
- **论文**：https://arxiv.org/abs/2605.06388
- **项目**：https://hskalin.github.io/semantic-wm/
- **代码**：https://huggingface.co/Nilaksh404/semantic-wm
- **评测**：仿真/离线 benchmark：固定协议下在 Bridge V2 上训练各类 latent diffusion world models，并从 visual fidelity、planning/downstream policy、representation quality 三轴评估；额外将 OpenVLA-7B rollout 到各 world model 中，在 20 个 Bridge V2 测试 episode、每 episode 8 次试验上做 policy-evaluation，并报告 OOD 干扰与指令扰动鲁棒性。真机：数据来源为真实 WidowX 250 示教数据集（约 60K demos，13 个任务家族）；并非新增独立真机闭环测试。设备配置：WidowX 250；每条轨迹含 RGB observation、7-DoF 末端动作与语言指令；正文未完整披露标准评测所用相机位姿数量，附加多视角实验明确使用 3 个 camera views。
- **算力**：正文/附录重点披露训练协议与模型结构，但未给出统一 GPU 型号、卡数或总 GPU-hours；因此记为“正文未披露”。
- **解读**：这篇很有价值，因为它把“latent 好不好”从生成画质，拉回到是否保留 action-relevant structure。结论偏明确：语义表征更适合机器人 world model，但 decoder recipe 很关键；局限是主要基于 BridgeV2 单一体系，复现时最好先复跑作者固定协议，再换你自己的 encoder family。

### C. 物理感知 RL / 动作迁移

#### 1) ReActor: Reinforcement Learning for Physics-Aware Motion Retargeting
- **标题**：ReActor: Reinforcement Learning for Physics-Aware Motion Retargeting
- **简称**：ReActor
- **领域**：强化学习 / 动作重定向 / 物理一致性 / Sim2Real
- **论文**：https://arxiv.org/abs/2605.06593
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：将 AMASS 过滤后的全量动作集重定向到 Unitree G1 与 Lima 两个类人机器人，并与 GMR、OmniRetarget 比较；下游 RL tracking 在完整过滤后的 AMASS 上训练与评测，单段 motion 随机起始帧后运行 5 秒，统计 success rate 与 root/joint 误差。真机：在物理 Lima 机器人上部署基于 ReActor 重定向数据训练的 goal-conditioned tracking policy，展示多种接近硬件极限的动作。设备配置：Unitree G1（1.27 m，35 kg，29 DoF）、Lima（0.84 m，16.2 kg，20 DoF）与 ANYmal D 作为目标形体；真实部署正文确认在 Lima 上完成，但相机系统、末端执行器与相机位姿数量不适用/未披露。
- **算力**：正文披露 Isaac Sim 中使用 4096 并行环境，在单张 RTX 5090 上训练 PPO tracking policy；方法运行 20k iterations，约 6 小时。AMASS 数据集上的整体 retargeting/training 开销约 6.5 小时/单 GPU。
- **解读**：ReActor 把“先做几何重定向，再拿去训策略”的串行流程改成双层优化：上层调参考动作、下层 RL 学 tracking，从一开始就把物理可执行性纳入目标。亮点是下游 policy 质量和真机 Lima 部署都更稳；局限是更偏 locomotion/全身动作场景，复现时建议先从 G1/Lima 的 correspondence 与 reward 正则化细节入手。
