### A. VLA / World Model / 长程规划

#### 1) MolmoAct2: Action Reasoning Models for Real-World Deployment
- **标题**：MolmoAct2: Action Reasoning Models for Real-World Deployment
- **简称**：MolmoAct2
- **领域**：VLA / Action Reasoning / Embodied Reasoning
- **论文**：https://arxiv.org/abs/2605.02881
- **项目**：https://allenai.org/blog/molmoact2
- **代码**：正文未披露（论文首页与 arXiv 摘要页可核验到项目博客，但未给稳定代码仓直链）
- **评测**：仿真 benchmark：LIBERO、RoboEval，并在文中汇总“7 个仿真+真机 benchmark、13 个 embodied-reasoning benchmark”。真机：Bimanual YAM、Franka-DROID、SO-100/101 三类平台。设备配置：Bimanual YAM 为双臂桌面平台 + 末端夹爪 + 多机位（文中配图与数据采集段落给出固定外置相机阵列）；DROID 为 Franka + 2 路外置相机 + 1 路 wrist 相机；SO-100/101 为低成本机械臂平台，具体相机位姿数量正文未统一汇总。
- **算力**：正文/附录披露：预训练阶段使用 2 nodes × 8 H100；BimanualYAM 微调用 64×H100、100K updates、约 2,300 GPU hours；序列长度 2100、global batch size 128。其余子模型有独立配置，但统一推理显存/时延账单未完整披露。
- **解读**：这篇把 VLM、动作 tokenizer 和“按需重推深度 token”的推理式控制结合起来，目标很明确：把 open VLA 真正拉到可部署区间。亮点是覆盖平台多、开放程度高；局限是多平台结果口径不完全统一，复现时建议先从 DROID 两外一腕相机设定与 OpenFAST tokenizer 开始还原。

#### 2) RoboAlign-R1: Distilled Multimodal Reward Alignment for Robot Video World Models
- **标题**：RoboAlign-R1: Distilled Multimodal Reward Alignment for Robot Video World Models
- **简称**：RoboAlign-R1
- **领域**：Robot World Model / Reward-Aligned Post-Training
- **论文**：https://arxiv.org/abs/2605.03821
- **项目**：正文给出 `alexander-wu.github.io/RoboAlign_R1`，但发布时页面返回 404
- **代码**：https://github.com/Alexander-wu/RoboAlign_R1
- **评测**：仿真/离线 benchmark：RobotWorldBench（10,000 组 video-instruction 标注样本）、并在 RT-1、BridgeData V2 rollout 上评估；RobotWorldBench 候选还来自 CALVIN、LIBERO。真机：无新增真机闭环执行，评测依赖现有机器人数据集。设备配置：RT-1/BridgeData V2 以第三人称主相机为主；BridgeData V2 对应 WidowX-250 六自由度机械臂 + 夹爪，轨迹记录第三人称 RGB、末端位姿与 gripper 状态，文中未统一给出 wrist/head 相机位姿数量。
- **算力**：正文披露训练与评测均在 NVIDIA A100 GPUs 上完成；SWR 推理仅增加约 1% latency。更细的卡数、总训练时长与显存曲线正文未统一披露。
- **解读**：作者把机器人视频世界模型从“像不像”对齐到“对任务是否有用”，用 judge→student reward distillation 做后训练，再用滑窗重编码抑制长程漂移。优点是方向很对；局限是评测仍以自动 judge 为主，复现时最好先重建 RobotWorldBench 标注口径，再验 SWR 的窗口长度收益。

#### 3) Anticipation-VLA: Solving Long-Horizon Embodied Tasks via Anticipation-based Subgoal Generation
- **标题**：Anticipation-VLA: Solving Long-Horizon Embodied Tasks via Anticipation-based Subgoal Generation
- **简称**：Anticipation-VLA
- **领域**：VLA / 长程任务 / 子目标生成
- **论文**：https://arxiv.org/abs/2605.01772
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：LIBERO 四个 suite（Spatial / Object / Goal / Long）与 VLABench。真机：ARX-X5 mobile manipulator，只用右臂执行 Rearrange Objects 与 Spell Words 两个长程任务。设备配置：ARX-X5 本体 + 右臂末端夹爪；2 路 RealSense D435i（1 路顶部 head 视角 + 1 路右臂相机/近腕视角）。
- **算力**：附录披露全部实验使用 4×NVIDIA H100；Anticipation Model 训练约 4–6 小时，Goal-conditioned VLA 训练约 1–2 小时；Goal-conditioned VLA total batch size 64，训练步数为 LIBERO 5k、VLABench 10k、真机任务 10k。
- **解读**：核心思路不是静态拆任务，而是递归预测“下一步该看到什么、该做什么”的中间子目标，再和低层 goal-conditioned VLA 闭环联动。它对长程任务确实更自然；局限是强依赖子目标质量，复现时应先单独验证 subgoal 文本/图像生成，再接策略层。

### B. 具身操控与接触智能

#### 1) CoRAL: Contact-Rich Adaptive LLM-based Control for Robotic Manipulation
- **标题**：CoRAL: Contact-Rich Adaptive LLM-based Control for Robotic Manipulation
- **简称**：CoRAL
- **领域**：LLM-based Manipulation / Contact-Rich Control / MPC
- **论文**：https://arxiv.org/abs/2605.02600
- **项目**：https://sites.google.com/view/lira-coral
- **代码**：正文未披露
- **评测**：仿真 benchmark：自建 4 类接触/受力关键任务 + LIBERO 标准 pick-and-place 任务，基于 RoboSuite / MuJoCo。真机：Franka Emika Panda 实机部署。设备配置：Franka Panda 7-DoF + parallel-jaw gripper；仿真用固定 RGB-D 相机、力/力矩与本体反馈；真机为 6 路 Vicon Vero 动捕做目标跟踪，正文未给额外 wrist/head 相机位姿数量。
- **算力**：正文披露系统运行在 Intel Core i9-13900K、64GB RAM、单张 NVIDIA RTX 4060 Ti；控制层级为 1kHz 阻抗控制、10Hz MPPI、约 1Hz LLM 重规划。训练型大规模算力不适用。
- **解读**：CoRAL的关键不是“让 LLM 直接控机械臂”，而是把 LLM 放到慢速语义重规划层，真正的接触执行交给物理世界模型 + MPPI。亮点是层次清楚、适合力接触任务；局限是视觉感知与物体模型依赖较强，复现时先从文中最简单的薄片拾取和靠墙翻转任务入手更稳。

#### 2) SigLoMa: Learning Open-World Quadrupedal Loco-Manipulation from Ego-Centric Vision
- **标题**：SigLoMa: Learning Open-World Quadrupedal Loco-Manipulation from Ego-Centric Vision
- **简称**：SigLoMa
- **领域**：四足移动操作 / Ego-Centric Vision / RL
- **论文**：https://arxiv.org/abs/2605.03846
- **项目**：https://11chens.github.io/SigLoMa/
- **代码**：正文未披露
- **评测**：仿真 benchmark：Isaac Gym 中按 short-axis grasp / long-axis grasp / target release 三类任务训练与消融，每项 100 次 trial、3 次独立评估。真机：Unitree Go2。设备配置：Go2 四足本体 + 超低成本双指夹爪末端 + 1 路固定俯视 RealSense D435i；机载 Jetson Orin NX，语义跟踪 5Hz、VO 60Hz；未使用 wrist/head 多相机阵列。
- **算力**：正文披露 PPO 训练在单张 NVIDIA RTX 4090 上约 6 小时收敛；附录给出 batch size、环境数与 Kalman filter / randomization 配置。真机部署侧为 Jetson Orin NX，但正文未统一给端到端显存/功耗表。
- **解读**：这篇抓住四足操作的真痛点：视觉慢、机体快、底盘还在动。作者用 Sigma Points 几何表示 + ego-centric KF 把低频语义感知补成高频控制输入。优点是硬件轻量、闭环完整；局限是任务仍偏 pick-and-place，复现时先对齐 5Hz 感知到 50Hz 控制的时序链路。

#### 3) Hydra-DP3: Frequency-Aware Right-Sizing of 3D Diffusion Policies for Visuomotor Control
- **标题**：Hydra-DP3: Frequency-Aware Right-Sizing of 3D Diffusion Policies for Visuomotor Control
- **简称**：Hydra-DP3
- **领域**：Diffusion Policy / 3D Visuomotor Control
- **论文**：https://arxiv.org/abs/2605.01581
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：RoboTwin2.0、Adroit、MetaWorld。真机：AgileX Piper 机械臂平台。设备配置：Piper 本体 + 末端夹爪；1 路全局安装的 Intel RealSense D455；文中未给 wrist/head 其他位姿。真机训练数据来自 leader-follower 遥操作，50 条训练轨迹。
- **算力**：正文披露全部模型在单张 NVIDIA RTX 5880 Ada 上训练 3,000 epochs，RoboTwin2.0 batch size 256、Adroit/MetaWorld batch size 128；真机推理部署在单张 NVIDIA RTX 4060。正文强调两步去噪即可，但未给完整跨任务显存曲线。
- **解读**：Hydra-DP3 用频域观点论证“机器人动作本来就低频、平滑”，因此没必要沿用图像扩散那套大解码器和长采样链。贡献很工程化：小模型、两步采样、真机也能跑；局限是主要利好平滑动作分布，复现时需检查高频接触任务是否仍会掉细节。

### C. Humanoid / 数据采集基础设施

#### 1) BifrostUMI: Bridging Robot-Free Demonstrations and Humanoid Whole-Body Manipulation
- **标题**：BifrostUMI: Bridging Robot-Free Demonstrations and Humanoid Whole-Body Manipulation
- **简称**：BifrostUMI
- **领域**：Humanoid Imitation / Robot-Free Demonstration / Whole-Body Manipulation
- **论文**：https://arxiv.org/abs/2605.03452
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：正文未设置统一公开仿真 benchmark，重点是从人类示教到 humanoid 真机执行。真机：Unitree G1 humanoid。设备配置：G1 本体；示教端为 PICO 4 头显 + 手柄 + 3 个下肢 tracker，配两只 UMI 风格手持 gripper，均带 wrist-mounted RGB cameras；执行端为全身控制器，低层 whole-body controller 50Hz。论文展示 cluttered tabletop pick-and-place 与 under-table waste disposal 两类真机任务。
- **算力**：正文未系统披露训练 GPU 型号、卡数、总时长与显存；仅给出策略分层结构与低层控制频率，因此记为“正文未披露”。
- **解读**：BifrostUMI的价值在于把“没有机器人也能采 humanoid 数据”这件事做通：VR 全身关键点 + 手持带腕视角 gripper，再做关键点重定向与全身执行。亮点是数据门槛低；局限是量化评测仍不充分，复现时应优先打通示教坐标系、关键点 retargeting 和 50Hz 全身控制。

#### 2) Phone2Act: A Low-Cost, Hardware-Agnostic Teleoperation System for Scalable VLA Data Collection
- **标题**：Phone2Act: A Low-Cost, Hardware-Agnostic Teleoperation System for Scalable VLA Data Collection
- **简称**：Phone2Act
- **领域**：数据采集 / Teleoperation / VLA Fine-Tuning
- **论文**：https://arxiv.org/abs/2605.01948
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：不适用。真机：在工业 Dobot CR5 与低成本双臂 LeRobot SO-101 平台验证，并用收集到的 130 条 episode 微调 GR00T-N1.5，在 Dobot CR5 多阶段 pick-and-place 上达到 90% 成功率。设备配置：Dobot CR5 / SO-101 机械臂 + 末端夹爪；手机作为 6-DoF 遥操作器；Universal Recorder 同步多路 RGB 相机与机器人状态，但正文未统一披露相机位姿数量。
- **算力**：正文披露 GR00T-N1.5-3B 微调在单张 NVIDIA RTX A6000 上完成；effective batch size 48、peak LR 1e-4、bf16 mixed precision、10-step action chunk。数据采集端手机姿态发布频率 50Hz。
- **解读**：Phone2Act的贡献不在新策略，而在把“低成本、跨硬件、可直接产出 LeRobot 格式数据”做成一个通用入口。对小团队很实用；局限是论文主要验证单任务微调收益，复现时最好先固定一台机械臂和双按钮映射，再逐步扩展多机位采集。
