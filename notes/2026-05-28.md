### A. VLA 架构、表示与训练范式

#### 1) PrimitiveVLA: Learning Reusable Motion Primitives for Efficient and Generalizable Robotic Manipulation
- **标题**：PrimitiveVLA: Learning Reusable Motion Primitives for Efficient and Generalizable Robotic Manipulation
- **简称**：PrimitiveVLA
- **领域**：VLA / Motion Primitive / Robot Manipulation
- **论文**：https://arxiv.org/abs/2605.28634
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：LIBERO-Spatial / Object / Goal、LIBERO-90、LIBERO-90-Novel、Libero-Long、RLBench 10 任务。代表结果：OpenVLA 在 LIBERO-90 相比基线提升 **9.2 个点**，π0.5 在 long-horizon 任务成功率从 **30.50% 提升到 80.25%**。真机：11 个桌面操作任务，含 ID（T1-T6）、Task Generalization（T7-T9）与 Compositional Generalization（T10-T11），每项 **20 trials**。设备配置：**UR5e 单臂 + Robotiq 2F-85**；**1 个第三人称 RealSense L515 + 1 个 wrist-mounted RealSense L515**，共 **2 个 RGB 机位**。真机结果：π0.5+Ours 在 ID / Task Gen / Compositional 三组均值分别为 **90% / 57% / 65%**，显著高于 π0.5 的 **70% / 20% / 10%**。
- **算力**：正文/附录披露较完整：OpenVLA 与 OpenVLA-OFT 微调在 **8×L40S 48GB** 上进行，耗时约 **60h / 100h**；π0.5 微调在 **8×A100 80GB** 上进行，约 **20h**；RLBench 与真机推理评测使用 **RTX 4090**。训练步数方面：OpenVLA 在 LIBERO-90 / RLBench 使用 **100k iterations**。
- **解读**：它把 VLA 从“整段轨迹硬记忆”改成“原子动作可复用”，长程泛化提升很硬。局限是仍依赖分割与规划链路。复现建议先单独复做 primitive disassembly，再验证增益究竟来自表示重组还是数据增广。

#### 2) ProgVLA: Progress-Aware Robot Manipulation Skill Learning
- **标题**：ProgVLA: Progress-Aware Robot Manipulation Skill Learning
- **简称**：ProgVLA
- **领域**：轻量级 VLA / Progress Modeling / Manipulation
- **论文**：https://arxiv.org/abs/2605.28231
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：LIBERO 40 任务（4 类×10 任务）、Meta-World MT50。正文总结其 **0.1B** 模型在长时程与较难任务上可超过更大预训练基线。真机：toy-kitchen + white-table 两个环境下的 **10 个任务 / 100 trials**。设备配置：**6-DoF AgileX PiPER 机械臂 + gripper**；**1 个 wrist RealSense D405 + 1 个 agent-view RealSense D435**，共 **2 个 RGB 机位**。真机平均成功率 **68%**。
- **算力**：正文/附录披露：训练使用 **单张 NVIDIA H100**；LIBERO 模型训练 **25h**，Meta-World **32h**，真机模型约 **8.5h**；部署可在 **Intel Ultra 7 + RTX 3500 Laptop GPU** 上实时运行。
- **解读**：亮点不是单纯做小模型，而是把“任务进度”显式塞进表示学习，让小模型在长序列上更稳。局限是当前真机只覆盖两类轻量场景。复现建议先验证 progress heads 对长时任务的边际收益，而别只看总体均分。

#### 3) Turning Video Models into Generalist Robot Policies
- **标题**：Turning Video Models into Generalist Robot Policies
- **简称**：VERA
- **领域**：Video Policy / World Model / Cross-Embodiment Control
- **论文**：https://arxiv.org/abs/2605.27817
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：PushT-Sim、MimicGen Panda-Sim、Allegro-Sim。闭环结果中，J-IDM 版在 **PushT-Sim 92.5% / 95.5% task progress**、**Panda-Sim 94.0% / 94.0%**、**Allegro-Sim 70.0% / 70.0%**，均优于 direct-IDM 基线。真机：Panda-Real 与 Allegro-Real 两套平台做 held-out 条件执行。设备配置：**Panda-Real 为 Franka-family 单臂 + parallel gripper**，**2 个 external RGB + 1 个 wrist RGB** 共 **3 机位**；**Allegro-Real 为横置 Allegro hand**，**2 个 calibrated RGB 机位**。正文展示 zero-shot Panda 语言操作与 Allegro in-hand reorientation，但未给统一汇总成功率表。
- **算力**：正文披露主视频规划器使用 **Wan-14B**，主实验训练硬件为 **1 台 H200 节点（8 GPUs）**；开发/消融使用 **1×H100**。推理默认使用 **40 个 UniPC denoising steps**；各真机 IDM 训练时长正文未统一披露。
- **解读**：这篇最有意思的点是把“视频世界模型”和“动作翻译器”硬拆开，保住了跨机体复用性。局限是实机结果仍偏展示型。复现建议优先复做 J-IDM 与 direct-IDM 的高 DoF 对比，先证明翻译器设计真的决定成败。

#### 4) What Frozen VLAs Already Know About Success: A Probing Study of Value-Like Structure in Foundation Robot Policies
- **标题**：What Frozen VLAs Already Know About Success: A Probing Study of Value-Like Structure in Foundation Robot Policies
- **简称**：Value-Probing for Frozen VLAs
- **领域**：VLA Analysis / Value Probing / Test-time Selection
- **论文**：https://arxiv.org/abs/2605.28527
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：LIBERO-Goal 主基准，并做 hard-3 online suite。作者从成功/失败轨迹上训练线性 probe，证明冻结特征里可读出 success-like signal；在 hard-3 聚合在线评测中，value-guided 选择达到 **42.44%**，高于 random **36.89%** 与 greedy **31.11%**。真机：**无真实机器人实验**。设备配置：不适用。
- **算力**：正文披露 probe 训练 **单 GPU、5 分钟内** 即可完成；主要额外成本来自 simulator-backed candidate evaluation。基础 VLA 的预训练算力不在本文新增披露范围内。
- **解读**：它说明很多 VLA 已经“知道自己会不会成功”，只是训练目标没把这种信息显式读出来。局限是当前仍依赖多候选评估。复现建议先测 probe 在不同任务切分下的稳定性，否则很容易把时间或任务泄漏误当价值信号。

### B. VLA 评测、失效监测与部署

#### 1) How VLAs Fail Differently: Black-Box Action Monitoring Reveals Architecture-Specific Failure Signatures
- **标题**：How VLAs Fail Differently: Black-Box Action Monitoring Reveals Architecture-Specific Failure Signatures
- **简称**：VLA Failure Signatures
- **领域**：VLA Safety / Failure Analysis / Action Monitoring
- **论文**：https://arxiv.org/abs/2605.28726
- **项目**：正文未披露
- **代码**：https://github.com/krishnam94/vla-edge
- **评测**：仿真 benchmark：PushT；另一评测设置为 **ALOHA 14-DoF bimanual cube transfer**。总计 **450 episodes**。在 PushT 上，direction reversal rate 对 VQ-BeT / Diffusion 的 AUROC 分别达 **0.93 / 0.79**；在 ALOHA 上对 ACT 达 **0.91**。ALOHA 条件下，ACT 无 SafeContract 成功率 **62% (31/50)**，有 SafeContract 为 **56% (28/50)**，velocity violation AUROC 仅 **0.52**，说明传统速度约束抓不到“动作方向错但物理上看似安全”的失败。真机：**无新增真实机器人实验**；ALOHA 为机器人平台评测。设备配置：**ALOHA 双臂 14-DoF 平台**；末端执行器与相机位姿数量正文未披露。
- **算力**：正文披露评测平台为 **NVIDIA A40 GPU（RunPod）**；监测开销 **<0.001% VLA inference time（约 13µs / call）**。训练算力正文未披露。
- **解读**：它最重要的贡献是把“失效监测要跟架构类型匹配”说透了：离散动作模型和连续动作模型坏法根本不同。局限是实验任务还少。复现建议一定复做相同数据同协议对照，否则不同 checkpoint 很容易把结论冲淡。

#### 2) Colosseum V2: Benchmarking Generalization for Vision Language Action Models
- **标题**：Colosseum V2: Benchmarking Generalization for Vision Language Action Models
- **简称**：Colosseum V2
- **领域**：VLA Benchmark / Generalization / Sim2Real Validation
- **论文**：https://arxiv.org/abs/2605.27759
- **项目**：https://sites.google.com/usc.edu/colosseum-v2/
- **代码**：正文未披露
- **评测**：仿真 benchmark：基于 ManiSkill 的 **28 个任务**，覆盖 **Single-Arm** 与 **Bimanual** 两种形态，并对视觉/语言/动作扰动做系统评测；每个 task-perturbation 对测 **200 episodes**。结果上，ACT 的视觉泛化掉点明显大于 π0.5：Single-Arm / Bimanual 平均下降 **28.57% / 29.06%**，而 π0.5 为 **7.73% / 9.86%**。真机：复现 **Raise Cube / Rotate Arrow / Lift Peg Upright** 3 个单臂任务，每任务训练 **60 demonstrations**。设备配置：**Franka Panda 单臂或双臂 + parallel-jaw gripper**；仿真/真机观测均对应 **2 个 external RGB**，单臂再加 **1 个 wrist RGB**、双臂为 **2 个 wrist RGB**，即单臂 **3 机位**、双臂 **4 机位**。
- **算力**：正文披露训练使用 **单张 H100**；ACT 单臂/双臂分别训练 **130k / 230k iterations，batch 256**，π0.5 为 **500k / 250k iterations，batch 4**。评测方面，ACT 双测试套总评测可在 **单张 RTX 4090** 上 **11.5h** 内完成。
- **解读**：这篇价值很直接：它把“VLA 真泛化了吗”从零散 case study 拉成统一压测台。亮点是 sim-real 趋势相关性被单独验证；局限是当前基线仍少。复现建议先照搬它的 perturbation protocol，不然不同论文的“泛化”几乎没法横向比。

#### 3) A Factory-Floor Deployment Case Study of VLA Pipelines for Industrial Packaging Task: Workflow, Failures, and Lessons
- **标题**：A Factory-Floor Deployment Case Study of VLA Pipelines for Industrial Packaging Task: Workflow, Failures, and Lessons
- **简称**：Factory VLA Deployment Study
- **领域**：VLA Deployment / Industrial Robotics / Reliability
- **论文**：https://arxiv.org/abs/2605.27461
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**无**。真机：西门子工厂包装任务，围绕“抓透明配件袋并放入纸盒腔体”做多轮现场迭代，累计 **2535 episodes（约 10 小时）**。设备配置：**双 UR7e + Robotiq 2F-85**，带自制 3D 打印指尖；**2 个 wrist-mounted cameras + 1 个 base camera**，共 **3 个机位**。正文未给统一最终成功率，但给出 Trials 2/3 的失败画像：内容物残留 **65%**、多袋误抓 **23%**、未完全入盒 **15%**、抓取差 **15%**，并明确指出最终成功率“未达预期”。
- **算力**：现场 rollout 运行在 **Siemens IPC + RTX 5090**；前两轮用 LoRA 微调，**batch 32 / 30k steps**；第三轮改全量微调，**batch 128 / 60k steps（约 4 epochs）**；额外内部集群细节正文未披露。
- **解读**：这篇不是刷分论文，而是把 VLA 真落工厂后最痛的东西摊开：延迟、抖动、透明袋分割和恢复样本都卡脖子。局限是最终成功率仍不够高。复现建议先把异步推理与失败分类打通，再谈继续堆数据。

### C. 接触丰富操作、触觉与显微操控

#### 1) Beyond Binary: Sim-to-Real Dexterous Manipulation with Physics-Grounded Contact Representation
- **标题**：Beyond Binary: Sim-to-Real Dexterous Manipulation with Physics-Grounded Contact Representation
- **简称**：Beyond Binary / CoP Contact Rep.
- **领域**：Sim2Real / Dexterous Manipulation / Tactile Representation
- **论文**：https://arxiv.org/abs/2605.28812
- **项目**：https://mpan31415.github.io/tactile_rep/
- **代码**：正文未披露
- **评测**：仿真 benchmark：IsaacLab 中的 **peg-in-hole insertion** 与 **ball-balancing** 两类 contact-rich 任务。真机：同两类任务直接 sim-to-real 迁移。设备配置：**16-DoF Allegro hand**，配 **XELA uSkin** 覆盖 fingertips / phalanges / palm；末端即灵巧手本体；策略为 **blind tactile policy**，**无相机位姿输入**。peg-in-hole 真实评测对每种形状 **10 trials**，CoP 表征整体最优；ball-balancing 对不同球体同样 **10 trials**，更高维 raw-taxel 表征反而不如紧凑 CoP 表征稳定。
- **算力**：正文/附录给出 PPO 细节：自适应学习率初值 **5e-4**、目标 KL **0.016**、插入任务每迭代 **64 steps/env**、平衡任务 **16 steps/env**、每轮 **5 epochs / 4 minibatches**。GPU 型号、卡数、总训练时长正文未披露。
- **解读**：这篇抓住了 tactile sim2real 的核心矛盾：信号太粗不够用，太原始又过不了仿真鸿沟。亮点是 CoP 表征在真机上赢过 raw taxel；局限是任务还集中在盲操。复现建议先做表征替换消融，不要一上来把全部收益都归因于 RL。

#### 2) Mag-VLA: Vision-Language-Action Model for Bimanual Magnetically Actuated Microrobot Manipulation
- **标题**：Mag-VLA: Vision-Language-Action Model for Bimanual Magnetically Actuated Microrobot Manipulation
- **简称**：Mag-VLA
- **领域**：VLA / Microrobot / Bimanual Manipulation
- **论文**：https://arxiv.org/abs/2605.28486
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**无标准仿真 benchmark**，主体是 teleoperated dataset 上的离线学习与真机评测。数据集含 **75 episodes / 20,724 RGB frames**，3 个难度递增任务。真机：三类显微血管通道操控任务分别统计 approach 与 transport。设备配置：**2 个 Sensapex uMp micromanipulators** 驱动双磁体执行，操作对象为磁驱微机器人；视觉为 **1 个 top-down microscope**，共 **1 个主机位**。结果上，approach success 在三任务均为 **90%**；transport success 随路径弯曲度增加从 **80% / 70% / 50%** 递减。
- **算力**：正文披露训练使用 **4×A100 80GB**，有效 batch size **64**；数据按 **60/9/6 episodes** 划分 train/val/test。统一总训练时长正文未披露。
- **解读**：亮点是把 VLA 推到了显微磁驱这种强非线性、弱观测的极端场景里，还做出分阶段成功率。局限是数据量仍小、任务模板单一。复现建议先保留 ACT-style chunk decoder，再去替换视觉骨干，否则很难复现其稳定性差异。

## Agent / OpenClaw 能力增强（固定栏目）

### 今日增量观察
- **PrimitiveVLA / ProgVLA / VERA** 一起说明：下一轮 VLA 竞争点不只是“更大”，而是 **更结构化地组织动作、进度与世界模型**。
- **VLA Failure Signatures / Colosseum V2 / Factory VLA Deployment** 形成了一条很实用的工程链：**先压测泛化，再监控动作异常，最后落到现场闭环修复**。
- **Beyond Binary / Mag-VLA** 则提示一个趋势：高价值增量开始从纯视觉 VLA 转向 **触觉、接触与特殊执行器耦合**。

### 对 OpenClaw / skill 工作流的直接启发
1. **把 skill 拆成可复用 primitive**：PrimitiveVLA 证明“原子动作库”比整段脚本记忆更利于迁移，OpenClaw 的复杂工作流也该往可重组 skillpack 走。
2. **监控不能只看越界**：VLA Failure Signatures 提醒我们，很多失败不是“超限”，而是“方向错但表面正常”；OpenClaw 工具调用也应加入轨迹/反转类监控。
3. **部署文档要保留失败画像**：Factory VLA Deployment 那种失败分类表很值得照搬到 agent/robot 项目复盘里，比只记“成功率多少”更能指导下一轮修复。
