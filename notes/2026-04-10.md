### A. VLA 架构、推理与部署效率

#### 1) A1: A Fully Transparent Open-Source, Adaptive and Efficient Truncated Vision-Language-Action Model
- **标题**：A1: A Fully Transparent Open-Source, Adaptive and Efficient Truncated Vision-Language-Action Model
- **简称**：A1
- **领域**：VLA / 截断推理 / 自适应预算推理
- **论文**：https://arxiv.org/abs/2604.05672
- **项目**：正文宣称全栈开源与可复现，但 arXiv abs 页未直接挂出独立项目页
- **代码**：正文宣称训练代码、数据处理、checkpoint、评测脚本均开放；当前可直接核到的是论文声明，仓库直链未在 abs 页显式给出
- **评测**：仿真 benchmark 覆盖 **LIBERO** 与 **VLABench**；正文给出 A1 在 LIBERO 上平均成功率 **96.6%**，在 VLABench 平均成功率 **53.5%**。真机方面，作者在 **Franka、AgiBot、OpenArm、Dobot-Arm** 四个平台上做了 7 类操作任务，并额外在 **RoboChallenge** 的 30 个真机任务上评测。设备配置方面，正文明确采用多机器人多平台设定，但各平台 camera 拓扑不完全统一公开；只可确认模型输入支持多相机图像加机器人 proprioception。
- **算力**：预训练采用两阶段流程。正文/附录披露主训练超参：预训练 **batch size 1024、total training steps 200K**；下游微调中，LIBERO 为 **batch size 128、50K steps**，VLABench 为 **batch size 64、50K steps**，RoboChallenge 不同平台在 **32~64 batch、50K/100K steps** 间变化。推理侧，正文强调通过跨层截断与动作头早停降低成本，在 LIBERO 上给出单 episode 推理时间约 **37.8s → 10.5s** 的量级改进，并称 flow-matching 推理 latency 可降 **72%**。GPU 型号正文未系统统一披露。
- **信息来源**：arXiv abs、PDF 正文与附录
- **解读**：A1最值得看的是它不是只卷 backbone 压缩，而是把 **VLM 主干 + action head** 一起纳入预算感知推理设计。对于真实部署，这比单纯做量化更务实，因为 VLA 真正的延迟瓶颈常在动作生成头。另一个价值是“全透明开源”姿态：如果后续仓库兑现，A1 可能会成为 2026 年这一波开源 VLA 里很适合工程复现的基线。

#### 2) Adaptive Action Chunking at Inference-time for Vision-Language-Action Models
- **标题**：Adaptive Action Chunking at Inference-time for Vision-Language-Action Models
- **简称**：AAC
- **领域**：VLA / Action Chunking / 推理时自适应控制
- **论文**：https://arxiv.org/abs/2604.04161
- **项目**：https://lance-lot.github.io/adaptive-chunking.github.io/
- **代码**：项目页称代码与视频公开；abs 页未直接给出单独 GitHub 仓库链接
- **评测**：仿真 benchmark 主要是 **RoboCasa Kitchen（24 个 kitchen manipulation tasks）** 与 **LIBERO（四个 task suites，共 40 个任务）**，并进一步测了 **LIBERO-Pro** 的 OOD 扰动。真机部分，使用 **Realman 单臂机器人 + Mycobot gripper**，视觉为 **1 路 wrist RGB + 1 路 tripod side RGB**，每任务收集 **50 条** 人类遥操作示范。真机任务共 3 个：**banana pick-and-place、emergency button pressing、long-horizon toy pick-place + close drawer**。
- **算力**：仿真训练中，作者在 **8× NVIDIA A800** 上进行 mixed-precision 训练；RoboCasa 与 LIBERO 均联合训练 **90 epochs**。真机微调同样使用 **8× A800**，每卡 batch size **32**，每任务训练 **300 epochs**。推理侧正文专门比较了 AAC 与固定 chunk size 的推理时间/成功率权衡，但未统一给出所有任务 FPS；方法本质上是利用 action entropy 自适应调 chunk，而非增加新的重模型。
- **信息来源**：arXiv abs、项目页、PDF 正文
- **解读**：这篇切得很准。Action chunking 一直是 VLA 系统里“看似小参数、实际强影响”的部署旋钮，但过去多半靠拍脑袋固定值。AAC 的意义在于把 chunk size 从静态超参变成推理时策略的一部分：该稳的时候长一点，该纠偏的时候短一点。对长时序 manipulation 尤其有用，因为很多失败并不是策略不会，而是 chunk 过长导致“看见错了也来不及改”。

#### 3) E-VLA: Event-Augmented Vision-Language-Action Model for Dark and Blurred Scenes
- **标题**：E-VLA: Event-Augmented Vision-Language-Action Model for Dark and Blurred Scenes
- **简称**：E-VLA
- **领域**：VLA / Event Camera / 低光与模糊鲁棒操控
- **论文**：https://arxiv.org/abs/2604.04834
- **项目**：正文未披露独立项目页
- **代码**：https://github.com/JJayzee/E-VLA
- **评测**：工作完全围绕真实机器人感知退化场景展开。作者构建了基于 **LeRobot SO100 机械臂 + angle-driven gripper + DAVIS346 event camera** 的开源遥操作平台，收集 **RGB-event-action** 同步数据。任务包括 **Pick-Place、Sorting、Stacking** 三类；测试覆盖多种照度（例如 30 lux、10 lux、低至 <1 lux 级别）和不同 motion blur/exposure 条件。部署端运行在 **NVIDIA AGX Orin**，同步推理采用 **action chunk size 40、30 Hz**。
- **算力**：正文披露基础训练在 **单张 NVIDIA A800** 上完成；主训练为 **20k iterations，batch size 256，lr 2e-4**。若启用 event adapter，两阶段额外训练分别为 **10k iterations（batch 256）** 和 **10k iterations（batch 128）**。推理侧强调无需高频 event-to-image reconstruction，而是直接做 event integration，以降低真实部署开销。
- **信息来源**：arXiv abs、GitHub、PDF 正文与附录
- **解读**：E-VLA 的工程价值非常直接：它不是把 event camera 当炫技传感器，而是正面解决 **低光、黑场、运动模糊** 这些 frame-based VLA 很脆的老问题。更关键的是，它没有绕回“先重建图像再喂策略”的老路，而是研究 event 与现有预训练 VLA 的轻量耦合方式。这条路线如果继续成熟，很可能成为户外、夜间、机载或高速操作场景里 VLA 感知增强的实用插件。

### B. VLA + RL / 视频价值建模 / 世界模型式策略增强

#### 4) ViVa: A Video-Generative Value Model for Robot Reinforcement Learning
- **标题**：ViVa: A Video-Generative Value Model for Robot Reinforcement Learning
- **简称**：ViVa
- **领域**：VLA+RL / Video Value Model / 长时序任务进度估计
- **论文**：https://arxiv.org/abs/2604.08168
- **项目**：正文未披露独立项目页
- **代码**：正文未披露代码仓库
- **评测**：作者把 value estimation 从静态 VLM 判分改成 **视频生成式价值模型**。真实任务共有 3 个：**shirt folding、box packaging/assembly、toilet paper organization**。论文强调最复杂的 **box assembly** 真机实验，并报告 success rate 与 throughput（每小时完成任务数）双指标。输入显式包含多视角视觉与 proprioception，正文图示可见 **left wrist / high / right wrist** 等视角组合；机器人为双臂操作平台，但具体型号在正文片段中未统一直接给出。
- **算力**：所有模型在相同三任务示范数据上训练 **1 epoch**，**batch size 192**，prediction horizon **K=50**；ViVa 采用 DDIM sampling。正文明确所有实验均在 **8× NVIDIA A800 GPUs** 上进行。推理侧重点不在高 FPS，而在更稳定的任务进度估计与 advantage 质量。
- **信息来源**：arXiv abs、PDF 正文
- **解读**：这篇是这批增量里我比较看重的一篇。很多“VLA+RL”工作还停留在给 VLM 套个 reward/value 头，但 ViVa 直接借用 **视频生成模型的时序先验** 去做 value grounding，本质上是在问：一个状态值不值钱，不能只看这一帧，要看它把机器人带向了什么未来。对长时序、部分可观测、错误会滞后暴露的任务，这比静态打分合理得多，也更像真正可部署的 critic。

#### 5) Veo-Act: How Far Can Frontier Video Models Advance Generalizable Robot Manipulation?
- **标题**：Veo-Act: How Far Can Frontier Video Models Advance Generalizable Robot Manipulation?
- **简称**：Veo-Act
- **领域**：Video Policy / 视频模型驱动操控 / VLA 替代与增强
- **论文**：https://arxiv.org/abs/2604.04502
- **项目**：正文未披露独立项目页
- **代码**：正文未披露代码仓库
- **评测**：核心设定是用前沿视频模型（文中以 **Veo-3** 为代表）生成未来图像轨迹，再由 **inverse dynamics model (IDM)** 还原动作。实验覆盖 **simulation + real robot** 两组，每组各 3 个设定；任务是偏语义与摆放约束的 **object-placement / grasp-and-place**。真机平台为 **7-DoF 机械臂 + 12-DoF dexterous hand**，带 **1 路 global-view camera + 1 路 wrist camera**；视频生成和 IDM 主要用 global-view 输入。
- **算力**：多头 IDM 在仿真采样 **300k frame-pair samples**；正文披露 IDM 训练使用 **4× NVIDIA GPU**（全文可核到为 4 卡训练）、训练约 **85,000 iterations**；VLA baseline 微调部分采用 **batch size 32、40K iterations**。推理层面，论文强调即使只需少量 sampling steps 也能获得接近最优闭环性能，但未系统给出统一 FPS/显存表。
- **信息来源**：arXiv abs、PDF 正文
- **解读**：这篇的价值不在于“视频模型马上替代 VLA”，而在于它认真回答了一个现在很热但经常被喊口号的问题：**大视频模型到底能不能直接变成机器人策略？** 结论相对克制——在高层任务轨迹、语义跟随、粗粒度规划上很有潜力，但在低层精度控制上仍不够。这反而是好消息，因为它把视频模型在机器人里的合理位置说清楚了：更像高层视觉规划器 / imagination 模块，而不是今天就能裸上生产线的 end-to-end controller。

#### 6) DriveVA: Video Action Models are Zero-Shot Drivers
- **标题**：DriveVA: Video Action Models are Zero-Shot Drivers
- **简称**：DriveVA
- **领域**：Autonomous Driving / Video Action Model / World Model
- **论文**：https://arxiv.org/abs/2604.04198
- **项目**：正文未披露独立项目页
- **代码**：正文未披露代码仓库
- **评测**：这是自动驾驶方向里很贴近“世界模型+动作联合生成”的一篇。作者在 **zero-shot** 设定下，从 **NAVSIM 训练** 直接评估到 **nuScenes**（cross-dataset）和 **Bench2Drive / CARLA**（cross-domain real→sim）。正文报告相较此前 SOTA 方法，在 zero-shot nuScenes 上平均 displacement error 与 collision rate 分别下降 **78.9% / 83.3%**，在 Bench2Drive(CARLA) 上下降 **52.5% / 52.4%**。输入既支持多相机，也专门分析了 front-camera-only 设定。
- **算力**：训练采用 distributed **bf16 mixed precision**；先以 **batch size 80 训练 20k steps**，再通过梯度累积把有效 batch 提到 **640**，继续训练 **10k steps**；学习率 **1e-4**。正文未统一披露 GPU 型号与总训练时长。推理方面，论文强调“视频预测与动作在共享潜空间中联合解码”，并指出 **as few as two sampling steps** 即可达到接近最优闭环表现。
- **信息来源**：arXiv abs、PDF 正文
- **解读**：DriveVA 的关键不是再做一个 driving foundation model，而是把 **未来视觉演化** 和 **动作规划** 放进同一生成过程，减少传统 world model → planner 两阶段接口错位。对自动驾驶来说，这非常重要：如果你想象出来的未来画面和你要执行的轨迹不是一个系统里长出来的，闭环经常会“想得很好、开得很歪”。这篇至少在 zero-shot 泛化上给了一个比较强的证据。

### C. Sim2Real / Real2Sim / 数据生成与扩展

#### 7) SIM1: Physics-Aligned Simulator as Zero-Shot Data Scaler in Deformable Worlds
- **标题**：SIM1: Physics-Aligned Simulator as Zero-Shot Data Scaler in Deformable Worlds
- **简称**：SIM1
- **领域**：Real2Sim / Sim2Real / 可变形体操控 / 数据引擎
- **论文**：https://arxiv.org/abs/2604.08544
- **项目**：https://internrobotics.github.io/sim1.github.io/
- **代码**：项目页当前可访问；代码开放状态需以项目页后续更新为准，abs 页未直挂 GitHub 仓库
- **评测**：论文主打 **R2S2R（real-to-sim-to-real）** 管线：从真实场景数字化、到变形物理校准、再到仿真生成大规模合成操控数据，最后零样本迁移回真机。代表性真实任务是 **T-shirt folding**，并扩展到更多布料操作任务。正文给出代表结果：在 **1:15 等价比** 的条件下，真实部署可做到 **90% zero-shot success**，相对真实数据基线带来 **50% generalization gains**。数据采集图示中可核到 **ARX X5 / ACONE / D405 camera** 等组件，仿真基础设施运行于 **RTX 4090 workstation**。
- **算力**：正文明确其 deformation-stable AVBD solver 与仿真生成运行在 **RTX 4090** 工作站；附录进一步提到大规模仿真数据引擎部署在 **8× NVIDIA RTX** 的服务器环境。论文重点在于用物理对齐仿真替代大规模真机采集，未像常规 VLA 一样给出单一“训练卡数+时长”账单。
- **信息来源**：arXiv abs、项目页、PDF 正文与附录
- **解读**：SIM1 很对当前具身痛点：**刚体任务的数据扩展已经卷得差不多了，真正缺的是可变形体**。布料、软体、接触拓扑变化这类任务靠传统 rigid sim 扩数据，经常越扩越偏。SIM1 的思路是先把模拟器“校正到像真的”，再谈 scale。若这条路成立，它不只是多一个 cloth benchmark，而是可能成为下一阶段 deformable manipulation 数据生产的基础设施。

#### 8) CRAFT: Video Diffusion for Bimanual Robot Data Generation
- **标题**：CRAFT: Video Diffusion for Bimanual Robot Data Generation
- **简称**：CRAFT
- **领域**：Video Diffusion / Bimanual Manipulation / Sim2Real 数据增强
- **论文**：https://arxiv.org/abs/2604.03552
- **项目**：https://craftaug.github.io/
- **代码**：项目页已公开，abs 页未直接给出单独 GitHub 链接
- **评测**：作者面向双臂学习，提出用 **video diffusion** 从少量真实示教和仿真轨迹生成大规模、带动作标签的合成示范。仿真使用 **RoboTwin**，覆盖 3 个双臂任务，并测试 **UR5→Franka** 的 cross-embodiment transfer；真机则在 **bimanual Franka Research 3** 上评测 **Lift Roller、Place Cans、Stack Bowls** 三个任务，并用 **bimanual xArm7 → Franka** 做跨本体迁移。视觉配置为 **1 或 3 路 Intel RealSense D435i**，视具体任务是否需要 wrist 视角而定。
- **算力**：论文披露较完整：ACT policy 训练与仿真推理可在 **单张 RTX 4090** 上完成；小规模视频生成使用 **单张 RTX 5090**，大规模生成分布在 **3× RTX 6000** 上；真机 ACT 训练/推理同样使用 **RTX 5090**。数据量方面，CRAFT 方案典型使用 **每任务 1000 条生成 demonstrations + 原始 real demonstrations**，与若干增强基线做公平比较。
- **信息来源**：arXiv abs、项目页、PDF 正文与附录
- **解读**：CRAFT 抓的是双臂学习最烧钱的一环：**真实示教太贵，而且视觉多样性太差**。它不是单纯做图像风格增强，而是把仿真轨迹、结构边缘、视频扩散和动作标签配对起来，尽量让生成数据在时序上仍可训练策略。对双臂和跨本体任务，这是很有实际吸引力的路线，因为只靠真实遥操作去堆规模，成本很快失控。
