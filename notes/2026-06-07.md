### A. VLA、具身表示与动作生成

#### 1) TempoVLA: Learning Speed-Controllable Vision-Language-Action Policies
- **标题**：TempoVLA: Learning Speed-Controllable Vision-Language-Action Policies
- **简称**：TempoVLA
- **领域**：VLA / speed control / manipulation
- **论文**：https://arxiv.org/abs/2606.06491
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**LIBERO**，表 3 在 4 个 suite 上评测，单速 baseline 平均成功率 **96.7%**；七速训练下 **0.5/0.75/1/1.25/1.5/1.75/2.0×** 的平均成功率分别为 **95.0/96.3/96.9/97.4/97.3/95.6/89.4**，峰值出现在 **1.25×**。真机：**7-DoF Franka arm**；末端为 **1-DoF parallel gripper**；视觉为 **主视角相机 + 腕部相机，共 2 个机位**；5 个真实任务、每任务 **50 条 teleop 训练轨迹**，每个指令速度 **10 次 rollout**。正文图示给出 overall success：baseline **80%**、**0.75× 66%**、**1.0× 88%**、**1.25× 84%**、**1.5× 70%**、**动态调速 96%**。
- **算力**：正文与附录**未披露训练 GPU 型号、卡数和总时长**。
- **解读**：把“速度”从控制器副作用变成显式条件变量，是这篇最实用的点。亮点是无需改底层控制器就能双向调速，且动态调速真机最好；局限是高速端仍明显掉点。复现时先严格实现 VSTA 的 merge/split 规则，再调文本式 speed prompt。

#### 2) AffordanceVLA: A Vision-Language-Action Model Empowering Action Generation through Affordance-Aware Understanding
- **标题**：AffordanceVLA: A Vision-Language-Action Model Empowering Action Generation through Affordance-Aware Understanding
- **简称**：AffordanceVLA
- **领域**：VLA / affordance / instruction-following manipulation
- **论文**：https://arxiv.org/abs/2606.06155
- **项目**：https://skywalker-yqz.github.io/AffordanceVLA/
- **代码**：https://github.com/Skywalker-yqz/AffordanceVLA
- **评测**：仿真 benchmark：**LIBERO** 上 4 个 suite、**50 rollouts** 评测，平均成功率 **95.8%**；**CALVIN ABC→D** 上 **1000 rollouts**，**1/2/3/4/5-step = 96.8 / 92.0 / 87.5 / 80.8 / 75.9**，平均链长 **4.33**。真机：**7-DoF Franka Emika Panda**；末端为 **DH-Robotics PGC-140 parallel jaw gripper**；视觉为 **1 个固定第三视角 + 1 个腕部 Intel RealSense D435，共 2 个机位**，RGB **15Hz**。真实任务表按 **15 trials/任务** 统计：Basic tasks 平均 **88.3%**，Complex tasks 平均 **82.9%**，如 Drawer-close **100%**、Toaster-toast **86.7%**。
- **算力**：附录披露全部训练在 **16× NVIDIA H200（141GB HBM3e/卡）** 集群完成；总训练时长正文未披露。
- **解读**：它的核心不是多加一个 affordance 头，而是把 Which/Where/How 三层中间表示内化进 MoT。亮点是 CALVIN 和真机都稳；局限是长程任务仍弱于真正带记忆的架构。复现时要保住三阶段 curriculum，否则 Stage II 缺失会明显伤 OOD 泛化。

### B. 世界模型与闭环评测

#### 1) PiL-World: A Chunk-Wise World Model for VLA Policy-in-the-Loop Evaluation
- **标题**：PiL-World: A Chunk-Wise World Model for VLA Policy-in-the-Loop Evaluation
- **简称**：PiL-World
- **领域**：world model / VLA evaluation / closed-loop rollout
- **论文**：https://arxiv.org/abs/2606.05773
- **项目**：https://pil-world.github.io/
- **代码**：正文未披露
- **评测**：仿真 benchmark：**无独立仿真 benchmark**；方法重点是替代真机闭环评测，而非在模拟器上刷榜。真实评测：双臂操作平台型号正文未披露；末端为 **双夹爪**（动作是 **14 维 absolute joint-space command**，含左右 gripper state）；视觉为 **top/head view + left wrist + right wrist，共 3 个机位**。任务为 **Sort Cubes / Stack Bowls / Stack Blocks**；测试划分分别为 **20/18/20 条 test trajectories**，对应 **60/54/40 个 subtasks**。相较 baseline，真实成功率与想象成功率的误差 **63.2% → 12.0%**；单步 LPIPS 在三任务上分别为 **0.0965 / 0.1100 / 0.1208**。
- **算力**：附录披露：在 **RealSource World** 上预训练 **2 epochs / 64× H20 GPU**，目标任务微调 **20 epochs / 8× H20 GPU**。
- **解读**：价值在于把 world model 从“开环预测器”改成真正可和 VLA 轮流闭环的评测环境。亮点是 success-rate 估计误差大幅收敛；局限是平台细节和腕视角遮挡仍限制 Stack Blocks。复现时要先对齐 head-view action projection 和 history memory。

### C. 灵巧操作、工具使用与数据接口

#### 1) DexFuture: Hierarchical Future-State Visuomotor Targeting for Bimanual Dexterous Tool Use
- **标题**：DexFuture: Hierarchical Future-State Visuomotor Targeting for Bimanual Dexterous Tool Use
- **简称**：DexFuture
- **领域**：dexterous manipulation / bimanual tool use / future-state targeting
- **论文**：https://arxiv.org/abs/2606.05699
- **项目**：https://blarklee.github.io/DexFuture_official_website/
- **代码**：正文未披露
- **评测**：仿真 benchmark：**OakInk2** 双手工具使用任务，覆盖 **cutting / pouring / wiping / shearing / stirring**。表 1 中 DexFuture（pred target）在代表性任务上取得：bread cutting **83.49%**、apple cutting **89.79%**、pour mug **41.05%**、big-brush wipe **56.96%**、small-brush wipe **67.17%**、shear paper **30.69% / 48.66%**；平均成功率约为特权基线的 **90%**，而 no-target policy 仅 **7%** 量级。真机：**无真实机器人实验**；机器人本体/末端/相机位姿数量 **不适用**。
- **算力**：正文**未披露训练 GPU 型号、卡数与时长**；推理部分披露 DexFuture 运行在 **60Hz**，对比的 DexWM-style CEM planning 在 **单张 RTX 3090Ti** 上约 **0.24Hz**。
- **解读**：它把“未来目标”从示教特权信息改成可预测的中层接口，避开高维在线规划。亮点是对双手工具任务仍能接近 oracle target policy；局限是倒液和剪切等高接触不连续任务掉点明显。复现时优先保住 multi-horizon target tokenization 与低层 per-link policy 对齐。

#### 2) RealDexUMI: A Wearable Universal Manipulation Interface for Dexterous Robot Learning
- **标题**：RealDexUMI: A Wearable Universal Manipulation Interface for Dexterous Robot Learning
- **简称**：RealDexUMI
- **领域**：dexterous learning / teleoperation / cross-embodiment interface
- **论文**：https://arxiv.org/abs/2606.06033
- **项目**：https://research.beingbeyond.com/realdexumi （摘要/正文给出，但当前访问 **404**）
- **代码**：正文未披露
- **评测**：仿真 benchmark：**无仿真 benchmark**。真机：默认评测平台为 **Franka FR3**；末端为 **与采集阶段相同的共享灵巧末端模块（6 actuated DoF，含在手视觉与指尖触觉）**；相机数量正文未按“机位数”单列，但硬件图明确包含 **in-hand / wrist vision**。Across **8 个真实任务**，每任务 **20 次 real-robot trials**，总体成功率 **88.75%**；去掉 tactile 后降至 **70.00%**，把 hand action 换成 state-as-action 后降至 **51.25%**。跨 embodiment 还在 **FR3 / RM65 / Adam-U** 上零重训迁移，Drawer 任务成功率分别 **0.85 / 0.75 / 0.80**。
- **算力**：正文/附录**未披露训练 GPU 型号、卡数与总时长**。
- **解读**：这篇真正解决的是“示教灵巧性无法部署”而不是单个 policy 架构问题。亮点是同一末端模块贯穿采集与部署，触觉收益也被量化出来；局限是视觉机位数量和训练算力披露不够完整。复现时末端一致性与 action-state correspondence 必须优先保证。

### D. 人形全身控制与极限运动

#### 1) HANDOFF: Humanoid Agentic Task-Space Whole-Body Control via Distilled Complementary Teachers
- **标题**：HANDOFF: Humanoid Agentic Task-Space Whole-Body Control via Distilled Complementary Teachers
- **简称**：HANDOFF
- **领域**：humanoid whole-body control / task-space interface / agentic planning
- **论文**：https://arxiv.org/abs/2606.06493
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**自建 humanoid velocity tracking + bilateral wrist workspace + fall-recovery 评测**。表 2 中 **Ours+Stab.** 的 **|Δvx|/|Δvy|/|Δωz| = 0.07 / 0.18 / 0.04**，feasibility **97.7%**，robust workspace **0.31 m³**；工作空间统计基于 **2000 个发现目标 + 400 个动作样本/目标**。真机：**29-DoF Unitree G1**；末端为双手/双腕 task-space target，不单列夹爪；感知为 **1 个头部固定 ZED-M RGB-D camera**；机载计算为 **NVIDIA Jetson Thor**。正文给出多项自然语言驱动任务：pick-and-place、pick-transport-place、squat-pick、bimanual-pick-and-hand-off、bilateral pick-and-place，但**未给出真机 trials 数与成功率表**。
- **算力**：附录披露训练采用 **单 GPU**、**4096 并行环境**、每 iteration **24 steps**（约 **98K transitions/iter**），PPO 训练；**GPU 型号与总训练时长未披露**。
- **解读**：重点是把规划器接口压成 10D task-space command，同时保住全身协调与恢复能力。亮点是 robust workspace 和 velocity tracking 一起强；局限是真机仍以代表性 rollout 为主、缺少系统化任务统计。复现时 context-gated teacher distillation 和 recovery routing 是成败关键。

#### 2) LadderMan: Learning Humanoid Perceptive Ladder Climbing
- **标题**：LadderMan: Learning Humanoid Perceptive Ladder Climbing
- **简称**：LadderMan
- **领域**：humanoid locomotion / ladder climbing / sim-to-real
- **论文**：https://arxiv.org/abs/2606.05873
- **项目**：https://ladderman-robot.github.io
- **代码**：正文未披露
- **评测**：仿真 benchmark：在 **IsaacSim** 中按 **倾角 55/60/65/70° × rung spacing 20/22/24/26/28/30cm** 网格评测，每个配置 **100 episodes**；LadderMan 在 **24–28cm、55–65°** 区间成功率 **>95%**，在更极端配置下平均成功率仍约 **57%**。真机：**1.3m、29-DoF Unitree G1**；末端执行器不适用；感知为 **1 个 onboard Intel RealSense D435i**；策略推理在 **Jetson Orin**，FoundationStereo 运行在 **RTX 4090**。真实 ladders 为 **A/B/C 三把梯子**，零样本 climbing 成功数分别 **9/10、6/10、7/10**；并完成 adjusting painting、replacing light bulb、box handover 等在梯操作。人类平均速度约 **3.2 s/rung**，LadderMan 为 **3.4 s/rung**。
- **算力**：正文/附录**未披露训练 GPU 卡数与总时长**；部署侧披露 **RTX 4090 + Jetson Orin**。
- **解读**：这篇把稀疏多接触、深度感知和人形全身协调真正绑到一起，难度明显高于楼梯。亮点是三把不同梯子上的零样本成功和接近人类爬梯速度；局限是真机样本量仍小，且依赖深度预处理链。复现时要先打通 VFM 深度桥接，再做 climbing expert distillation。
