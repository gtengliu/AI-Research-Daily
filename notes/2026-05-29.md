### A. 通用 VLA、表示学习与跨任务泛化

#### 1) Qwen-VLA: Unifying Vision-Language-Action Modeling across Tasks, Environments, and Robot Embodiments
- **标题**：Qwen-VLA: Unifying Vision-Language-Action Modeling across Tasks, Environments, and Robot Embodiments
- **简称**：Qwen-VLA
- **领域**：Generalist VLA / Cross-Embodiment / Navigation + Manipulation
- **论文**：https://arxiv.org/abs/2605.30280
- **项目**：https://qwen.ai/blog?id=qwenvla
- **代码**：https://github.com/QwenLM/Qwen-VLA
- **评测**：仿真/离线 benchmark：LIBERO **97.9%**、Simpler-WidowX **73.7%**、RoboTwin-Easy/Hard **86.1% / 87.2%**、R2R **69.0% OSR**、RxR **59.6% SR**。真机：ALOHA 双臂平台，覆盖 pick-and-place、table cleaning、bowl stacking、bowl pick & place、towel folding、fine-grained manipulation 六类任务，并评测颜色、实例、位置、背景、指令五类 OOD 泛化；预训练版 in-domain 平均 **83.6%**、OOD 平均 **76.9%**。设备配置：**双 6-DoF 机械臂 + parallel-jaw grippers**；**2 个 wrist RGB + 1 个 first-person RGB**，共 **3 个机位**。另在 DOMINO 动态操作上给出 **26.6% zero-shot success**。
- **算力**：正文披露了大规模数据与并行 rollout / 轨迹生成细节，但**未统一披露主模型训练 GPU 型号、卡数与总训练时长**；仅说明部分模拟轨迹生成与 RL rollout 运行在 **multiple GPUs** 上。
- **解读**：它的价值在于把导航、桌面操作、双臂操作和多机体提示统一进一个 VLA 配方，证明“共享主干+具身提示”可行。局限是训练预算未完全公开，复现门槛仍高。建议先复做 ALOHA 子集与 embodiment prompt，再逐步加导航数据。

#### 2) DynaFLIP: Rethinking Robotics Perception via Tri-Modal-Dynamics Guided Representation
- **标题**：DynaFLIP: Rethinking Robotics Perception via Tri-Modal-Dynamics Guided Representation
- **简称**：DynaFLIP
- **领域**：Robot Perception / VLA Backbone / OOD Generalization
- **论文**：https://arxiv.org/abs/2605.30350
- **项目**：https://dynaflip-robotics.github.io
- **代码**：正文未披露
- **评测**：仿真 benchmark：MetaWorld、RLBench、LIBERO，作为可插拔视觉表征增强多种策略/ VLA。真机：3 个任务——Pick `<object>` into Sink、Pour almonds into `<object>`、Unfold Towel，并测 in-distribution 与 OOD 扰动；相对强基线在真实 OOD 设置下最高提升 **22.5%**。设备配置：**固定底座 UR3 单臂 + two-finger gripper**；**1 个第三人称 RGB + 1 个 wrist RGB**，共 **2 个机位**；每个真实设置 **20 rollouts**。
- **算力**：正文披露视觉预训练约 **4 天 / 4× NVIDIA L40S**；真机与 LIBERO 策略训练使用 task-specific fine-tuning，但未统一披露全部策略训练卡数与总时长。
- **解读**：这篇不是再堆视觉 backbone，而是把图像、语言、动力学变化联合成“控制相关表征”，对真实分布偏移更有针对性。局限是最终仍依赖下游 policy 注入设计。复现建议先测 frozen encoder 替换收益，再做端到端联合训练。

#### 3) Gaze2Act: Gaze-Conditioned Vision-Language-Action Policies for Interactive Robot Manipulation
- **标题**：Gaze2Act: Gaze-Conditioned Vision-Language-Action Policies for Interactive Robot Manipulation
- **简称**：Gaze2Act
- **领域**：Human-in-the-loop VLA / Gaze Interaction / Humanoid Manipulation
- **论文**：https://arxiv.org/abs/2605.30282
- **项目**：https://zuo-kuangji.github.io/Gaze2Act/
- **代码**：正文未披露
- **评测**：真机主评测为 **Unitree G1 humanoid** 上 **7 类 16 个任务**，覆盖 object disambiguation、part-level intent、dynamic intent steering 等场景；主表多数任务 **50 trials**，Hammer 等关键消融任务 **60 trials**。设备配置：**Unitree G1 humanoid**；**1 个机载 RealSense D455 RGB**，操作者侧使用 **Meta Aria** 提供 gaze；单机位机器人视角。论文核心结果表明在 intent accuracy 与 task success 上均优于语言基线和点提示基线。
- **算力**：每个任务组训练 **20,000 steps / 单张 NVIDIA RTX PRO 6000 96GB**，batch size **80**。
- **解读**：它把 gaze 从“辅助 UI 信号”提升为 VLA 控制条件，尤其适合目标歧义和部位级交互。局限是依赖眼动设备与在线对齐稳定性。复现建议优先复做 perception-level prompting 与 action-level conditioning 的分工消融。

#### 4) VLA-Pro: Cross-Task Procedural Memory Transfer for Vision-Language-Action Models
- **标题**：VLA-Pro: Cross-Task Procedural Memory Transfer for Vision-Language-Action Models
- **简称**：VLA-Pro
- **领域**：VLA + Procedural Memory / Cross-Task Transfer
- **论文**：https://arxiv.org/abs/2605.29562
- **项目**：https://geniuslouis.github.io/VLA-Pro
- **代码**：https://github.com/ketchup45/VLA-Pro
- **评测**：仿真 benchmark：自定义 RoboTwin 任务套件（**8 个训练任务、9 个 held-out 测试任务**，每个 seen task **50 trajectories**）与 RLBench X-ICM 划分（**18 个 seen / 23 个 held-out**，主表列出 9 个非零测试任务，**25 trials**）。真机：**UR7e + Robotiq 2F-85**，**1 个 wrist-mounted RealSense D405**，共 **1 个机位**；**6 个训练任务 + 6 个 held-out 测试任务**，每个训练任务 **50 demonstrations**、每个测试任务 **20 trials**。真实平均成功率从 **5.8% 提升到 65.0%**。
- **算力**：正文披露训练使用 **2× NVIDIA A100 80GB**；Base LoRA **10,000 steps**，各 task-specific adapter 额外 **3,000 steps**。
- **解读**：核心不是加更多数据，而是把 task-specific LoRA 当作“过程记忆”，在推理时检索并融合。亮点是跨任务迁移收益非常直观；局限是 memory source 仍需人工定义训练/测试结构。复现建议先固定 π0.5 backbone，只替换 memory retrieval 组件做对照。

#### 5) 3DVLA: Enhancing Vision-Language-Action Models via 3D Spatial and Instance Understanding
- **标题**：3DVLA: Enhancing Vision-Language-Action Models via 3D Spatial and Instance Understanding
- **简称**：3DVLA
- **领域**：3D-aware VLA / Multi-view Spatial Reasoning
- **论文**：https://arxiv.org/abs/2605.29416
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：LIBERO-Plus 与 RoboTwin 2.0。LIBERO-Plus 主表覆盖 **7 类 perturbation**（Camera / Robot / Language / Light / Background / Noise / Layout），zero-shot 平均成功率达到 **86.0%**；RoboTwin 2.0 上对 π0 与 X-VLA 的 Easy/Hard 均有稳定增益，例如 π0 在 Easy/Hard 从 **46.4% / 16.3%** 提升到 **54.5% / 23.2%**。真机：**无正文级真实机器人实验**，主要定位为多视角 3D 表征增强模块。
- **算力**：两阶段训练总计约 **14 小时 / 4× NVIDIA A800**；其中 Stage 1 约 **5h**，Stage 2 约 **9h**。
- **解读**：它抓的是 VLA 的老毛病——只会 2D 相关性，不会真正 3D grounding。优势是模块可插拔、增益稳定；局限是尚未给真实机器人闭环验证。复现建议优先核对多视角标定与 3D instance token 质量，否则收益很难复现。

### B. VLA + RL、后训练与接触丰富操作

#### 1) BORA: Bridging Offline Reinforcement Learning and Online Residual Adaptation for Real-World Dexterous VLA Models
- **标题**：BORA: Bridging Offline Reinforcement Learning and Online Residual Adaptation for Real-World Dexterous VLA Models
- **简称**：BORA
- **领域**：Dexterous VLA / Offline RL / Online Residual Adaptation
- **论文**：https://arxiv.org/abs/2605.30226
- **项目**：https://chenzhongxi-sjtu.github.io/BORA/
- **代码**：正文未披露
- **评测**：真机为主：**Franka 单臂 + DexHand021 12-DoF dexterous hand**；**Intel RealSense D435 RGB-D**，正文未明确相机数量，至少核验到该视觉链路。任务包含 **Pick the plush toy / Pick and Place / Open the box / Pull the tissue / Press the button** 五项；每任务 **20 次标准配置 + 20 次 unseen object**。相对纯 imitation 与 decoupled RL，标准设置平均成功率绝对提升 **33%**，unseen object 泛化最高提升 **43%**。
- **算力**：正文附录给出分阶段工作站配置：在线阶段使用 **8× NVIDIA H100 + 1× RTX 4090**；但各阶段训练时长未完整统一汇总。
- **解读**：它把“离线 token-action RL”和“在线残差纠偏”接起来，比较贴近真实灵巧手系统的训练现实。局限是系统复杂、算力要求偏高。复现建议先独立验证 offline critic 对 chunk action 的收益，再接在线 residual 模块。

#### 2) VE2VF: Vision-Enabled to Vision-Free Distillation via Real-world Reinforcement Learning for Robust Contact-Rich Manipulation
- **标题**：VE2VF: Vision-Enabled to Vision-Free Distillation via Real-world Reinforcement Learning for Robust Contact-Rich Manipulation
- **简称**：VE2VF
- **领域**：Contact-Rich Manipulation / Real-world RL / Distillation
- **论文**：https://arxiv.org/abs/2605.29564
- **项目**：https://tuwien-asl.github.io/VE2VF/
- **代码**：正文未披露
- **评测**：真机：基于 **NIST Assembly Board I** 的接触丰富插接任务。设备配置：**Franka FR3**；**2 个 wrist-mounted RealSense D435i**，共 **2 个机位**；配 3D Space mouse 人类遥操作。训练任务为 medium peg / ethernet / medium gear 三项，评测扩展到 **8 个 assembly tasks**，并包含 disturbed 与 OOD 几何泛化。最终 vision-free distilled policy 总体成功率 **95.0%**，对最难任务再经额外蒸馏微调可达满分表现。
- **算力**：训练使用 **NVIDIA RTX A4000**；总训练 **50 分钟**，其中 teacher **40 分钟**、student **10 分钟**；每任务预填充 **5 demonstrations**。
- **解读**：思路很漂亮：训练时用视觉老师学对接触策略，部署时把视觉丢掉，只保留 pose/twist/wrench 学生策略来抗分布偏移。局限是任务仍集中在插接类。复现建议严格区分 teacher 与 student 训练时长，否则很容易误判蒸馏贡献。

#### 3) MonoDuo: Using One Robot Arm to Learn Bimanual Policies
- **标题**：MonoDuo: Using One Robot Arm to Learn Bimanual Policies
- **简称**：MonoDuo
- **领域**：Bimanual Policy Learning / Single-to-Bimanual Transfer
- **论文**：https://arxiv.org/abs/2605.29298
- **项目**：https://bimanual-monoduo.github.io
- **代码**：正文未披露
- **评测**：源机器人为 **Franka 单臂**，目标机器人为 **双 UR5e**。视觉配置：**1 个固定式 ZED2 RGB-D + 1 个 wrist-mounted ZED-mini**，共 **2 个机位**。评测任务共 **5 个双臂任务**：Box Lifting、Backpack Packing、Jacket Zipping、Plate Handover 等。zero-shot 在未见过的双臂机器人配置上成功率最高 **70%**；few-shot 阶段额外收集 **25 条 target robot 轨迹** 后，相比从零训练成功率再提升 **65–70%**。
- **算力**：正文披露离线 augmentation 约 **106 秒 / trajectory / NVIDIA A100**；主策略训练总卡数与时长未统一披露。
- **解读**：它把昂贵的双臂示教问题改写成“单臂+人协作示教，再合成双臂数据”，非常务实。局限是依赖较强的视觉分割/修补质量。复现建议先验证 cross-painting 生成数据是否真的保留接触关系，再谈 zero-shot 转移。

### C. Benchmark、校准与部署效率

#### 1) PhAIL: A Real-Robot VLA Benchmark and Distributional Methodology
- **标题**：PhAIL: A Real-Robot VLA Benchmark and Distributional Methodology
- **简称**：PhAIL
- **领域**：Real-Robot Benchmark / VLA Evaluation / Distributional Metrics
- **论文**：https://arxiv.org/abs/2605.29710
- **项目**：https://phail.ai
- **代码**：https://github.com/Positronic-Robotics/phail-paper
- **评测**：真机 benchmark 为主：DROID-style 工位，**Franka Research 3 + Robotiq 2F-85**；**1 个 over-the-shoulder external RGB + 1 个 wrist-mounted RGB**，共 **2 个机位**。任务为 bin-to-bin order picking，物体含 wooden spoons、towels、scissors、batteries。公开基准数据约 **990 episodes**，其中 **396 条**为同夹具人类参考 rollout；评测包含 blind scheduling、30 s/item time cap、成功/安全/超时等分布统计。所有参赛 VLA 共用 **449-episode、约 13 小时** 的微调集。
- **算力**：本文聚焦 benchmark 方法学，**未统一披露各 VLA 微调 GPU 型号**；仅明确所有模型采用各自默认 fine-tuning recipe，并共享同一 449-episode 数据集。
- **解读**：这篇最重要的是把“真实机器人 benchmark”从单点成功率升级为可审计的分布方法学，更适合比较实际可部署性。局限是当前任务仍较聚焦拣选。复现建议连同视频、事件日志和统计脚本一起用，否则只看均值会丢失很多信息。

#### 2) VLAConf: Calibrated Task-Success Confidence for Vision-Language-Action Models
- **标题**：VLAConf: Calibrated Task-Success Confidence for Vision-Language-Action Models
- **简称**：VLAConf
- **领域**：VLA Confidence / Failure Prediction / Calibration
- **论文**：https://arxiv.org/abs/2605.29605
- **项目**：https://sites.google.com/view/vlaconf
- **代码**：https://sites.google.com/view/vlaconf
- **评测**：仿真 benchmark：standard LIBERO，以及 shifted **LIBERO-Pro / LIBERO-Plus**。后两者系统包含 task、object、layout、camera、robot-state、lighting、background、sensor 扰动。真机：**Franka 单臂 + RealSense L515**，演示由 **Gello teleoperator** 收集；任务为 **cup-on-table / duck-in-cooker / towel-folding**。共 **151 次 physical rollouts**，其中 cup-on-table **48/50** 成功、towel-folding **24/48**，并报告 ECE / Brier / NLL 等校准指标。
- **算力**：正文强调单前向 confidence head 的效率优势，但**未披露训练 GPU 型号、卡数与总时长**。
- **解读**：相比反复采样式不确定性估计，VLAConf 更像给 VLA 加了一个可部署的“成功率温度计”。亮点是同时覆盖仿真扰动与真机校准；局限是仍主要做 post-hoc confidence 而非主动恢复。复现建议先检查 calibration split 与 online evidence 设计，避免信息泄漏。

#### 3) ElegantVLA: Learning When to Think for Efficient Vision-Language-Action Models
- **标题**：ElegantVLA: Learning When to Think for Efficient Vision-Language-Action Models
- **简称**：ElegantVLA
- **领域**：Efficient VLA / Dynamic Compute Scheduling / Real-time Control
- **论文**：https://arxiv.org/abs/2605.29438
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：GR00T in SimplerEnv 与 CogACT，分别实现最高 **2.55×** 与 **3.77×** 平均加速。真机：**Franka Research 3** 上 6 个任务（Phone Stand、Pen Holder、Stack Bowls、Pineapple Bun、Toast、Chocolate Bun）；设备配置：**Intel RealSense D435i**，包含 **1 个 front-mounted eye-to-hand + 1 个 wrist-mounted eye-in-hand**，共 **2 个机位**；每任务 **10 trials**。真实部署中计算量降低 **2.18×**，控制频率从 **13.8 Hz 提升到 26.3 Hz**，同时维持或提升成功率。
- **算力**：真实推理服务器为 **NVIDIA GeForce RTX 4090**；正文未完整披露训练阶段 GPU 配置。
- **解读**：它不是做更小的 VLA，而是让视觉、LLM、动作头都按阶段动态省算，特别适合实时控制。局限是调度器依赖时序稳定性假设。复现建议先测不同 compute mode 对末端精度的副作用，再追求极限频率。

## Agent / OpenClaw 能力增强（固定栏目）

### 今日增量观察
- 今天的主线很清楚：**VLA 正从“大一统模型”走向“结构化增强”**，包括过程记忆、3D 空间理解、gaze 条件、动态算力调度与置信度估计。
- 真机论文明显更重视 **OOD、部署效率、可恢复性与评测协议**，单纯刷静态成功率的比重在下降。
- “Moonlight 协作增强 + 原文核验闭环”这套流程依然有意义：Moonlight 公开入口不稳定，但它提醒我们先做结构化候选，再回 PDF 审事实，这对减少摘要幻觉很有用。

### 对 OpenClaw / skill 工作流的直接启发
1. **给复杂 skill 加“过程记忆”**：VLA-Pro 这类工作说明，把历史成功子程序做成可检索模块，比单次长提示更稳，适合沉淀成 OpenClaw skill 组合件。
2. **给自动化任务加“置信度与回退”**：VLAConf/ElegantVLA 的思路可迁到 agent 工具链——在低置信度时自动降速、重试或切换更保守路径。
3. **把多视角/多源信息真正对齐**：3DVLA 和 Gaze2Act 都在强调“输入不只是堆更多模态，而是做可行动的对齐”，这对多工具、多上下文 agent 也同样成立。

### 一个实用的 OpenClaw 小技巧
- 对这类长链研究整理任务，优先采用：**本地批量抓取 → 结构化中间 JSON → 正文 grep/抽取 → 最终 Markdown 定稿**。这样比直接在聊天里来回总结更稳，也更容易做二次复核。
