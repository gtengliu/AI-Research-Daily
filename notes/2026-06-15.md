### A. VLA 系统栈、物理约束与实时推理

#### 1) Hy-Embodied-0.5-VLA: From Vision-Language-Action Models to a Real-World Robot Learning Stack
- **标题**：Hy-Embodied-0.5-VLA: From Vision-Language-Action Models to a Real-World Robot Learning Stack
- **简称**：HyVLA-0.5
- **领域**：VLA / robot learning stack / UMI data / RL post-training
- **论文**：https://arxiv.org/abs/2606.14409
- **项目**：https://tairos.tencent.com/openSourceModels/hy-embodied-0.5-vla
- **代码**：https://github.com/Tencent-Hunyuan/Hy-Embodied-0.5-VLA
- **模型/数据**：https://huggingface.co/tencent/Hy-Embodied-0.5-VLA-UMI ；https://huggingface.co/datasets/tencent/Hy-Embodied-0.5-VLA-Data
- **评测**：RoboTwin 2.0 上按 50-task suite 评测，每任务 100 次随机 rollout；Clean / Randomized 成功率为 **90.9% / 90.1%**，高于 JoyAI-RA 的 **90.5% / 89.3%** 与 π0.5 的 **82.7% / 76.8%**。真机部分覆盖 Dobot X-Trainer、Astribot S1、Unitree G1 等平台，含 6 个双臂/类双臂任务与力觉验证任务。
- **算力/数据**：数据侧披露自研 fingertip UMI + motion-capture cage，采集 **10K+ 小时、100 万+ episodes、70 个任务**；RoboTwin 评测公开成功率，训练 GPU 总账单正文未完整披露。
- **解读**：这篇不是单点模型，而是把数据采集、VLA action expert、SFT、离线 RL 后训练和部署管线打包成一个可落地栈。亮点是 UMI 数据规模、跨平台实机部署和开源模型/数据入口都比较完整；局限是大规模训练成本披露仍不够细。复现优先级应放在 UMI 数据格式、delta-chunk 动作表示和异步部署链路，而不是只看 backbone。

#### 2) PhysVLA: Towards Physically-Grounded VLA for Embodied Robotic Manipulation
- **标题**：PhysVLA: Towards Physically-Grounded VLA for Embodied Robotic Manipulation
- **简称**：PhysVLA
- **领域**：VLA / physics-aware inference / runtime correction / sim2real
- **论文**：https://arxiv.org/abs/2606.13886
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：LIBERO-Spatial 10 个空间推理 pick-and-place 任务，Franka Emika Panda，闭环 MuJoCo 20 Hz，每任务 5 trials。PhysVLA 在 OpenVLA 上将成功率 **36% → 53%**，OpenVLA-OFT **92% → 95%**，Force-VLA **40% → 53%**，Generalist-VLA **36% → 50%**。真机 Agilex Piper 6-DoF pick-and-place 中，OpenVLA baseline **45%**，PhysVLA **95%**，每模式 20 trials。
- **算力**：LIBERO 实验在 **单张 RTX 4090** 上运行；PhysVLA 额外开销约 **0.6 ms/step**，小于 20 Hz 控制周期的 50 ms。
- **解读**：PhysVLA 的实用点在于不改 VLA 权重，而是在运行时用 phase-aware FSM 和 Euler-Lagrange gate 修正物理不一致动作。它适合当成 frozen VLA 的部署补丁；局限是依赖系统/模拟器状态和动力学 oracle，真实复杂接触下的泛化还要继续看。

#### 3) ReactVLA: Fast and Lightweight Reactive Robot Manipulation via Improved Mean Flow Action Generation
- **标题**：ReactVLA: Fast and Lightweight Reactive Robot Manipulation via Improved Mean Flow Action Generation
- **简称**：ReactVLA
- **领域**：VLA / low-latency policy / mean flow / reactive manipulation
- **论文**：https://arxiv.org/abs/2606.14255
- **项目**：https://game-loader.github.io/ReactVLA/
- **代码**：正文未披露
- **评测**：仿真覆盖 LIBERO 与自建 RoboIMI 双臂平台；LIBERO 平均成功率 **88.0%**，策略延迟 **18.3 ms**。RoboIMI 的 Peg-in-Socket 中，ReactVLA reward **1513.56**，较 Diffusion Policy 高 **48.5%**，延迟 **15.1 ms** vs Diffusion Policy **398.0 ms**。真机采用 Agile Robots Diana 7 单臂，Orange Pick-and-Place 与 Block Stack 各 20 trials；Orange P&P 为 **95%**，Block Stack 为 **90%**，SmolVLA 在 Block Stack 为 **75%**。
- **算力/数据**：真机每任务采集 **50 条 teleoperated demonstrations**；输入为 wrist RGB + third-person RGB + joint states，输出 joint commands，部署控制 **60 Hz**。训练 GPU 总账单正文未披露。
- **解读**：ReactVLA 直接击中扩散式 VLA 的推理延迟痛点，用 improved Mean Flow 把多步采样压到少步生成。亮点是真机延迟降到 **38.6 ms**，在 Block Stack 这种精细闭环任务上体现出更明显收益；局限是真机任务规模仍小。

### B. World Model / World Action Model 与 VLA 后训练

#### 1) WEAVER, Better, Faster, Longer: An Effective World Model for Robotic Manipulation
- **标题**：WEAVER, Better, Faster, Longer: An Effective World Model for Robotic Manipulation
- **简称**：WEAVER
- **领域**：robot world model / policy evaluation / policy improvement / test-time planning
- **论文**：https://arxiv.org/abs/2606.13672
- **项目/代码/模型**：https://arnavkj1995.github.io/WEAVER/
- **评测**：WEAVER 在真实 Franka manipulation 设置上用于三类下游任务：policy evaluation 与真实成功率相关性 **ρ = 0.870**；在 π0.5 上做 policy improvement，真实成功率提升 **38%**；test-time planning 平均成功率提升约 **15%**，最高提升 **20%**。真机任务包括 Stack Bowls、PnP Bag、PnP Marker、PnP Towel、Pour Beans，每任务采集 50 条 fine-tuning rollouts 与 20 条 evaluation rollouts。
- **算力**：WEAVER 为 **928M** 参数；DROID 预训练 **1M steps**，batch size 32，**4 × H100 训练 10 天**；任务 fine-tuning **16k steps**。test-time planning 在 RTX A6000 Ada 上，horizon 15、batch size 4 时 dynamics prediction **1.4547s**，比 Ctrl-World **29.4244s** 快 **20.2×**。
- **解读**：WEAVER 的价值是把世界模型从“生成好看视频”推进到能做 policy evaluation、finetuning data selection 和 test-time planning。它的瓶颈也很清楚：虽然比 Ctrl-World 快很多，但在线规划仍是秒级延迟，只适合短 action chunk 或低频决策。

#### 2) World Pilot: Steering Vision-Language-Action Models with World-Action Priors
- **标题**：World Pilot: Steering Vision-Language-Action Models with World-Action Priors
- **简称**：World Pilot
- **领域**：VLA / World Action Model / latent steering / action steering
- **论文**：https://arxiv.org/abs/2606.12403
- **项目**：https://world-pilot.github.io/
- **代码**：正文未披露
- **评测**：LIBERO-Plus OOD benchmark 覆盖 **10,030 个扰动任务**、7 类扰动轴；World Pilot Total 成功率 **84.7%**，较最强 baseline 高 **2.6 points**。真机覆盖 Stack Blocks、Fold Towel、Fruit-to-Plate、Container-Lid Alignment 四任务，每任务 1 个 ID + 2 个 OOD 设定，每设定 20 trials；World Pilot 在 12 个真机设定中均为最高成功率。
- **算力/数据**：fine-tuning 使用 **8 × RTX PRO 6000**；真机每任务采集 **100 条 ID teleoperated demonstrations**，训练 **10,000 steps**。
- **解读**：World Pilot 的关键是把视频预训练 WAM 的“未来场景 latent”和“动作先验 token”接入 VLA，而不是只用 VLM 静态语义。它对 OOD 视觉/几何扰动有明显帮助，但论文也承认 LIBERO-Plus 的 Language、Robot、Layout 轴和真机 OOD 仍会掉点。

#### 3) Elastic Queries Reinforcement Learning: Self-Aware Policy Execution for VLA Models
- **标题**：Elastic Queries Reinforcement Learning: Self-Aware Policy Execution for VLA Models
- **简称**：EQRL
- **领域**：VLA / RL post-training / adaptive inference / compute-aware control
- **论文**：https://arxiv.org/abs/2606.14375
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真在 LIBERO-4Tasks 与 ALOHA-Cube 上评估 frozen π0 base policy 的 query-level RL 调度。LIBERO 上 EQRL final success **0.96**，Success AUC **0.90**，relative NFE **0.303**，相对最佳 baseline 减少 **24.3%** NFE；ALOHA-Cube final success **1.00**，relative NFE 减少 **20.0%**。真机离线适配使用 DISCOVER Robotics PTK2，在线 RL 使用 DISCOVER Robotics TOK4。
- **算力/数据**：真机离线每任务 **500k gradient steps**，使用 **50-100 expert trajectories**；在线 RL 初始化 **20 expert trajectories**，约 **15k real-environment steps** 后停止。GPU 型号正文未披露。
- **解读**：EQRL 关注的不是让 VLA 更大，而是让每次 query 自己决定 denoising budget 和 action chunk 长度。它适合部署成本敏感的 VLA：简单阶段省计算，接触/对齐阶段加计算并缩短开环承诺。局限是每任务适配和在线 RL 成本仍不低。

#### 4) NavWAM: A Navigation World Action Model for Goal-Conditioned Visual Navigation
- **标题**：NavWAM: A Navigation World Action Model for Goal-Conditioned Visual Navigation
- **简称**：NavWAM
- **领域**：navigation world action model / image-goal navigation / real robot navigation
- **论文**：https://arxiv.org/abs/2606.13494
- **项目**：https://dachii-azm.github.io/navwam/
- **代码**：正文未披露
- **评测**：离线基准使用 GO STANFORD 与 SIT image-goal navigation；真机在 Direct Drive Tech Diablo 移动机器人上进行 24 个闭环 image-goal episodes，覆盖 Office、Storage、Meeting room、Hallway 四个室内环境。NavWAM 到达 **19/24**，成功率 **79.2%**，高于 NWM 与 OmniVLA。
- **算力/设备**：训练使用 **4 × RTX PRO 6000**，effective batch 32；真机为 Diablo + Intel RealSense D455 RGB-D + Livox Mid-360 LiDAR + NVIDIA Jetson AGX Orin。推理效率在 RTX PRO 6000 96GB 上测得，NavWAM 约 **5 Hz**，CEM-style world-model planner 则为 sub-Hz。
- **解读**：NavWAM 把“预测未来视角”直接变成可执行动作，而不是再外接 CEM 搜索。对导航类 world model 来说，这是从 planner module 走向 policy module 的一个重要方向；限制是真机规模只有单平台 24 episodes。

### C. 真实机器人 RL 与 Agent / Benchmark 构建

#### 1) UniIntervene: Agentic Intervention for Efficient Real-World Reinforcement Learning
- **标题**：UniIntervene: Agentic Intervention for Efficient Real-World Reinforcement Learning
- **简称**：UniIntervene
- **领域**：real-world RL / human-in-the-loop RL / agentic intervention / manipulation
- **论文**：https://arxiv.org/abs/2606.12372
- **项目**：https://denghaoyuan123.github.io/UniIntervene-project/
- **代码**：正文未披露
- **评测**：UR7e + parallel-jaw gripper 真机平台，任务为 Pick Eggplant、Tube Insertion、RAM Insertion、Wipe Whiteboard、Fold Towel。UniIntervene 平均成功率 **88%**，平均人工 intervention rate **14.6%**；HiL-SERL 为 **81% / 34.3%**，HiL-SERL + FA-RL 为 **77% / 24.6%**。每任务训练后 20 evaluation episodes，intervention rate 由 3 次独立训练统计。
- **设备/数据**：视觉输入来自 wrist RGB + fixed third-person RGB，两路均 resize 到 224 × 224；人工通过 3Dconnexion SpaceMouse 接管。SFT baseline 每任务使用 20 demonstrations。
- **解读**：UniIntervene 把“人类什么时候该介入”改写成 agentic recovery 问题：先预测 rollout 是否停滞，再从高价值记忆中取 recovery target 并执行纠偏。它最适合接触丰富、失败不一定立刻灾难但会慢慢卡死的真机 RL；局限是 memory/recovery policy 仍依赖历史 intervention 数据。

#### 2) Embodied-BenchClaw: An Autonomous Multi-Agent System for Embodied Spatial Intelligence Benchmark Construction
- **标题**：Embodied-BenchClaw: An Autonomous Multi-Agent System for Embodied Spatial Intelligence Benchmark Construction
- **简称**：E-BenchClaw
- **领域**：embodied benchmark / multi-agent benchmark construction / spatial intelligence
- **论文**：https://arxiv.org/abs/2606.11909
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：系统生成标准化 benchmark package，包含 benchmark items、input observations、reference answers、evidence records、scoring protocols、metadata、provenance traces 与 update suggestions。覆盖六类场景：indoor spatial reasoning、outdoor spatial reasoning、object manipulation、quadruped robot navigation、UAV/aerial-view understanding、static benchmark enhancement。代表性 UAV/aerial-view benchmark 含 **5,000 questions**，vision-language 平均 **65.39%**，blind setting **29.82%**，random baseline **33.30%**；GPT-5.5 overall **74.13%**。
- **算力**：正文主要报告构建流程、评测维度和 benchmark utility，未披露统一训练/推理 GPU 账单。
- **解读**：E-BenchClaw 与 OpenClaw 类 agent 工作流高度相关：它把 benchmark 生产拆成 intent parsing、resource matching、skill library、quality gate、repair/update 的多 agent 流程。最值得借鉴的是 provenance trace 和 evidence-grounded scoring，这能降低自动 benchmark 生成时的幻觉和不可复核风险。

## Agent / OpenClaw 能力增强（固定栏目）

### 今日新增 / 值得关注
- **Embodied-BenchClaw** 提供了一个可复用的自动 benchmark 构建范式：用户意图 → 资源匹配 → skill/library 调用 → benchmark package → evidence/provenance → 质量门禁与修复。这个结构很适合迁移到 OpenClaw 的科研评测、机器人任务集构建和长期自动更新。
- **UniIntervene** 的 agentic intervention 思路值得用于真实设备控制：把“需要人接管”的判断显式建模成 value-stagnation detection + memory retrieval + recovery policy，而不是只依赖硬阈值或人工盯守。

### 能力提升方法 / 工作流要点
- 自动科研日报可加入“证据包”概念：每篇论文旁边保存 abs、PDF 文本、项目页、代码页、关键表格行号与链接状态，生成正文时只引用证据包字段，减少二次核验成本。
- 对机器人/设备控制任务，建议把 intervention 记录结构化保存为：触发状态、失败类型、人工修正轨迹、恢复目标、最终成功/失败。这类数据后续可训练 recovery policy 或用于自动诊断。

### 机器人 / 设备控制能力相关进展
- VLA 部署方向出现三条很明确的路线：**物理约束运行时修正**（PhysVLA）、**低延迟生成式动作头**（ReactVLA）、**按状态难度分配推理预算**（EQRL）。三者互补，后续可以组合成更稳的真实设备控制栈。
