## 主清单（按技术重点分类）

### A. VLA 记忆与长时序推理

#### 1) ReMem-VLA: Empowering Vision-Language-Action Model with Memory via Dual-Level Recurrent Queries
- **标题**：ReMem-VLA: Empowering Vision-Language-Action Model with Memory via Dual-Level Recurrent Queries
- **简称**：ReMem-VLA
- **领域**：VLA / 记忆增强 / 长时序操作
- **论文**：https://arxiv.org/abs/2603.12942
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark 为 MemoryBench（含 Put Block Back、Rearrange Block、Reopen Drawer）并加入复合长时序任务；真机为 UR5 + Robotiq 夹爪，双机位 RealSense D435（wrist 1 + third-person 1，共 2 个相机位姿）。
- **算力**：训练使用 8×A100（总 batch size 64）；推理在单卡部署，文中另给单 RTX 4090 约 10Hz 推理频率（复用 chunk 可接近 20Hz）。
- **解读**：核心贡献是帧级与chunk级双层循环查询，把“短期可见信息+跨步骤状态”统一进动作生成。优势在记忆相关任务显著提升；局限是公开工程与更复杂双臂场景细节不足。复现建议先按同分辨率双视角和 action chunk=30 复刻，再做记忆消融与频率对照。

#### 2) RoboStream: Weaving Spatio-Temporal Reasoning with Memory in Vision-Language Models for Robotics
- **标题**：RoboStream: Weaving Spatio-Temporal Reasoning with Memory in Vision-Language Models for Robotics
- **简称**：RoboStream
- **领域**：VLM规划 / 机器人时空推理 / 记忆机制
- **论文**：https://arxiv.org/abs/2603.12939
- **项目**：https://robostream123.github.io/
- **代码**：正文未披露
- **评测**：仿真 benchmark 覆盖 SIMPLER、长时序 RLBench、6-DoF SpatialBench、Open6DOR V2；真机为 Franka Research 3（FR3）+ parallel gripper，前视 RealSense D435i（front 1，相机位姿共 1）。
- **算力**：正文给出单 NVIDIA A100 上 RoboStream-8B 的延迟分解实验；未统一披露完整训练集群规模与总训练时长。
- **解读**：方法把时空token、对象级状态追踪与规划闭环耦合，重点解决遮挡后目标持续识别与步骤顺序稳定性。亮点是跨仿真到真机一致收益；局限是训练账单和开源细节未完整公开。复现建议优先在 RLBench 复刻 token 设计，再迁移到 FR3 桌面任务验证遮挡恢复。

#### 3) AnchorVLA4D: an Anchor-Based Spatial-Temporal Vision-Language-Action Model for Robotic Manipulation
- **标题**：AnchorVLA4D: an Anchor-Based Spatial-Temporal Vision-Language-Action Model for Robotic Manipulation
- **简称**：AnchorVLA4D
- **领域**：VLA / 时空锚定 / 机器人操控
- **论文**：https://arxiv.org/abs/2603.12730
- **项目**：正文未披露
- **代码**：https://github.com/Vector-Wangel/XLeRobot
- **评测**：仿真 benchmark 为 Simpler WidowX（文中报告较基线提升）；真机为 xLerobot 单臂平台，末端夹爪执行抓取放置，主实验 3 个相机位姿（并加入 anchor 视角输入，推理使用 4 图像通道）。
- **算力**：仿真训练提到单 A800；推理在 RTX 4090 Ti；预训练阶段披露 NPU 资源但口径与GPU实验不同，统一总训练成本正文未披露。
- **解读**：通过 anchor frame 缓解抓取后目标被机械臂遮挡导致的策略犹豫，将空间参照显式注入动作决策。贡献是提升长期步骤连贯性；局限是硬件与训练口径混杂。复现建议固定相机标定后先做“有无anchor”对照，再测多任务泛化与失败恢复。

### B. 视觉注入与跨具身迁移

#### 1) PVI: Plug-in Visual Injection for Vision-Language-Action Models
- **标题**：PVI: Plug-in Visual Injection for Vision-Language-Action Models
- **简称**：PVI
- **领域**：VLA 适配 / 跨平台迁移 / 双臂操作
- **论文**：https://arxiv.org/abs/2603.12772
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark 使用 RoboTwin 2.0（单任务与多任务协议，100 rollouts/任务口径）；真机为 Airbot 双臂平台，末端双夹爪，视觉为环境相机+左右腕相机（共 3 个相机位姿）。
- **算力**：主对比采用 4×NVIDIA H20Z（20k/60k steps）；更长训练设置用 8×H20Z，正文未统一披露总训练时长与推理时延。
- **解读**：PVI 通过可插拔视觉注入层，把预训练VLA主干与具身特定观测对齐，降低跨机器人迁移成本。优势是不改主干也能稳步增益；局限在公开代码与真实复杂场景规模仍有限。复现建议先在 RoboTwin 严格对齐同训练步数，再迁移到双臂实机验证策略稳定性。

### C. 全景占据感知与具身评测基准

#### 1) Panoramic Multimodal Semantic Occupancy Prediction for Quadruped Robots
- **标题**：Panoramic Multimodal Semantic Occupancy Prediction for Quadruped Robots
- **简称**：PanoMMOcc / VoxelHound
- **领域**：机器人感知 / 语义占据预测 / 四足平台
- **论文**：https://arxiv.org/abs/2603.13108
- **项目**：正文未披露
- **代码**：https://github.com/SXDR/PanoMMOcc
- **评测**：仿真/离线 benchmark 以作者构建的 PanoMMOcc 全景多模态占据数据与基准进行；真实采集基于四足机器人平台，传感系统含四路环视鱼眼相机与 LiDAR 融合（相机位姿 4），末端执行器不适用。
- **算力**：训练采用 4×NVIDIA RTX 3090，文中给出训练 epoch 与优化器设置；推理时延/FPS/显存正文未统一披露。
- **解读**：工作把全景视觉、激光与语义占据统一到四足场景，补齐车载范式难覆盖的近地障碍和大视角盲区。亮点是数据与基准并行开放；局限是尚未闭环到下游控制策略。复现建议先复现实验配置与标定，再验证占据增益对导航/巡检任务成功率提升。

#### 2) ESPIRE: A Diagnostic Benchmark for Embodied Spatial Reasoning of Vision-Language Models
- **标题**：ESPIRE: A Diagnostic Benchmark for Embodied Spatial Reasoning of Vision-Language Models
- **简称**：ESPIRE
- **领域**：具身空间推理评测 / VLM 基准
- **论文**：https://arxiv.org/abs/2603.13033
- **项目**：https://spatigen.github.io/espire.io/
- **代码**：https://github.com/spatigen/espire
- **评测**：仿真 benchmark 构建于 Isaac Sim，任务分解为 localization+execution；机器人为 Franka 机械臂，末端夹爪，视觉配置为 wrist 相机 1 + 固定世界视角 2（共 3 个相机位姿）。真机：正文未报告系统性真机对比。
- **算力**：正文给出部署侧 RTX 4090 的推理/执行时间统计；统一训练算力与总训练时长正文未披露。
- **解读**：ESPIRE 价值在于把“会答题”转成“可执行”的空间推理诊断，能细粒度定位模型在定位、约束理解和动作执行上的薄弱环节。局限是主要在仿真闭环，真机外推仍待验证。复现建议先按其三相机设定和任务模板复刻，再接入自家VLM做误差分桶分析。
