### A. VLA 后训练与解码改进

#### 1) VLA-OPD: Bridging Offline SFT and Online RL for Vision-Language-Action Models via On-Policy Distillation
- **标题**：VLA-OPD: Bridging Offline SFT and Online RL for Vision-Language-Action Models via On-Policy Distillation
- **简称**：VLA-OPD
- **领域**：VLA / Post-training / On-policy Distillation
- **论文**：https://arxiv.org/abs/2603.26666
- **项目**：https://irpn-lab.github.io/VLA-OPD/
- **代码**：项目页已公开；正文未见独立代码仓库链接
- **评测**：仿真 benchmark：LIBERO 四个 suite（Spatial/Object/Goal/Long），采用极低数据设定的 1-traj SFT 初始化；RoboTwin2.0 选取 4 个双臂协调任务，采用 1,000-traj SFT 初始化，并与 GRPO、离线 SFT、student init 比较。真机：正文未报告独立真机实验。设备配置：不适用；论文仅说明 RoboTwin2.0 用于复杂双臂协调评测，但未给机器人本体、末端与相机位姿细节。
- **算力**：正文给出主实验 batch size 64、group size G=8；教师策略采用 SimpleVLA-RL，学生基于 OpenVLA-OFT 初始化。GPU 型号、卡数、总训练时长与显存账单正文/附录未披露。
- **解读**：VLA-OPD把离线 SFT 的密集监督和在线 RL 的 on-policy 覆盖结合起来，关键在用教师对学生自采样轨迹提供 token 级蒸馏，并用 reverse-KL 避免熵爆炸或塌缩。结果对样本效率和抗遗忘都友好，但目前闭环验证仍限于仿真；复现时要重点核对 teacher 质量与 student 初始化，否则蒸馏收益会明显波动。

#### 2) DFM-VLA: Iterative Action Refinement for Robot Manipulation via Discrete Flow Matching
- **标题**：DFM-VLA: Iterative Action Refinement for Robot Manipulation via Discrete Flow Matching
- **简称**：DFM-VLA
- **领域**：VLA / Action Token Refinement / Robot Manipulation
- **论文**：https://arxiv.org/abs/2603.26320
- **项目**：https://chris1220313648.github.io/DFM-VLA/
- **代码**：项目页已公开；正文未见独立代码仓库链接
- **评测**：仿真 benchmark：CALVIN 长时序语言操控基准与 LIBERO 四个 suite；文中报告 CALVIN Avg. Success Length 最高 4.44、LIBERO 平均成功率最高 95.7%。真机：双臂 AgileX 平台，3 个任务——Pot Lift、Place Veg. to Pot、Place Block to Plate；每任务采集 100 条训练轨迹，评测 40 次 trial，并与 π0-FAST、Dream-VLA、RDT 对比。设备配置：机器人本体为 AgileX 双臂平台；每臂 6-DoF + parallel gripper；相机为 1 个高位固定 RGB 相机 + 2 个腕部 RGB 相机，共 3 个位姿。
- **算力**：训练与推理均在 8× NVIDIA H100 上进行；默认学习率 1e-4、batch size 8。仿真任务训练 20k–32k steps；每个真机任务训练 5k steps。
- **解读**：DFM-VLA的核心不是更长 action chunk，而是允许离散 action token 在迭代中被整体修正，补上自回归和离散扩散“早错难返工”的短板。它在 CALVIN/LIBERO 和真机双臂任务上都给出稳定增益，但工程复杂度比普通解码器更高；复现时建议先跑两阶段解码与 velocity-field 两个关键消融。

#### 3) Realtime-VLA V2: Learning to Run VLAs Fast, Smooth, and Accurate
- **标题**：Realtime-VLA V2: Learning to Run VLAs Fast, Smooth, and Accurate
- **简称**：Realtime-VLA V2
- **领域**：VLA Deployment / Real-time Control / System Optimization
- **论文**：https://arxiv.org/abs/2603.26360
- **项目**：https://dexmal.github.io/realtime-vla-v2/
- **代码**：正文未披露独立代码仓库
- **评测**：仿真 benchmark：正文未设置标准仿真 benchmark，重点是系统级真实部署评估。真机：DOS W1 系统，文中明确使用 RealSense D435 相机与 Airbot Play 机械臂；任务包括 shirt-folding、place-into-fixture、pick-and-latch。结果以端到端执行时长对比 human demonstration / VLA execution / human operation 报告：如 shirt-folding 为 75.3s / 18.9s / 19.0s，place-into-fixture 为 89.5s / 37.8s / 37.6s，pick-and-latch 为 98.6s / 42.6s / 36.0s。设备配置：机器人本体为 DOS W1；末端执行器随 Airbot Play 机械臂配置，正文未单列夹爪型号；相机为 1 个前视 RealSense D435，腕部/头部多相机位姿未披露。
- **算力**：论文说明 VLA 采用 client-server 架构部署，模型运行在独立 GPU server、机器人连到 client 侧；但 server GPU 型号、卡数与训练资源正文未披露。文中披露系统延迟量级为约 33ms、55ms、50ms、150ms 等分项分析值，用于速度上界讨论；完整算力账单正文未披露。
- **解读**：这篇更像“VLA 真机加速手册”，把时延校准、轨迹后处理、client-server 拆分和速度自适应串成一套工程链路，价值在让机器人接近人类操作节奏。局限是缺少统一公开仿真 benchmark 和完整算力披露；复现时应先量化相机/网络/控制回路延迟，再谈更快执行。

### B. 语言交互驱动的双臂技能适配

#### 1) Adapt as You Say: Online Interactive Bimanual Skill Adaptation via Human Language Feedback
- **标题**：Adapt as You Say: Online Interactive Bimanual Skill Adaptation via Human Language Feedback
- **简称**：BiSAIL
- **领域**：Bimanual Manipulation / Interactive Adaptation / Language Feedback
- **论文**：https://arxiv.org/abs/2603.26466
- **项目**：https://rip4kobe.github.io/BiSAIL/
- **代码**：项目页已公开实验视频与说明；正文未见独立代码仓库链接
- **评测**：仿真 benchmark：正文未设置标准仿真 benchmark，主评测为真机双臂在线适配。真机：在两套双臂平台上做 6 个 bimanual task，并补充双臂 Franka 的 cross-embodiment 实验 4 项任务。主平台配置从正文图表与实验设置可核对为：2× Dobot Nova2 机械臂；感知为 1 个顶部 RealSense D435i + 左右各 1 个 RealSense D405，共 3 个相机位姿。交叉本体验证平台为双臂 Franka。末端执行器：主平台按双臂夹持/操作末端执行，正文未单列夹爪型号；Franka 平台同样未单列末端型号。
- **算力**：所有实验在配备 NVIDIA RTX 4090 的工作站上运行；训练采用 AdamW，学习率 1e-4，batch size 256；在线 motion modulation 的初始 population size 为 32，去噪步数与在线调制步数均设为 10。
- **解读**：BiSAIL把双臂技能适配拆成“先理解用户语言想改什么，再约束式改动作”的分层流程，避免端到端方法在高维双臂轨迹上直接失控。优点是零样本在线改动作和跨本体可迁移都做出来了，但仍主要依赖作者自建真机协议；复现时要先打稳任务语义解析和双臂先验轨迹，否则语言纠错很容易只改嘴上不改动作。
