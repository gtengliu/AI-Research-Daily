## 主清单（单一合并版，按技术重点分类）

### A. VLA / World Action Model / 结构化泛化

#### 1) OA-WAM: Object-Addressable World Action Model for Robust Robot Manipulation
- **标题**：OA-WAM: Object-Addressable World Action Model for Robust Robot Manipulation
- **简称**：OA-WAM
- **领域**：World Action Model / VLA / 对象中心表征 / 鲁棒操控
- **论文**：https://arxiv.org/abs/2605.06481
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：LIBERO 四个标准 suites、SimplerEnv WidowX（Bridge tasks, Visual Matching）、LIBERO-Plus 七个扰动轴（Camera / Robot init / Layout / Light / Background / Language / Sensor noise）。真机：未做独立真机闭环评测；论文只在 SimplerEnv 中对齐真实 WidowX 场景进行视觉匹配。设备配置：策略输入为 third-person + wrist 两视角 RGB，外加 7 维末端状态与语言；真实机器人本体可对应 WidowX 场景来源，但机器人本体、末端夹爪型号与真实相机安装细节正文未逐项披露；相机位姿数量可核验为 2。
- **算力**：正文/附录披露三阶段训练。Stage 0 预训练使用 384×A100-80GB，约 18 天、约 166k A100-hours；Stage I 在 8×A100-80GB 上 3–4 天；Stage II 在 8×A100-80GB 上 3–4 天。下游复现实验可不重做 Stage 0，作者称仅 Stage I/II 约需 8×A100 上约 60 GPU-days。单张 A100 推理约 233 ms/chunk。
- **解读**：核心不是再堆更大视频 latent，而是把“对象是谁”和“对象当前状态”强制拆开，用 address-only cross-slot routing 稳住目标选择。优势在几何扰动下更稳；局限是前端槽提取仍怕传感器噪声，复现时应先缓存 SAM3+DINOv3 槽特征并重点检查 sensor-noise 退化。

#### 2) TriRelVLA: Triadic Relational Structure for Generalizable Embodied Manipulation
- **标题**：TriRelVLA: Triadic Relational Structure for Generalizable Embodied Manipulation
- **简称**：TriRelVLA
- **领域**：VLA / 关系建模 / 泛化操控
- **论文**：https://arxiv.org/abs/2605.05714
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：LIBERO，用于跨场景、跨物体、跨任务泛化评测。真机：CSOT-Bench，自建真实机器人操作数据与评测集，覆盖 scene reasoning、object understanding、task goal 三类泛化套件。设备配置：明确包含 multi-view RGB，其中 third-view + wrist-view 两个相机位姿；记录指令、关节角、速度、末端位姿、gripper state、动作标签与 object/hand masks。机器人本体型号与末端执行器型号正文未披露，相机位姿数量可核验为 2。
- **算力**：正文披露训练使用 8×A800 GPUs；三阶段 batch size / learning rate 分别为 64 / 1e-4、32 / 2e-5、32 / 1e-5，对应 Stage 1/2/3。统一 wall-clock、总 token 或总 GPU-hours 正文未披露。
- **解读**：这篇抓得很准：操控泛化真正缺的不是更多语义标签，而是 object-hand-task 三元关系的稳定瓶颈表示。优点是跨场景/跨物体更自然；局限是长时序组合任务仍弱，复现时建议先把 CSOT-Bench 的双视角与 mask 监督链路跑通，再看 relational bottleneck 带来的增益。

### B. Sim2Real / Dexterous Manipulation

#### 1) DexSim2Real: Foundation Model-Guided Sim-to-Real Transfer for Generalizable Dexterous Manipulation
- **标题**：DexSim2Real: Foundation Model-Guided Sim-to-Real Transfer for Generalizable Dexterous Manipulation
- **简称**：DexSim2Real
- **领域**：Sim2Real / 灵巧手操作 / Foundation Model + RL
- **论文**：https://arxiv.org/abs/2605.05241
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：基于 Isaac Sim / PhysX 5 的六任务 dexterous manipulation 训练与评测环境。真机：六项真实任务——Pick-Place、Stacking、Peg Insertion、In-Hand Rotation、Tool Use、Pouring；每任务 50 次真实试验、3 个随机种子、双盲人工判定。设备配置：Franka Emika Panda 机械臂 + Allegro Hand（16-DoF）+ XELA tactile sensors（每个指尖 15 taxels，4 指）；2 路 Intel RealSense D435，相机位姿为 front view + wrist-mounted view，共 2 个。
- **算力**：正文披露仿真训练在 4096 并行环境上运行，使用 4×NVIDIA A100 GPUs，控制频率 120 Hz；FM-DR 额外开销约 2 GPU-hours，总流程约 50 A100 GPU-hours，其中 RL 训练约 48 GPU-hours。
- **解读**：它把 VLM 当“视觉真实性裁判”来反向调 DR，再叠加视触觉交叉注意力和逐级课程，组合拳很完整。亮点是把 sim-real gap 压到 8.3%；局限是任务规模仍偏中小、且未公开工程细节，复现时建议先复刻 Franka+Allegro 双相机与 tactile 输入，再单独验证 FM-DR 的收益。

### C. Offline-to-Online RL / Chunked Policy

#### 1) Adaptive Q-Chunking for Offline-to-Online Reinforcement Learning
- **标题**：Adaptive Q-Chunking for Offline-to-Online Reinforcement Learning
- **简称**：AQC
- **领域**：Offline-to-Online RL / Action Chunking / Chunked Critic
- **论文**：https://arxiv.org/abs/2605.05544
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：OGBench 五个 manipulation domains（scene-sparse、puzzle-3x3-sparse、cube-double、cube-triple、cube-quadruple），Robomimic 三任务（lift、can、square），以及 RoboCasa-GR1 24 个 tabletop manipulation tasks。真机：无独立真机实验。设备配置：OGBench 为 UR5e 机械臂、5D end-effector control；Robomimic 任务沿用标准仿真操作设定，机器人本体/末端/相机安装按 benchmark 默认配置，正文未新增真机参数；RoboCasa-GR1 为 GR1 humanoid upper-body 仿真平台，44 维动作，覆盖 torso、arms、grippers 等 8 个 body parts。真机设备与真实相机位姿数量不适用。
- **算力**：正文/附录披露全部实验使用 NVIDIA RTX 4090 GPUs。单次 OGBench offline-to-online 约 5–7 GPU hours，100M transition 的 cube-quadruple 额外约 2 小时；单次 RoboCasa-GR1 训练约 20 小时。作者估算复现实验总量约 1,600 GPU hours。RoboCasa-GR1 两阶段中，actor fine-tuning 用 global batch size 32、60,000 steps；critic 训练用 h=16、K={1,4,8,16}。
- **解读**：AQC 的关键贡献是按状态自适应选 chunk size，但不是直接比 Q，而是比经 horizon baseline 与 discount 归一化后的 advantage，规避短 chunk 天生占便宜。优点是对接触前后尺度切换很合理；局限是目前仍全在 benchmark 内验证，复现时先从 OGBench 稀疏奖励场景验证选择器是否稳定最划算。

### D. 仿真基准 / 视觉真实性

#### 1) Toward Visually Realistic Simulation: A Benchmark for Evaluating Robot Manipulation in Simulation
- **标题**：Toward Visually Realistic Simulation: A Benchmark for Evaluating Robot Manipulation in Simulation
- **简称**：VISER
- **领域**：机器人仿真基准 / Sim2Real 评测 / 视觉真实性
- **论文**：https://arxiv.org/abs/2605.06311
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：VISER，包含 14 个 curated tasks、8 个从真实场景重建的任务，以及自动生成任务；基础技能覆盖 pick up、put in、push near、pick from、open，并额外评长时程抽象目标任务。真机：用于相关性核验的真实 BridgeData-v2 / WidowX 场景，论文报告仿真与真实成功率平均 Pearson 相关系数 0.92。设备配置：仿真采用 SimplerEnv + SAPIEN，策略评测对应 WidowX 机器人；真实侧机器人本体可核验为 WidowX，但末端执行器型号、相机数量与安装位姿正文未完整披露。
- **算力**：正文披露高保真渲染在单张 NVIDIA 4090 GPU 上约 20 FPS。统一训练算力正文未披露；该工作重点是评测基准与渲染资产构建，而非训练单一策略。
- **解读**：VISER 最有价值的地方是把“材质高光+阴影”从美术细节提升为 VLA 评测变量，证明不少策略在假而亮的仿真里被高估。局限是物理真实性与多机器人覆盖还不够，复现时建议先复用其重建任务去验证你自己的 VLA 是否真吃视觉 realism。
