### A. World Action Model：生成可靠性、调度与测试时选择

#### 1) NoiseGate: Learning Per-Latent Timestep Schedules as Information Gating in World Action Models
- **标题**：NoiseGate: Learning Per-Latent Timestep Schedules as Information Gating in World Action Models
- **简称**：NoiseGate
- **领域**：World Action Model / 视频-动作联合建模 / 测试时调度
- **论文**：https://arxiv.org/abs/2605.07794
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：RoboTwin 2.0 random-scene 条件，正文与附录核验为 **50 个双臂 manipulation tasks**，统一采用 **每任务 100 episodes** 的 success rate 评测。真机：无独立真实机器人闭环实验。设备配置：仿真对象基于 RoboTwin 双臂系统；真实机器人本体、末端执行器与相机位姿数量正文未作为独立真机系统披露。
- **算力**：正文披露两阶段训练：Stage 1 为 demonstration-trained WAM backbone，Stage 2 冻结 backbone、仅训练 GPN；附录给出 GRPO rollout workers（GPUs）配置，但**统一 GPU 型号、总卡时与 wall-clock 正文未完整披露**，记为“正文未披露”。
- **解读**：这篇抓住了 WAM 常被忽略的点：不是所有未来帧都该以同样速度去噪。NoiseGate 用独立 per-latent noise 训练底座，再让 GPN+GRPO 学“该信哪几帧”。亮点是 RoboTwin random-scene 上稳定增益；局限是仍停留仿真验证，复现时先对齐两阶段训练与 schedule ablation。

#### 2) Is the Future Compatible? Diagnosing Dynamic Consistency in World Action Models
- **标题**：Is the Future Compatible? Diagnosing Dynamic Consistency in World Action Models
- **简称**：Dynamic Consistency for WAM
- **领域**：World Action Model / 可靠性诊断 / test-time selection
- **论文**：https://arxiv.org/abs/2605.07514
- **项目**：正文未披露
- **代码**：https://github.com/nvlabs/cosmos-policy （Cosmos-Policy 基线）｜ https://github.com/robbyant/lingbot-va （LingBot-VA 基线）
- **评测**：仿真 benchmark：RoboCasa（**24 个 kitchen manipulation tasks**，单臂 **Franka Emika Panda**，**每任务 50 trials**）与 RoboTwin 2.0（**50+ 个 bimanual manipulation tasks**，沿 LingBot-VA 设置）；测试时默认 **N=8 candidate rollouts**。真机：无新增真实机器人实验。设备配置：RoboCasa 明确为单臂 Franka Panda；RoboTwin 为双臂仿真任务集合；末端执行器、相机位姿数量按原 benchmark 设定，本文未单独展开。
- **算力**：正文明确写明**全部实验运行于 8× NVIDIA RTX 5090 GPUs**；总 GPU-hours 正文未披露。
- **解读**：作者把“预测未来是否可信”形式化成 action-state consistency，并证明它既能诊断 rollout 质量，也能直接做 value-free 的测试时分支选择。亮点是不加额外 reward model 就把 RoboTwin 从 90.2% 拉到 93.0%；局限是仍依赖多候选 rollout，真实系统延迟与相机失真影响还没被系统展开。

### B. VLA：触觉注入、长时程规划与世界模型耦合

#### 1) AT-VLA: Adaptive Tactile Injection for Enhanced Feedback Reaction in Vision-Language-Action Models
- **标题**：AT-VLA: Adaptive Tactile Injection for Enhanced Feedback Reaction in Vision-Language-Action Models
- **简称**：AT-VLA
- **领域**：VLA / 触觉融合 / 接触丰富操作
- **论文**：https://arxiv.org/abs/2605.07308
- **项目**：https://sites.google.com/view/at-vla
- **代码**：正文未披露
- **评测**：真机主评测。正文核验：**4 个 contact-rich tasks + 2 个 non-contact-rich tasks**。设备配置：**AgiBot Genie1**，双 **7-DoF** 机械臂；**front-view camera + 两个 wrist cameras**，共 **3 个相机位姿**；末端为**带 Xense Robotics 触觉传感器的 gripper**。仿真 benchmark：正文未设置独立仿真 benchmark。
- **算力**：正文与附录重点披露结构、任务与推理时延，但**训练 GPU 型号、卡数、总 GPU-hours 正文未披露**；推理侧明确实现 **0.04 s closed-loop** 反应。
- **解读**：AT-VLA 不只是“给 VLA 加触觉 token”，而是把触觉只在接触阶段自适应注入，同时用双流结构把慢视觉语义和快物理反应拆开。亮点是真机接触任务收益明确；局限是训练与硬件开销披露不足，复现时先复刻 tactile gate 的接触判定与 dual-stream 时序对齐。

#### 2) BrickCraft: Visuomotor Skill Composition with Situated Manual Guidance for Long-Horizon Interlocking Brick Assembly
- **标题**：BrickCraft: Visuomotor Skill Composition with Situated Manual Guidance for Long-Horizon Interlocking Brick Assembly
- **简称**：BrickCraft
- **领域**：长时程装配 / 视觉-运动技能组合 / 机器人装配
- **论文**：https://arxiv.org/abs/2605.07605
- **项目**：https://intelligent-control-lab.github.io/BrickCraft
- **代码**：正文未披露
- **评测**：真机主评测。单技能评测：**3 类 assembly skills**，每类在 **8 个 seen + 8 个 unseen structural designs** 上测试；长时程评测：**4 个 structural designs** 的 multi-step assembly。设备配置：**Kinova Gen3** 机械臂 + **Robotiq 2F-85 gripper**；**1 个 wrist camera + 1 个 workspace camera**，共 **2 个相机位姿**。仿真：使用 **Blender** 渲染 situated manual 参考，不作为独立 benchmark 报分。
- **算力**：正文披露训练数据为 **692 demonstration trajectories**、覆盖 **60+ assembly configurations**，10 Hz 同步采样，扩散策略训练用 **50 diffusion iterations / 10 inference iterations**；但**GPU 型号、卡数与总 GPU-hours 正文未披露**。
- **解读**：BrickCraft 的核心不是端到端硬扛长时装配，而是把“符号步骤→情境化视觉手册→原子装配技能”串起来，显式缩短规划到控制的语义鸿沟。亮点是 unseen 结构也能组装；局限是目前 brick 域较窄，复现时优先核对 situated manual 与 YOLO/SAM2 掩码对齐误差。

#### 3) One Token Per Frame: Reconsidering Visual Bandwidth in World Models for VLA Policy
- **标题**：One Token Per Frame: Reconsidering Visual Bandwidth in World Models for VLA Policy
- **简称**：OneWM-VLA
- **领域**：VLA / 世界模型 / 长时程规划 / 参数高效适配
- **论文**：https://arxiv.org/abs/2605.07931
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**MetaWorld MT50（50 tasks，最长规划 horizon H=30）**；**LIBERO 四套件（Spatial / Object / Goal / Long）**。真机：**Piper 6-DoF arm + parallel gripper**，任务为 **Pick Banana / Fold Cloth / Pull Drawer**，**每任务 20 trials**，并额外做 lighting shifts、positional perturbations、4–6 unseen distractors 鲁棒性检查。设备配置：正文模型输入明确为 **third-person view + two wrist views**，共 **3 个相机位姿**。
- **算力**：正文明确为在 **π0 (2B)** backbone 上训练，新增 **14.71M LoRA parameters**，**30K steps**，使用 **8× NVIDIA A800 GPUs**；总 GPU-hours 正文未披露。
- **解读**：这篇很值得看，因为它反直觉地证明了“给世界模型更多视觉 token”在冻结 VLA 适配 regime 下未必更好；1 token/frame 反而更利于长时规划。亮点是 LIBERO-Long 和真机 Fold Cloth 提升明显；局限是结论强依赖其 π0+LoRA 设定，复现时要严格复刻 inference horizon 与 replan 配置。

### C. Embodied Agent / OpenClaw 能力增强

#### 1) MemCompiler: Compile, Don't Inject -- State-Conditioned Memory for Embodied Agents
- **标题**：MemCompiler: Compile, Don't Inject -- State-Conditioned Memory for Embodied Agents
- **简称**：MemCompiler
- **领域**：Embodied Agent Memory / 记忆编译 / Agent 能力增强
- **论文**：https://arxiv.org/abs/2605.07594
- **项目**：正文未披露
- **代码**：README / Homepage 级直接仓库链接正文未稳定披露；文中显式讨论 OpenClaw（https://openclaw.ai/）与 Claude Code 作为 rule-based memory injection 代表系统。
- **评测**：benchmark：**EmbodiedBench（EB-ALFRED / EB-Habitat）**、**AlfWorld**、**ScienceWorld**。真机：无；均为仿真/交互环境评测。设备配置：非机器人真机系统，不适用；机器人本体、末端执行器、相机位姿数量不适用。
- **算力**：附录明确所有训练在 **8× NVIDIA H20 (96GB)** 上完成；SFT 使用 DDP，RL 阶段使用 FSDP。统一总 GPU-hours 正文未披露。
- **解读**：它把 agent memory 从“提前整段塞上下文”改成“按当前状态即时编译”，这对 OpenClaw 一类长期运行 agent 很有启发。亮点是在多 benchmark 上普遍优于静态注入，还显式讨论延迟与 executor 负担；局限是主要验证于仿真环境，迁移到真实机器人/工具流还需要检验 memory trigger 与 observation schema。
