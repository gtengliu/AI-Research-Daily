### A. VLA 架构、表示与测试时适配

#### 1) PointAction: 3D Points as Universal Action Representations for Robot Control
- **标题**：PointAction: 3D Points as Universal Action Representations for Robot Control
- **简称**：PointAction
- **领域**：VAM / 4D world modeling / cross-embodiment manipulation
- **论文**：https://arxiv.org/abs/2606.03943
- **项目**：https://oriontmt.github.io/pointaction/
- **代码**：正文未披露
- **评测**：仿真 benchmark：RoboCasa365，10 个 seen tasks 各 100 teleoperated episodes 训练，按 ID / OOD-Env / OOD-Task 三个 regime、每个 cell **100 rollouts** 评测；PointAction 平均成功率 **47.7% / 44.1% / 17.0%**。真机：两套 pretraining 未见机体。**xArm7**：末端执行器正文未披露，**1 个 frontal ZED 2i stereo**，共 **1 机位**；3 个任务、每任务 **50 expert trajectories** 微调、**100 rollouts** 测试，平均 **43.0%**。**YAM arm**：末端执行器正文未披露，**1 个 Intel RealSense D455 stereo**，共 **1 机位**；3 个任务、每任务 **20 expert trajectories**、**20 evals**，在 Stack Cubes / Pick Pens / Insert Cups 上达 **20 / 60 / 50**。
- **算力**：4D 生成推理单次约 **6 分钟 / 1× NVIDIA B200**；YAM 上 action decoder 从零训练约 **1 小时 / 3,000 epochs**；其余主训练 GPU 卡数正文未统一披露。
- **解读**：它把视频动作接口从 2D 像素改成显式 3D 点运动，跨机体迁移更自然，实机也比 VLA 基线稳。局限是视频生成推理仍慢。复现建议先复做 robot-only point mask，再看增益是否真来自 4D 接口而非数据规模。

#### 2) Revisiting Embodied Chain-of-Thought for Generalizable Robot Manipulation
- **标题**：Revisiting Embodied Chain-of-Thought for Generalizable Robot Manipulation
- **简称**：ERVLA
- **领域**：Embodied CoT / VLA pretraining / OOD manipulation
- **论文**：https://arxiv.org/abs/2606.03784
- **项目**：正文未披露
- **代码**：正文写明将发布，当前未给可核验链接
- **评测**：仿真 benchmark：LIBERO-Plus 总成功率 **86.9%**；VLABench 平均 **SR 53.2 / IS 70.4 / PS 65.9**。真机：两类任务族——drawer placing 与 table clearing，共 **4 个难度层级 × 5 指令 = 20 tasks**，每 task **5 trials**，合计 **100 rollouts / 方法**。设备配置：机器人本体与末端型号正文未披露；**1 个 third-person + 1 个 wrist camera**，共 **2 机位**。按难度层级统计，ERVLA 在 Basic / Distractor / Semantic / Long-Horizon 上的 **SR/PS** 为 **96/97、44/58、42/58、38/55**，总体 **55/67**，高于 π0.5 的 **53/60**。
- **算力**：真实系统微调采用 **8× NVIDIA A100**，**per-GPU batch size 16**，**20,000 training steps**；构建的 embodied CoT 数据规模为 **978,743 trajectories / 226.3M samples / 2592.5 hours**。
- **解读**：论文核心不是让机器人“说更多”，而是把 CoT 变成表征塑形监督，显著补 semantic 与 long-horizon 短板。局限是硬件公开不完整。复现建议先做 reasoning-dropout 消融，否则容易退回脆弱的前缀式 CoT。

#### 3) TTT-VLA: Test-Time Latent Prompt Optimization for Vision-Language-Action Models
- **标题**：TTT-VLA: Test-Time Latent Prompt Optimization for Vision-Language-Action Models
- **简称**：TTT-VLA
- **领域**：Test-time training / latent prompt / foundation manipulation policy
- **论文**：https://arxiv.org/abs/2606.03127
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：SimplerEnv。单机体 WidowX 上，平均成功率从 π0.5 的 **51.1%** 提升到 **63.5%**，再经 TTT 提升到 **67.4%**；Google Robot visual-matching 从 **67.5% → 68.9% → 72.4%**，variant aggregation 从 **58.1% → 58.6% → 60.1%**。多机体设定使用 OXE-Aug Bridge V2 九种 embodiment 训练、WidowX 测试，均值从 **22.8% → 28.5% → 31.6%**。真机：**无真实机器人实验**。设备配置：不适用。
- **算力**：主训练使用 **32× NVIDIA H100**，约 **20 小时 / run**；测试时仅更新 latent prompt，使用 **8× H100**，通常 **15–30 分钟**；WidowX 评测按官方协议每任务 **200 episodes**。
- **解读**：优点是把部署期适配压缩成“只调 prompt、不动主策略”，对 foundation policy 很实用。局限是当前仍限仿真与离线缓冲式 TTT。复现建议先固定 π0.5 权重，只验证 proxy task 与 prompt 更新是否真在修关键决策。

#### 4) PHASER: Phase-Aware and Semantic Experience Replay for Vision-Language-Action Models
- **标题**：PHASER: Phase-Aware and Semantic Experience Replay for Vision-Language-Action Models
- **简称**：PHASER
- **领域**：Continual learning / VLA replay / catastrophic forgetting
- **论文**：https://arxiv.org/abs/2606.03598
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：LIBERO continual learning 套件，聚焦 **Goal / Long** 两套、每套 **10 tasks**；最终 checkpoint 以 **50 zero-shot rollouts / task** 统计 ASR，并计算 **10×10** task-by-checkpoint NBT。PHASER 在 OpenVLA-OFT-7B 上达到 **Goal 87.8 / Long 85.8 ASR**，相较 ER 的 **77.6 / 54.6** 明显提升；在三类 backbone 上最高给出 **+31% ASR** 增益。真机：**无真实机器人实验**。设备配置：不适用。
- **算力**：正文未统一披露 GPU 型号与总时长；仅报告在 accuracy–efficiency 分析中同步跟踪 wall-clock 与 peak GPU memory，并强调总 replay 预算与 ER 基本匹配。
- **解读**：它抓住了 VLA 持续学习的真正痛点：短但关键的 phase 容易被均匀 replay 饿死。亮点是 phase floor + semantic routing 简洁有效；局限是只在 LIBERO 族验证。复现建议优先复做 phase 切分质量，再看 replay 设计收益。

### B. 鲁棒性、安全性与对抗分析

#### 1) Partially Observable Adversarial Patch Attacks on Vision-Language-Action Models in Robotics
- **标题**：Partially Observable Adversarial Patch Attacks on Vision-Language-Action Models in Robotics
- **简称**：PO-AdvPatch for VLA
- **领域**：VLA robustness / adversarial attack / safety
- **论文**：https://arxiv.org/abs/2606.03556
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：LIBERO 四套 **Spatial / Object / Goal / Long**，每套 **10 tasks × 100 trials**，总 **1,000 rollouts**。其方法在 ASR 上于多数组合最优，例如 **Object / K=30 为 90.7%**、**Long / K=40 为 89.1%**；对应 nASR 在 **Object / K=30** 达 **96.0%**。真机：**ROKAE xMate ER7 Pro**，末端为 **Robotiq 2F-85**，**1 个 overhead Intel RealSense D455 RGB**，共 **1 机位**；物理贴片大小 **18cm × 18cm**，文中展示两项真实任务，成功率从 benign 条件下的 **72% / 58%** 降到 attacked 条件下的 **12% / 8%**。
- **算力**：全部仿真实验运行于 **8× H100 GPUs** 的工作站；其余攻击生成时长正文未统一披露。
- **解读**：贡献在于把对抗补丁威胁从“全轨迹可见”收紧到更现实的部分可观测场景，依旧能稳定拖垮长程执行。局限是真机任务数量不多。复现建议同时看 ASR 与 nASR，否则只看最终成败会低估轨迹扰动强度。

### C. 具身扩展：人形、灵巧手与无人机

#### 1) Humanoid-GPT: Scaling Data and Structure for Zero-Shot Motion Tracking
- **标题**：Humanoid-GPT: Scaling Data and Structure for Zero-Shot Motion Tracking
- **简称**：Humanoid-GPT
- **领域**：Humanoid control / motion tracking / large-scale transformer policy
- **论文**：https://arxiv.org/abs/2606.03985
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：在 **AMASS test split** 上比较架构与数据规模，Humanoid-GPT-L（2B tokens）达到 **SR 92.58**，优于 Humanoid-GPT-B（2B）**90.43** 与小模型基线；文中还在 MuJoCo 中系统分析 data/model scaling。真机：**Unitree G1 humanoid（29 DoF retargeted joint space）**，末端执行器不适用；感知链路以 live MoCap / 视频重定向为主，**相机位姿数量正文未披露**。真实展示包含 whole-body teleoperation 与 zero-shot dancing，作者强调训练外动作可直接跟踪执行。
- **算力**：总计算量约 **15,000 GPU-hours**，其中 **PPO experts（约 384 experts）= 12,000 GPU-hours / RTX 4090**，**distillation（S/B/L）= 3,000 GPU-hours / H100**；部署端延迟 **<1.5 ms / 单张 RTX 4090**。
- **解读**：这篇把 humanoid tracking 明确拉向“大数据+GPT 式因果 Transformer”路线，零样本泛化比传统 MLP 轨迹器更像真正 foundation policy。局限是偏 tracking 而非任务交互。复现建议先复做 2B→200M 数据缩放曲线，再判断收益来自结构还是数据。

#### 2) EaDex: A Cross-Embodiment Dexterous Manipulation Framework from Low-Cost Demonstrations
- **标题**：EaDex: A Cross-Embodiment Dexterous Manipulation Framework from Low-Cost Demonstrations
- **简称**：EaDex
- **领域**：Dexterous manipulation / low-cost demos / cross-embodiment RL
- **论文**：https://arxiv.org/abs/2606.03268
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：自建 low-cost dexterous 数据集，覆盖 **3 种灵巧手（Inspire / Allegro / XHand）× 3 个 articulated object-opening tasks（box / waffle iron / drawer）= 9 个 cross-embodiment settings**；平均成功率 **36.5%**，相对无 annealing 基线 **23.5%** 提升 **55.3%**，部分任务最高 **93.3%**。附加评测：ARCTIC / DexMachina 设定下，Ability Hand 的 ketchup / waffleiron / mixer 上 ADD-AUC3 均有提升。真机：**无真实机器人实验**；但演示采集链路为 **单个 Intel RealSense D435i RGB-D**，共 **1 机位**。设备配置：机器人本体为三类双手灵巧手平台，末端即灵巧手本体。
- **算力**：策略训练在 **Genesis simulator + 1× NVIDIA RTX 3090** 上完成；采集使用 **Intel i9-10900KF + 1× D435i**；部分任务从演示采集到成功训练约 **1 小时**。
- **解读**：亮点是把昂贵灵巧手数据收集压成“单 RGB-D 相机+MANO 重建+退火式 demo 引导”，很接地气。局限是平均成功率仍不高、尚未上真机。复现建议优先验证 contact-reward annealing，而不是先替换视觉或 PPO 细节。

#### 3) AirDreamer: Generalist Drone Navigation with World Models
- **标题**：AirDreamer: Generalist Drone Navigation with World Models
- **简称**：AirDreamer
- **领域**：Drone navigation / world model / sim-to-real
- **论文**：https://arxiv.org/abs/2606.03252
- **项目**：正文写明代码将公开，当前未给可核验链接
- **代码**：正文未披露
- **评测**：仿真 benchmark：基于 OmniDrones / Isaac Sim 4.1.0，成功率基准中 AirDreamer 为 **59.3%**，高于 DepthNav **54.0%**、NavRL **9.3%**、EgoPlanner **6.7%**。真机：**280 mm motor-to-motor span quadrotor**，末端执行器不适用，**1 个机载 RealSense D455 深度相机**，共 **1 机位**；另有 lidar 仅用于定位、不进入策略。共 **6 个真实场景演示运行**，覆盖 Forest、1m Passage、5m Wall、U-shape、C-shape 等；论文给出每场景 **3 trials** 平均飞行统计，例如 **5m Wall** 的平均/最大速度 **1.1 / 1.8 m/s**、直线/路径距离 **9.0 / 12.6 m**。
- **算力**：训练与仿真运行在 **128GB RAM + 2× RTX 4090D 24GB**，完整训练约 **38 小时 43 分钟**；部署在 **Jetson Orin NX Super 16GB**，推理最高 **88 Hz**，ROS 实际部署 **20 Hz**。
- **解读**：它把导航核心从手工规则改成“世界模型理解 + 稀疏奖励策略”，最有说服力的是 C-shape 这类局部最优逃逸。局限是真机统计规模仍小。复现建议先核对 yaw 行为与 reward 设计，否则很难复现其绕障 emergent 行为。
