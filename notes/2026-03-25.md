## 主清单（单一合并版，按技术重点分类）

### A. World Action Model / 多模态具身生成

#### 1) VTAM: Video-Tactile-Action Models for Complex Physical Interaction Beyond VLAs
- **标题**：VTAM: Video-Tactile-Action Models for Complex Physical Interaction Beyond VLAs
- **简称**：VTAM
- **领域**：World Action Model / 视触觉具身生成
- **论文**：https://arxiv.org/abs/2603.23481
- **项目**：https://plan-lab.github.io/projects/vtam/
- **代码**：https://github.com/haorany7/VTAM
- **评测**：仿真 benchmark：正文主评为视频预测与真实接触操作对比，未提供统一公开仿真 benchmark 名称。真机：6-DoF xArm6 接触密集操作平台。设备配置：xArm6 + parallel gripper + 指尖 GelSight 触觉传感器 + 2 路 Intel RealSense D455 RGB-D（third-person 与 first-person），总观测位姿数=3（2 视觉 + 1 触觉流）；任务含 cucumber peeling、whiteboard wiping、potato chip pick-and-place。
- **算力**：训练资源：正文/附录披露 VTAM 两阶段训练均使用 4× NVIDIA A100 40GB；Stage 1 为 world model 训练，Stage 2 为 action head 训练，per-GPU batch size 16，采用 FSDP 与 bfloat16。推理开销：正文未给统一 latency/FPS/显存横表。披露状态：训练资源披露较完整，推理账单正文未披露。
- **解读**：这篇把 GelSight 直接接入 video-action model，补足视觉看不见的接触形变与受力线索，对脆弱抓取和擦拭类任务提升很实在。局限是仿真基准不统一、硬件耦合较强；复现时建议先独立跑通触觉同步与跨模态 attention 稳定性。

#### 2) ABot-PhysWorld: Interactive World Foundation Model for Robotic Manipulation with Physics Alignment
- **标题**：ABot-PhysWorld: Interactive World Foundation Model for Robotic Manipulation with Physics Alignment
- **简称**：ABot-PhysWorld
- **领域**：具身世界模型 / 物理对齐生成
- **论文**：https://arxiv.org/abs/2603.23376
- **项目**：https://github.com/amap-cvlab/ABot-PhysWorld
- **代码**：正文未披露
- **评测**：仿真 benchmark：PBench 与作者新建的 EZSbench；其中 EZSbench 为 training-independent 的 embodied zero-shot benchmark，混合 synthetic 与 real 的 unseen robot-task-scene 组合。真机：真实数据来源整合自 AgiBot、RoboMimic、RoboSet、OXE、RoboMind 等五类公开具身数据；机器人本体、末端与相机位姿配置跨数据源混合，正文未按单一硬件表逐项枚举，因此真实设备仅能确认其覆盖多机器人、多场景、多相机录制设定。
- **算力**：训练资源：正文披露全流程在 128× Nvidia H20 GPUs 上完成，基于 Wan2.1-I2V-14B 做全量微调并加入 DPO 式物理后训练。推理开销：正文未给统一 latency/FPS/显存。披露状态：训练硬件规模披露明确，推理账单未公开。
- **解读**：核心价值是把“视觉像真”推进到“物理也像真”，用 decoupled discriminators 和 DPO 压制穿模、反重力等问题，并顺带给出零样本评测基准。局限是真实硬件拆解不细；复现时应优先复刻其物理偏好数据构造和评测协议，而非只看生成效果图。

### B. 数据与仿真基础设施

#### 1) WildWorld: A Large-Scale Dataset for Dynamic World Modeling with Actions and Explicit State toward Generative ARPG
- **标题**：WildWorld: A Large-Scale Dataset for Dynamic World Modeling with Actions and Explicit State toward Generative ARPG
- **简称**：WildWorld
- **领域**：世界模型数据集 / Action-conditioned video
- **论文**：https://arxiv.org/abs/2603.23497
- **项目**：https://shandaai.github.io/wildworld-project/
- **代码**：https://github.com/ShandaAI/WildWorld
- **评测**：仿真 benchmark：WildBench，面向交互式世界模型，重点评 Action Following、State Alignment，以及相机轨迹一致性等指标。真机：无真实机器人实验；数据来源为《Monster Hunter: Wilds》AAA 游戏环境中的自动化采集，因此机器人本体/末端/相机位姿不适用。设备配置：非真机论文，采用游戏内第三人称相机轨迹、深度、骨架与状态标注。
- **算力**：训练资源：本文重点是数据集与 benchmark，正文未披露统一模型训练 GPU 账单；仅给出数据规模 108M+ frames、450+ actions。推理开销：不适用。披露状态：正文未披露。
- **解读**：WildWorld 把“动作—状态—相机—深度”一起打包，补上现有视频世界模型数据里最缺的显式状态层。对交互式生成和动作条件视频都很有价值；局限是仍属游戏域，复现时应把 state-aware metric 与现实机器人数据分开看，别直接等同具身落地能力。

#### 2) SIMART: Decomposing Monolithic Meshes into Sim-ready Articulated Assets via MLLM
- **标题**：SIMART: Decomposing Monolithic Meshes into Sim-ready Articulated Assets via MLLM
- **简称**：SIMART
- **领域**：仿真资产生成 / Embodied 3D 基础设施
- **论文**：https://arxiv.org/abs/2603.23386
- **项目**：https://simart-mllm.github.io/
- **代码**：正文未披露（项目页标注 Code coming soon）
- **评测**：仿真 benchmark：PartNet-Mobility 与作者构建的 SIMART-Bench，并在 Isaac Sim 等 physics-based simulator 下展示下游机器人交互可用性。真机：无真机实验。设备配置：非真机论文，核心对象是 articulation-ready mesh 与 URDF/物理属性生成，机器人本体、末端与相机位姿不适用。
- **算力**：训练资源：正文披露稀疏 3D VQ-VAE 训练使用 8× NVIDIA A100，核心 reasoning engine 使用 32× NVIDIA A100。推理开销：正文未给统一 latency/FPS/显存。披露状态：训练资源披露较清楚，推理账单未公开。
- **解读**：它解决的是具身社区一个很现实的卡点：高质量网格很多，但能直接进仿真器的可动资产太少。SIMART 用 MLLM 同时做部件拆分与运动学预测很实用；局限是代码未放出，复现时建议先验证生成 URDF 的关节语义和物理参数一致性。

### C. 医疗机器人 / 感知与估计

#### 1) PinPoint: Monocular Needle Pose Estimation for Robotic Suturing via Stein Variational Newton and Geometric Residuals
- **标题**：PinPoint: Monocular Needle Pose Estimation for Robotic Suturing via Stein Variational Newton and Geometric Residuals
- **简称**：PinPoint
- **领域**：医疗机器人 / 单目针位姿估计
- **论文**：https://arxiv.org/abs/2603.23365
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：无统一公开仿真 benchmark，主体是受控真实序列与消融评测。真机：同心管机器人（concentric tube robot）缝合平台。设备配置：CTR 双臂系统，其中右臂配 C-Lock 抓持缝针；单目内窥镜相机提供 RGB 观测；总相机位姿数=1。实验含 slow-speed、normal-speed、induced-rotation 与 ex vivo tissue suturing 序列。
- **算力**：训练资源：本文为概率推断与几何残差方法，正文未披露 GPU 型号、卡数与训练时长。推理开销：正文突出实时单目估计与遮挡下跟踪稳定性，但未给统一 FPS/显存。披露状态：正文未披露。
- **解读**：PinPoint 的亮点不是堆网络，而是正面承认单目缝针位姿天然多模态，用 SVN 保留多假设而不是硬压成单解。对遮挡和入组织阶段尤其关键；复现时要先标定好 C-Lock 几何约束与相机内参，否则论文优势很难跑出来。
