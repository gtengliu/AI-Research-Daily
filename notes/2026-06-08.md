### A. VLA、动作中间表示与部署

#### 1) RhinoVLA Technical Report
- **标题**：RhinoVLA Technical Report
- **简称**：RhinoVLA
- **领域**：VLA / edge deployment / cross-embodiment manipulation
- **论文**：https://arxiv.org/abs/2606.07383
- **项目**：正文未单列项目页
- **代码**：https://github.com/HuixiAI/RhinoVLA
- **评测**：仿真 benchmark：正文表 4 在 **LIBERO** 四个 suite 上报告 **Spatial 93.0 / Object 91.0 / Goal 93.4 / Long 82.4 / Avg 90.0**。真机：跨 **Galbot G1 / AgiBot G2 / AgiBot G1** 三个平台评测；正文明确部署输入为 **1 个 head camera + 2 个 wrist cameras，共 3 个机位**；末端类型按统一 72D 槽位兼容 **parallel gripper / dexterous hand**，但各平台逐项末端配置正文未完全展开。Galbot G1 的 “red bag to the far bin” unseen **100%**；AgiBot G2 的 three-step sequence 为 **seen 58% / unseen 24%**；AgiBot G1 towel folding 为 **seen 67% / unseen 43%**。
- **算力**：部署侧正文披露 **Huixi R1 7nm edge SoC，500 TOPS INT8**，端到端 **11.69 Hz**；训练 GPU 型号、卡数与总时长 **正文未披露**。
- **解读**：这篇最有价值的是把 VLA 的“能跑”前移到板端部署设计本身：少 token 的 Qwen3-VL、统一 72D 动作槽位和实例 LoRA 一起做协同。亮点是真机跨机型迁移与 11.69Hz 闭环；局限是训练算力与各平台末端细节披露不足。复现时优先对齐 3 视角输入协议与实例适配层。

#### 2) Coarse-to-Control: Action-Token Planning for Vision-Language-Action Models
- **标题**：Coarse-to-Control: Action-Token Planning for Vision-Language-Action Models
- **简称**：Coarse-to-Control
- **领域**：VLA / action-token planning / long-horizon manipulation
- **论文**：https://arxiv.org/abs/2606.07107
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**LIBERO** 四个 suite、每任务 **50 rollouts**，正文与摘要给出 **Avg 97.90%**；**SimplerEnv-WidowX** 四个任务、每任务 **24 rollouts**，**Avg 83.3%**。真机：4 个物理任务（carrot、carrot+button、multi-carrot、fruits+vegetables clearing），每任务 **50 demonstrations** 训练、**20 rollout trials** 评测；输入明确为 **primary + wrist 共 2 个机位**；机器人本体与末端型号 **正文未披露**。正文给出真机平均成功率 **62.5%**，且在 4 个任务中的 3 个上最好。
- **算力**：附录披露 action tokenizer 预训练使用 **8× H200 GPU**；下游 real-world policy 训练到 **30k steps**，但完整训练总时长 **正文未披露**。
- **解读**：核心不是“多一步规划”，而是把规划压进和执行共词表的 action token 空间，避免文本/视觉 CoT 到控制信号的再翻译损失。亮点是 LIBERO 与 SimplerEnv 都强，长程真机也更稳；局限是真机平台细节披露偏少。复现时 joint tokenizer 与 coarse-plan horizon 设定比换 backbone 更关键。

#### 3) LARA: Latent Action Representation Alignment for Vision-Language-Action Models
- **标题**：LARA: Latent Action Representation Alignment for Vision-Language-Action Models
- **简称**：LARA
- **领域**：VLA / latent action model / post-training enhancement
- **论文**：https://arxiv.org/abs/2606.07100
- **项目**：https://lmy1001.github.io/ICML26_LARA/
- **代码**：https://github.com/lmy1001/LARA （论文与项目页给出，但当前访问 **404**）
- **评测**：仿真 benchmark：正文表 1 在 **LIBERO** 上，LARA(full) 达到 **Avg 88.6**；在 **SIMPLER-ENV** 上达到 **Avg 65.2**。另在 **GR1-Sim-24(30)**（24 个双手仿真任务、每任务 30 demos 微调）上，表 2 给出平均成功率 **48.5**。真机：**Unitree G1 humanoid**；末端为 **2× Inspire Robots RH56DFTP 6-DoF hands**；视觉为 **1 个 head-mounted Intel RealSense D455 RGB，相机机位 1 个**；在 **Pick-n-Place** 与 **Grasp-n-Pour** 两个多阶段任务上，每任务 **50 real-robot demos**、**50 trials** 评测，G1-Real 平均成功率从 **54.0 → 68.0**，对应提升 **25.9%**。
- **算力**：附录披露：LAM 预训练 **350k steps / 4× NVIDIA A100**；LARA joint training **200k steps / 4× NVIDIA A100**；下游 post-training 通常约 **20k steps / 4× NVIDIA A100**。
- **解读**：LARA 的真正贡献是把 LAM latent 不再当冻结伪标签，而是和扩散式 VLA 通过表示对齐共同塑形。亮点是既能做 full training，也能做 post-training 插件；局限是绝对性能仍受上游预训练数据规模限制。复现时先保住 alignment 注入层位与 joint optimization，再谈换更大 backbone。

### B. 世界模型、推演路由与闭环执行

#### 1) Dreaming when Necessary: Advancing World Action Models with Adaptive Multi-Modal Reasoning
- **标题**：Dreaming when Necessary: Advancing World Action Models with Adaptive Multi-Modal Reasoning
- **简称**：AdaWAM
- **领域**：world action model / multimodal reasoning / long-horizon manipulation
- **论文**：https://arxiv.org/abs/2606.07089
- **项目**：https://adawam.github.io/
- **代码**：正文未单列代码仓库
- **评测**：仿真 benchmark：**LIBERO** 上正文表 1 报告 **Overall 98.5**，其中 **LIBERO-Long 99.1**；同时在 **RoboTwin 2.0** 上比较长程与精细操作子集。真机：**AgileX Split-Type ALOHA** 主从平台；末端为 **PiPER 6-DoF arms**；相机位姿数量 **正文未披露**。真实任务为 **Clean Up Trash** 与 **Wipe Table** 两项，表 3 对应 AdaWAM 成功率分别为 **70%** 与 **60%**；推理时间分析另外在 StackThreeBowls / PutObjectCabinet 上各跑 **100 trajectories**。
- **算力**：附录披露整套框架在 **8× NVIDIA A100 80GB** 上训练；Stage 1 联训 **50,000 steps**，Stage 2 文本模块微调 **10,000 steps**。
- **解读**：这篇的关键不是再做一次“梦境预测”，而是让系统学会何时该用文本推理、何时该用视觉推演、何时直接动作解码。亮点是 LIBERO-Long 很强且真机两类任务都能落地；局限是相机配置和真实试验次数披露不够完整。复现时要先还原 dynamic router 与轨迹事件标注流程。

### C. 操作鲁棒性、坐标理解与数据生成

#### 1) AxisGuide: Grounding Robot Action Coordinate System in RGB Observations for Robust Visuomotor Manipulation
- **标题**：AxisGuide: Grounding Robot Action Coordinate System in RGB Observations for Robust Visuomotor Manipulation
- **简称**：AxisGuide
- **领域**：visuomotor manipulation / coordinate grounding / generalization
- **论文**：https://arxiv.org/abs/2606.06761
- **项目**：https://jiyunjang-24.github.io/AxisGuide-project/
- **代码**：项目页给出 `https://github.com/jiyunjang-24/axisguide`，当前访问 **404**
- **评测**：仿真 benchmark：**LIBERO** 单视角实验以 **75 rollouts（25×3 seeds）** 评测，Pick&Place(Bowl) **69.33 → 82.67**、Drawer **86.67 → 93.33**、Stove **92.00 → 100.0**。真机：**UR5e**；末端为 **OnRobot RG2 gripper**；视觉为 **固定前视 RGB + wrist-mounted camera，共 2 个机位**，单视角设置则只用前视 1 机位。真实任务为 Pick&Place(Grape)、Flip Pot、Close Pot，各配置 **30 rollouts**；单视角下分别达到 **96.66 / 93.33 / 53.33**，相对 DP 提升 **+3.27 / +20.00 / +16.67 pct**；多视角下 Pick&Place 达 **93.33**、Flip Pot **50.00**、Close Pot **40.00**。
- **算力**：正文未披露训练 GPU 型号、卡数与总时长。
- **解读**：AxisGuide 抓住了一个经常被忽略的问题：模型知道“抓哪个物体”，却不一定知道图像里 +x/+y/+z 在机器人基座下到底意味着什么。亮点是几乎不改 policy 结构就显著提泛化；局限是仍依赖准确相机内外参与末端位姿。复现时几何投影与 cue 渲染精度要先于模型调参。

#### 2) Task Editing for Generalizable 3D Visuomotor Policy Learning
- **标题**：Task Editing for Generalizable 3D Visuomotor Policy Learning
- **简称**：Task-Edit
- **领域**：3D visuomotor policy / Real2Sim2Real / demonstration generation
- **论文**：https://arxiv.org/abs/2606.07012
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：在 **Meal Preparation / Handover / Cup Placement** 三个模拟任务上，DP3 用 Task-Edit 数据训练后 **in-distribution Avg 0.937**、**out-of-distribution Avg 0.887**，均显著高于 DemoGen 与 human-collected。真机：覆盖 **ABB YuMi / AgileX Cobot Magic / Unitree G1 / Franka Panda** 四类平台；末端为 **YuMi、Cobot Magic、Franka 的 parallel-jaw grippers** 与 **G1 dexterous hands**；视觉为 **YuMi/G1/Panda 使用 1 个第三视角 RealSense L515，Cobot Magic 模拟使用 1 个第三视角 RealSense D435，共 1 个机位**。九个真实任务的 in-distribution 评测里，Task-Edit 平均成功率 **0.813**，高于 DemoGen **0.656** 与 human-collected **0.719**。
- **算力**：正文披露训练 **2000 epochs / 4× NVIDIA RTX 4090**；部署侧 DP3 + depth restoration 控制频率 **5.26 Hz（laptop RTX 4090）**。
- **解读**：这篇最强的点不是“多生成点数据”，而是把任务拆成 scene / skill / object 三个可编辑因子，再用模拟器重组出真实世界难采的组合。亮点是真机跨多本体都有效；局限是当前仍主要适用于刚体，衣物和抽屉类对象受限。复现时要先打通高质量 Real2Sim 重建，再做 DAG 级 task edit。
