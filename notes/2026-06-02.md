### A. 通用 VLA、跨具身迁移与可泛化操作

#### 1) HARP-VLA: Human-Robot Aligned Representation Learning for Vision-Language-Action Model
- **标题**：HARP-VLA: Human-Robot Aligned Representation Learning for Vision-Language-Action Model
- **简称**：HARP-VLA
- **领域**：Cross-Embodiment VLA / Human Video Pretraining
- **论文**：https://arxiv.org/abs/2605.31234
- **项目**：正文未披露
- **代码**：https://github.com/anonymity35/HARP-VLA
- **评测**：仿真 benchmark：RLBench **18 tasks**，冻结视觉编码器设定下平均成功率 **46.59%**；CALVIN **ABC→D**，Avg. Len. **4.481**。真机：**xArm7 + Robotera XHand**，共 **18 DoF**；**1 个 third-view RGB + 1 个 wrist RGB**，共 **2 个机位**；任务为 pick-and-place、push object、press button、flip cup，**每任务 60 trials**，平均成功率 **76.3%**。
- **算力**：正文未披露。
- **解读**：核心是先把人类视频与机器人视频在视觉表征和 latent action 上对齐，再做 VLA 预训练。优点是跨具身迁移链路清晰；局限是训练账单没公开。复现建议先单独验证对齐模块，再接下游 VLA 微调。

#### 2) DeMaVLA: A Vision-Language-Action Foundation Model for Generalizable Deformable Manipulation
- **标题**：DeMaVLA: A Vision-Language-Action Foundation Model for Generalizable Deformable Manipulation
- **简称**：DeMaVLA
- **领域**：Deformable Manipulation / Bimanual VLA
- **论文**：https://arxiv.org/abs/2605.31286
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：RoboTwin **50 simulation tasks**，Clean / Randomized 平均成功率 **88.42% / 86.78%**。真机：**ALOHA-style dual-arm robot**；正文明确为 **Head-Wrist Image Sequence / 3 operational views**，共 **3 个机位**；末端执行器型号正文未披露。任务含 shirt / skirt / pants / towel 四类折叠，均从“篮中随机掉落→上桌展开→完成折叠”全流程评测，**每任务 20 trials**；混合多任务平均成功率 **92.5%**。
- **算力**：正文披露预训练数据约 **5,000 小时真实双臂演示**，后训练混合折叠数据 **37 小时**；训练 GPU/卡数/时长正文未披露。
- **解读**：这篇抓住了可形变操作最痛的“多品类干扰”问题，用单 checkpoint 覆盖多类衣物折叠，且把 RTC 与人类纠偏 DAgger 合进训练。局限是硬件细节公开不全。复现建议先复做单任务 shirt，再叠加 mixed folding 与 DAgger。

#### 3) ELAN4D: Embodiment-Centric 4D Supervision for Vision-Language-Action Models via Plug-and-Play Adaptation
- **标题**：ELAN4D: Embodiment-Centric 4D Supervision for Vision-Language-Action Models via Plug-and-Play Adaptation
- **简称**：ELAN4D
- **领域**：4D-aware VLA / OOD Generalization
- **论文**：https://arxiv.org/abs/2605.30484
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：LIBERO 总体成功率将 π0 从 **94.2% 提升到 95.0%**、π0.5 提升到 **97.0%**；LIBERO-Plus 将 π0 / π0.5 的 overall 从 **53.6% / 73.6%** 提升到 **67.6% / 78.2%**；RoboTwin2.0 使用 **AgileX Piper dual-arm setup**，在 **8 个 unseen 设置**上、**每设置 100 trials**评测。真机：**AgileX Piper arm**；夹爪/末端型号与相机位姿数量正文未披露；三类任务 visual robustness、spatial generalization、temporal reasoning，**每类 50 expert trajectories 训练、20 trials 测试**。
- **算力**：训练 **30K steps / 8× NVIDIA GH200 / batch size 64**。
- **解读**：它不是预测全场景 future RGB，而是直接预测机器人关键点未来位移，把 4D 监督压到控制真正关心的部分。亮点是对 OOD 扰动很稳；局限是真机硬件配置公开较少。复现建议优先保证 forward kinematics 与 keypoint 追踪定义一致。

#### 4) Primitive Subspaces Mediate Few-Shot Transfer in VLAs
- **标题**：Primitive Subspaces Mediate Few-Shot Transfer in VLAs
- **简称**：Primitive Subspaces
- **领域**：Few-shot VLA Transfer / Contact-rich Assembly
- **论文**：https://arxiv.org/abs/2605.30695
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：主评测为 **REASSEMBLE**（基于 **NIST Assembly Task Board 1**，**4,551 demonstrations**），另做 **LIBERO-Long** 复核。对 **6 个 held-out tasks**、**每 cell 50 rollouts**，primitive-trained 模型在 **m=3** demonstrations 时达到 **0.62–0.66** success，flat-trained 仅 **0.31–0.34**；并在 LIBERO-Long 上复现相同趋势。真机：**无正文级真实机器人实验**。
- **算力**：约 **3,600 GPU-hours on H100s**；OpenVLA 单卡 H100 可训，π0.5 需 **2× H100** model parallelism。
- **解读**：最有意思的是把 primitive-aware 训练看成一种“可在推理时复用的程序子空间”，从而减少新任务再微调。局限是验证仍停留在仿真装配域。复现建议先稳定 primitive segmentation，再比较 flat/primitive 在小样本下的拐点差异。

### B. VLA 可靠性、监控与安全执行

#### 1) Hide-and-Seek in Trajectories: Discovering Failure Signals for VLA Runtime Monitoring
- **标题**：Hide-and-Seek in Trajectories: Discovering Failure Signals for VLA Runtime Monitoring
- **简称**：Hide-and-Seek
- **领域**：VLA Runtime Monitoring / Failure Detection
- **论文**：https://arxiv.org/abs/2605.30834
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**LIBERO-10** 与 **VLABench**，均采用 **10 选 3 held-out tasks** 协议，覆盖 OpenVLA、π0、π0.5。真机：**UFactory xArm 6**；末端执行器型号正文未披露；**1 个 Intel RealSense D435**，共 **1 个机位**；8 个任务分为 **CUBE / KITCHEN** 两组，各组 **25 demonstrations/task** 微调 π0.5，并收集 **98 成功 + 62 失败**（CUBE）与 **102 成功 + 58 失败**（KITCHEN）轨迹。相对基线以 **0.001s/step** 的代价实现更优检测，较重采样式方法快 **2000×+**。
- **算力**：训练与推理均在 **单张 NVIDIA A6000 48GB** 上完成；训练通常约 **5 分钟**，**100 epochs / batch size 64**。
- **解读**：它把“整条轨迹成败标签”重新拆成时序上的失败信号发现问题，比一刀切地给所有 timestep 贴失败标签更合理。局限是当前仍偏监测而非恢复。复现建议先标定 failure onset 代理，再接在线报警阈值。

#### 2) GSAM: A Generalizable and Safe Robotic Framework for Articulated Object Manipulation
- **标题**：GSAM: A Generalizable and Safe Robotic Framework for Articulated Object Manipulation
- **简称**：GSAM
- **领域**：Articulated Object Manipulation / Safe Planning
- **论文**：https://arxiv.org/abs/2605.30740
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：**无正文级仿真 benchmark 主结果**。真机：**Ranger Mini 3.0 mobile base + Realman 63-F robotic arm**；末端执行器型号正文未披露；**1 个 wrist-mounted Intel RealSense D435i RGB-D**，共 **1 个机位**；**5 类未见 articulated objects × 5 instances × 2 base placements = 50 trials**。平均操作成功率 **88.0%**，较最佳基线提升 **36.0%**，标准差下降 **3.1%**。
- **算力**：系统运行平台包含 **NVIDIA RTX 3090**；训练/微调总时长与卡数正文未披露。
- **解读**：它把“感知参数修正 + 约束函数生成 + 运动规划验证”串成安全闭环，比直接让 VLM 端到端出轨迹稳得多。局限是当前任务族主要是门/抽屉类结构件。复现建议优先核对 kinematic parameter refiner 与约束函数生成器。

#### 1) AR Forcing: Towards Long-Horizon Robot Navigation World Model
- **标题**：AR Forcing: Towards Long-Horizon Robot Navigation World Model
- **简称**：AR Forcing
- **领域**：Navigation World Model / Autoregressive Training
- **论文**：https://arxiv.org/abs/2605.31314
- **项目**：正文未披露
- **代码**：正文未披露（论文写明 code soon）
- **评测**：仿真 benchmark：多域导航数据集 **RECON / SCAND / HuRoN / TartanDrive**；目标条件视觉导航对比中，ATE / RTE 达到 **1.22 / 0.34**，优于 NWM 的 **1.38 / 0.35** 与 NoMaD 组合基线。真机：**无正文级真实机器人实验**。
- **算力**：**6× A100 80GB / total batch size 96 / 50,000 steps**。
- **解读**：关键改动很克制：不换 diffusion 世界模型骨架，只把训练过程改成真正见过自己预测分布的自回归式 rollout。优点是易接现有框架；局限是仍未证明真机闭环收益。复现建议先做长时域 rollout stability 诊断，再看下游 planner 指标。

#### 2) Can Aerial VLA Models Cooperate? Evaluating Closed-Loop Air-Ground Coordination with CARLA-Air
- **标题**：Can Aerial VLA Models Cooperate? Evaluating Closed-Loop Air-Ground Coordination with CARLA-Air
- **简称**：CARLA-Air
- **领域**：Aerial VLA Evaluation / Air-Ground Coordination
- **论文**：https://arxiv.org/abs/2605.31066
- **项目**：正文未披露
- **代码**：https://github.com/louiszengCN/CarlaAir
- **评测**：仿真 benchmark：**CARLA-Air** 统一 **CARLA + AirSim** 的闭环平台，主任务为 **moving-platform landing** 与 **occlusion-recovery escort**，结果以 **3 seeds × 50 episodes** 报告，并给出 TSR / LSR / CCR / CG 等协同指标；平台验证显示相较桥接式运行时，时间戳失配从均值 **12.4 ms** 降到 **0 ms**。真机：**无正文级真实机器人实验**。
- **算力**：实验工作站为 **Intel Core Ultra 9 275HX + NVIDIA RTX 5090 Laptop GPU 24GB + 128GB RAM**。
- **解读**：这篇最重要的不是再训一个空地协作模型，而是把“闭环协作评测环境”先搭扎实，直接暴露出单机体 aerial VLA 向协作迁移的断层。局限是仍停留在模拟器。复现建议优先验证 tick 对齐与 latency 统计，再比较模型优劣。
