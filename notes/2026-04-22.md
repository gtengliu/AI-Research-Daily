## 主清单（单一合并版，按技术重点分类）

### A. 人形/VLA 与统一动作表示

#### 1) UniT: Toward a Unified Physical Language for Human-to-Humanoid Policy Learning and World Modeling
- **标题**：UniT: Toward a Unified Physical Language for Human-to-Humanoid Policy Learning and World Modeling
- **简称**：UniT
- **领域**：Humanoid VLA / World Model / 跨 embodiment 统一动作表示
- **论文**：https://arxiv.org/abs/2604.19734
- **项目**：https://xpeng-robotics.github.io/unit/
- **代码**：正文未披露
- **评测**：仿真 benchmark：RoboCasa GR1 Tabletop，24 个任务（18 个 pick-place 重排 + 6 个 articulation 交互），每任务 50 个 episode；同时报告 EgoDex 人类示范共训与 world-model 可控生成结果。真机：IRON-R01-1.11 humanoid。设备配置：本体为 IRON-R01-1.11，人形全身 50 维动作空间，覆盖 arms / hands / waist / head / wrist poses；末端为双手/手部执行器；正文未见外置/腕部相机位姿数量的完整披露。
- **算力**：正文未披露统一训练硬件账单；正文明确 world-model 采用 10-step autoregressive rollout、10 秒视频生成，DROID 分辨率 192×320，RoboCasa-GR1 / EgoDex 为 192×336；VLA / WM 的 GPU 型号、卡数、训练时长未系统给出。
- **解读**：核心是把人类与人形动作都压到统一 UniT token 空间，再由轻量 flow head 解码回具体控制，因而同时服务策略学习和世界模型。亮点在于少样本、OOD 和零样本迁移都提升；局限是算力账单与感知配置披露不全，复现时应先锁定 EgoDex/RoboCasa 数据配比与下游 head 细节。

#### 2) VLA Foundry: A Unified Framework for Training Vision-Language-Action Models
- **标题**：VLA Foundry: A Unified Framework for Training Vision-Language-Action Models
- **简称**：VLA Foundry
- **领域**：VLA 训练框架 / 开源基础设施
- **论文**：https://arxiv.org/abs/2604.19728
- **项目**：https://tri-ml.github.io/vla_foundry/
- **代码**：https://github.com/TRI-ML/vla_foundry
- **评测**：仿真 benchmark：lbm_eval_oss / lbm_eval_cs，主文评测 16 个 seen simulation tasks + 3 个 unseen tasks，统一 200 rollout episodes 预算；42 个仿真任务、361 个真实任务用于训练混合。真机：本文未给独立真机成绩，仅说明数据中含 real robot subset。设备配置：仿真输入使用两路 wrist camera + 两路 external camera，共 4 视角；真实机器人本体/末端型号未在本文统一列出。
- **算力**：吞吐测试覆盖最多 128 GPUs / 16 nodes，硬件为 AWS P5 节点、每节点 8× NVIDIA H100；示例 global batch size 256、per-GPU batch size 8。附录表 4 还给出 MT 约 100M samples@batch 1024，ST 为 2–5M samples@batch 512，FT 为 1M samples。单次完整训练时长正文未统一披露。
- **解读**：这篇更像“开源 VLA 工程底座”而非单一模型：把 LLM→VLM→VLA 三段训练、数据转换、评测和分布式伸缩串成一条线。价值在于复现门槛明显下降；但当前主结果仍偏 LBM 仿真闭环，真机基准和多 embodiment 外推还要靠社区继续补齐。

### B. 高效动作桥接与鲁棒规划

#### 1) SpanVLA: Efficient Action Bridging and Learning from Negative-Recovery Samples for Vision-Language-Action Model
- **标题**：SpanVLA: Efficient Action Bridging and Learning from Negative-Recovery Samples for Vision-Language-Action Model
- **简称**：SpanVLA
- **领域**：Driving VLA / 高效动作桥接 / 后训练鲁棒性
- **论文**：https://arxiv.org/abs/2604.19710
- **项目**：https://spanvla.github.io/
- **代码**：正文未披露
- **评测**：仿真 benchmark：NAVSIM v1（navtest）与 NAVSIM v2（navtest、navhard），指标为 PDMS / EPDMS；v2 的 navhard 采用 pseudo closed-loop simulation。真机：无整车闭环真机部署结果，但引入真实道路 mReasoning 数据与 negative-recovery 样本。设备配置：默认 3 路相机视角（front、front-left、front-right），历史 4 帧/每路、2Hz；数据集原始采集含 8 路环视相机。机器人本体/末端不适用。
- **算力**：SFT 使用 8× NVIDIA A100；VLM 训练 per-GPU batch size 1、gradient accumulation 4；action bridge batch size 16。RFT 采用 LoRA，固定 6K RFT budget（2K positive warmup + 4K mixed training）。推理侧正文披露当前整套系统约 1.5Hz，仍不足以直接上车。
- **解读**：SpanVLA把“长链路自回归动作生成”改成 VLM + flow-matching action bridge，核心收益是把推理延迟从随轨迹长度线性增长改成并行生成，同时再用负样本/恢复样本做 GRPO 式后训练补鲁棒性。局限是仍未达到车端实时部署；复现重点应放在 mReasoning 清洗、奖励函数和 action bridge 接口对齐。

#### 2) Mask World Model: Predicting What Matters for Robust Robot Policy Learning
- **标题**：Mask World Model: Predicting What Matters for Robust Robot Policy Learning
- **简称**：MWM
- **领域**：World Model / Diffusion Policy / 语义掩码预测
- **论文**：https://arxiv.org/abs/2604.19683
- **项目**：正文未披露
- **代码**：正文未披露
- **评测**：仿真 benchmark：LIBERO（Spatial / Object / Goal / LIBERO-10，500 episodes/套件）与 RLBench。真机：Franka Emika Panda 四项桌面任务。设备配置：Franka Panda 7-DoF + parallel gripper；两路 Intel RealSense D435i，同步第三视角固定相机 + wrist eye-in-hand 相机，共 2 个位姿。
- **算力**：附录披露训练使用 8× NVIDIA A100 80GB + DeepSpeed ZeRO-2；Stage 1 约 3.5 天 / 30k steps，Stage 2 约 1.5 天 / 18k steps（per task suite）。部署频率正文给出真机 10Hz raw RGB 闭环；显存与单步时延未统一披露。
- **解读**：MWM不是直接预测 RGB 未来，而是先预测任务相关语义 mask，再让 policy head 消费 mask-centric predictive features，因此天然更抗背景、光照和颜色扰动。优点是现实视觉泛化更强；局限是需要离线语义标注链路，复现时要先打通 mask 渲染和两阶段训练，再谈真机收益。

### C. 真实系统与平台验证

#### 1) Autonomous UAV Pipeline Near-proximity Inspection via Disturbance-Aware Predictive Visual Servoing
- **标题**：Autonomous UAV Pipeline Near-proximity Inspection via Disturbance-Aware Predictive Visual Servoing
- **简称**：ESKF-PRE-VMPC
- **领域**：UAV 视觉伺服 / 真实系统控制
- **论文**：https://arxiv.org/abs/2604.19618
- **项目**：https://eskf-pre-vmpc.github.io/
- **代码**：正文超链接可核验到作者仓库主页，但论文未给单一总仓链接；已见相关代码域名碎片：Crazyflie / VS / C 系列 GitHub 链接。
- **评测**：仿真 benchmark：高保真 Gazebo + PX4 + ROS 的 SITL 管线巡检仿真，两组对比场景（无风 / 有风）。真机：nano quadrotor 管线巡检。设备配置：改装 Crazyflie，本体带 downward-facing onboard camera；真实实验另配四相机动作捕捉系统 ChingMu MC1300 做状态获取，机外 NUC PC（Intel i5-12450H）运行算法；机载微型下视相机分辨率 756×560。机器人末端不适用。
- **算力**：仿真在 Intel i5-13600KF CPU@5.1GHz 上运行；真实实验在 NUC PC + Intel i5-12450H 上执行算法。正文未披露 GPU 训练资源，因为方法为控制/估计框架而非深度大模型训练。
- **解读**：这篇亮点不在“大模型”，而在把低频视觉、扰动估计与 predictive visual servoing 扎实拼成可飞的近距管线巡检系统，真实风扰下仍优于基线。局限是对 mocap 和外部算力有依赖；复现建议先复刻 Gazebo 管线与相机-机体系外参，再逐步下放到机载计算。
